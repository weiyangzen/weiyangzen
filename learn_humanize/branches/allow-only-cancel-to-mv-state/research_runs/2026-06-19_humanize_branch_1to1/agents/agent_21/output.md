### ALLOW_ONLY_CANCEL_TO_MV_STATE-HZ-021 `file` `hooks/loop-write-validator.sh`
- cursor: `[_]`
- core_role: PreToolUse validator for Claude `Write` operations during an active RLCR loop. It protects loop-owned control files and round artifacts from direct writes, especially `state.md`, loop `plan.md` backup, prompt files, todos files, goal tracker after Round 0, and summaries in the wrong round or directory. It is registered as the `Write` matcher in `hooks/hooks.json:14-21`.
- algorithmic_behavior: The hook reads JSON hook input from stdin, extracts `.tool_name` and `.tool_input.file_path` with `jq`, and exits immediately unless `tool_name == "Write"` (`hooks/loop-write-validator.sh:23-30`). It lowercases the target path for type matching (`:31`), blocks round todos files unless they are allowlisted in the active loop directory (`:37-44`), and blocks round prompt files unconditionally (`:47-50`). It then classifies whether the target is a summary file and whether it is inside `.humanize/rlcr/` (`:56-61`). Non-summary files outside the loop directory are allowed. Non-summary loop files are allowed unless they are `state.md`, `goal-tracker.md`, or `plan.md` backup (`:64-72`). For protected loop files, it finds the newest active loop directory and parses `state.md` frontmatter to get `current_round` (`:78-89`). It blocks state file writes (`:95-98`), blocks loop `plan.md` backup writes using the plan-backup template (`:104-110`), blocks `goal-tracker.md` after Round 0 and redirects the user to the current summary file (`:117-120`), blocks summaries outside `.humanize/rlcr/` with a corrected path (`:127-134`), rejects wrong-round summary writes unless allowlisted (`:153-170`), and finally enforces that loop summary writes land inside the active loop directory (`:177-188`).
- inputs_outputs_state: Inputs are the hook JSON payload, `CLAUDE_PROJECT_DIR` or current working directory, active loop files under `.humanize/rlcr`, and `state.md` frontmatter. Outputs are process exit codes plus stderr guidance. Exit `0` allows the write; exit `2` blocks the tool call with a rendered template or fallback message. The hook does not mutate state itself. It reads RLCR state through `parse_state_file` in `hooks/lib/loop-common.sh:110-137`, using `STATE_CURRENT_ROUND` as its round gate. State transition effect is indirect: attempted `Write` transitions into protected loop state are denied before the write occurs. The branch’s explicit `mv state.md` cancel allowance is not owned by this Write hook; that exception lives in Bash-command validation through `is_cancel_authorized` (`hooks/lib/loop-common.sh:251-389`) and is invoked by `hooks/loop-bash-validator.sh:81-89`, `:218-225`, and `:231-240`.
- gates_or_invariants: The `state.md` invariant is absolute for `Write`: any target matching `state.md` is blocked (`hooks/loop-write-validator.sh:95-98`), so only loop-managed paths can end or rename state. The loop `plan.md` backup is read-only while it is under `.humanize/rlcr/` (`:104-110`). Round prompt files are read-only because they are generated instructions (`:47-50`). Todos files should not be written directly except a small legacy/compatibility allowlist checked by `is_allowlisted_file` (`hooks/lib/loop-common.sh:165-186`). After Round 0, `goal-tracker.md` becomes Codex-owned and Claude must request updates through the summary file (`hooks/loop-write-validator.sh:117-120`). Summary writes must match the active loop round and active loop directory unless allowlisted (`:153-188`).
- dependencies_and_callers: Called by Claude hook configuration in `hooks/hooks.json:14-21`. Depends on `hooks/lib/loop-common.sh` for active-loop discovery, state parsing, type/path predicates, allowlist checks, and message helpers (`hooks/loop-write-validator.sh:15-17`). `loop-common.sh` in turn loads `hooks/lib/template-loader.sh`, resolves `prompt-template`, and provides safe rendering (`hooks/lib/loop-common.sh:46-55`; `hooks/lib/template-loader.sh:24-31`, `:167-203`). Shell/tool dependencies include `bash`, `jq`, `sed`, `grep`, `tr`, `basename`, `ls`, `sort`, and `head`. Related sibling validators enforce the same plan backup invariant for `Edit` (`hooks/loop-edit-validator.sh:88-95`) and Bash writes (`hooks/loop-bash-validator.sh:249-253`).
- edge_cases_or_failure_modes: If the hook input is malformed or `jq` is unavailable, `set -euo pipefail` can fail the hook rather than producing a graceful block. If no active loop exists, round todos and prompt gates still run early, but protected `.humanize/rlcr` state/plan/summary validation exits allowed once no active loop is found (`hooks/loop-write-validator.sh:83-85`). `find_active_loop` only considers the newest timestamp-sorted loop directory with `state.md`, so older loop directories are intentionally ignored (`hooks/lib/loop-common.sh:58-79`). Non-numeric corrupted `current_round` could break numeric comparisons because this validator assumes `state.md` schema has already been protected by upstream validation.
