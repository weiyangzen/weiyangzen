- core_role: UserPromptSubmit validator for the RLCR loop state machine. It runs before user prompts while a loop is active and enforces three state-machine invariants: the loop state schema is current, the git branch has not changed, and the plan file still has the tracking/cleanliness posture recorded at loop start. Registered as the UserPromptSubmit command in `hooks/hooks.json:4-10`.
- algorithmic_behavior: The hook reads stdin into `INPUT` but does not inspect it; behavior is driven by `CLAUDE_PROJECT_DIR` or `pwd`, `.humanize/rlcr`, git state, and `state.md` frontmatter (`hooks/loop-plan-file-validator.sh:13-31`). It finds only the newest active loop via `find_active_loop`, then exits allow/quietly if none exists (`hooks/loop-plan-file-validator.sh:30-36`; `hooks/lib/loop-common.sh:58-79`). For an active loop, it parses `state.md`, maps `STATE_PLAN_TRACKED`, `STATE_PLAN_FILE`, and `STATE_START_BRANCH` to local variables, and validates required fields (`hooks/loop-plan-file-validator.sh:38-82`).
- inputs_outputs_state: Inputs are hook stdin, `CLAUDE_PROJECT_DIR`, active loop path `.humanize/rlcr/<timestamp>/state.md`, git branch, git tracked-file index, and git porcelain status. Outputs are either no stdout with exit 0 for allow, or a JSON object with `"decision": "block"` and a human-readable `"reason"` (`hooks/loop-plan-file-validator.sh:64-69`, `92-97`, `101-106`, `124-129`, `149-154`, `168-183`, `214-219`). It does not mutate files or advance loop state. The relevant state transition is gate-level: active `state.md` plus violated invariant causes the next user prompt to be blocked; no active `state.md`, including a renamed end-state file, allows the hook to exit with no decision.
- gates_or_invariants: Requires `plan_tracked` and `start_branch` fields; missing fields produce a schema-outdated block rendered through `block/schema-outdated.md` with inline fallback (`hooks/loop-plan-file-validator.sh:52-80`). If `plan_tracked=true`, the plan must still be tracked and have empty `git status --porcelain` output (`hooks/loop-plan-file-validator.sh:116-185`). Git operations are capped by `GIT_TIMEOUT=30` through `run_with_timeout` (`hooks/loop-plan-file-validator.sh:19-24`; `scripts/portable-timeout.sh:31-71`).
