- core_role: PreToolUse `Read` validator for RLCR loop files. It protects the active loop state machine from stale or misplaced round artifacts by blocking reads of wrong-round `round-*-summary.md` and `round-*-prompt.md` files, blocking most `round-*-todos.md` reads, and requiring current round files to come from the active `.humanize/rlcr/<session>` directory.
- algorithmic_behavior: The hook reads JSON from stdin, extracts `.tool_name`, and exits successfully unless the tool is `Read` (`hooks/loop-read-validator.sh:22-27`). It extracts `.tool_input.file_path`, lowercases it for pattern matching, and first applies a special todos gate (`hooks/loop-read-validator.sh:29-44`). Non-round summary/prompt files are allowed immediately (`hooks/loop-read-validator.sh:50-52`). For summary/prompt files, it resolves the active loop directory, parses `state.md`, extracts `current_round`, extracts the requested round from the filename, infers `summary` versus `prompt`, then applies location, round, and exact-path validation (`hooks/loop-read-validator.sh:54-140`).
- inputs_outputs_state: Inputs are Claude hook JSON on stdin, the requested `file_path`, environment-derived project roots (`PROJECT_ROOT`, `CLAUDE_PROJECT_DIR`, `LOOP_BASE_DIR`, optional `LOOP_DIR`), and the active loop `state.md` frontmatter field `current_round`. Outputs are process exit codes: `0` permits the read; `2` blocks the read and writes a rendered block message to stderr. The script does not mutate loop state. It is a guard around state-machine reads, not a state transition executor.
- gates_or_invariants: Todos files matching `round-[0-9]+-todos.md` are blocked unless allowlisted in the active loop directory (`hooks/loop-read-validator.sh:36-43`; allowlist in `hooks/lib/loop-common.sh:168-186`). Summary/prompt round files must be inside `.humanize/rlcr/`, match the active `current_round`, and have the exact active-loop path `$ACTIVE_LOOP_DIR/round-$CURRENT_ROUND-$FILE_TYPE.md` (`hooks/loop-read-validator.sh:94-140`). The hardcoded allowlist permits only `round-1-todos.md`, `round-2-todos.md`, `round-0-summary.md`, and `round-1-summary.md` in the active loop dir (`hooks/lib/loop-common.sh:172-177`).

Sources `hooks/lib/loop-common.sh` (`hooks/loop-read-validator.sh:14-16`), which in turn sources `hooks/lib/template-loader.sh` and provides `find_active_loop`, `parse_state_file`, `is_round_file_type`, `extract_round_number`, `is_allowlisted_file`, and block-message helpers (`hooks/lib/loop-common.sh:46-52`, `62-79`, `110-137`, `140-197`). Output is markdown reason text embedded in the Stop-hook JSON block response. The template itself has no state mutation; the gate prevents the active RLCR loop from proceeding to the expensive review/exit path until over-limit changed files are split.
