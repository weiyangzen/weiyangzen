### ALLOW_ONLY_CANCEL_TO_MV_STATE-HZ-012 `file` `commands/cancel-rlcr-loop.md`
- cursor: `[_]`
- core_role: Slash-command workflow for terminating the active RLCR loop by transitioning the loop state file from active `state.md` to preserved `cancel-state.md`. The command is hidden from the slash-command tool list and tightly scopes its declared tools to listing loop dirs, touching the cancel signal, moving the state file, reading state, and `Read` (`commands/cancel-rlcr-loop.md:1-4`).
- algorithmic_behavior: It selects the current loop as the newest timestamped directory under `.humanize/rlcr/` using `ls -1d ... | sort -r | head -1` (`commands/cancel-rlcr-loop.md:11-16`). It treats missing loop dirs as no active loop (`commands/cancel-rlcr-loop.md:18`). It then checks for `state.md`; if absent, it reports that the loop already ended or never started properly (`commands/cancel-rlcr-loop.md:20-27`). If present, it reads round/max-iteration state, creates `.cancel-requested` first, then renames `state.md` to `cancel-state.md` (`commands/cancel-rlcr-loop.md:28-32`).
- inputs_outputs_state: Inputs are `.humanize/rlcr/<timestamp>/` directories plus the active loop’s `state.md` frontmatter containing values such as current round and max iterations. Outputs are `.humanize/rlcr/<timestamp>/.cancel-requested`, `.humanize/rlcr/<timestamp>/cancel-state.md`, and a user-facing cancellation report (`commands/cancel-rlcr-loop.md:30-32`). The state transition is active loop present to canceled loop preserved; the active-loop predicate is explicitly the presence of `state.md` in the newest loop dir (`commands/cancel-rlcr-loop.md:34-36`). This matches `find_active_loop`, which only returns the newest directory if it contains `state.md` (`hooks/lib/loop-common.sh:58-79`).
- gates_or_invariants: The command’s key invariant is “newest timestamp wins” and “active only if `state.md` exists” (`commands/cancel-rlcr-loop.md:34`). The branch-specific safety gate is the cancel signal: the Bash validator blocks state-file modifications generally (`hooks/loop-bash-validator.sh:71-89`) and blocks `mv`/`cp` where `state.md` is the source unless `is_cancel_authorized` accepts it (`hooks/loop-bash-validator.sh:91-105`, `hooks/loop-bash-validator.sh:218-224`). Authorization requires `.cancel-requested` in the active loop dir, rejects command substitution, backticks, newlines, and shell chaining, requires a bare `mv`, requires exactly two args, and requires exact active-loop `state.md -> cancel-state.md` paths (`hooks/lib/loop-common.sh:251-388`).
- dependencies_and_callers: Depends on the `.humanize/rlcr` layout created by `start-rlcr-loop`, whose stop conditions include user cancellation (`commands/start-rlcr-loop.md:51-55`). README documents the same lifecycle: loop state is controlled by `state.md`, cancellation renames it to `cancel-state.md`, and artifacts are preserved (`README.md:112-129`). Hook failure messages also direct users to cancel when Codex is missing or review fails (`hooks/loop-codex-stop-hook.sh:720-728`, `hooks/loop-codex-stop-hook.sh:817-828`).
- edge_cases_or_failure_modes: If there are no loop dirs or the newest dir lacks `state.md`, the command reports no active loop rather than searching older dirs (`commands/cancel-rlcr-loop.md:18-27`). That avoids reviving older “zombie” loops, consistent with `find_active_loop` comments (`hooks/lib/loop-common.sh:58-61`). If `state.md` exists but lacks expected fields, the command says to read round/max values but does not define parse fallback behavior in the markdown. A race is possible if a newer loop dir appears between selection and move; no lock is defined in this command file. The signal file is intentionally created before `mv`, and wrong-directory signals do not authorize cancellation.
- validation_or_tests: `tests/test-cancel-signal-file.sh` directly validates the branch’s cancel-signal mechanism: allowed exact move with signal (`tests/test-cancel-signal-file.sh:78-91`), blocked move without signal (`tests/test-cancel-signal-file.sh:119-132`), blocked edits/removes/wrong destinations despite signal (`tests/test-cancel-signal-file.sh:139-235`), blocked command chaining and substitution (`tests/test-cancel-signal-file.sh:241-315`), and many bypass cases later in the file. `tests/test-state-exit-naming.sh` validates that `cancel-state.md` is not active and only `state.md` indicates active state (`tests/test-state-exit-naming.sh:74-107`).
