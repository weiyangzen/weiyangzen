The manifest-level algorithm is declarative event routing: match lifecycle/tool name, execute command hook, let the command return allow/block JSON or exit status according to Claude hook semantics.
- inputs_outputs_state: Input is Claude hook runtime JSON passed to the configured command on stdin. The manifest itself has no internal state, but it routes events into scripts that read tool-specific fields. The `Write`, `Edit`, and `Read` validators parse `.tool_name` and `.tool_input.file_path`; the `Bash` validator parses `.tool_input.command`; the `Stop` hook reads the full hook input and optionally uses `transcript_path` through `check-todos-from-transcript.py`. Output is the invoked command’s decision: validator scripts usually `exit 0` to allow or `exit 2` with stderr guidance to block, while the stop hook emits JSON such as `{"decision":"block", ...}` when loop continuation is required.
