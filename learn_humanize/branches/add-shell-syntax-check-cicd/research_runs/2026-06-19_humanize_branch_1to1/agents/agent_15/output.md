### ADD_SHELL_SYNTAX_CHECK_CICD-HZ-015 `file` `hooks/loop-write-validator.sh`
- cursor: `[_]`
- core_role: `hooks/loop-write-validator.sh` is the Claude `PreToolUse` Write-path gate for the RLCR loop state machine. It prevents Claude Write operations from mutating generated/managed loop files incorrectly, especially `round-*-todos.md`, `round-*-prompt.md`, `state.md`, `goal-tracker.md` after round 0, and summary files for the wrong round or wrong loop directory. It is registered as the `Write` matcher command in `hooks/hooks.json:4-11`.
- algorithmic_behavior: The script reads the hook JSON payload from stdin, extracts `.tool_name`, and exits successfully for non-`Write` tools (`hooks/loop-write-validator.sh:23-28`). For Write calls, it extracts `.tool_input.file_path`, lowercases it via `to_lower`, and applies ordered validation gates (`hooks/loop-write-validator.sh:30-45`). The first hard gates block any write to `round-*-todos.md` and `round-*-prompt.md` using shared user-facing messages (`hooks/loop-write-validator.sh:37-45`; shared implementations in `hooks/lib/loop-common.sh:75-108`). It then classifies whether the target is a summary file or is inside `.humanize-loop.local/` (`hooks/loop-write-validator.sh:51-59`). Files that are neither summary files nor inside the loop state directory are allowed immediately (`hooks/loop-write-validator.sh:61-64`). Non-summary files inside `.humanize-loop.local/` are allowed unless they are `state.md` or `goal-tracker.md` (`hooks/loop-write-validator.sh:66-72`). For guarded loop files, it locates the newest active loop directory and reads the current round from `state.md` (`hooks/loop-write-validator.sh:78-87`). If no active loop exists, the write is allowed (`hooks/loop-write-validator.sh:82-84`). Once active state exists, it blocks `state.md` writes unconditionally (`hooks/loop-write-validator.sh:92-95`), blocks `goal-tracker.md` writes after round 0 (`hooks/loop-write-validator.sh:101-105`), blocks summary writes outside the active loop directory (`hooks/loop-write-validator.sh:111-120`), extracts the loop-relative filename (`hooks/loop-write-validator.sh:127-133`), rejects mismatched `round-N-summary.md` writes (`hooks/loop-write-validator.sh:139-154`), and finally enforces that the requested path exactly equals `$ACTIVE_LOOP_DIR/$CLAUDE_FILENAME` (`hooks/loop-write-validator.sh:161-170`).
- inputs_outputs_state: Input is a Claude hook event JSON on stdin with at least `tool_name` and, for Write, `tool_input.file_path` (`hooks/loop-write-validator.sh:23-30`). It depends on `jq` for JSON parsing and on shell utilities including `tr`, `grep`, `sed`, `find`-style path checks through shared helpers. Output is either exit `0` to allow the Write or exit `2` with a markdown diagnostic on stderr to block the Write. State read is external and repository-local: `PROJECT_ROOT` defaults to `CLAUDE_PROJECT_DIR` or `pwd`, `LOOP_BASE_DIR` is `$PROJECT_ROOT/.humanize-loop.local`, `find_active_loop` selects the newest child directory containing `state.md`, and `get_current_round` parses `current_round:` from YAML frontmatter (`hooks/loop-write-validator.sh:78-87`; `hooks/lib/loop-common.sh:15-47`). The script does not mutate state itself; it only protects state-machine-owned files. The loop state files it guards are created by setup code under `.humanize-loop.local/<timestamp>/`, including `state.md`, `goal-tracker.md`, and round prompt/summary files (`scripts/setup-rlcr-loop.sh:210-240`, `scripts/setup-rlcr-loop.sh:334-347`; README structure at `README.md:241-251`).
- gates_or_invariants: The primary invariant is that Claude may not use Write to create or alter generated todo files; it must use native TodoWrite instead (`hooks/loop-write-validator.sh:37-40`; message text in `hooks/lib/loop-common.sh:80-89`). Prompt files are immutable instructions from Codex to Claude (`hooks/loop-write-validator.sh:42-45`; `hooks/lib/loop-common.sh:93-107`). `state.md` is loop-managed and blocked for all Write attempts once an active loop exists (`hooks/loop-write-validator.sh:92-95`; `hooks/lib/loop-common.sh:110-124`). `goal-tracker.md` has a round-dependent gate: Write is allowed during round 0 setup, but after `CURRENT_ROUND > 0` Claude must request changes through the current summary file instead (`hooks/loop-write-validator.sh:101-105`; `hooks/lib/loop-common.sh:203-235`). Summary writes must target the active loop directory, must use the current round number, and must match the computed canonical path exactly (`hooks/loop-write-validator.sh:111-170`). The active-loop invariant intentionally uses only the newest loop directory with `state.md` to avoid reviving older sessions, documented in `hooks/lib/loop-common.sh:11-15`.
- dependencies_and_callers: Direct dependency is `hooks/lib/loop-common.sh`, sourced at `hooks/loop-write-validator.sh:15-17`. Shared helper roles include active loop discovery (`hooks/lib/loop-common.sh:15-33`), current round parsing (`hooks/lib/loop-common.sh:35-47`), round filename matching (`hooks/lib/loop-common.sh:54-61`), round extraction (`hooks/lib/loop-common.sh:63-73`), protected-path predicates (`hooks/lib/loop-common.sh:160-176`), and standard block messages (`hooks/lib/loop-common.sh:75-124`, `hooks/lib/loop-common.sh:203-235`). - edge_cases_or_failure_modes: If hook input is malformed JSON or `jq` is missing/fails, `set -euo pipefail` causes the hook to fail rather than silently allow, which is conservative but may block tool use unexpectedly (`hooks/loop-write-validator.sh:13`, `hooks/loop-write-validator.sh:23-30`). The path matching for round files is lowercase and filename-suffix based, so case-insensitive `Round-1-Summary.md` still classifies after lowercasing (`hooks/loop-write-validator.sh:30-31`; `hooks/lib/loop-common.sh:49-61`). If no `.humanize-loop.local` active directory exists, guarded writes are allowed after active-loop lookup returns empty (`hooks/loop-write-validator.sh:80-84`); this avoids blocking outside active sessions but means protection depends on setup state existing. `find_active_loop` only checks the lexicographically newest child directory and ignores older loops even if they contain state (`hooks/lib/loop-common.sh:11-33`), so a newer incomplete directory without `state.md` can cause no active loop to be found. `get_current_round` defaults to `0` if parsing fails (`hooks/lib/loop-common.sh:35-47`), which would allow round-0-style `goal-tracker.md` Write behavior. Path canonicalization is string-based: symlinks, `..`, relative paths, or different absolute spellings are not normalized before comparing to `CORRECT_PATH` (`hooks/loop-write-validator.sh:161-170`). The filename extraction assumes `.humanize-loop.local/<session>/<filename>` or `.humanize-loop.local/<filename>` and only captures a single remaining component pattern via `sed` (`hooks/loop-write-validator.sh:127-133`), so unusual nested loop paths may be allowed or redirected depending on how the regex captures.
