- core_role: `hooks/hooks.json` is the Claude Code hook manifest for the Humanize RLCR loop. The description explicitly scopes it to validation hooks and stop hooks for `/start-rlcr-loop` at `hooks/hooks.json:2`.
- algorithmic_behavior: The manifest registers three event classes. `UserPromptSubmit` runs `loop-plan-file-validator.sh` before user prompts are accepted (`hooks/hooks.json:4-13`), making plan/state/branch checks a prompt-entry gate. `PreToolUse` dispatches tool-specific validators by matcher: `Write` -> `loop-write-validator.sh`, `Edit` -> `loop-edit-validator.sh`, `Read` -> `loop-read-validator.sh`, and `Bash` -> `loop-bash-validator.sh` (`hooks/hooks.json:14-51`). `Stop` runs `loop-codex-stop-hook.sh` with a 7200 second timeout (`hooks/hooks.json:52-61`), making attempted loop termination pass through Codex review and loop-state transition logic.
- inputs_outputs_state: Input is Claude Code hook event JSON supplied by the host runtime to each registered command. Outputs are indirect: each invoked hook returns either allow/block status or JSON block decisions depending on hook type. State touched by downstream scripts includes `.humanize/rlcr/<timestamp>/state.md`, `goal-tracker.md`, round prompt/summary/todos files, Codex review prompt/result files, and cache files, but the manifest itself has no mutable state.
- gates_or_invariants: The manifest enforces coverage over all RLCR-sensitive interaction points: prompt submission, file reads/writes/edits, bash commands, and stop attempts. The key invariant is that protected RLCR files cannot be bypassed by switching tools: direct writes go through Write/Edit validators, shell-based writes go through the Bash validator, reads go through the Read validator, and exit attempts go through the Stop hook. A second invariant is event-to-script separation: the manifest uses exact matcher names (`Write`, `Edit`, `Read`, `Bash`) and each script independently ignores nonmatching tool names, for example `loop-bash-validator.sh` exits unless `.tool_name == "Bash"` at `hooks/loop-bash-validator.sh:22-27`.
