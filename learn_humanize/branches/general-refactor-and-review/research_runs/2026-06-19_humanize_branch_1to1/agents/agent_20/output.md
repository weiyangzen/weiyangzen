### GENERAL_REFACTOR_AND_REVIEW-HZ-020 `file` `hooks/loop-read-validator.sh`
- cursor: `[_]`
- core_role: PreToolUse read-access validator for RLCR loop artifacts. It protects the loop state machine from stale or misplaced round context by allowing ordinary reads, but gating reads of `round-*-summary.md`, `round-*-prompt.md`, and `round-*-todos.md` files. It is registered for Claude `Read` tools in `hooks/hooks.json:33-39`.
- algorithmic_behavior: The hook reads JSON hook input from stdin, extracts `.tool_name`, and exits immediately unless the tool is `Read` (`hooks/loop-read-validator.sh:22-27`). For `Read`, it extracts `.tool_input.file_path`, lowercases it, and first checks for `round-*-todos.md` access (`hooks/loop-read-validator.sh:29-43`). Todos reads are blocked unless the file is explicitly allowlisted for the active loop. For summary/prompt files, it finds the active loop, parses current round state, extracts the requested round number, classifies the file as `summary` or `prompt`, then enforces location, round number, and exact directory invariants (`hooks/loop-read-validator.sh:50-140`). Successful reads exit `0`; policy violations render a block template/fallback to stderr and exit `2`.
- inputs_outputs_state: Input is Claude hook JSON on stdin with at least `tool_name` and `tool_input.file_path`. Environment inputs are `PROJECT_ROOT`, `CLAUDE_PROJECT_DIR`, `LOOP_BASE_DIR`, and `LOOP_DIR`, with defaults rooted at `$PROJECT_ROOT/.humanize/rlcr` (`hooks/loop-read-validator.sh:37-39`, `hooks/loop-read-validator.sh:61-64`). State comes from the newest active loop directory containing `state.md`, discovered by `find_active_loop` (`hooks/lib/loop-common.sh:58-79`), then parsed via YAML frontmatter into `STATE_CURRENT_ROUND` by `parse_state_file` (`hooks/lib/loop-common.sh:96-137`). Outputs are process status plus any rendered diagnostic message. No repository files are modified.
- gates_or_invariants: Non-Read tools are ignored (`hooks/loop-read-validator.sh:25-27`). Non-round summary/prompt/todos files are allowed (`hooks/loop-read-validator.sh:50-52`). Todos files are generally blocked, with allowlist exceptions (`hooks/loop-read-validator.sh:36-43`). Summary/prompt files must be inside `.humanize/rlcr/` (`hooks/loop-read-validator.sh:54-55`, `hooks/loop-read-validator.sh:94-103`), must match the active `current_round` unless allowlisted (`hooks/loop-read-validator.sh:110-122`), and must be exactly `$ACTIVE_LOOP_DIR/$CLAUDE_FILENAME`, preventing reads from older loop sessions or parallel loop directories (`hooks/loop-read-validator.sh:129-140`). Round file detection is strict, case-normalized, and based on `round-[0-9]+-{summary|prompt|todos}.md` (`hooks/lib/loop-common.sh:139-163`). The allowlist is limited to `round-1-todos.md`, `round-2-todos.md`, `round-0-summary.md`, and `round-1-summary.md` in the active loop directory only (`hooks/lib/loop-common.sh:165-186`).
- dependencies_and_callers: Depends on `jq`, Bash with `set -euo pipefail`, and shared helpers from `hooks/lib/loop-common.sh` (`hooks/loop-read-validator.sh:12-16`). That library depends on `hooks/lib/template-loader.sh` and computes `TEMPLATE_DIR` for message rendering (`hooks/lib/loop-common.sh:46-56`). It uses templates `block/todos-file-access.md`, `block/wrong-file-location.md`, `block/wrong-round-file.md`, and `block/wrong-directory-path.md` via `load_and_render_safe` (`hooks/loop-read-validator.sh:41`, `hooks/loop-read-validator.sh:99-102`, `hooks/loop-read-validator.sh:116-121`, `hooks/loop-read-validator.sh:135-138`). Caller registration is `hooks/hooks.json:33-39`. `tests/test-template-references.sh:57-64` includes this validator in template-reference scanning and `tests/test-template-references.sh:174-190` checks critical validators use safe rendering.
- edge_cases_or_failure_modes: If there is no active loop, round summary/prompt validation exits allowed (`hooks/loop-read-validator.sh:65-67`), and todos validation blocks unless allowlisted because the allowlist check cannot pass without a loop directory (`hooks/loop-read-validator.sh:36-43`). If `parse_state_file` cannot read `state.md`, the script does not inspect its return code; because `find_active_loop` only returns directories with `state.md`, the main practical fallback is `STATE_CURRENT_ROUND` defaulting to `0` from the helper (`hooks/lib/loop-common.sh:113-137`). If a filename contains `round-N-summary.md` or `round-N-prompt.md` but is outside `.humanize/rlcr/`, it is blocked as wrong location (`hooks/loop-read-validator.sh:94-103`). If it is in `.humanize/rlcr/` but not the newest active loop directory, it is blocked by exact path comparison (`hooks/loop-read-validator.sh:129-140`). The active-loop algorithm intentionally ignores older loop directories, even if they still contain state, to avoid reviving stale sessions (`hooks/lib/loop-common.sh:58-60`).
