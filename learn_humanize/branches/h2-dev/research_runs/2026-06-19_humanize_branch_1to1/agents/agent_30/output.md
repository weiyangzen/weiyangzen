- algorithmic_behavior: `detect_timeout_impl` selects in priority order: Homebrew `gtimeout`, GNU-compatible `timeout`, `python3`, `python`, then `none` (`scripts/portable-timeout.sh:10-32`). GNU `timeout` is accepted only if `timeout --version` output looks like GNU/coreutils/timeout version text, avoiding shims (`scripts/portable-timeout.sh:15-21`). The result is stored once at source time in `TIMEOUT_IMPL` (`scripts/portable-timeout.sh:34`). `run_with_timeout` preserves command arguments in an array and dispatches by implementation (`scripts/portable-timeout.sh:38-43`).
- inputs_outputs_state: Inputs are timeout seconds plus command argv (`scripts/portable-timeout.sh:36-41`). Outputs are the wrapped command’s stdout/stderr and exit status. On Python timeout it exits `124` to match GNU timeout (`scripts/portable-timeout.sh:61-62`). The only durable shell state is exported `TIMEOUT_IMPL`, so sourcing scripts can inspect which backend was selected (`scripts/portable-timeout.sh:78-81`).
- gates_or_invariants: The wrapper is a workflow safety gate for potentially hanging commands. Native `gtimeout`/GNU `timeout` get direct delegation (`scripts/portable-timeout.sh:43-50`). Python fallback uses `subprocess.run(..., timeout=seconds)` and returns the child exit code on success (`scripts/portable-timeout.sh:52-67`). If no timeout tool exists, it warns and runs without a timeout (`scripts/portable-timeout.sh:69-73`), so bounded execution degrades rather than failing closed.

It is also used in timeout and unified-config tests.
- edge_cases_or_failure_modes: Timeout seconds are not validated by this wrapper; invalid values are passed to the selected backend. If no timeout implementation exists, commands can hang despite the warning (`scripts/portable-timeout.sh:69-73`). Python fallback catches generic exceptions, prints `Error: ...` to stderr, and exits `1` (`scripts/portable-timeout.sh:63-65`). Backend behavior can differ for process groups and signal handling.
- validation_or_tests: Related tests include `tests/robustness/test-timeout-robustness.sh`, which exercises success, timeout, argument preservation, nonzero exits, pipelines, missing commands, empty command, special characters, and repeated use. `tests/test-unified-codex-config.sh` also depends on `run_with_timeout` for bounded setup/ask-codex calls.
