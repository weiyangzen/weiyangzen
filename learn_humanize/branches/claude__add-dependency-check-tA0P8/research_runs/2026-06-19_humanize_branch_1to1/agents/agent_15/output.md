- cursor: `[_]`
- core_role: Runtime PR-review polling script. It is the stop hook’s API adapter for detecting new bot issue comments, review comments, and PR reviews after a timestamp, normalized into one JSON result consumed by the PR loop.
- algorithmic_behavior: The script parses required CLI input `poll-pr-reviews.sh <pr_number> --after <timestamp> --bots <bot1,bot2>` and rejects missing/unknown args and nonnumeric PR numbers (`scripts/poll-pr-reviews.sh:23-103`). It requires `gh` and `jq` (`scripts/poll-pr-reviews.sh:108-116`). It resolves the correct PR repository by trying the current repo first, then parent repo for fork PRs (`scripts/poll-pr-reviews.sh:122-158`). It maps logical bot names to GitHub authors, with special handling for `codex -> chatgpt-codex-connector[bot]` and default `<bot>[bot]` (`scripts/poll-pr-reviews.sh:164-187`). It creates a temp directory, installs cleanup trap, and uses `fetch_with_retry` to call `gh api --paginate` up to three times; API failure returns `[]` and exit 0 so polling can continue (`scripts/poll-pr-reviews.sh:193-235`). It fetches issue comments, review comments, and PR reviews, normalizes them into common fields, and supplies a placeholder body for approval-only/empty review bodies (`scripts/poll-pr-reviews.sh:240-294`). It combines arrays, filters by timestamp and author regex, and emits final JSON with `comments`, `bots_responded`, `has_new_comments`, and `comment_count` (`scripts/poll-pr-reviews.sh:296-326`).
- inputs_outputs_state: Inputs are PR number, ISO-like `--after` timestamp, comma-separated bot list, current git/GitHub repo context from `gh repo view`, and GitHub API endpoint data. Output is JSON to stdout; warnings/errors go to stderr. State is temporary only: `mktemp -d` files `issue.json`, `review.json`, `reviews.json`, `all_comments.json`, and `filtered.json`, all removed by trap. It does not mutate repository files.
- gates_or_invariants: It enforces required arguments, numeric PR id, availability of `gh` and `jq`, resolvable current or parent repository, and parseable owner/name. API endpoint failures are intentionally downgraded to empty arrays, preserving loop liveness. Bot author filtering is restricted to configured bots and comments with `.created_at >= --after`.
- dependencies_and_callers: Direct caller is `hooks/pr-loop-stop-hook.sh`, which sets `POLL_SCRIPT="$PLUGIN_ROOT/scripts/poll-pr-reviews.sh"` (`hooks/pr-loop-stop-hook.sh:812`) and invokes it inside a polling loop with waiting bots (`hooks/pr-loop-stop-hook.sh:908-963`). The stop hook then maps `bots_responded` back to logical bot names, deduplicates comments by id, handles Codex `+1`, and updates `active_bots` or writes `approve-state.md` (`hooks/pr-loop-stop-hook.sh:968-1126`). External dependencies are GitHub CLI authentication/network and `jq`.
- edge_cases_or_failure_modes: Fork PRs are handled by parent-repo fallback, but failure to find the PR in current or parent repo is fatal. Timestamp comparison is string-based in jq (`.created_at >= $after`), which works for normalized ISO 8601 UTC timestamps but is not a true date parse. Retry behavior prevents transient API failure from crashing the loop, but also means a full outage appears as `has_new_comments:false`; the stop hook’s timeout logic must decide whether to continue, block, or auto-remove bots. Empty PR review bodies are converted to `[Review state: <state>]`, avoiding silent loss of approval-only reviews. Bot regex escapes brackets but assumes simple bot names; unexpected regex characters in bot names could broaden matches.
- validation_or_tests: Robustness tests cover help output, missing args rejection, JSON shape with mocked `gh`, slow API response, and API failure returning valid JSON with boolean `has_new_comments:false` (`tests/robustness/test-pr-loop-api-robustness.sh:630-855`). Stop-hook tests exercise poll integration, raw GitHub API fixture shape, fork PR base repo behavior, approval-only paths, timestamp filtering, and timeout behavior (`tests/test-pr-loop-stophook.sh:755-891`, `tests/test-pr-loop-stophook.sh:1390-1515`).
