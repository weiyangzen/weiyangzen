### USE_REALPATH4EVERYTHING-HZ-046 `file` `hooks/loop-write-validator.sh`

- core_role: Claude Code `PreToolUse` validator for the RLCR loop `Write` tool. It protects `.humanize/rlcr/<loop>/` control artifacts by allowing only the active loop’s authorized write targets and blocking writes that would corrupt round state, generated prompts, todos discipline, finalize state, methodology-analysis state, immutable goal-tracker content, stale round summaries/contracts, or equivalent paths outside the active loop. It is installed as the `Write` matcher in `hooks/hooks.json:14-21`.
- algorithmic_behavior: Reads the hook JSON from stdin, validates structure with shared helpers, ignores non-`Write` tools, requires `tool_input.file_path`, and extracts `session_id` for session-scoped loop lookup (`hooks/loop-write-validator.sh:24-52`). It first blocks round todos unless the path is allowlisted in the session’s active loop, and always blocks round prompt writes (`hooks/loop-write-validator.sh:57-70`). It then applies methodology-analysis phase isolation before normal file-type early exits: when `methodology-analysis-state.md` exists in the session-matched loop, only `methodology-analysis-report.md` and `methodology-analysis-done.md` inside that loop may be written; all other writes are blocked (`hooks/loop-write-validator.sh:79-133`). After classifying summary, contract, finalize-summary, and `.humanize/rlcr` paths, it exits early for unrelated writes (`hooks/loop-write-validator.sh:139-147`). For loop-local non-summary/non-contract files it allows ordinary files but retains validation for `state.md`, `finalize-state.md`, `methodology-analysis-state.md`, `goal-tracker.md`, and `plan.md` backups (`hooks/loop-write-validator.sh:149-157`). It resolves the active loop, resolves normal/finalize/methodology state, strict-parses current round, blocks state-file writes, conditionally allows only the active loop’s `finalize-summary.md`, blocks contracts in finalize phase, blocks `plan.md` backups, validates goal-tracker path/content mutability, blocks summaries/contracts outside `.humanize/rlcr`, validates round numbers, then validates the canonicalized directory path (`hooks/loop-write-validator.sh:163-354`).
- inputs_outputs_state: Input is Claude hook JSON on stdin with at minimum `tool_name`; for `Write`, `tool_input.file_path` is mandatory and `tool_input.content` is mandatory only for post-round goal-tracker writes (`hooks/loop-write-validator.sh:27-47`, `hooks/loop-write-validator.sh:253-259`). Optional input `session_id` scopes active-loop discovery through `extract_session_id` and `find_active_loop`, preventing unrelated sessions in the same repo from being restricted by another loop (`hooks/loop-write-validator.sh:50-51`, `hooks/loop-write-validator.sh:82-87`; helper behavior in `hooks/lib/loop-common.sh:308-332`). Outputs are exit codes and stderr block messages: `0` allows the write or ignores irrelevant contexts; `1` signals malformed hook input, deeply nested JSON, missing required fields, or malformed state; `2` blocks the Claude operation with a rendered/fallback explanation. The script does not mutate repo state itself. It reads state from `.humanize/rlcr/<loop>/{methodology-analysis-state.md,finalize-state.md,state.md}` via `resolve_active_state_file` and `parse_state_file_strict`, where the active state precedence is methodology-analysis, finalize, then normal state (`hooks/lib/loop-common.sh:268-280`, `hooks/lib/loop-common.sh:533-570`).
- gates_or_invariants: JSON must be syntactically valid, non-null-byte, valid UTF-8 when `iconv` is present, include `tool_name`, and not exceed nesting depth 30 (`hooks/lib/loop-common.sh:91-165`). Write paths to `round-*-prompt.md` are always blocked (`hooks/loop-write-validator.sh:67-70`).
