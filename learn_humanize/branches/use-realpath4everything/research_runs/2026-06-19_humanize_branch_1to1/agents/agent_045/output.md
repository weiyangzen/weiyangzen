### USE_REALPATH4EVERYTHING-HZ-045 `file` `hooks/loop-read-validator.sh`
- cursor: `[_]`
- core_role: `hooks/loop-read-validator.sh` is the Claude `PreToolUse` read gate for RLCR loop artifacts. It decides whether a `Read` tool call may access RLCR prompt, summary, contract, todos, goal-tracker, or methodology-analysis files. It is registered for the `Read` matcher in `hooks/hooks.json:33-39`, runs before Claude reads a file, and returns hook exit codes rather than mutating repository state.
- algorithmic_behavior: The hook reads JSON from stdin, validates it with shared hook-input guards, ignores non-`Read` tool calls, requires `.tool_input.file_path`, lowercases a copy for pattern matching, and extracts optional `.session_id` for session-scoped active-loop lookup (`hooks/loop-read-validator.sh:23-50`). It first blocks most `round-*-todos.md` reads, allowing only active-loop allowlisted files (`hooks/loop-read-validator.sh:56-64`). It then applies methodology-analysis isolation before normal round-file checks: when the active state is `methodology-analysis-state.md`, it canonicalizes the requested path and loop/project roots, allows only `methodology-analysis-report.md`, `methodology-analysis-done.md`, and `methodology-analysis-state.md` inside the loop directory, blocks other project-root reads, and allows files outside the project root (`hooks/loop-read-validator.sh:76-168`). For normal/finalize loop files, it classifies goal tracker and round prompt/summary/contract reads, resolves the active loop, parses the active state strictly, blocks contract reads in finalize phase, validates the active `goal-tracker.md` path, extracts the filename round number and type, blocks round files outside `.humanize/rlcr`, blocks stale round numbers unless explicitly allowlisted, and finally compares the requested path to the expected active-loop path using prefix canonicalization (`hooks/loop-read-validator.sh:175-324`).
- inputs_outputs_state: Inputs are stdin hook JSON with `tool_name`, `tool_input.file_path`, and optional `session_id`; environment/project context from `CLAUDE_PROJECT_DIR` or `git rev-parse`; active RLCR directories under `.humanize/rlcr`; active state files `methodology-analysis-state.md`, `finalize-state.md`, or `state.md`; and message templates under `prompt-template/block`. Outputs are only process status plus stderr guidance: exit `0` allows the read or ignores irrelevant tools, exit `1` rejects malformed hook input or malformed state, and exit `2` blocks a policy violation with a rendered message. The hook itself does not transition files, but its behavior transitions by loop phase: no relevant file or no active loop generally allows; methodology-analysis state enforces sanitized-report-only reads; finalize state blocks round contracts; normal state allows only current-round prompt/summary/contract plus narrow allowlist.
- gates_or_invariants: JSON must be valid, UTF-8 if `iconv` is available, include `tool_name`, stay at nesting depth `<=30`, and include `tool_input.file_path` for `Read` (`hooks/lib/loop-common.sh:91-165`, `hooks/loop-read-validator.sh:25-43`). Active-loop lookup is session-aware and deliberately does not fall back to unrelated sessions in validators (`hooks/loop-read-validator.sh:79-83`; `hooks/lib/loop-common.sh:332-425`). State parsing is strict and fail-closed for missing frontmatter, missing required fields, non-numeric rounds, non-numeric max iterations, or invalid `review_started` (`hooks/lib/loop-common.sh:533-600`). Round file patterns are limited to `round-[0-9]+-(summary|prompt|todos|contract).md` helpers (`hooks/lib/loop-common.sh:797-814`). The allowlist is exact-path and limited to `round-1-todos.md`, `round-2-todos.md`, `round-0-summary.md`, and `round-1-summary.md` in the active loop (`hooks/lib/loop-common.sh:819-837`). Final path equality is checked against `$ACTIVE_LOOP_DIR/$CLAUDE_FILENAME` after `canonicalize_path_prefix` on both paths (`hooks/loop-read-validator.sh:305-321`; helper in `hooks/lib/project-root.sh:72-96`).
- dependencies_and_callers: Called by Claude hook configuration `hooks/hooks.json:33-39`. It sources `hooks/lib/loop-common.sh` (`hooks/loop-read-validator.sh:15-17`), which in turn sources project-root resolution and template loading (`hooks/lib/loop-common.sh:171-243`). - edge_cases_or_failure_modes: If project root cannot be resolved, the todos branch exits allow because it cannot locate `.humanize/rlcr` (`hooks/loop-read-validator.sh:56-58`), and the methodology-analysis branch also allows (`hooks/loop-read-validator.sh:76-77`). If a session-scoped active loop is not found, normal restricted-file checks allow rather than block (`hooks/loop-read-validator.sh:197-202`), while todos reads with no active loop are blocked unless allowlisted, because `LOOP_DIR` is empty (`hooks/loop-read-validator.sh:56-63`). During methodology analysis, unresolved paths with `..` or unresolved symlinks are blocked to avoid traversal when canonicalization fails (`hooks/loop-read-validator.sh:91-129`). `is_in_humanize_loop_dir` is a raw substring check for `.humanize/rlcr/` (`hooks/lib/loop-common.sh:1252-1256`), so relative, symlinked, or unusual-case paths depend on later canonicalization or may bypass classification if they do not contain that literal segment. The final directory-path check uses `canonicalize_path_prefix`, which resolves the parent and preserves the basename (`hooks/lib/project-root.sh:55-96`); this matches symlinked project prefixes, but because it does not dereference the leaf, a leaf symlink behavior should be verified against the `Read` tool semantics. Goal-tracker validation only normalizes `/./` and repeated slash forms (`hooks/loop-read-validator.sh:227-244`; `hooks/lib/loop-common.sh:1027-1031`), not full realpath canonicalization. Malformed active state files block with exit `1`, which is intentionally fail-closed (`hooks/loop-read-validator.sh:211-215`).
