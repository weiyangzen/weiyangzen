- core_role: `scripts/portable-timeout.sh` is a shared Bash library that normalizes timeout execution across macOS and Linux. It is sourced by workflow scripts and hooks before running long or blocking external commands, especially `git`, `codex`, `gemini`, `claude`, and validation hooks. Its core abstraction is `run_with_timeout <seconds> <command> [args...]`, plus a detected global `TIMEOUT_IMPL`.
- algorithmic_behavior: The script first detects the best available timeout implementation in priority order: Homebrew/GNU `gtimeout`, GNU `timeout`, `python3`, `python`, then `none` ([portable-timeout.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/scripts/portable-timeout.sh:10)). GNU `timeout` is accepted only if `timeout --version` succeeds, which filters out non-GNU variants ([portable-timeout.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/scripts/portable-timeout.sh:13)). The detected value is computed once at source time into `TIMEOUT_IMPL` ([portable-timeout.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/scripts/portable-timeout.sh:29)). `run_with_timeout` shifts the first argument into `timeout_secs`, stores the remaining command and arguments in a Bash array, and dispatches through a `case` on `TIMEOUT_IMPL` ([portable-timeout.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/scripts/portable-timeout.sh:33)). For `gtimeout` and GNU `timeout`, it directly executes the command with the requested duration and returns the native exit code ([portable-timeout.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/scripts/portable-timeout.sh:38)). For Python fallback, it runs `subprocess.run(sys.argv[1:], timeout=<seconds>)`, exits with the child return code on completion, exits `124` on `subprocess.TimeoutExpired`, and exits `1` for other Python-level exceptions ([portable-timeout.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/scripts/portable-timeout.sh:47)). For `none`, it emits a warning to stderr and runs the command without a timeout ([portable-timeout.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/scripts/portable-timeout.sh:64)).
- inputs_outputs_state: Inputs are positional: timeout seconds as `$1`, then the command vector as `"$@"` after `shift` ([portable-timeout.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/scripts/portable-timeout.sh:34)). Outputs are the wrapped command’s stdout/stderr, a normalized timeout exit code of `124` for Python fallback, native `gtimeout`/GNU `timeout` behavior otherwise, or the raw command exit code when no timeout implementation exists. State is minimal and process-local: `TIMEOUT_IMPL` is set once when sourced and exported for child processes ([portable-timeout.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/scripts/portable-timeout.sh:73)). The function itself has no files, locks, caches, or persistent state transitions.
- gates_or_invariants: The implementation gate is the detection chain: prefer real timeout binaries, accept `timeout` only when it behaves like GNU coreutils, then fall back to Python, then degrade to no-timeout mode. The command vector is held in a Bash array, preserving argument boundaries and special characters for normal calls ([portable-timeout.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/scripts/portable-timeout.sh:36)).
