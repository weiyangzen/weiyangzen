### USE_REALPATH4EVERYTHING-HZ-167 `file` `prompt-template/codex/full-alignment-review.md`
- cursor: `[_]`
- core_role: This file is a Codex review prompt template for RLCR “Full Alignment Check” rounds. It is algorithmic because it defines the mandatory review contract that controls loop continuation, drift detection, completion, and stagnation stopping. The template is selected by `hooks/loop-codex-stop-hook.sh` when `CURRENT_ROUND % FULL_REVIEW_ROUND == FULL_REVIEW_ROUND - 1`, so default interval 5 means rounds 4, 9, 14, etc. See `prompt-template/codex/full-alignment-review.md:1-3` and `hooks/loop-codex-stop-hook.sh:977-987`.
- algorithmic_behavior: The template forces Codex through six ordered phases. First, it anchors review to the original plan and requires reading `@{{PLAN_FILE}}` before judging work (`prompt-template/codex/full-alignment-review.md:5-10`). Second, it embeds Claude’s round summary and commit history context (`prompt-template/codex/full-alignment-review.md:13-19`). Third, it audits the goal tracker against acceptance criteria, forgotten items, deferrals, and a completion summary (`prompt-template/codex/full-alignment-review.md:21-50`). Fourth, it classifies mainline drift and requires the exact `Mainline Progress Verdict: ADVANCED / STALLED / REGRESSED` line (`prompt-template/codex/full-alignment-review.md:52-66`). Fifth, it performs implementation review against claims, docs, gaps, and bugs (`prompt-template/codex/full-alignment-review.md:68-74`). Sixth, it checks stagnation over historical round files and decides whether the final marker should be `STOP` or `COMPLETE` (`prompt-template/codex/full-alignment-review.md:77-114`).
- inputs_outputs_state: Inputs are rendered placeholders supplied by the stop hook: `CURRENT_ROUND`, `PLAN_FILE`, `SUMMARY_CONTENT`, `GOAL_TRACKER_FILE`, `DOCS_PATH`, `GOAL_TRACKER_UPDATE_SECTION`, `COMMIT_HISTORY_SECTION`, `COMPLETED_ITERATIONS`, `LOOP_TIMESTAMP`, `PREV_ROUND`, `PREV_PREV_ROUND`, and `REVIEW_RESULT_FILE`; these are passed at `hooks/loop-codex-stop-hook.sh:1055-1069`. The rendered output is written to `round-N-review-prompt.md` (`hooks/loop-codex-stop-hook.sh:965-968`, `hooks/loop-codex-stop-hook.sh:1057-1069`). Codex is then expected to write its review to `@{{REVIEW_RESULT_FILE}}`, and the stop hook reads that review result, extracts the last non-empty marker line, and updates loop state or blocks/continues accordingly (`hooks/loop-codex-stop-hook.sh:1770-1827`). State affected downstream includes `mainline_stall_count`, `last_mainline_verdict`, and `drift_status`, updated after parsing the required verdict (`hooks/loop-codex-stop-hook.sh:1779-1824`, `hooks/loop-codex-stop-hook.sh:1952-1956`).
- gates_or_invariants: The strongest invariant is the required `Mainline Progress Verdict` line. The template explicitly says omission blocks the round (`prompt-template/codex/full-alignment-review.md:59-66`), and the hook enforces this by calling `block_missing_mainline_verdict` when the verdict cannot be extracted and the last line is not `STOP` (`hooks/loop-codex-stop-hook.sh:1785-1791`). `COMPLETE` is valid only when all original plan ACs are fully met and no deferrals remain (`prompt-template/codex/full-alignment-review.md:112-114`). `STOP` is reserved for stagnation/circuit-breaker outcomes and must be the last line when development is stagnating (`prompt-template/codex/full-alignment-review.md:94-103`, `hooks/loop-codex-stop-hook.sh:1914-1945`). The hook also uses strict last-line matching: only a standalone final `COMPLETE` or `STOP` marker is accepted, preventing accidental matches such as “CANNOT COMPLETE” (`hooks/loop-codex-stop-hook.sh:1773-1777`).
- dependencies_and_callers: Primary caller is `hooks/loop-codex-stop-hook.sh`, which loads this template through `load_and_render_safe` when `FULL_ALIGNMENT_CHECK=true` (`hooks/loop-codex-stop-hook.sh:1055-1069`). Rendering depends on `hooks/lib/template-loader.sh`, whose `render_template` implements single-pass `{{VAR}}` replacement and leaves missing placeholders intact (`hooks/lib/template-loader.sh:48-56`, `hooks/lib/template-loader.sh:69-135`). The full-review interval originates from setup CLI/state: `--full-review-round` validates numeric values and minimum `2` (`scripts/setup-rlcr-loop.sh:256-270`) and writes `full_review_round` into loop state (`scripts/setup-rlcr-loop.sh:880-889`). User-facing command docs describe the same configurable full-alignment behavior (`commands/start-rlcr-loop.md:147-154`). The template also coordinates with sibling templates: `codex/commit-history-section.md` can supply `{{COMMIT_HISTORY_SECTION}}`, `codex/goal-tracker-update-section.md` supplies `{{GOAL_TRACKER_UPDATE_SECTION}}`, and `claude/post-alignment-action-items.md` is appended to the next Claude prompt after a full alignment round (`hooks/loop-codex-stop-hook.sh:971-1026`, `hooks/loop-codex-stop-hook.sh:2114-2120`).
- edge_cases_or_failure_modes: If `FULL_REVIEW_ROUND` is absent or invalid at hook runtime, the hook warns and defaults to `5`; values below `2` are also rejected/defaulted in runtime logic (`hooks/loop-codex-stop-hook.sh:977-983`). Previous-round references are clamped to zero so early alignment rounds, especially interval `2`, do not produce negative filenames (`hooks/loop-codex-stop-hook.sh:989-995`). If the template is missing or renders empty, `load_and_render_safe` falls back to a much shorter alignment prompt, which preserves loop operation but weakens the detailed audit contract (`hooks/loop-codex-stop-hook.sh:1028-1040`, `hooks/lib/template-loader.sh:188-210`). If Codex writes `STOP` during a non-alignment round, the hook still honors it but labels it unexpected; during full alignment it is treated as the intended stagnation circuit breaker (`hooks/loop-codex-stop-hook.sh:1914-1938`). A wording mismatch exists inside the prompt: Part 6 says “see Part 4” for stagnation even though stagnation is Part 5 (`prompt-template/codex/full-alignment-review.md:77`, `prompt-template/codex/full-alignment-review.md:111`); this is a documentation/prompt precision issue, not a runtime failure.
- validation_or_tests: Setup robustness tests cover valid full-review intervals, minimum `2`, rejecting `1`, rejecting non-numeric values, and rejecting a missing value (`tests/robustness/test-setup-scripts-robustness.sh:770-859`). State parsing tests cover reading `full_review_round`, defaulting missing values to `5`, and accepting the minimum value `2` (`tests/robustness/test-state-file-robustness.sh:399-473`). Finalize/drift tests exercise the mandatory mainline verdict behavior: stalled verdicts trigger drift recovery state after repeated rounds, missing verdict blocks exit and preserves prior state, and repeated stalled/regressed rounds trigger circuit-breaker behavior (`tests/test-finalize-phase.sh:900-1015`). The hook’s runtime parsing and blocking logic is at `hooks/loop-codex-stop-hook.sh:1770-1827`.
