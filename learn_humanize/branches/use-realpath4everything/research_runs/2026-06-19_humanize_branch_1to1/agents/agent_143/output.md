### USE_REALPATH4EVERYTHING-HZ-143 `file` `prompt-template/block/unpushed-commits.md`
- cursor: `[_]`
- core_role: This is a stop-gate prompt template used by the RLCR loop when a worker/agent attempts to stop while `--push-every-round` is active and the current branch is ahead of its upstream. The template is short, but it encodes the human/agent-facing recovery contract for the “unpushed commits” gate: explain the block, name the branch, report the ahead count, and prescribe `git push origin {{CURRENT_BRANCH}}`.
- algorithmic_behavior: The file itself is declarative markdown with two placeholders. Its behavior is supplied by the template renderer and stop hook. `prompt-template/block/unpushed-commits.md:1` sets the block title, `prompt-template/block/unpushed-commits.md:3` interpolates `{{AHEAD_COUNT}}` and `{{CURRENT_BRANCH}}`, `prompt-template/block/unpushed-commits.md:5` ties the rule specifically to `--push-every-round`, and `prompt-template/block/unpushed-commits.md:8-10` emits the required recovery command. The stop hook computes `GIT_AHEAD` with `git -C "$PROJECT_ROOT" status -sb`, extracts the numeric ahead count, resolves the branch via `git rev-parse --abbrev-ref HEAD`, renders this template, and returns a JSON block decision instead of allowing exit (`hooks/loop-codex-stop-hook.sh:717-741`).
- inputs_outputs_state: Inputs are `AHEAD_COUNT`, derived from the `ahead N` token in short branch status, and `CURRENT_BRANCH`, derived from the current Git branch (`hooks/loop-codex-stop-hook.sh:719-731`). The rendered output is a markdown reason string embedded as the `reason` field in the stop hook’s JSON response, with a separate `systemMessage` summarizing the block (`hooks/loop-codex-stop-hook.sh:733-740`). The state transition is: stop requested -> worktree cleanliness gate passes -> `PUSH_EVERY_ROUND == true` -> branch ahead detected -> stop is blocked -> user/agent must push -> next stop attempt may proceed if no longer ahead. The setup path initializes `PUSH_EVERY_ROUND=false` by default, flips it to true only for `--push-every-round`, and writes `push_every_round: $PUSH_EVERY_ROUND` into loop state (`scripts/setup-rlcr-loop.sh:46`, `scripts/setup-rlcr-loop.sh:232-234`, `scripts/setup-rlcr-loop.sh:880-887`). State parsing reads `push_every_round` as `STATE_PUSH_EVERY_ROUND` and defaults it to false when absent (`hooks/lib/loop-common.sh:456`, `hooks/lib/loop-common.sh:590`).
- gates_or_invariants: The template is reachable only after earlier stop-hook Git cleanliness checks pass; dirty or uncommitted work blocks first (`hooks/loop-codex-stop-hook.sh:687-711`). The unpushed-commit gate is conditional on exact string truth, `[[ "$PUSH_EVERY_ROUND" == "true" ]]`, so false, empty, or malformed optional state does not activate the push requirement (`hooks/loop-codex-stop-hook.sh:717`). The gate invariant is that a loop configured to push every round cannot exit while the local branch is ahead of remote. Template rendering preserves missing placeholders rather than failing, because the renderer keeps unresolved `{{VAR}}` text if no variable is supplied (`hooks/lib/template-loader.sh:7-13`, `hooks/lib/template-loader.sh:115-121`). Rendering is single-pass to avoid placeholder injection through values (`hooks/lib/template-loader.sh:48-55`, `hooks/lib/template-loader.sh:69-128`).

- edge_cases_or_failure_modes: The ahead detection is based on `grep -o 'ahead [0-9]*'` over `git status -sb`; if Git status wording differs, no upstream is configured, the command times out, or the branch status does not include an `ahead N` marker, the template will not be triggered (`hooks/loop-codex-stop-hook.sh:719`). If multiple `ahead N` matches somehow appear, `AHEAD_COUNT=$(echo "$GIT_AHEAD" | grep -o '[0-9]*')` could produce multi-line numeric content, though normal Git short branch output has one ahead count (`hooks/loop-codex-stop-hook.sh:719-721`). If branch resolution fails at this specific point, `CURRENT_BRANCH` falls back to `unknown`, which would render `git push origin unknown` in the template fallback path and may be misleading (`hooks/loop-codex-stop-hook.sh:722`). If the template file is missing or empty, `load_and_render_safe` emits the inline fallback message instead of failing the stop hook (`hooks/loop-codex-stop-hook.sh:724-729`, `hooks/lib/template-loader.sh:194-210`). If a branch name contains shell-special characters, the template prints it unquoted in `git push origin {{CURRENT_BRANCH}}`; branch names normally avoid whitespace, but this is still a presentation-level sharp edge.
