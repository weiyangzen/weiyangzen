- algorithmic_behavior: The template imposes a deterministic plan-generation shape:

  - `Feasibility Hints and Suggestions` is marked non-prescriptive, so generated plans can include possible approaches without converting hints into hard requirements (`prompt-template/plan/gen-plan-template.md:46` to `prompt-template/plan/gen-plan-template.md:55`).

  - `Task Breakdown` requires every task to carry exactly one routing tag, either `coding` for Claude implementation or `analyze` for Codex via `/humanize:ask-codex` (`prompt-template/plan/gen-plan-template.md:69` to `prompt-template/plan/gen-plan-template.md:79`).

  - `Pending User Decisions` captures unresolved decision topics with Claude position, Codex position, tradeoff, and `PENDING` or final user decision state (`prompt-template/plan/gen-plan-template.md:91` to `prompt-template/plan/gen-plan-template.md:97`).
  - `Implementation Notes` prevents plan workflow labels from leaking into implementation code/comments (`prompt-template/plan/gen-plan-template.md:99` to `prompt-template/plan/gen-plan-template.md:104`).
