- core_role: This file is a shared Claude prompt-template fragment for RLCR Agent Teams mode. It defines the behavioral contract for a Claude “team leader” agent: coordinate and delegate only, never directly implement. The strongest role gate is at `prompt-template/claude/agent-teams-core.md:1-3`, where the template states that the leader’s only job is coordination/delegation and it must never write code, edit files, or implement work. It is core prompt infrastructure rather than application runtime code: it shapes how downstream Claude agents split work, assign ownership, prevent collisions, review outputs, and commit.
- algorithmic_behavior: The template encodes a delegation algorithm for parallel work. First, it requires decomposing work into independent, parallelizable units with clear scope and acceptance criteria (`prompt-template/claude/agent-teams-core.md:16`). Second, it requires creating agent teams using the Task tool with a `team_name` parameter (`prompt-template/claude/agent-teams-core.md:7`, `prompt-template/claude/agent-teams-core.md:27`). Third, it requires explicit coordination to prevent overlapping file changes (`prompt-template/claude/agent-teams-core.md:8`, `prompt-template/claude/agent-teams-core.md:18`). Fourth, it requires monitoring progress, resolving blockers, and reassigning stuck work via TaskList (`prompt-template/claude/agent-teams-core.md:9`, `prompt-template/claude/agent-teams-core.md:19`, `prompt-template/claude/agent-teams-core.md:28`). Fifth, it requires review of member output before considering tasks complete (`prompt-template/claude/agent-teams-core.md:20`). The file also includes a self-interruption rule: if the leader is about to implement directly, it must stop and delegate (`prompt-template/claude/agent-teams-core.md:12`, `prompt-template/claude/agent-teams-core.md:30`).
- inputs_outputs_state: Inputs are not function parameters; they are the prompt context surrounding this fragment. The leader consumes the implementation plan, relevant goals, file paths, prior work state, review feedback, and any project-level `CLAUDE.md`/MCP context that teammates load independently. The cold-start rule explicitly says teammates do not inherit conversation history, so the leader must include enough context in each spawned task: goals, paths, completed work, and exact acceptance criteria (`prompt-template/claude/agent-teams-core.md:17`). Outputs are delegated Task-tool work items, team assignments with strict file ownership, TaskList progress updates, sequenced dependencies through `blockedBy`, reviewed teammate results, teammate commits, and a final merged summary. The file itself has no persisted state, but it operates inside RLCR state controlled by `agent_teams`. Setup writes `agent_teams: $AGENT_TEAMS` into RLCR state (`scripts/setup-rlcr-loop.sh:884-898`), and common config defaults `agent_teams` from merged config or false (`hooks/lib/loop-common.sh:232-235`, `config/default_config.json:1-5`).
- gates_or_invariants: The primary invariant is “leader does not implement”; the template repeats this at role, stop-rule, and important sections (`prompt-template/claude/agent-teams-core.md:3`, `prompt-template/claude/agent-teams-core.md:12`, `prompt-template/claude/agent-teams-core.md:30`). The second invariant is file-ownership isolation: two teammates must not edit the same file in parallel because the system treats that as silent overwrite risk rather than a merge conflict (`prompt-template/claude/agent-teams-core.md:18`). Same-file work must be serialized with `blockedBy` (`prompt-template/claude/agent-teams-core.md:18`). The third invariant is cold-start completeness: spawned teammates need sufficient explicit context and should not be assumed to know the leader’s conversation (`prompt-template/claude/agent-teams-core.md:17`). The fourth invariant is quality gating: teammate output must be reviewed for correctness, conflict absence, and acceptance criteria before completion (`prompt-template/claude/agent-teams-core.md:20`). An external activation gate exists in setup: `--agent-teams` is accepted only when `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`; otherwise setup exits with an error (`scripts/setup-rlcr-loop.sh:385-396`).
