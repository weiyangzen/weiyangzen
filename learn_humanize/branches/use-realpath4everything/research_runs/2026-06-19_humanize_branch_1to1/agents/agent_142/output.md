- core_role: This is a blocking prompt-template fragment used by the RLCR/humanize hook system when an agent attempts to create, read, edit, write, or bash-modify `round-*-todos.md` files directly. The file is intentionally small and directive: it names the blocked artifact class at `prompt-template/block/todos-file-access.md:1`, forbids direct todo-file creation/access at `prompt-template/block/todos-file-access.md:3`, and redirects work to native task tools at `prompt-template/block/todos-file-access.md:5`.
- algorithmic_behavior: The template itself does not compute state, but it participates in the access-control algorithm as the emitted reason for a gate failure. `hooks/lib/loop-common.sh:839-848` defines `todos_blocked_message`, loads this template through `load_and_render_safe`, and falls back to an inline equivalent if the template is missing or empty. Hook validators call that function when path classification detects a todos round file outside the permitted active-loop allowlist, then terminate with a blocking exit code. The read validator blocks disallowed reads at `hooks/loop-read-validator.sh:56-63`; write blocks at `hooks/loop-write-validator.sh:57-64`; edit blocks at `hooks/loop-edit-validator.sh:40-47`; bash modification blocks at `hooks/loop-bash-validator.sh:543-550`.
- inputs_outputs_state: Inputs are indirect: a hook receives tool input with a `file_path` or shell `command`, normalizes it, classifies whether it matches `round-[0-9]+-todos.md`, finds the active loop directory using the session id, and checks whether the target is allowlisted. The template receives no variables in its current contents, even though `todos_blocked_message` accepts an action string such as `Read`, `Write`, `Edit`, or `Bash`. Output is a markdown block sent to stderr explaining the denial and instructing use of `TaskCreate`, `TaskUpdate`, and `TaskList`. State transition is external to the template: validator success path continues, but blocked path exits `2`, preventing the attempted operation. No repository files or loop state are mutated by this template.
- gates_or_invariants: The invariant enforced by this block is that agents should not use `round-*-todos.md` as the task-state surface. Direct file access is rejected unless it is one of the narrow legacy allowlist entries in the active loop directory. The allowlist is hard-coded in `hooks/lib/loop-common.sh:823-836` as `round-1-todos.md`, `round-2-todos.md`, `round-0-summary.md`, and `round-1-summary.md`; wrong directories are rejected even if the basename is allowlisted.
