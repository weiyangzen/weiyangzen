### USE_REALPATH4EVERYTHING-HZ-134 `file` `prompt-template/block/plan-file-modified.md`
- cursor: `[_]`
- core_role: This file is a blocking prompt template used when an active RLCR session detects that the original plan file no longer matches the loop's saved plan backup. The template is not executable logic itself, but it is part of the algorithmic control surface: it becomes the user-facing `reason` in a stop-hook JSON block response. The direct template content is at [plan-file-modified.md](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/prompt-template/block/plan-file-modified.md:1): title, modified-plan diagnosis, forbidden-change rule, cancel/update/restart recovery steps, and backup path.
- algorithmic_behavior: The template defines the remediation contract for a plan-integrity failure. It interpolates `{{PLAN_FILE}}` into the diagnosis and restart command, and `{{BACKUP_PATH}}` into the backup notice. In the stop hook, the relevant transition is: active loop found, review phase not started, backup exists, original plan exists, git-status checks pass or are skipped as applicable, then `diff -q "$FULL_PLAN_PATH" "$BACKUP_PLAN"` fails. At that point [loop-codex-stop-hook.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/hooks/loop-codex-stop-hook.sh:377) renders this template through `load_and_render_safe` and emits `{"decision":"block", ...}` with system message `Loop: Blocked - plan file modified`. The template therefore participates in an exit-block transition rather than a plan rewrite transition.
- inputs_outputs_state: Inputs are `PLAN_FILE`, the repo-relative path from loop state, and `BACKUP_PATH`, the saved backup at `.humanize/rlcr/<timestamp>/plan.md`. The immediate output is rendered Markdown. The caller then wraps that Markdown as the JSON `reason` field via `jq -n` at [loop-codex-stop-hook.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/hooks/loop-codex-stop-hook.sh:391). State read by the caller includes `STATE_PLAN_FILE`, `STATE_PLAN_TRACKED`, and `STATE_REVIEW_STARTED`, parsed earlier in the same stop hook. Setup creates the backup copy with `cp "$FULL_PLAN_PATH" "$LOOP_DIR/plan.md"` and persists `plan_file` / `plan_tracked` in state at [setup-rlcr-loop.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/scripts/setup-rlcr-loop.sh:829) and [setup-rlcr-loop.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/scripts/setup-rlcr-loop.sh:880).
- gates_or_invariants: The invariant expressed by the template is that plan files must not be modified during an active session; recovery requires canceling the active loop, changing the plan, then starting a fresh loop. The stop hook enforces related gates before this template can fire: skip the entire plan-integrity check once `review_started` is true, require `$LOOP_DIR/plan.md` backup to exist, require the original plan file to exist, reject uncommitted changes for tracked plan files, then compare original plan to backup. The template text says "session", while the fallback in the caller says "RLCR loop"; both encode the same invariant. The setup script also constrains plan paths before a loop starts: relative path only, no spaces, no shell metacharacters, no symlink file or symlink parent path, must exist, must be readable, must resolve inside the project, and must not be inside a submodule at [setup-rlcr-loop.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/scripts/setup-rlcr-loop.sh:455).
- dependencies_and_callers: The direct caller found is [loop-codex-stop-hook.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/hooks/loop-codex-stop-hook.sh:391), which passes `"PLAN_FILE=$PLAN_FILE"` and `"BACKUP_PATH=$BACKUP_PLAN"` to `load_and_render_safe`. Rendering is implemented in [template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/hooks/lib/template-loader.sh:56): placeholders use `{{VARIABLE_NAME}}`, rendering is single pass, missing variables remain as placeholders, and safe loading falls back when the template is missing, empty, or renders empty. The template path is also included in template-reference coverage because tests scan `load_template`, `load_and_render`, and `load_and_render_safe` calls against files under `prompt-template` in [test-template-references.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/use-realpath4everything/tests/test-template-references.sh:83).
- edge_cases_or_failure_modes: If the template file is missing or empty, `load_and_render_safe` uses the inline fallback message from the stop hook, so the block decision still works. If `PLAN_FILE` or `BACKUP_PATH` were not supplied, the template renderer would leave `{{PLAN_FILE}}` or `{{BACKUP_PATH}}` visible rather than failing, because missing variables are preserved. If the backup file is absent, the hook blocks earlier with "plan backup missing" and this template is not used. If the original plan was deleted, the hook blocks earlier with "plan file deleted" and includes the backup path directly. If the loop is already in review phase, plan integrity is intentionally skipped. For tracked plan files, uncommitted modifications are blocked before the content-diff branch, so this template mainly covers content mismatch cases not caught by that earlier tracked-file status check or gitignored plan-file changes.
