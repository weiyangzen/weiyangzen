### USE_REALPATH4EVERYTHING-HZ-149 `file` `prompt-template/block/wrong-round-number.md`
- cursor: `[_]`
- core_role: This is a reusable validator block-message template for the round-file protection algorithm. It does not perform validation itself; it renders the user-facing rejection when an agent tries to write or edit a `round-N-summary.md` or `round-N-contract.md` file whose `N` does not match the active loop round. The template title is defined at `prompt-template/block/wrong-round-number.md:1`, the mismatch explanation is at `prompt-template/block/wrong-round-number.md:3`, the corrected destination is emitted at `prompt-template/block/wrong-round-number.md:5`, and the invariant instruction is at `prompt-template/block/wrong-round-number.md:7`.

- algorithmic_behavior: The template is selected by hook validators after they extract a round number from the attempted path and compare it to `CURRENT_ROUND`. In `hooks/loop-write-validator.sh`, the write path derives `CLAUDE_FILENAME`, checks summary/contract files, computes `CLAUDE_ROUND`, computes `FILE_TYPE`, then renders this template if `CLAUDE_ROUND` is non-empty, differs from `CURRENT_ROUND`, and the file is not allowlisted; see `hooks/loop-write-validator.sh:295`, `hooks/loop-write-validator.sh:307`, `hooks/loop-write-validator.sh:308`, `hooks/loop-write-validator.sh:309`, `hooks/loop-write-validator.sh:311`, and `hooks/loop-write-validator.sh:318`. The edit path performs the same algorithm for summary/contract edits at `hooks/loop-edit-validator.sh:236`, `hooks/loop-edit-validator.sh:244`, `hooks/loop-edit-validator.sh:245`, `hooks/loop-edit-validator.sh:247`, and `hooks/loop-edit-validator.sh:254`. In both callers, the rendered message is written to stderr and the validator exits with status `2`, making this template part of a hard block rather than a warning; see `hooks/loop-write-validator.sh:323` through `hooks/loop-write-validator.sh:324` and `hooks/loop-edit-validator.sh:259` through `hooks/loop-edit-validator.sh:260`.

- inputs_outputs_state: Inputs are template variables passed through `load_and_render_safe`: `ACTION`, `CLAUDE_ROUND`, `FILE_TYPE`, `CURRENT_ROUND`, and `CORRECT_PATH` (`hooks/loop-write-validator.sh:318` through `hooks/loop-write-validator.sh:323`; `hooks/loop-edit-validator.sh:254` through `hooks/loop-edit-validator.sh:259`). `ACTION` distinguishes “write to” from “edit”; `CLAUDE_ROUND` is parsed from the attempted filename; `FILE_TYPE` is either `summary` or `contract`; `CURRENT_ROUND` is the authoritative state round; `CORRECT_PATH` is built as `$ACTIVE_LOOP_DIR/round-${CURRENT_ROUND}-${FILE_TYPE}.md`. Output is a Markdown block telling the agent which stale or future round file it tried to access and the exact correct path to use. Internal state transitions are absent in this file; the state transition occurs in the validator’s control flow: allowed operation continues, rejected operation renders this block and exits `2`.

- gates_or_invariants: The main invariant expressed by the template is that agents must not self-increment or otherwise choose the round number; `prompt-template/block/wrong-round-number.md:7` states this directly. The caller-side gate only applies when the target is a recognized round summary or contract file and a numeric round can be extracted. The shared helper `is_round_file_type` matches lowercase paths ending in `round-[0-9]+-${file_type}.md` (`hooks/lib/loop-common.sh:797` through `hooks/lib/loop-common.sh:802`). The helper `extract_round_number` lowercases the filename and extracts only `summary`, `prompt`, `todos`, or `contract` numeric round filenames (`hooks/lib/loop-common.sh:807` through `hooks/lib/loop-common.sh:814`). The wrong-round block is bypassed for allowlisted files through `is_allowlisted_file`, whose contract begins at `hooks/lib/loop-common.sh:816` through `hooks/lib/loop-common.sh:819`; caller checks include `! is_allowlisted_file "$FILE_PATH" "$ACTIVE_LOOP_DIR"` in both write and edit validators.

- dependencies_and_callers: Direct production callers are `hooks/loop-write-validator.sh` and `hooks/loop-edit-validator.sh`, each using `load_and_render_safe` so a fallback message exists if the template file is missing. The write validator fallback is defined at `hooks/loop-write-validator.sh:313` through `hooks/loop-write-validator.sh:317`, then replaced by this template when available at `hooks/loop-write-validator.sh:318`. The edit validator fallback is defined at `hooks/loop-edit-validator.sh:249` through `hooks/loop-edit-validator.sh:253`, then replaced at `hooks/loop-edit-validator.sh:254`. Functional dependencies include `hooks/lib/template-loader.sh` for variable rendering, `hooks/lib/loop-common.sh` for filename classification and round extraction, and active-loop state variables such as `CURRENT_ROUND`, `ACTIVE_LOOP_DIR`, and `TEMPLATE_DIR` established by the surrounding hook scripts.

- edge_cases_or_failure_modes: If the attempted filename does not match the expected `round-<digits>-<type>.md` pattern, `CLAUDE_ROUND` can be empty and this specific block is not used; other location/path gates may still reject the operation. Negative, non-numeric, or malformed round strings are not parsed by `extract_round_number`, because it only accepts `[0-9][0-9]*` (`hooks/lib/loop-common.sh:813`). Case variation is normalized by lowercasing before extraction (`hooks/lib/loop-common.sh:809` through `hooks/lib/loop-common.sh:810`). Allowlisted historical files bypass the wrong-round gate even if their number differs, which is intentional for special loop files. If the template is missing or unreadable, `load_and_render_safe` uses the caller fallback, so validation still blocks but the wording may be less specific than `prompt-template/block/wrong-round-number.md`.
