- algorithmic_behavior: The test sources `scripts/lib/config-loader.sh` after deriving `SCRIPT_DIR`, `PROJECT_ROOT`, and `CONFIG_LOADER` from the test script location (`tests/test-config-merge.sh:16-33`). It then creates isolated temporary project and user config layouts for seven scenarios, calls `load_merged_config "$PROJECT_ROOT" "$PROJECT_DIR"`, and reads individual keys with `get_config_value`.
  The implementation under test resolves paths in `scripts/lib/config-loader.sh`: defaults from `$plugin_root/config/default_config.json`, user config from `$XDG_CONFIG_HOME/humanize/config.json` or `$HOME/.config/humanize/config.json`, and project config from `$HUMANIZE_CONFIG` or `$project_root/.humanize/config.json` (`scripts/lib/config-loader.sh:77-88`). It prepares each layer as a JSON object or `{}` (`scripts/lib/config-loader.sh:24-61`), then merges layers with jq object multiplication in this order: empty base, defaults, user, project (`scripts/lib/config-loader.sh:113-132`). Each layer is run through `strip_nulls`, so null values are removed before merge and cannot erase lower-priority values (`scripts/lib/config-loader.sh:119-126`).
  The seven scenarios encode the expected state transitions of the merged config:
  Default-only starts with an empty project and missing user config, then expects defaults for `bitlesson_model`, `agent_teams`, and `gen_plan_mode` (`tests/test-config-merge.sh:39-64`). Project override writes `.humanize/config.json` with `bitlesson_model: opus`, expecting that key to change while default `agent_teams` remains (`tests/test-config-merge.sh:70-89`). Priority ordering writes the same key in user and project configs and expects the project value to win (`tests/test-config-merge.sh:95-109`). Additive merge writes distinct keys in user and project configs and expects both to survive (`tests/test-config-merge.sh:115-132`). Null stripping writes project `bitlesson_model: null` and expects default `haiku` to remain (`tests/test-config-merge.sh:138-150`). `HUMANIZE_CONFIG` supplies a custom project config path and expects that file to override the default `.humanize/config.json` path (`tests/test-config-merge.sh:156-173`). The final scenario checks all concrete layers contributing separate keys at once: user `gen_plan_mode`, project `agent_teams`, default `bitlesson_model` (`tests/test-config-merge.sh:179-198`).
