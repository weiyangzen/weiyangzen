The shared renderer resolves templates under `prompt-template` via `hooks/lib/template-loader.sh:24-31`, loads file content with `load_template` at `hooks/lib/template-loader.sh:33-46`, and safely falls back if a template is missing or empty at `hooks/lib/template-loader.sh:185-211`.

Runtime behavior is: a caller computes gate context, selects a block template by stable relative path such as `block/wrong-directory-path.md`, passes placeholder assignments, and emits the rendered Markdown either to stderr with a nonzero/blocked hook exit or inside JSON `decision: block`. Rendering is single-pass through `awk`; variable names are documented as uppercase letters, numbers, and underscores, missing variables remain literal, and injected values containing `{{OTHER}}` are not rescanned (`hooks/lib/template-loader.sh:7-14`, `hooks/lib/template-loader.sh:48-56`, `hooks/lib/template-loader.sh:69-128`). `validate_template_dir` requires `block`, `codex`, `claude`, and `plan` subdirectories, so this folder is a required template namespace rather than optional text (`hooks/lib/template-loader.sh:213-238`).
