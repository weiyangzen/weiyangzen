- core_role: `hooks/lib` is the shared shell-library layer for the RLCR hook system. It centralizes hook input validation, state-file parsing, active-loop discovery, path canonicalization, guarded loop-state transitions, template rendering, background-task parking, and methodology-analysis phase transitions. The directory is flat and contains five files: `project-root.sh`, `loop-common.sh`, `loop-bg-tasks.sh`, `template-loader.sh`, and `methodology-analysis.sh`. The branch’s relevant core change is concentrated in path resolution: `project-root.sh` resolves project roots through `CLAUDE_PROJECT_DIR` or git and canonicalizes with `realpath`, while validators and cancel authorization compare user paths through `canonicalize_path_prefix` so symlinked ancestors are normalized without dereferencing the leaf.
- algorithmic_behavior: `project-root.sh` defines the path-resolution algorithm used across hooks and scripts. `resolve_project_root` first uses `CLAUDE_PROJECT_DIR`, falls back to `git rev-parse --show-toplevel`, returns non-zero if neither exists, then emits a canonical path via `canonicalize_path` (`hooks/lib/project-root.sh:41`). `canonicalize_path` realpaths the full path when possible, canonicalizes parent plus basename for non-existent targets, falls back to Python `os.path.realpath`, then raw input (`hooks/lib/project-root.sh:113`). `canonicalize_path_prefix` deliberately canonicalizes only the parent directory and reattaches the basename (`hooks/lib/project-root.sh:72`), which is the important safety distinction for authorization checks against paths like `state.md`, `finalize-state.md`, and loop round files. `loop-common.sh` is the aggregator and policy library. It validates hook JSON, rejects null bytes, optionally rejects invalid UTF-8 through `iconv`, requires `tool_name`, and extracts `tool_input` (`hooks/lib/loop-common.sh:91`). It computes config-backed defaults after sourcing `scripts/lib/config-loader.sh`, preserving shell options around that source operation (`hooks/lib/loop-common.sh:180`). It discovers loop state with `resolve_active_state_file`, preferring `methodology-analysis-state.md`, then `finalize-state.md`, then `state.md` (`hooks/lib/loop-common.sh:268`). `find_active_loop` selects the newest active loop, optionally scoped by `session_id`, with zombie-loop protection and a stop-hook-only `bg-pending.marker` fallback (`hooks/lib/loop-common.sh:332`). It parses state frontmatter in tolerant and strict modes; strict mode requires YAML separators, `current_round`, `max_iterations`, `review_started`, `base_branch`, numeric iteration fields, and boolean `review_started` (`hooks/lib/loop-common.sh:533`). It extracts Codex review issues by scanning only the last 50 log lines for `[P?]` markers near the start of a line and writes an audit result file (`hooks/lib/loop-common.sh:740`). It validates cancel transitions with `is_cancel_authorized`: requires `.cancel-requested`, rejects substitution, newlines, shell chaining, hidden `$`, mixed quote styles, extra arguments, wrong source/destination, and source symlinks; it canonicalizes the loop dir and compares source/destination using `canonicalize_path_prefix` (`hooks/lib/loop-common.sh:1051`). It also protects `.humanize/` from `git add`, including force and broad-scope forms (`hooks/lib/loop-common.sh:1280`), blocks direct hook execution, detects Bash file modifications, preserves goal-tracker immutable sections, previews Edit operations with Perl, and ends loops by renaming state files (`hooks/lib/loop-common.sh:1540`). `loop-bg-tasks.sh` owns stop-hook short-circuiting while asynchronous Claude background work is still running. It expands leading `~`, extracts transcript paths, converts loop-dir local timestamps into UTC ISO timestamps for transcript filtering, derives Claude task-output directories, checks liveness with output files plus `lsof`, computes pending task ids as launched minus completed, and runs four stop-hook guards in order: ambiguous caller, cross-session parked loop, pending-task early exit, same-session stale-marker cleanup (`hooks/lib/loop-bg-tasks.sh:29`, `hooks/lib/loop-bg-tasks.sh:65`, `hooks/lib/loop-bg-tasks.sh:193`, `hooks/lib/loop-bg-tasks.sh:311`). `template-loader.sh` computes the prompt-template directory, loads files, performs single-pass `{{VAR}}` substitution through environment-prefixed `awk`, preserves unknown placeholders, and provides safe fallback rendering when templates are missing or empty (`hooks/lib/template-loader.sh:26`, `hooks/lib/template-loader.sh:56`, `hooks/lib/template-loader.sh:188`). `methodology-analysis.sh` moves a terminal RLCR loop into a methodology-analysis phase, renaming active state to `methodology-analysis-state.md`, recording the original exit reason, creating a done placeholder, rendering the analysis prompt, and later requiring non-empty done and report artifacts before renaming back to `complete-state.md`, `stop-state.md`, or `maxiter-state.md` (`hooks/lib/methodology-analysis.sh:38`, `hooks/lib/methodology-analysis.sh:114`).
