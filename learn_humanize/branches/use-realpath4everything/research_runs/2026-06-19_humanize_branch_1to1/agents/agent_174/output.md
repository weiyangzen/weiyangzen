- core_role: Shared monitor utility library for Humanize shell commands. It is sourced by `scripts/humanize.sh` at startup when present (`scripts/humanize.sh:6-10`) and provides reusable algorithms for monitor display, session discovery, state-file classification, lightweight YAML frontmatter extraction, timestamp/string formatting, and goal-tracker summary parsing. It is also a declared dependency of `scripts/lib/monitor-skill.sh` for skill invocation metadata parsing and timestamp display (`scripts/lib/monitor-skill.sh:8-10`).
- algorithmic_behavior: The file is a sourced Bash library, not an executable workflow entrypoint (`scripts/lib/monitor-common.sh:1-6`). Its algorithms are small, state-reading helpers:
  - ANSI color helpers return escape strings for consistent monitor rendering (`scripts/lib/monitor-common.sh:12-23`).
  - `monitor_get_file_size` tries GNU `stat -c%s`, then BSD/macOS `stat -f%z`, then returns `0` if neither works or the file is absent (`scripts/lib/monitor-common.sh:29-35`).
  - `monitor_find_latest_session` scans one directory level under a loop directory, accepts only timestamp-named directories matching `YYYY-MM-DD_HH-MM-SS`, and selects the lexicographically greatest timestamp name as latest (`scripts/lib/monitor-common.sh:37-63`). The caller `_find_latest_session` in `scripts/humanize.sh` delegates to it (`scripts/humanize.sh:281` per usage search).
  - Terminal helpers clear/setup a fixed-header scroll region and restore it with ANSI scroll reset (`scripts/lib/monitor-common.sh:69-92`).
  - The signal-handling section documents expected cleanup semantics but intentionally does not implement a generic trap because local monitor variables need function-local scope (`scripts/lib/monitor-common.sh:98-122`).
  - `monitor_get_status_color` maps known loop statuses to colors: active/methodology-analysis green, completed cyan, failed/error/timeout/max-iterations red, cancelled yellow, unknown dim, and other states yellow (`scripts/lib/monitor-common.sh:128-141`).
  - `monitor_find_state_file` classifies a session by state file. It first treats `methodology-analysis-state.md` as `methodology-analysis`, then `state.md` as `active`, then picks the first `*-state.md` found and derives status by removing `-state.md`; absent/invalid session yields `|unknown` (`scripts/lib/monitor-common.sh:147-192`). `scripts/humanize.sh` wraps this in `_find_state_file` and uses the returned `path|status` pair to drive monitor status rendering (`scripts/humanize.sh:377-429`).
  - `monitor_get_yaml_value` extracts frontmatter between `---` delimiters, finds a `key:` line, strips the key and optional spaces, and removes double quotes (`scripts/lib/monitor-common.sh:198-213`). Skill monitor uses it for `tool`, `status`, `model`, `effort`, `duration`, and `started_at` from `.humanize/skill/*/metadata.md` (`scripts/lib/monitor-skill.sh:53-63`, `scripts/lib/monitor-skill.sh:119-142`, `scripts/lib/monitor-skill.sh:246-280`).
  - `monitor_format_timestamp` handles blank/`N/A` as `N/A`, otherwise converts ISO-ish `T` and trailing `Z` into display text ending in `UTC` (`scripts/lib/monitor-common.sh:219-232`).
  - `monitor_truncate_string` keeps either the start or end of long strings, adding `...`, based on a direction argument (`scripts/lib/monitor-common.sh:234-256`).
  - `parse_goal_tracker_issue_counts` counts markdown table data rows in Blocking/Queued Side Issues sections, falls back to legacy Open Issues as blocking when no blocking/queued rows exist, and emits `blocking|queued|open` (`scripts/lib/monitor-common.sh:262-295`).
  - `parse_goal_tracker` summarizes a goal tracker into `total_acs|completed_acs|active_tasks|completed_tasks|deferred_tasks|open_issues|goal_summary`, using section-bounded `sed` windows and table-row counting (`scripts/lib/monitor-common.sh:297-383`).
- inputs_outputs_state: Inputs are filesystem paths and status strings. Outputs are stdout values intended for caller parsing, especially pipe-delimited records. No persistent state is written by this file. `monitor_find_latest_session` reads a loop directory and emits one path or blank (`scripts/lib/monitor-common.sh:40-63`). `monitor_find_state_file` reads session state files and emits `state_file_path|loop_status` (`scripts/lib/monitor-common.sh:154-192`). Goal tracker parsers read markdown files and emit numeric summaries with defaults for missing files (`scripts/lib/monitor-common.sh:265-304`). Terminal helpers mutate terminal state only via `clear`, `printf`, and `tput` (`scripts/lib/monitor-common.sh:71-92`). In `scripts/humanize.sh`, the monitor splits `monitor_find_state_file` output, parses state fields, then combines state, goal tracker, and git data into the fixed status bar (`scripts/humanize.sh:424-470`).
- gates_or_invariants: Session discovery requires basename timestamp format exactly matching `^[0-9]{4}-[0-9]{2}-[0-9]{2}_[0-9]{2}-[0-9]{2}-[0-9]{2}$` (`scripts/lib/monitor-common.sh:54-56`). State-file priority is invariant: active methodology analysis, active normal state, stopped `*-state.md`, then unknown (`scripts/lib/monitor-common.sh:162-190`). Markdown table counts subtract two header/separator rows and clamp empty sections to zero (`scripts/lib/monitor-common.sh:272-279`, `scripts/lib/monitor-common.sh:307-315`).
