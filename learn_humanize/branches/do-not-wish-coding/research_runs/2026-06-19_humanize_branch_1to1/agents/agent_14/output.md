- cursor: `[_]`

- algorithmic_behavior: The hook reads JSON from stdin, validates it with `validate_hook_input`, rejects deeply nested JSON above depth 30, exits successfully for non-`Read` tools, and requires `tool_input.file_path` for Read. It lowercases the path for pattern matching, extracts `session_id`, blocks non-allowlisted `round-*-todos.md` reads, and then only continues special validation for `round-*-summary.md` or `round-*-prompt.md`. For round files, it finds the active loop under `.humanize/rlcr`, strict-parses the active state file, extracts the current round, validates that the path is inside a humanize loop dir, validates the requested round equals `current_round` unless allowlisted, and finally validates exact directory equality to `$ACTIVE_LOOP_DIR/$CLAUDE_FILENAME`.
- inputs_outputs_state: Inputs are Claude hook JSON (`tool_name`, `tool_input.file_path`, optional `session_id`) plus environment `CLAUDE_PROJECT_DIR`, `PROJECT_ROOT`, `LOOP_BASE_DIR`, or `LOOP_DIR`. It reads active state via `resolve_active_state_file` and `parse_state_file_strict`, but does not mutate files. Exit `0` allows the read, exit `1` rejects malformed hook/state input, and exit `2` blocks with rendered markdown on stderr. It transitions attempted reads into allow/block decisions without changing RLCR state.
- gates_or_invariants: Only active-loop current round prompt/summary files are readable by default. Todos files are blocked unless allowlisted. Historical allowlist is shared through `is_allowlisted_file`, currently `round-1-todos.md`, `round-2-todos.md`, `round-0-summary.md`, and `round-1-summary.md`. Malformed active state fails closed with “Malformed state file” before current-round comparisons.
- dependencies_and_callers: Registered as the PreToolUse Read hook in `hooks/hooks.json` lines 33-39. It sources `hooks/lib/loop-common.sh` for JSON validation, session filtering, round-file detection, allowlist, active-loop lookup, state parsing, and template rendering. It renders `prompt-template/block/todos-file-access.md`, `wrong-file-location.md`, `wrong-round-file.md`, and `wrong-directory-path.md`.
- edge_cases_or_failure_modes: If there is no active loop, round prompt/summary validation exits `0`, allowing normal reads. If the read target is a round file outside `.humanize/rlcr`, it blocks with wrong-location guidance. If a filename has no extractable round number, it exits `0`. If a session ID is supplied, active-loop discovery is session-aware through `find_active_loop`; otherwise only the newest loop directory is considered, protecting against stale zombie loops. Exact path comparison means equivalent symlinked or differently normalized paths may be blocked unless they match the canonical active-loop string used by the hook.
- validation_or_tests: `tests/robustness/test-hook-input-robustness.sh` exercises valid JSON, malformed JSON, missing `tool_name`, missing `file_path`, Unicode, deep nesting, and non-UTF8 input. `tests/test-allowlist-validators.sh` lines 234-283 tests read allowlist behavior and blocks non-allowlisted todos/summary files. `tests/test-finalize-phase.sh` has a read-validator check for finalize/current-round summary access. `tests/robustness/test-hook-system-robustness.sh` covers concurrent hook invocations and path behavior.
