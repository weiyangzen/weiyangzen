State transition: `state.md session_id empty + pending signal` -> `state.md session_id recorded + signal removed`.
- gates_or_invariants: `matches_setup_command_signature` accepts only exact quoted or unquoted setup-script invocation forms followed by end-of-string or POSIX whitespace (`hooks/loop-post-bash-hook.sh:43`, `hooks/loop-post-bash-hook.sh:67`). - dependencies_and_callers: `scripts/setup-rlcr-loop.sh` initializes `state.md` with `session_id:` (`scripts/setup-rlcr-loop.sh:885`) and writes `.humanize/.pending-session-id` with the state path and resolved setup-script path (`scripts/setup-rlcr-loop.sh:914`). If the signal points to a missing state file, it is removed as stale (`hooks/loop-post-bash-hook.sh:156`). If the command does not match, the signal is preserved (`hooks/loop-post-bash-hook.sh:162`). If `session_id` is absent, the signal is left for a later attempt (`hooks/loop-post-bash-hook.sh:175`).
