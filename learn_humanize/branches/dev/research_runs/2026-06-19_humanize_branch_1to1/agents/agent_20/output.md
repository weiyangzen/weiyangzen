### DEV-HZ-200 `file` `scripts/lib/config-loader.sh`
- cursor: `[_]`
- core_role: Runtime configuration merger and accessor for Humanize features. It is the canonical implementation for layered config semantics used by hooks, commands, and BitLesson routing.
- algorithmic_behavior: `load_merged_config <plugin_root> <project_root>` resolves four layers: empty base, required plugin default config, optional user config from `${XDG_CONFIG_HOME:-$HOME/.config}/humanize/config.json`, and optional project config from `$HUMANIZE_CONFIG` or `<project>/.humanize/config.json` (`scripts/lib/config-loader.sh:63`, `scripts/lib/config-loader.sh:77`, `scripts/lib/config-loader.sh:84`). It validates each layer as a JSON object, recursively strips nulls, then merges with jq object multiplication so later layers override earlier layers (`scripts/lib/config-loader.sh:113`). `get_config_value` extracts top-level scalar-like values (`scripts/lib/config-loader.sh:139`).
- inputs_outputs_state: Inputs are plugin root, project root, environment variables `XDG_CONFIG_HOME`, `HOME`, and `HUMANIZE_CONFIG`, plus JSON files. Output is merged JSON on stdout or a scalar value from `get_config_value`. It creates a temp directory during merge and cleans it via trap; no persistent state is mutated.
- gates_or_invariants: `jq` is mandatory (`scripts/lib/config-loader.sh:17`). Default config is required and malformed/missing default config is fatal (`scripts/lib/config-loader.sh:109`). Optional user/project configs may be missing; malformed optional configs warn and collapse to `{}` (`scripts/lib/config-loader.sh:40`, `scripts/lib/config-loader.sh:52`). All layers must be JSON objects, not arrays or scalars.
- dependencies_and_callers: Called by `hooks/lib/loop-common.sh` for Codex/BitLesson defaults (`hooks/lib/loop-common.sh:199`), `scripts/bitlesson-select.sh` for provider routing config (`scripts/bitlesson-select.sh:113`), and documented as required behavior for `commands/gen-plan.md` and `commands/refine-plan.md`. Defaults live in `config/default_config.json`.
- edge_cases_or_failure_modes: Missing arguments are fatal. Non-string scalar config values are converted with `tostring`; missing/null values return empty. Null stripping means a higher layer cannot erase a lower-layer value by setting it to null. The implementation intentionally exits inside `_config_loader_prepare_layer` for required-layer failures because nested function returns under `set -e` are unreliable in this context (`scripts/lib/config-loader.sh:43`).
- validation_or_tests: `tests/test-config-merge.sh` covers precedence, additive merge, null stripping, and `HUMANIZE_CONFIG` override. `tests/test-config-error-handling.sh` covers missing default fatal behavior and optional malformed/missing config fallback.
