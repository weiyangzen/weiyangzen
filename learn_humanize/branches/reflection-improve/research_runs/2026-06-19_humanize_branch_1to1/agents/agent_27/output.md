- gates_or_invariants: The key invariant is that the working branch must remain constant across all RLCR rounds (`agents/plan-compliance-checker.md:46`). The prompt also enforces output cardinality: exactly one verdict, never zero or multiple (`agents/plan-compliance-checker.md:56-60`). Both relevance and branch-switch detection lean toward pass when uncertain to avoid false positives (`agents/plan-compliance-checker.md:58-61`).

- core_role: Runtime polling helper for PR-loop review orchestration. - algorithmic_behavior: The script parses `<pr_number> --after <timestamp> --bots <bot1,bot2>` and validates required arguments plus numeric PR number (`scripts/poll-pr-reviews.sh:19-102`). It requires `gh` and `jq` (`scripts/poll-pr-reviews.sh:104-116`). Bot names are mapped to GitHub author logins, with `codex` mapped specially to `chatgpt-codex-connector[bot]` and other names mapped to `<bot>[bot]` (`scripts/poll-pr-reviews.sh:164-187`). It fetches three endpoints with retry-tolerant `fetch_with_retry`: issue comments, pull review comments, and PR reviews (`scripts/poll-pr-reviews.sh:207-294`). It combines those arrays, filters by `created_at >= --after` and author regex, then emits JSON containing `comments`, `bots_responded`, `has_new_comments`, and `comment_count` (`scripts/poll-pr-reviews.sh:296-326`).
- inputs_outputs_state: Inputs are CLI arguments, current GitHub repo context from `gh repo view`, PR metadata from `gh pr view`, and API responses from `gh api`. Outputs are JSON to stdout and warnings/errors to stderr. It creates a temporary directory via `mktemp -d` and removes it on exit (`scripts/poll-pr-reviews.sh:193-199`). It does not mutate repository files or PR-loop state directly; state transitions happen in the caller.
- gates_or_invariants: It fail-fast rejects missing arguments, unknown options, nonnumeric PR numbers, missing `gh`, and missing `jq` (`scripts/poll-pr-reviews.sh:23-116`). It preserves JSON output even when individual GitHub API fetches fail: retry failures increment `API_FAILURES`, output `[]`, and return success so downstream polling can continue (`scripts/poll-pr-reviews.sh:200-235`). Empty PR review bodies are converted to a placeholder like `[Review state: APPROVED]`, preserving approval-only reviews as actionable comments (`scripts/poll-pr-reviews.sh:276-289`).
- dependencies_and_callers: The main caller is `hooks/pr-loop-stop-hook.sh`, which sets `POLL_SCRIPT="$PLUGIN_ROOT/scripts/poll-pr-reviews.sh"` (`hooks/pr-loop-stop-hook.sh:812`) and invokes it inside the poll loop for bots still waiting (`hooks/pr-loop-stop-hook.sh:961-966`). The stop hook maps returned author names back to configured bot names, marks per-bot response state, deduplicates comments by ID, handles Codex thumbs-up reactions separately, and writes `round-N-pr-comment.md` for local Codex validation (`hooks/pr-loop-stop-hook.sh:968-1062`, `hooks/pr-loop-stop-hook.sh:1182-1231`). Bot mapping helpers are supplied by `hooks/lib/loop-common.sh` as noted by the stop hook (`hooks/pr-loop-stop-hook.sh:35-37`).
- edge_cases_or_failure_modes: Fork PRs are handled by resolving the parent repo when the PR number is absent from the current repo (`scripts/poll-pr-reviews.sh:132-150`). API outages do not terminate the poll script, which may produce a valid empty result while warnings precede JSON on stderr (`scripts/poll-pr-reviews.sh:221-235`).
