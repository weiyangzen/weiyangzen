### REFLECTION_IMPROVE-HZ-058 `file` `scripts/portable-timeout.sh`
- cursor: `[_]`
- core_role: Provides a cross-platform timeout abstraction for macOS/Linux so hooks and setup scripts can bound external commands consistently (`scripts/portable-timeout.sh:1-7`).
- algorithmic_behavior: `detect_timeout_impl` selects the first available backend in this order: Homebrew `gtimeout`, GNU `timeout`, `python3`, `python`, then `none` (`scripts/portable-timeout.sh:10-27`). `run_with_timeout` dispatches to the detected backend, preserving command arguments as a Bash array for native timeout tools and using Python `subprocess.run(..., timeout=...)` for fallback (`scripts/portable-timeout.sh:33-62`).
- inputs_outputs_state: Inputs are `timeout_seconds` plus command/arguments (`scripts/portable-timeout.sh:31-36`). Outputs are the child command stdout/stderr and exit status, `124` for Python timeout expiration to match GNU timeout, `1` for Python wrapper exceptions, or a warning plus unbounded command execution when no timeout implementation exists (`scripts/portable-timeout.sh:56-68`). State is the sourced global/exported `TIMEOUT_IMPL`, computed once at source time (`scripts/portable-timeout.sh:29`, `scripts/portable-timeout.sh:73-76`).
- gates_or_invariants: GNU `timeout` is accepted only if `timeout --version` succeeds, which avoids assuming BSD-compatible behavior (`scripts/portable-timeout.sh:13-19`). The wrapper does not validate that the timeout value is numeric or positive; callers are responsible for sane values.
- dependencies_and_callers: Used broadly by runtime-critical scripts and hooks, including `scripts/setup-rlcr-loop.sh`, `scripts/setup-pr-loop.sh`, `scripts/check-bot-reactions.sh`, `scripts/ask-codex.sh`, `hooks/loop-codex-stop-hook.sh`, `hooks/pr-loop-stop-hook.sh`, and validators. Examples include PR reaction polling through `run_with_timeout "$GH_TIMEOUT" gh api ...` (`scripts/check-bot-reactions.sh:273-275`) and Codex execution in stop hooks.
- edge_cases_or_failure_modes: If no backend exists, commands run without a deadline and only emit a warning (`scripts/portable-timeout.sh:64-68`). The Python fallback times out the launched subprocess but may not emulate all GNU `timeout` process-group behavior. Empty commands, zero timeout, missing commands, and commands that handle signals are left to backend-specific behavior.
- validation_or_tests: `tests/robustness/test-timeout-robustness.sh` validates backend detection, accepted backend names, quick command success, timeout exit `124`, argument preservation, command exit-code preservation, pipelines via `sh -c`, zero-timeout variance, large output, rapid cycles, missing command behavior, special characters, signal handling, fallback detection, subshells, and `TIMEOUT_IMPL` export (`tests/robustness/test-timeout-robustness.sh:31-272`).
- skip_candidate: `no`
