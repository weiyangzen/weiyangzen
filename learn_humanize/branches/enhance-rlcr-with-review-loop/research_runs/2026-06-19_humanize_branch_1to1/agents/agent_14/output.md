### ENHANCE_RLCR_WITH_REVIEW_LOOP-HZ-074 `file` `hooks/lib/template-loader.sh`
- cursor: `[_]`
- core_role: Shared template resolution and rendering library for RLCR and PR-loop hooks. It is a core hook dependency because validators and stop hooks use it to produce user-facing block prompts, review prompts, fallback messages, and loop instructions without duplicating template text.
- algorithmic_behavior: `get_template_dir` maps a hooks/lib path to the plugin’s `prompt-template` directory (`hooks/lib/template-loader.sh:24-31`). `load_template` reads a named template or emits a warning plus empty content for missing files (`hooks/lib/template-loader.sh:33-48`). `render_template` accepts `VAR=value` assignments, exports them with `TMPL_VAR_` prefixes, and runs an awk single-pass scanner over `{{VAR}}` placeholders (`hooks/lib/template-loader.sh:50-132`). `load_and_render` composes load plus render (`hooks/lib/template-loader.sh:134-147`), `append_template` concatenates a second template (`hooks/lib/template-loader.sh:149-161`), and `load_and_render_safe` renders a fallback when the template is missing or produces empty output (`hooks/lib/template-loader.sh:167-203`). `validate_template_dir` checks for required `block`, `codex`, and `claude` subdirectories (`hooks/lib/template-loader.sh:205-222`).
- inputs_outputs_state: Inputs are a template directory, template name, template content, and variable assignments. Output is rendered markdown on stdout; warnings/errors go to stderr. The library does not mutate persistent state. During rendering, assignment names become environment keys with a `TMPL_VAR_` prefix (`hooks/lib/template-loader.sh:62-69`), and awk builds a lookup table from those variables (`hooks/lib/template-loader.sh:74-82`).
- gates_or_invariants: Rendering is intentionally single-pass: substituted values are written directly to output and not rescanned (`hooks/lib/template-loader.sh:54-57`, `hooks/lib/template-loader.sh:116-125`). Missing variables are preserved as their original placeholders (`hooks/lib/template-loader.sh:119-122`). Unterminated `{{` sequences are treated literally (`hooks/lib/template-loader.sh:102-108`). The loader’s directory validation is structural, not exhaustive: it requires only top-level template categories (`hooks/lib/template-loader.sh:211-219`).
- dependencies_and_callers: `hooks/lib/loop-common.sh` sources this library and initializes `TEMPLATE_DIR` (`hooks/lib/loop-common.sh:138-147`). Runtime callers include `hooks/loop-read-validator.sh`, `hooks/loop-write-validator.sh`, `hooks/loop-edit-validator.sh`, `hooks/loop-bash-validator.sh`, `hooks/loop-codex-stop-hook.sh`, `hooks/pr-loop-stop-hook.sh`, and `scripts/setup-pr-loop.sh`. Example: wrong-round read blocks are rendered through `load_and_render_safe` in `hooks/loop-read-validator.sh:138-150`.
- edge_cases_or_failure_modes: Newlines and shell metacharacters in values are passed through environment variables and awk output, not shell-evaluated by the loader. However, very large values/templates depend on shell environment and awk limits. Duplicate variable assignments resolve to the last environment value. Malformed placeholder names are still parsed as whatever text appears between braces; only provided variable names are replaced. `load_template` returns empty for missing files, so callers that use unsafe `load_and_render` may silently omit a message.
- validation_or_tests: `tests/test-template-loader.sh` validates path resolution, existing/missing template loading, single and multiple substitutions, multiline rendering, unreplaced variables, safe fallbacks, directory validation, and shell metacharacter literal handling (`tests/test-template-loader.sh:41-244`, `tests/test-template-loader.sh:280-430`). Stress tests cover large values/templates, regex characters, newlines, empty values, and placeholder-injection prevention (`tests/robustness/test-template-stress-robustness.sh:82-132`, `tests/robustness/test-template-stress-robustness.sh:195-228`). Template reference tests require critical validators to use `load_and_render_safe` (`tests/test-template-references.sh:172-200`).
