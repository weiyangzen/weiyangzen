- core_role: Runtime cancellation algorithm for active RLCR loops. It converts the active loop state into a preserved cancellation terminal state and emits machine-readable first-line status tokens consumed by `commands/cancel-rlcr-loop.md`.
- algorithmic_behavior: Parses only `--force` and help at `scripts/cancel-rlcr-loop.sh:24-63`. - inputs_outputs_state: Inputs are optional `--force`, `CLAUDE_PROJECT_DIR`, loop directory contents, and YAML-like `current_round`/`max_iterations` lines. Output is stdout status plus filesystem state transition: active `state.md` or `finalize-state.md` becomes `cancel-state.md`, with `.cancel-requested` left as an authorization signal. Missing round fields degrade to `?` at `scripts/cancel-rlcr-loop.sh:105-111`.
- gates_or_invariants: Only the newest RLCR loop directory is considered. A loop is active only if that newest directory contains `state.md` or `finalize-state.md`. The broader hook invariant is that state-file moves are normally blocked; `hooks/lib/loop-common.sh:545-704` authorizes only a signal-backed `mv` from `state.md` or `finalize-state.md` to `cancel-state.md`, rejecting command substitution, shell chaining, wrong paths, extra args, mixed quotes, remaining variables, and symlink sources. `hooks/loop-bash-validator.sh:139-155` uses that exception.
- dependencies_and_callers: Called by `commands/cancel-rlcr-loop.md:13-15` and optionally again with `--force` at `commands/cancel-rlcr-loop.md:31`. Coupled to active loop detection in `hooks/lib/loop-common.sh:150-173`, Finalize Phase creation/completion in `hooks/loop-codex-stop-hook.sh:716-720` and `hooks/loop-codex-stop-hook.sh:996-1003`, monitor phase detection in `scripts/lib/monitor-common.sh`, and state-file protection hooks.
- edge_cases_or_failure_modes: If `.humanize/rlcr` has older active loops but the newest directory is inactive, this script still picks newest then reports `NO_ACTIVE_LOOP`; that matches the repo’s zombie-loop avoidance policy in `hooks/lib/loop-common.sh:150-153`. `ls | sort -r` is timestamp-name dependent and path-newline fragile. `touch` or `mv` failures exit under `set -euo pipefail` without custom recovery. A simultaneous hook read may observe `.cancel-requested` before the rename, but the signal exists specifically to permit the rename operation.
