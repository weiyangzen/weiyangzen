### ENHANCE_RLCR_WITH_REVIEW_LOOP-HZ-039 `file` `scripts/portable-timeout.sh`
- cursor: `[_]`
- core_role: Runtime portability wrapper that gives setup scripts and hooks a uniform bounded-execution primitive across macOS and Linux.
- algorithmic_behavior: `detect_timeout_impl` selects the first available implementation in priority order: Homebrew `gtimeout`, GNU `timeout`, `python3`, `python`, then `none` (`scripts/portable-timeout.sh:10`). `run_with_timeout` dispatches to that implementation, preserving command argv as an array for shell safety (`scripts/portable-timeout.sh:33`). The Python fallback uses `subprocess.run(sys.argv[1:], timeout=timeout_secs)` and maps `TimeoutExpired` to exit `124` to match GNU timeout (`scripts/portable-timeout.sh:49`). With no implementation, it emits a warning and runs unbounded (`scripts/portable-timeout.sh:64`).
- inputs_outputs_state: Input is timeout seconds plus command/args. Output is the child command’s stdout/stderr and exit status, or `124` on timeout for real timeout backends. State is global `TIMEOUT_IMPL`, computed at source time and exported for child visibility (`scripts/portable-timeout.sh:29`, `scripts/portable-timeout.sh:76`).
- gates_or_invariants: Valid GNU `timeout` is accepted only if `timeout --version` succeeds, avoiding non-GNU/BSD ambiguity (`scripts/portable-timeout.sh:13`). Command arguments are never eval’d. Exit codes are propagated for normal child failures and Python exceptions become exit `1`.
- dependencies_and_callers: Used by GitHub and git-heavy workflow scripts and hooks, including `scripts/setup-rlcr-loop.sh`, `scripts/setup-pr-loop.sh`, `scripts/check-pr-reviewer-status.sh`, `scripts/check-bot-reactions.sh`, `hooks/loop-codex-stop-hook.sh`, `hooks/pr-loop-stop-hook.sh`, and `hooks/loop-plan-file-validator.sh`. For example, setup validates git repo/HEAD through `run_with_timeout` (`scripts/setup-rlcr-loop.sh:257`), PR status fetching wraps `gh` calls (`scripts/check-pr-reviewer-status.sh:114`), and Codex review/exec are bounded by stop hooks (`hooks/loop-codex-stop-hook.sh:943`).
- edge_cases_or_failure_modes: `none` mode silently loses enforcement except for a warning. The Python fallback does not explicitly kill process groups, so commands spawning children may leave descendants depending on platform behavior. `timeout_secs` is interpolated into the Python snippet, so callers should pass numeric values; current callers use controlled timeout variables. In this sandbox, only `python3` was detected, and invoking it produced macOS developer-tool cache warnings because `/tmp` writes are restricted.
