- core_role: Edit-path gate for Claude tool usage in both RLCR and PR review loops. It is registered as the `Edit` `PreToolUse` hook in `hooks/hooks.json:24-30`, so it runs before every Claude Edit operation and decides whether an attempted file edit is allowed, blocked as user-correctable, or blocked as unsafe.
- algorithmic_behavior: The script reads hook JSON from stdin, extracts `.tool_name` and `.tool_input.file_path` via `jq`, and exits immediately unless the tool is `Edit` (`hooks/loop-edit-validator.sh:24-31`). It lowercases the path for classifier checks (`hooks/loop-edit-validator.sh:31-32`). It first blocks generated RLCR todos unless the target is allowlisted in the current active loop (`hooks/loop-edit-validator.sh:38-45`), blocks prompt files unconditionally (`hooks/loop-edit-validator.sh:48-51`), then applies PR-loop protections before RLCR-loop protections. For PR loop paths, it blocks `state.md`, read-only PR artifacts, and validates `round-N-pr-resolve.md` against the current PR round (`hooks/loop-edit-validator.sh:57-76`). For RLCR paths, it locates the active loop, parses `state.md` or `finalize-state.md`, blocks state-file edits, blocks `plan.md` backups, blocks `goal-tracker.md` after Round 0, and validates summary filename round numbers (`hooks/loop-edit-validator.sh:83-183`).
- inputs_outputs_state: Input is the Claude hook payload JSON. Primary input fields are `tool_name` and `tool_input.file_path` (`hooks/loop-edit-validator.sh:24-31`). It reads repository state from `.humanize/rlcr/<timestamp>/state.md` or `finalize-state.md` using `find_active_loop` and `parse_state_file_strict` (`hooks/loop-edit-validator.sh:91-110`). Outputs are process exit codes and stderr guidance: exit `0` allows, exit `2` blocks with a templated correction message, and exit `1` fails closed on malformed state (`hooks/loop-edit-validator.sh:106-108`). It does not mutate state.
- gates_or_invariants: The hook enforces that generated prompts are read-only, state files are hook-managed, `goal-tracker.md` becomes read-only to Claude after Round 0, PR state/read-only files are protected, and summary or PR-resolve edits must target the current round unless explicitly allowlisted. It uses `finalize-state.md` preferentially over `state.md` when present (`hooks/loop-edit-validator.sh:99-103`), and checks `finalize-state.md` before generic `state.md` because the generic state matcher also matches finalize-state names (`hooks/loop-edit-validator.sh:113-124`). Malformed state is a safety stop, not a permissive fallback (`hooks/loop-edit-validator.sh:105-109`).
- dependencies_and_callers: Depends on `hooks/lib/loop-common.sh` (`hooks/loop-edit-validator.sh:16-18`), especially `find_active_loop`, `parse_state_file_strict`, `is_round_file_type`, `is_allowlisted_file`, PR path classifiers, `is_goal_tracker_path`, `extract_round_number`, and templated message helpers. `find_active_loop` chooses only the newest timestamped loop containing `state.md` or `finalize-state.md`, preventing stale loops from reviving (`hooks/lib/loop-common.sh:150-173`). Strict state parsing requires frontmatter separators plus `current_round`, `max_iterations`, `review_started`, and `base_branch`, and validates numeric/boolean fields (`hooks/lib/loop-common.sh:251-323`). The allowlist is limited to `round-1-todos.md`, `round-2-todos.md`, `round-0-summary.md`, and `round-1-summary.md` in the active loop only (`hooks/lib/loop-common.sh:414-435`). Goal tracker blocking renders `block/goal-tracker-modification.md` with current round and summary path (`hooks/lib/loop-common.sh:1092-1104`).
- edge_cases_or_failure_modes: Non-Edit hook payloads pass through (`hooks/loop-edit-validator.sh:27-29`). If no active RLCR loop exists, RLCR path edits pass because there is no authoritative current round to enforce (`hooks/loop-edit-validator.sh:91-97`). A malformed active state blocks with exit `1`, which is stricter than ordinary policy block exit `2` (`hooks/loop-edit-validator.sh:105-108`). Todos have a narrow active-loop allowlist; same filenames in old or wrong directories are blocked through `is_allowlisted_file` equality against `active_loop_dir` (`hooks/lib/loop-common.sh:421-434`). Summary round extraction has two sed fallbacks for timestamped and non-timestamped RLCR paths (`hooks/loop-edit-validator.sh:155-160`). PR loop `pr-resolve` edits exit early after round validation, avoiding later RLCR logic (`hooks/loop-edit-validator.sh:72-76`).
- validation_or_tests: `tests/test-allowlist-validators.sh:191-227` directly verifies the Edit validator allows allowlisted active-loop files and blocks disallowed todos. `tests/test-finalize-phase.sh:369-378` verifies Edit blocks `finalize-state.md`. `tests/test-pr-loop-hooks.sh:1499-1527` verifies wrong-round PR resolve edits are blocked. `tests/test-template-references.sh:176-190` includes `loop-edit-validator.sh` among critical validators checked for safe template rendering usage.
