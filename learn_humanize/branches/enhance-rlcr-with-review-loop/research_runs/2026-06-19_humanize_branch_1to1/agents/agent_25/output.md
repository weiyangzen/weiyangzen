- core_role: PreToolUse validator for Bash commands during active RLCR and PR loops. It is a guard layer preventing shell commands from mutating loop-managed state, prompts, summaries, todo files, and `.humanize` control files. It is registered as the Bash hook in `hooks/hooks.json:47`.
- algorithmic_behavior: Reads hook JSON from stdin, validates JSON and required fields, exits early for non-Bash tools, lowercases the command, locates active RLCR and PR loop dirs, then applies ordered blocking checks. The RLCR branch starts by choosing `state.md` or `finalize-state.md` as the authoritative state file and strictly parsing it before making decisions (`hooks/loop-bash-validator.sh:74-86`). State read includes `.humanize/rlcr/<timestamp>/{state.md,finalize-state.md}` and `.humanize/pr-loop/<timestamp>/state.md`; the script does not mutate state itself.
- gates_or_invariants: Requires valid JSON, no excessive nesting above depth 30, and `tool_input.command` for Bash (`hooks/loop-bash-validator.sh:26-45`). Fails closed on malformed state (`hooks/loop-bash-validator.sh:81-85`). Blocks `git push` unless `push_every_round: true` (`hooks/loop-bash-validator.sh:93-105`). Blocks `git add` targeting `.humanize` through direct, broad, force, or all-scope add patterns (`hooks/loop-bash-validator.sh:115-117`; helper in `hooks/lib/loop-common.sh:924-1019`). Blocks state/finalize-state modification as both destination and source, including `mv`/`cp` from protected state files (`hooks/loop-bash-validator.sh:139-156`, `158-328`). Blocks plan backup writes, goal-tracker writes, prompt writes, summary writes, and most todos writes during RLCR (`hooks/loop-bash-validator.sh:336-393`). - gates_or_invariants: Encodes the review-phase transition contract: run `codex review --base {{BASE_BRANCH}}`, scan output for `[P0-9]`, return remediation prompt when issues exist, and transition to Finalize Phase when none are found (`prompt-template/codex/code-review-phase.md:12-18`).
