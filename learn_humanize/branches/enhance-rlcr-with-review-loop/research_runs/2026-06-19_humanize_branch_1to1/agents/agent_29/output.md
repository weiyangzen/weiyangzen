### ENHANCE_RLCR_WITH_REVIEW_LOOP-HZ-029 `file` `hooks/loop-read-validator.sh`
- cursor: `[_]`
- core_role: PreToolUse `Read` gate for RLCR loop files, registered in `hooks/hooks.json:33-39`. It prevents Claude from reading stale or misplaced `round-N-summary.md`, `round-N-prompt.md`, and non-allowlisted `round-N-todos.md` files while permitting ordinary reads and PR-loop reads.
- algorithmic_behavior: Reads hook JSON from stdin, validates JSON structure and nesting via shared helpers, ignores non-`Read` tools, requires `tool_input.file_path`, lowercases only for file-type matching, and exits early for non-round summary/prompt/todos paths (`hooks/loop-read-validator.sh:25-71`). For todos files it finds the active loop and blocks unless the file is in the explicit active-loop allowlist (`hooks/loop-read-validator.sh:55-63`). For summary/prompt reads it resolves active loop state, parses `state.md` or `finalize-state.md`, extracts the requested round, validates location, validates current round, then validates exact active-loop directory path (`hooks/loop-read-validator.sh:80-168`).
- inputs_outputs_state: Inputs are stdin JSON, `CLAUDE_PROJECT_DIR` or current directory, optional `PROJECT_ROOT`, `LOOP_BASE_DIR`, `LOOP_DIR`, the active `.humanize/rlcr/<timestamp>/` directory, `state.md` or `finalize-state.md`, and prompt templates. Outputs are exit `0` allow, exit `1` malformed hook input or malformed state fail-closed, and exit `2` block with rendered stderr guidance. It mutates no state; it reads loop state to enforce the current round.
- gates_or_invariants: Invariant is that summary/prompt reads must target the active loop directory and current round unless explicitly allowlisted. Todos reads are generally blocked with “use native TodoWrite” semantics except active-loop allowlisted files. `parse_state_file_strict` requires YAML frontmatter plus `current_round`, `max_iterations`, `review_started`, and `base_branch` (`hooks/lib/loop-common.sh:251-323`). Exact path equality is enforced at the end (`hooks/loop-read-validator.sh:157-168`).
- dependencies_and_callers: Depends on `hooks/lib/loop-common.sh` for `validate_hook_input`, `is_deeply_nested`, `require_tool_input_field`, `find_active_loop`, `parse_state_file_strict`, `is_round_file_type`, `extract_round_number`, `is_allowlisted_file`, and template-backed block messages (`hooks/lib/loop-common.sh:58-132`, `155-173`, `251-323`, `395-445`, `706-710`). Uses `jq`, `basename`, `grep`, `sed`, and template loader functions. Related tests cover allowlist behavior in `tests/test-allowlist-validators.sh:231-284`, finalize-state parsing in `tests/test-finalize-phase.sh:839-850`, hook input robustness in `tests/robustness/test-hook-input-robustness.sh:35-115` and `177-235`, and template reference safety in `tests/test-template-references.sh:56-64` and `176-200`.
- edge_cases_or_failure_modes: No active loop allows reads, which avoids blocking normal use (`hooks/loop-read-validator.sh:84-86`). Malformed active state blocks safely. Files matching round patterns outside `.humanize/rlcr/` are blocked with “wrong location.” Same filename in an old loop directory is blocked by exact active-loop path comparison. Round filenames that do not yield a numeric round are ignored. Path comparison is string-based rather than canonicalized, so alternate spellings, symlinks, or normalization differences can block otherwise equivalent paths.

It mutates no files; it controls the stop-hook transition from “exit requested” to “blocked pending refactor.”
- gates_or_invariants: The rendered contract requires code files to be split below the line limit with behavior strictly unchanged, documentation files to be split into logical sections with cross-references and navigation updates, and changes to be committed before retrying (`prompt-template/block/large-files.md:13-25`).
