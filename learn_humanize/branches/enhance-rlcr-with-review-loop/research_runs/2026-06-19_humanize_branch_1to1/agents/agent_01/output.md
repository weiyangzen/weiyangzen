### ENHANCE_RLCR_WITH_REVIEW_LOOP-HZ-031 `file` `hooks/pr-loop-stop-hook.sh`
- cursor: `[_]`
- core_role: This file is the Stop hook for the PR loop state machine. It intercepts Claude exit attempts while `.humanize/pr-loop/<timestamp>/state.md` is active, ensures the user has completed the required PR-loop handoff, polls remote bot reviews, runs local Codex to classify per-bot review status, and either ends the loop or blocks exit with next-round feedback.
- algorithmic_behavior: The hook reads stdin hook input but primarily derives behavior from `CLAUDE_PROJECT_DIR`, `.humanize/pr-loop`, `state.md`, Git/GitHub/Codex state, and helper scripts. It exits cleanly when no active loop exists (`hooks/pr-loop-stop-hook.sh:64-75`). It parses YAML frontmatter into PR fields and bot arrays, supporting both YAML-list and legacy inline formats (`hooks/pr-loop-stop-hook.sh:87-174`). It resolves PR base/lookup repositories for fork PRs before any PR-state check (`hooks/pr-loop-stop-hook.sh:196-237`). It moves state to terminal state files when PR is merged or closed (`hooks/pr-loop-stop-hook.sh:244-255`).
- inputs_outputs_state: Required state inputs include `current_round`, `max_iterations`, `pr_number`, `start_branch`, `configured_bots`, `active_bots`, Codex config, polling config, `started_at`, `startup_case`, `latest_commit_sha`, `latest_commit_at`, `last_trigger_at`, and `trigger_comment_id`. Required round input is `round-N-pr-resolve.md`; missing resolution summary returns a JSON block decision (`hooks/pr-loop-stop-hook.sh:262-277`). Outputs include generated `round-N-pr-comment.md`, `round-N-codex-prompt.md`, `round-N-pr-check.md`, `round-N-pr-feedback.md`, updated `state.md`, and terminal moved states such as `approve-state.md`, `closed-state.md`, `merged-state.md`, `maxiter-state.md`, and `usage-limit-state.md`.
- gates_or_invariants: The first gates are resolution summary presence, clean non-`.humanize` git status, and no unpushed commits (`hooks/pr-loop-stop-hook.sh:258-362`). It detects force pushes by checking whether `latest_commit_sha` remains an ancestor of current HEAD; on rewrite it updates commit fields, clears trigger fields, and blocks for a fresh bot mention (`hooks/pr-loop-stop-hook.sh:364-424`). It enforces max iterations (`hooks/pr-loop-stop-hook.sh:426-435`). Trigger requirement logic depends on round, startup case, and newly detected commits: later rounds always require trigger; cases 4/5 require trigger; cases 1/2/3 do not unless new commits were detected (`hooks/pr-loop-stop-hook.sh:673-708`). If trigger is required and absent, it blocks (`hooks/pr-loop-stop-hook.sh:710-736`).
- dependencies_and_callers: Called as a Claude Stop hook from `hooks/hooks.json:52-64`. It sources `hooks/lib/loop-common.sh` for active loop discovery, bot-author mapping, YAML list construction, mention strings, and goal tracker updates (`hooks/pr-loop-stop-hook.sh:51-53`). It sources `scripts/portable-timeout.sh` for bounded `git` and `gh` calls (`hooks/pr-loop-stop-hook.sh:55-62`). It calls `scripts/check-bot-reactions.sh` for Codex thumbs-up and Claude eyes checks, `scripts/poll-pr-reviews.sh` for new comments, and `scripts/check-pr-reviewer-status.sh` for updated startup-case calculation. It calls `codex exec` with configured model/effort and project root (`hooks/pr-loop-stop-hook.sh:1309-1320`).
- edge_cases_or_failure_modes: Fork PRs are resolved through current and parent repos. Missing PR number or invalid `current_round` causes quiet exit with warning rather than corrupting state (`hooks/pr-loop-stop-hook.sh:185-194`). Missing `git` or no git repo skips git gates. Failed `git status`, uncommitted work, and unpushed commits block. If PR head SHA cannot be fetched for branch comparison, the hook fails closed by assuming unpushed commits (`hooks/pr-loop-stop-hook.sh:331-344`). New commits clear old trigger state to avoid stale bot review reuse (`hooks/pr-loop-stop-hook.sh:598-629`). Claude eyes checks are only run after a confirmed trigger exists, avoiding stale `trigger_comment_id` use (`hooks/pr-loop-stop-hook.sh:738-789`). Poll timeouts are anchored to trigger time, except startup cases 2/3 use current time to avoid instant timeout on historical comments (`hooks/pr-loop-stop-hook.sh:858-868`).
- validation_or_tests: The assigned test file and companion suites cover the hook’s YAML parsing, trigger filtering, timeout anchoring, pagination, wrong-round protection, approval-only reviews, fixture-backed fetch/poll, and monitor display. Specific direct coverage appears in `tests/test-pr-loop-hooks.sh:271-466` for gatekeeper units, `tests/test-pr-loop-hooks.sh:606-754` for trigger/timeout/round naming logic, `tests/test-pr-loop-hooks.sh:766-1148` for mocked end-to-end stop-hook paths, and `tests/test-pr-loop-hooks.sh:1160-1419` for approval-only and fixture-backed polling.
