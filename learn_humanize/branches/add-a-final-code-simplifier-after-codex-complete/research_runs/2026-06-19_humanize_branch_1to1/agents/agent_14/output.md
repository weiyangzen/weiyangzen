- algorithmic_behavior: Reads hook input JSON from stdin, extracts `transcript_path`, scans the transcript JSONL line by line, and keeps only the most recent non-empty TodoWrite `todos` array. It supports three transcript shapes: Claude assistant message content blocks, alternative `type: message` blocks, and direct `type: tool_use` entries. Any todo whose `status` is not exactly `completed` is treated as incomplete.
- inputs_outputs_state: Input is JSON stdin containing optional `transcript_path` (`hooks/check-todos-from-transcript.py:90-104`). Secondary input is a UTF-8 JSONL transcript (`:30-39`). Output is process exit state plus stdout/stderr: exit `0` means no blocking todos, exit `1` prints `INCOMPLETE_TODOS` and each incomplete todo, exit `2` prints `PARSE_ERROR` to stderr for invalid hook JSON (`:7-13`, `:92-127`). It does not write repo state.
- gates_or_invariants: Missing transcript path, missing transcript file, empty transcript, no TodoWrite calls, or empty latest todos all allow exit (`:25-26`, `:99-111`). Invalid JSONL transcript lines are ignored, so malformed history cannot block by itself (`:36-39`). The invariant is strict status matching: only `completed` clears a todo; `pending`, `in_progress`, missing status, and empty status block (`:113-119`).
- dependencies_and_callers: Uses only Python stdlib `json`, `sys`, and `pathlib.Path` (`:15-17`). Called from `hooks/loop-codex-stop-hook.sh:258-306`, which passes the full hook input, converts exit `2` into a blocking parse-error JSON response, and converts exit `1` into a `block/incomplete-todos.md` rendered block without running Codex review.
- edge_cases_or_failure_modes: If the latest TodoWrite has an empty `todos` list, an earlier incomplete list is effectively forgotten because `latest_todos` only updates when `todos` is truthy (`:61-64`, `:74-76`, `:83-85`). Transcript file open errors other than nonexistence are not caught. Todo entries that are not dict-like would fail at `todo.get(...)` (`:115-118`). Because invalid JSONL lines are skipped, a corrupted final TodoWrite line may leave an older todo state authoritative.
- validation_or_tests: `tests/test-todo-checker.sh` covers invalid stdin, missing transcript path, nonexistent transcript, all-complete, incomplete, in-progress, invalid JSONL lines, latest TodoWrite precedence, direct `tool_use`, `type: message`, missing status, empty content, and Unicode content (`tests/test-todo-checker.sh:51-293`). `tests/test-finalize-phase.sh:515-540` verifies the stop hook blocks finalize exit on incomplete todos and does not invoke Codex in that path.
