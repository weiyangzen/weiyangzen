- cursor: `[_]`
- core_role: Block-message template used by the RLCR stop hook when Claude attempts to stop while its latest TodoWrite state still contains unfinished todos. It is not executable code, but it defines a user-facing gate contract for the stop/review transition: finish todos before Codex review or loop finalization proceeds.
- algorithmic_behavior: The template contains one placeholder, `{{INCOMPLETE_LIST}}`, embedded under “You are trying to stop, but you still have incomplete todos” (`prompt-template/block/incomplete-todos.md:1-5`). Its instructions require completing remaining todos, marking each with TodoWrite, and only then writing the summary and stopping (`prompt-template/block/incomplete-todos.md:7-10`). The stop hook invokes the todo checker before expensive Codex review, and when the checker returns incomplete status, it renders this template with the incomplete list and returns a JSON block decision (`hooks/loop-codex-stop-hook.sh:252-306`). In that flow, the template is the explanatory payload for the hard stop gate.
- inputs_outputs_state: Template input is `INCOMPLETE_LIST`, passed from `loop-codex-stop-hook.sh` after stripping the first marker line from `check-todos-from-transcript.py` output (`hooks/loop-codex-stop-hook.sh:284-295`). The incomplete list itself is derived from the latest TodoWrite tool call in the transcript JSONL, where every todo whose status is not `completed` is formatted as `  - [status] content` (`hooks/check-todos-from-transcript.py:20-87`, `hooks/check-todos-from-transcript.py:113-126`). Output is rendered markdown used as the `reason` field in a Claude hook response with `"decision": "block"` and system message “Loop: Blocked - incomplete todos detected, please finish all tasks first” (`hooks/loop-codex-stop-hook.sh:297-304`). The template does not update state; it prevents state transition to Codex review/stop by causing the stop hook to block.
- gates_or_invariants: The gate runs before Codex review and is explicitly described as a quick check to avoid expensive review when Claude has incomplete todos (`hooks/loop-codex-stop-hook.sh:252-256`). If the checker reports parse error, the stop hook blocks with a parse-error reason instead of using this template (`hooks/loop-codex-stop-hook.sh:265-281`). If the checker reports incomplete todos, the hook blocks immediately and exits without continuing to git checks or Codex review (`hooks/loop-codex-stop-hook.sh:284-306`). The template’s invariant is that stopping is not allowed until all TodoWrite items are in the completed state, and its final sentence forbids proceeding to Codex review until that is true (`prompt-template/block/incomplete-todos.md:7-12`). Rendering is single-pass through `load_and_render_safe`; missing variables remain placeholders, while missing template files fall back to an inline “Incomplete Todos” message (`hooks/lib/template-loader.sh:50-132`, `hooks/lib/template-loader.sh:170-203`, `hooks/loop-codex-stop-hook.sh:289-295`).
- dependencies_and_callers: The direct caller is `hooks/loop-codex-stop-hook.sh`, registered as the Claude `Stop` hook in `hooks/hooks.json:52-59`. The stop hook depends on `hooks/check-todos-from-transcript.py` for transcript parsing and todo status classification (`hooks/loop-codex-stop-hook.sh:258-263`, `hooks/check-todos-from-transcript.py:1-14`). Rendering depends on `hooks/lib/template-loader.sh` via the shared loop common setup. The template is indirectly part of the RLCR state machine because the stop hook decides whether the loop may advance toward review, finalization, max-iteration handling, or completion after the todo gate.
- edge_cases_or_failure_modes: If `check-todos-from-transcript.py` is absent, the stop hook skips this gate entirely because the check is wrapped in `if [[ -f "$TODO_CHECKER" ]]` (`hooks/loop-codex-stop-hook.sh:258-260`). If hook input has no `transcript_path`, or the transcript path does not exist, the checker exits success and the template is not used (`hooks/check-todos-from-transcript.py:99-111`). If the transcript has no TodoWrite calls, the checker also allows stopping (`hooks/check-todos-from-transcript.py:107-111`). Malformed JSONL lines in the transcript are ignored, not fatal (`hooks/check-todos-from-transcript.py:36-40`). The checker only considers the latest non-empty TodoWrite payload it sees, so a later TodoWrite call can override earlier incomplete tasks (`hooks/check-todos-from-transcript.py:28-87`).
