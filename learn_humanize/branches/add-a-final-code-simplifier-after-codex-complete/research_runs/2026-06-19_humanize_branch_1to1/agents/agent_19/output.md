The hook is registered in `hooks/hooks.json:4-10` as the `UserPromptSubmit` command.
- algorithmic_behavior: Reads stdin hook JSON into `INPUT` but does not inspect it, then discovers the newest active loop under `.humanize/rlcr` via `find_active_loop` from `hooks/lib/loop-common.sh` (`hooks/loop-plan-file-validator.sh:26-35`, `hooks/lib/loop-common.sh:58-81`). If no active loop exists, it silently allows. If `finalized-state.md` exists, it parses that instead of `state.md`, allowing the Finalize Phase to keep the same guard behavior (`hooks/loop-plan-file-validator.sh:38-45`). It maps parsed state fields into local names, then runs three gates in order: schema, branch, plan tracking (`hooks/loop-plan-file-validator.sh:47-51`, `hooks/loop-plan-file-validator.sh:52-226`).
- inputs_outputs_state: Inputs are `CLAUDE_PROJECT_DIR` or `pwd` for `PROJECT_ROOT`, `.humanize/rlcr/<loop>/state.md` or `finalized-state.md`, state fields `plan_tracked`, `plan_file`, `start_branch`, and git repository state (`hooks/loop-plan-file-validator.sh:13-31`, `hooks/lib/loop-common.sh:98-138`). Outputs are either no output with exit 0, or JSON with `"decision": "block"` and a rendered/inline reason; even blocks exit 0 because UserPromptSubmit uses JSON decision output (`hooks/loop-plan-file-validator.sh:68-73`, `hooks/loop-plan-file-validator.sh:96-111`, `hooks/loop-plan-file-validator.sh:128-143`, `hooks/loop-plan-file-validator.sh:172-188`, `hooks/loop-plan-file-validator.sh:218-224`). It writes no files and advances no checklist cursor itself.
- gates_or_invariants: Requires `plan_tracked` and `start_branch`; missing values are treated as old schema and blocked through `block/schema-outdated.md` fallback rendering (`hooks/loop-plan-file-validator.sh:56-86`). Requires current branch from `git rev-parse --abbrev-ref HEAD` to equal the recorded `start_branch`, blocking branch switches during the loop (`hooks/loop-plan-file-validator.sh:92-112`). If `plan_tracked=true`, the plan must still be tracked and have empty `git status --porcelain` output (`hooks/loop-plan-file-validator.sh:120-189`). Write/Edit/Bash validators emit it to stderr and exit with a blocking status for PreToolUse (`hooks/loop-write-validator.sh:145-148`, `hooks/loop-edit-validator.sh:114-117`, `hooks/loop-bash-validator.sh:285-293`).
