- gates_or_invariants: The registry enforces that write/edit/read/bash access is mediated before tool execution and that loop completion is mediated at stop time. It preserves separation between prompt submission validation, tool-use protection, and terminal stop-loop transitions.
- dependencies_and_callers: Depends on the hook scripts named in the JSON. If a hook target is missing or not executable, the lifecycle gate becomes unavailable or fails at runtime. If both stop hooks are installed, each stop path must safely no-op when its own loop state is absent; both inspected stop hooks do this by checking for active loop directories before acting.
- validation_or_tests: `hooks/hooks.json` parsed successfully as JSON with `python3 -m json.tool`. Related validators are covered by many test files in `tests/`, including hook-system, hook-input, plan-file, PR-loop, and bash-validator suites.
