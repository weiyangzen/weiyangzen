It does not mutate repository or loop state directly; its exit status drives the stop hook’s block decision.
- gates_or_invariants: Missing transcript/task directory is fail-open and considered no incomplete work (`hooks/check-todos-from-transcript.py:64-65`, `hooks/check-todos-from-transcript.py:121-122`). Malformed transcript lines and malformed/unreadable task files are skipped (`hooks/check-todos-from-transcript.py:76-80`, `hooks/check-todos-from-transcript.py:143-145`). Missing task status defaults to `pending`; missing todo status becomes incomplete because it is not `completed` (`hooks/check-todos-from-transcript.py:91-99`, `hooks/check-todos-from-transcript.py:130-142`).
- dependencies_and_callers: Called by `hooks/loop-codex-stop-hook.sh` via `python3 "$TODO_CHECKER"` with the original hook input (`hooks/loop-codex-stop-hook.sh:347-352`). Exit `1` is converted into a rendered `block/incomplete-todos.md` message and blocks without Codex review (`hooks/loop-codex-stop-hook.sh:373-395`). Exit `2` blocks as task-checker parse failure (`hooks/loop-codex-stop-hook.sh:354-370`).
- edge_cases_or_failure_modes: Empty stdin exits `0`; invalid top-level hook JSON exits `2`; absent task/session/transcript sources exit `0`. Empty `TodoWrite.todos` does not replace the previous todo snapshot because the code only updates `latest_todos` when `todos` is truthy (`hooks/check-todos-from-transcript.py:84-88`). Task output order depends on filesystem glob order (`hooks/check-todos-from-transcript.py:125`). Transcript file open/read errors are not caught around the top-level `open`, so unreadable transcript paths can raise rather than return a clean checker code.
