Otherwise it creates `.cancel-requested`, moves the active state file to `cancel-state.md`, and prints either `CANCELLED` or `CANCELLED_FINALIZE` (`scripts/cancel-rlcr-loop.sh:134-152`).
- inputs_outputs_state: Inputs are optional `--force`, environment variable `CLAUDE_PROJECT_DIR`, and filesystem state under `.humanize/rlcr/<timestamp>/`. State transition is `state.md -> cancel-state.md` for normal loops or `finalize-state.md -> cancel-state.md` for forced Finalize Phase cancellation, plus creation of `.cancel-requested` (`scripts/cancel-rlcr-loop.sh:134-138`).
- gates_or_invariants: A loop is active only if the newest loop directory has `state.md` or `finalize-state.md` (`scripts/cancel-rlcr-loop.sh:89-99`). Finalize Phase cancellation requires explicit `--force` after caller confirmation (`scripts/cancel-rlcr-loop.sh:117-128`), coordinated with `commands/cancel-rlcr-loop.md:23-35`. The `.cancel-requested` signal is also part of the broader hook invariant for allowing state-file moves: `hooks/lib/loop-common.sh:557-716` authorizes only a signal-backed `mv` from `state.md` or `finalize-state.md` to `cancel-state.md` and rejects command substitution, chaining, extra args, wrong source/dest, mixed quotes, and symlink source.
- dependencies_and_callers: Called by `commands/cancel-rlcr-loop.md:3-15`; referenced as the stopping path from `commands/start-rlcr-loop.md:54-58` and startup conflict messages in `scripts/setup-pr-loop.sh`/`scripts/setup-rlcr-loop.sh`. If the newest directory exists but has no active state file, it emits `NO_ACTIVE_LOOP` and exits 1 (`scripts/cancel-rlcr-loop.sh:95-99`). Malformed or missing `current_round`/`max_iterations` are not fatal; report fields become `?` (`scripts/cancel-rlcr-loop.sh:109-112`). Directory choice is newest lexicographic path, not mtime.
