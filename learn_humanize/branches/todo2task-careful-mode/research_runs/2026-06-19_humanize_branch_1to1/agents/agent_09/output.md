### TODO2TASK_CAREFUL_MODE-HZ-039 `file` `scripts/portable-timeout.sh`
- cursor: `[_]`
- core_role: Cross-platform timeout abstraction for all shell workflows that need bounded git, GitHub CLI, Codex, validator, or test execution. It normalizes behavior across macOS and Linux where GNU `timeout` may not exist.
- algorithmic_behavior: `detect_timeout_impl` selects the first usable timeout implementation in priority order: Homebrew `gtimeout`, GNU `timeout`, `python3`, `python`, then `none` (lines 9-27). It verifies `timeout --version` so non-GNU or incompatible implementations do not get selected (lines 13-19). `TIMEOUT_IMPL` is computed once at source time (line 29). `run_with_timeout` receives `<seconds> <command> [args...]`, shifts the seconds argument, preserves the command as an array, then dispatches by implementation (lines 31-71). GNU wrappers directly execute `gtimeout "$timeout_secs" "${cmd[@]}"` or `timeout "$timeout_secs" "${cmd[@]}"` and return the wrapped status (lines 38-46). The Python fallback runs `subprocess.run(sys.argv[1:], timeout=$timeout_secs)`, propagates the command return code, maps `TimeoutExpired` to exit `124`, and maps other exceptions to exit `1` with stderr text (lines 47-63). If no timeout exists, it warns and runs the command unbounded (lines 64-69). It exports `TIMEOUT_IMPL` for sourcing scripts (lines 73-76).
- inputs_outputs_state: Inputs are the detected local command inventory, timeout seconds, command executable, and command arguments. Outputs are the wrapped command’s stdout/stderr and exit status, with `124` as the timeout convention for the Python path. State is limited to the sourced global `TIMEOUT_IMPL`; the function itself does not write files.
- gates_or_invariants: The wrapper preserves argument boundaries through Bash arrays (`local cmd=("$@")`, line 36). It attempts to preserve GNU timeout semantics, especially exit `124` on Python timeouts (lines 56-58). It treats missing timeout support as degraded but nonfatal behavior, with a visible warning (lines 64-67).
- dependencies_and_callers: Depends on Bash, `command -v`, optional `gtimeout`, optional GNU `timeout`, and optional Python. The repository-wide reference scan shows it sourced by `hooks/loop-codex-stop-hook.sh`, `hooks/pr-loop-stop-hook.sh`, `hooks/loop-plan-file-validator.sh`, `scripts/setup-rlcr-loop.sh`, `scripts/setup-pr-loop.sh`, `scripts/check-bot-reactions.sh`, `scripts/check-pr-reviewer-status.sh`, and robustness tests. `scripts/setup-rlcr-loop.sh` uses `run_with_timeout` throughout git setup and base-branch detection, including the production auto-detection block at lines 601-668.
- edge_cases_or_failure_modes: `TIMEOUT_IMPL` is fixed when the script is sourced, so later PATH changes will not change the selected implementation unless the file is re-sourced. The Python command string interpolates `$timeout_secs` directly into Python source (line 54); callers should pass numeric seconds. The `none` path can hang if the wrapped command hangs because it intentionally runs without a timeout (lines 64-68). Bash arrays mean this is not POSIX-sh compatible despite the file being a small portability shim.
