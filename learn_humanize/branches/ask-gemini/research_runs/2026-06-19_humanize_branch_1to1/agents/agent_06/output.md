### ASK_GEMINI-HZ-006 `directory` `hooks`
- cursor: `[_]`
- core_role: The `hooks` directory is the enforcement and transition layer for the Humanize RLCR and PR-loop algorithms. `hooks/hooks.json` wires Claude Code hook events to validators and stop hooks: `UserPromptSubmit` runs `loop-plan-file-validator.sh`; `PreToolUse` runs Write/Edit/Read/Bash validators; `PostToolUse` runs `loop-post-bash-hook.sh`; `Stop` runs both `loop-codex-stop-hook.sh` and `pr-loop-stop-hook.sh` with 7200s timeouts (`hooks/hooks.json:4-80`).
- algorithmic_behavior: The directory implements a multi-stage loop state machine. RLCR starts from `.humanize/rlcr/<timestamp>/state.md`, validates every read/write/bash action against active loop state, blocks bypass attempts, and on Stop runs Codex summary review followed by Codex code review before finalization. PR-loop starts from `.humanize/pr-loop/<timestamp>/state.md`, validates PR-loop artifacts, polls GitHub bot comments/reactions, uses local Codex to classify bot feedback, and transitions active bots until approval or terminal states.
- inputs_outputs_state: Inputs are Claude hook JSON on stdin, repository-local state files, task transcripts or `~/.claude/tasks/<session_id>`, Git status, prompt templates, Codex CLI output, and GitHub CLI/API data for PR loops. - gates_or_invariants: Shared input validation rejects malformed JSON, missing tool fields, null bytes, invalid UTF-8, and excessive nesting (`hooks/lib/loop-common.sh:91-165`). Active RLCR loop selection is session-aware and only returns the newest active loop for the session, preventing stale loop revival (`hooks/lib/loop-common.sh:289-365`). Read/write/edit validators block stale round files, protected state files, prompt files, plan backups, and immutable goal-tracker edits (`hooks/loop-read-validator.sh:156-303`, `hooks/loop-write-validator.sh:205-363`, `hooks/loop-edit-validator.sh:158-272`).
