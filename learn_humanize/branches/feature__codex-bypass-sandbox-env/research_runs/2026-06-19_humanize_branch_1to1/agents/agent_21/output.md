- inputs_outputs_state: Inputs are a round number plus globals `CACHE_DIR` and `LOOP_DIR`; the test creates logs at `$CACHE_DIR/round-N-codex-review.log` and expects results at `$LOOP_DIR/round-N-review-result.md` (`tests/test-codex-review-merge.sh:47-54`, `tests/test-codex-review-merge.sh:311-329`). Outputs are stdout prefixed with `## Codex Review Issues`, stderr audit messages, result-file creation when issues exist, and exit codes: `0` issues found, `1` no issues, `2` missing/empty log hard error (`hooks/lib/loop-common.sh:332-391`).
- gates_or_invariants: Only `[P?]` markers inside the first 10 characters count (`tests/test-codex-review-merge.sh:57-107`). The scan must cover the entire file, not just the last N lines, including markers early or late in long logs (`tests/test-codex-review-merge.sh:171-227`). Missing or empty logs are hard errors with exit code `2`, not “clean review” (`tests/test-codex-review-merge.sh:133-169`). The first matching marker defines the extraction start, and later content is preserved (`tests/test-codex-review-merge.sh:229-255`).
