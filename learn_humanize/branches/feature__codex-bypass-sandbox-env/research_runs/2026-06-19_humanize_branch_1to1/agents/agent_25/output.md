### FEATURE__CODEX_BYPASS_SANDBOX_ENV-HZ-025 `file` `hooks/loop-bash-validator.sh`
- cursor: `[_]`
- core_role: PreToolUse Bash validator for RLCR and PR loops. It is wired for the Bash matcher in `hooks/hooks.json:42-48`, reads hook JSON from stdin, and decides whether a proposed Bash command may run. Its main role is preventing shell-based bypasses of loop-managed files and validation hooks.
- algorithmic_behavior: The script validates hook JSON via `validate_hook_input`, rejects deeply nested JSON, ignores non-Bash tools, requires `tool_input.command`, lowercases the command, then discovers active RLCR and PR loops under `.humanize/rlcr` and `.humanize/pr-loop` (`hooks/loop-bash-validator.sh:24-66`). If no active loop exists, it allows the command. For active RLCR loops, it selects `state.md` or `finalize-state.md`, parses state strictly, derives `CURRENT_ROUND`, and applies ordered block rules (`hooks/loop-bash-validator.sh:74-107`). It then enforces shared protections including `.humanize` git-add prevention (`hooks/loop-bash-validator.sh:109-118`), RLCR state/file protection (`hooks/loop-bash-validator.sh:125-395`), and PR-loop state/read-only file protection (`hooks/loop-bash-validator.sh:397-443`).
- inputs_outputs_state: Input is Claude hook JSON on stdin with `tool_name` and `tool_input.command`; environment input is `CLAUDE_PROJECT_DIR`, falling back to `pwd` (`hooks/loop-bash-validator.sh:24-55`). Outputs are process exit status and stderr block messages rendered through templates. Exit `0` means allowed, exit `1` means malformed/unsafe hook input or malformed active state, and exit `2` means policy block. State consumed includes active loop dirs, `state.md` / `finalize-state.md` frontmatter, `.cancel-requested`, and `push_every_round`. It does not mutate state itself.
- gates_or_invariants: Key invariants are fail-closed hook input validation (`hooks/loop-bash-validator.sh:26-35`), Bash-only scope (`hooks/loop-bash-validator.sh:36-45`), no push unless `push_every_round: true` (`hooks/loop-bash-validator.sh:96-105`), no `git add` of `.humanize` (`hooks/loop-bash-validator.sh:115-118`), no Bash mutation of `state.md`/`finalize-state.md` except authorized cancel rename (`hooks/loop-bash-validator.sh:139-156`), no mv/cp source exfiltration of state files (`hooks/loop-bash-validator.sh:158-328`), no plan backup mutation (`hooks/loop-bash-validator.sh:330-341`), no direct goal-tracker Bash mutation (`hooks/loop-bash-validator.sh:343-358`), no prompt or summary Bash writes (`hooks/loop-bash-validator.sh:360-379`), todos writes only for active loop `round-1-todos.md` or `round-2-todos.md` with full active-loop path (`hooks/loop-bash-validator.sh:381-393`), and PR-loop state/read-only files are blocked by both full-path and basename patterns (`hooks/loop-bash-validator.sh:403-442`).
- dependencies_and_callers: Sources `hooks/lib/loop-common.sh` (`hooks/loop-bash-validator.sh:16-18`). It depends on `jq`, `sed`, `grep`, `tr`, `find_active_loop`, `find_active_pr_loop`, `parse_state_file_strict`, `command_modifies_file`, `git_adds_humanize`, `is_cancel_authorized`, and rendered block-message helpers. Shared helpers define JSON validation and nesting checks (`hooks/lib/loop-common.sh:59-133`), active-loop discovery (`hooks/lib/loop-common.sh:156-174`, `hooks/lib/loop-common.sh:798-814`), strict state schema parsing (`hooks/lib/loop-common.sh:255-330`), broad command-modification patterns (`hooks/lib/loop-common.sh:1071-1096`), `.humanize` git-add detection (`hooks/lib/loop-common.sh:930-1026`), and cancel authorization (`hooks/lib/loop-common.sh:551-710`). Template dependency for Round 0 goal tracker Bash blocks is `prompt-template/block/goal-tracker-bash-write.md`, rendered by `goal_tracker_bash_blocked_message` (`hooks/lib/loop-common.sh:492-501`).
- edge_cases_or_failure_modes: It blocks malformed active RLCR state instead of allowing operations (`hooks/loop-bash-validator.sh:81-85`). It checks `finalize-state.md` before `state.md` to avoid substring overlap (`hooks/loop-bash-validator.sh:127-146`). The mv/cp source-protection path splits commands on shell operators while preserving redirection placeholders, strips leading redirections, and checks wrapped `sh -c` / `bash -c` payloads (`hooks/loop-bash-validator.sh:169-328`). Authorized cancel is intentionally narrow: requires `.cancel-requested`, exact `mv` from active `state.md` or `finalize-state.md` to `cancel-state.md`, rejects command substitution, shell operators, remaining `$`, mixed quotes, extra args, symlink sources, and malformed spacing (`hooks/lib/loop-common.sh:551-710`).
