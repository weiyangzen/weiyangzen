- core_role: Shared primitive library for the RLCR hook state machine. It is sourced by the read/write/edit/bash validators, the plan validator, the Codex stop hook, and setup script, giving all of them one implementation for active-loop discovery, state-file shutdown, current-round parsing, portable relative paths, loop-file classification, shell-write detection, and templated block messages. - algorithmic_behavior: The library implements small deterministic gates used across the larger RLCR loop. `find_active_loop` checks only the lexicographically newest subdirectory under `.humanize-loop.local` and returns it only if it contains `state.md`, intentionally avoiding older "zombie" loops ([hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/lib/loop-common.sh:67)). `get_current_round` extracts YAML frontmatter between `---` delimiters and returns `current_round` or `0` ([hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/lib/loop-common.sh:88)). `check_start_commit_ancestry` uses `git merge-base --is-ancestor "$start_commit" HEAD` to verify the current history still descends from loop start ([hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/lib/loop-common.sh:43)). `stop_loop` ends a loop by renaming `state.md` to a prefix-qualified state file, preserving evidence where possible and falling back to removal if rename fails ([hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/lib/loop-common.sh:31)). `get_relative_path` uses GNU `realpath --relative-to`, then `grealpath`, then a shell fallback that normalizes absolute path components and computes `..` segments ([hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/lib/loop-common.sh:146)). Round-file and protected-file helpers classify `round-N-summary.md`, `round-N-prompt.md`, `round-N-todos.md`, `goal-tracker.md`, `state.md`, and `.humanize-loop.local/` paths ([hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/lib/loop-common.sh:234)). `command_modifies_file` scans lowercased Bash commands for common write forms: redirection, `tee`, in-place `sed`/`awk`/`perl`, `mv`/`cp`, and `dd of=` ([hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/lib/loop-common.sh:327)).

- inputs_outputs_state: Inputs are shell arguments, hook-provided paths/commands, `state.md` frontmatter, filesystem presence under `.humanize-loop.local`, the current git repository, and template files under `prompt-template`. Outputs are stdout path/round values, function exit status gates, rendered block-message text to callers, and state-file rename side effects for loop termination. The state-machine transitions encoded here are: active loop exists when the newest loop dir has `state.md`; terminated loop exists when `state.md` is renamed to `completed-state.md`, `stopped-state.md`, `unexpected-state.md`, or `cancelled-state.md`; current round is parsed from active state and defaults to round zero if absent. Setup creates the consumed state fields: `current_round`, `max_iterations`, Codex settings, `push_every_round`, `commit_plan_file`, `plan_file`, `plan_file_tracked`, `start_commit`, and `started_at` in [scripts/setup-rlcr-loop.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/scripts/setup-rlcr-loop.sh:326). The stop hook advances rounds by rewriting `current_round` only after review finds more work, and uses `stop_loop` for completed, stopped, and unexpected terminal states ([hooks/loop-codex-stop-hook.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/loop-codex-stop-hook.sh:865)).

- gates_or_invariants: Invariants enforced through this library include: only the newest loop directory can be active; `state.md` is the active-loop sentinel; old directories do not revive; prompt/summary/todos filenames must match `round-[0-9]+-{type}.md`; todos files are blocked in favor of native todos; prompt files are read-only; state files are hook-managed; goal tracker is editable only during round zero via non-Bash tools; summary file writes must target the active round and directory; Bash cannot bypass Write/Edit restrictions for protected files; loop history must remain descended from `start_commit` for Bash commands and is conditionally checked for plan prompts.
