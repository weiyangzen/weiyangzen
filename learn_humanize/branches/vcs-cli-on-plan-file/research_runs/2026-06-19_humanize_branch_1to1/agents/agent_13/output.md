- core_role: This file is the slash-command entrypoint for starting the RLCR loop. It is intentionally thin: frontmatter defines the user-facing command metadata, argument contract, and an allowlist limited to the setup script at `scripts/setup-rlcr-loop.sh`; the body invokes that setup script with `$ARGUMENTS`. - algorithmic_behavior: The command’s direct behavior is `setup-rlcr-loop.sh <plan-file> [options]`, where the command body executes `"${CLAUDE_PLUGIN_ROOT}/scripts/setup-rlcr-loop.sh" $ARGUMENTS` (`commands/start-rlcr-loop.md:10-14`). The delegated setup script parses one plan file plus options such as `--max`, `--codex-model`, `--codex-timeout`, `--push-every-round`, and `--commit-plan-file` (`scripts/setup-rlcr-loop.sh:31-170`). It validates prerequisites, creates `.humanize-loop.local/<timestamp>/`, copies the plan to `plan-backup.md`, writes `state.md`, generates `goal-tracker.md`, and emits `round-0-prompt.md` as the initial work instruction (`scripts/setup-rlcr-loop.sh:305-493`). The runtime loop is then enforced by Claude Code hooks in `hooks/hooks.json`: `UserPromptSubmit` validates plan stability, `PreToolUse` validators constrain reads/writes/bash/edit actions, and the `Stop` hook runs Codex review (`hooks/hooks.json:4-63`). The stop hook is the main transition algorithm: it blocks if local gates fail, builds a regular or full-alignment Codex prompt, runs `codex exec`, then either renames state to completed/stopped or increments `current_round` and writes the next prompt (`hooks/loop-codex-stop-hook.sh:506-608`, `hooks/loop-codex-stop-hook.sh:672-699`, `hooks/loop-codex-stop-hook.sh:820-931`).

- inputs_outputs_state: Inputs are the plan path and flags described in the command frontmatter (`commands/start-rlcr-loop.md:1-4`) and setup help (`scripts/setup-rlcr-loop.sh:41-98`). The setup input also depends on git state: repository required, at least one commit required, and optional `--commit-plan-file` changes whether the plan must be tracked, clean, inside the repo, and not ignored (`scripts/setup-rlcr-loop.sh:199-241`, `scripts/setup-rlcr-loop.sh:251-303`). Outputs are local loop artifacts under `.humanize-loop.local/<timestamp>/`: `state.md`, `plan-backup.md`, `goal-tracker.md`, `round-N-prompt.md`, `round-N-summary.md`, `round-N-review-prompt.md`, and `round-N-review-result.md`; README documents the same artifact set (`README.md:283-295`). State transitions are file-renames and round-number edits: active loop is the newest directory whose `state.md` exists (`hooks/lib/loop-common.sh:63-84`); successful completion renames `state.md` to `completed-state.md`, STOP/max-iteration termination to `stopped-state.md`, and corruption/legacy/branch-change cases to `unexpected-state.md` (`hooks/lib/loop-common.sh:23-38`, `hooks/loop-codex-stop-hook.sh:512-515`, `hooks/loop-codex-stop-hook.sh:820-858`). If Codex reports issues, the stop hook updates `current_round` to `NEXT_ROUND`, creates the next prompt and summary target, then returns a hook block with that prompt as feedback (`hooks/loop-codex-stop-hook.sh:865-931`).
