### VCS_CLI_ON_PLAN_FILE-HZ-032 `file` `prompt-template/block/branch-changed.md`
- cursor: `[_]`
- core_role: This is a user-facing prompt block for the RLCR loop’s branch/history invalidation gate. It does not run logic itself; it defines the rendered explanation shown when a loop is terminated because the current Git history no longer matches the loop’s recorded `start_commit`. The title and invariant are declared at [prompt-template/block/branch-changed.md](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/prompt-template/block/branch-changed.md:1), with the key message that `{{CURRENT_HEAD}}` is not a descendant of `{{START_COMMIT}}` at line 3.
- algorithmic_behavior: The template is selected after validators detect an invalid branch/commit-history condition. It explains the transition as: current HEAD is outside the expected ancestry, the loop state is no longer valid for this history, and the active state file has been renamed to `unexpected-state.md` to stop the loop. The remediation contract is embedded in the prompt: checkout the original branch and rename `unexpected-state.md` back to `state.md` to resume, start a fresh loop with `/humanize:start-rlcr-loop <your-plan.md>`, or continue without the loop. See the operational text at [prompt-template/block/branch-changed.md](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/prompt-template/block/branch-changed.md:5) and the continuation options at [prompt-template/block/branch-changed.md](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/prompt-template/block/branch-changed.md:13).
- inputs_outputs_state: Inputs are two template variables, `CURRENT_HEAD` and `START_COMMIT`, rendered into the markdown using `{{...}}` syntax. The renderer is the shared `load_and_render_safe` path, which loads a template, substitutes variables, and falls back to an inline message if the template is missing or renders empty; that behavior is implemented at [hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/lib/template-loader.sh:152). The output is markdown written to stderr by the calling validators, followed by exit code `2`. The state transition is performed by `stop_loop "$STATE_FILE" "unexpected"`, whose implementation renames `state.md` to `unexpected-state.md` when possible; see [hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/lib/loop-common.sh:23) and the rename line at [hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/lib/loop-common.sh:35).
- gates_or_invariants: The core invariant is ancestry: the loop’s `start_commit` must be an ancestor of current `HEAD`. Shared gate logic uses `git merge-base --is-ancestor "$start_commit" HEAD`; missing `start_commit` or non-Git contexts are treated as pass-through, not termination. See [hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/lib/loop-common.sh:40). In `loop-bash-validator.sh`, failed ancestry immediately stops the loop and renders this template; see [hooks/loop-bash-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/loop-bash-validator.sh:49) and [hooks/loop-bash-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/loop-bash-validator.sh:55).
