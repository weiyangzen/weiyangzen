The directory has exactly three files, with no nested subdirectories: [regular-review.md](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/prompt-template/codex/regular-review.md:1), [full-alignment-review.md](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/prompt-template/codex/full-alignment-review.md:1), and [goal-tracker-update-section.md](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/prompt-template/codex/goal-tracker-update-section.md:1).

First, it renders the shared goal-tracker update section from `codex/goal-tracker-update-section.md` into `GOAL_TRACKER_UPDATE_SECTION` [hooks/loop-codex-stop-hook.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/loop-codex-stop-hook.sh:536). Then it selects a review mode: regular review for most rounds, or full alignment when `CURRENT_ROUND % 5 == 4` [hooks/loop-codex-stop-hook.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/loop-codex-stop-hook.sh:542). The selected template is rendered into `round-N-review-prompt.md` [hooks/loop-codex-stop-hook.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/loop-codex-stop-hook.sh:577), then passed to `codex exec` [hooks/loop-codex-stop-hook.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/loop-codex-stop-hook.sh:672). The runtime interprets the review result’s last non-empty line: exact `COMPLETE` stops successfully, exact `STOP` stops as circuit breaker, anything else advances `current_round` and generates the next Claude prompt [hooks/loop-codex-stop-hook.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/loop-codex-stop-hook.sh:814).
