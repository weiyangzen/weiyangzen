### VCS_CLI_ON_PLAN_FILE-HZ-031 `file` `hooks/lib/template-loader.sh`
- cursor: `[_]`
- core_role: Shared Bash template-loading and rendering library for RLCR hook/validator output. It is not the RLCR state machine itself, but it is on the hook/validator critical path because validators use it to generate block/termination messages for state-machine gates. `hooks/lib/loop-common.sh` sources it and initializes `TEMPLATE_DIR` before shared hook helpers run (`hooks/lib/loop-common.sh:11-20`), and validators such as `hooks/loop-plan-validator.sh` and `hooks/loop-bash-validator.sh` call its safe renderer when blocking prompt or Bash actions.
- algorithmic_behavior: `get_template_dir` computes the plugin template root by resolving `script_dir/../..` and appending `prompt-template` (`hooks/lib/template-loader.sh:11-16`). `load_template` joins `template_dir/template_name`, emits the file contents if present, and otherwise prints a warning to stderr plus empty stdout (`hooks/lib/template-loader.sh:21-33`). `render_template` performs single-pass `{{VAR}}` substitution by exporting all `VAR=value` inputs as `TMPL_VAR_VAR` environment entries, then running an awk scanner over each input line (`hooks/lib/template-loader.sh:43-117`). The awk loop finds `{{`, searches for the next `}}`, extracts the exact variable name, substitutes only if the variable exists, preserves unknown placeholders literally, and does not rescan substituted values (`hooks/lib/template-loader.sh:56-114`). `load_and_render` composes load then render only when loaded content is non-empty (`hooks/lib/template-loader.sh:121-132`). `append_template` concatenates base content and an additional loaded template (`hooks/lib/template-loader.sh:136-146`). `load_and_render_safe` adds the gate-safe path: if the template is missing, empty, or renders empty, it renders or echoes the fallback message instead (`hooks/lib/template-loader.sh:155-188`). `validate_template_dir` checks that the template root exists and contains `block`, `codex`, and `claude` subdirectories (`hooks/lib/template-loader.sh:193-207`).
- inputs_outputs_state: Inputs are a caller-supplied script/library directory for `get_template_dir`, template directory and relative template name for loading, and a content string plus shell arguments formatted as `VAR=value` for rendering. Outputs are rendered Markdown/plain-text prompt fragments on stdout and warnings/errors on stderr. Persistent RLCR state is not mutated by this library; it only influences hook behavior by producing the human/agent-facing `reason` text that accompanies a gate decision. The important state transition is therefore indirect: validators decide `exit 0` versus `exit 2` or loop termination, then this library supplies the explanatory block text. For example, the plan validator stops legacy loops by renaming `state.md` to `unexpected-state.md` and renders `block/pre-112-state-file.md` before `exit 2` (`hooks/loop-plan-validator.sh:59-78`), blocks `--commit-plan-file` with an outside-repo plan through `block/plan-file-outside-repo-conflict.md` (`hooks/loop-plan-validator.sh:173-198`), blocks untracked/dirty committed plan files through `block/plan-file-not-tracked.md` and `block/plan-file-uncommitted.md` (`hooks/loop-plan-validator.sh:204-250`), and blocks plan content drift through `block/plan-file-changed-prompt-block.md` (`hooks/loop-plan-validator.sh:291-317`). The Bash validator similarly blocks `git commit` when the plan file is staged and `commit_plan_file` is false through `block/plan-file-staged.md` (`hooks/loop-bash-validator.sh:99-146`).
- gates_or_invariants: Template placeholders are exact `{{VARIABLE_NAME}}` tokens; unknown variables are preserved instead of deleted (`hooks/lib/template-loader.sh:100-107`). Rendering is intentionally single-pass to prevent injected values containing `{{OTHER_VAR}}` from being substituted later (`hooks/lib/template-loader.sh:35-42`, `hooks/lib/template-loader.sh:56-114`). Missing templates do not crash callers: `load_template` returns empty and `load_and_render_safe` falls back to inline text (`hooks/lib/template-loader.sh:26-32`, `hooks/lib/template-loader.sh:161-188`). Template-directory validation is advisory in `loop-common.sh`: it warns and allows inline fallbacks instead of failing hook startup (`hooks/lib/loop-common.sh:18-21`). The structural invariant for a valid template root is presence of `block/`, `codex/`, and `claude/` (`hooks/lib/template-loader.sh:201-204`). Tests also encode syntax expectations for template files: placeholders should be uppercase double-brace names without extra braces or embedded spaces (`tests/test-templates-comprehensive.sh:122-194`).
- dependencies_and_callers: Direct dependency is standard shell plus `awk`, `cat`, `env`, and filesystem checks. `loop-common.sh` is the central importer and sets `TEMPLATE_DIR` from `get_template_dir` (`hooks/lib/loop-common.sh:11-20`). Downstream callers include `loop-bash-validator.sh`, `loop-read-validator.sh`, `loop-write-validator.sh`, `loop-edit-validator.sh`, `loop-plan-validator.sh`, and `loop-codex-stop-hook.sh`; `rg` found validator and stop-hook calls using `load_and_render_safe`, plus a few direct `load_template` calls for optional appended notes. The branch-specific plan-file surfaces are present under `prompt-template/block/`, including `plan-file-staged.md`, `plan-file-committed.md`, `plan-file-changed-prompt-block.md`, `plan-file-uncommitted.md`, `plan-file-not-tracked.md`, `plan-file-outside-repo-conflict.md`, and `pre-112-state-file.md`. Template reference coverage is enforced by `tests/test-template-references.sh`, which scans hook scripts for `load_template`, `load_and_render`, and `load_and_render_safe` calls and fails if referenced templates are missing (`tests/test-template-references.sh:56-112`, `tests/test-template-references.sh:214-230`).
- edge_cases_or_failure_modes: Missing template file or directory returns empty from `load_template`; this is safe only when callers use `load_and_render_safe`, because raw `load_and_render` can silently produce no reason text (`hooks/lib/template-loader.sh:21-33`, `hooks/lib/template-loader.sh:121-132`). `tests/test-error-scenarios.sh` explicitly documents this risk: missing templates, missing directories, and empty content all exit 0 and return empty, with the noted risk that an empty reason can be sent if the safe wrapper is not used (`tests/test-error-scenarios.sh:21-120`). `render_template` accepts arbitrary variable names from the left side of `VAR=value` without validating against uppercase syntax; invalid names simply become environment keys and only match exact placeholders. If a value contains `=`, only the first `=` splits name from value, so additional `=` characters are preserved in the value (`hooks/lib/template-loader.sh:50-53`). If template content has `{{` without closing `}}`, awk treats only the opening braces literally and continues scanning the rest (`hooks/lib/template-loader.sh:87-94`). Multiline variable values are passed through environment variables and can expand into multiline rendered output; tests cover this behavior (`tests/test-template-loader.sh:473-486`). Shell metacharacters, backticks, pipes, redirections, globs, quotes, JSON-like strings, and regex-like strings are intended to render literally because substitution is performed inside awk from environment values rather than shell-evaluated text (`tests/test-template-loader.sh:257-486`). Direct `load_template` use for optional notes can degrade to omitted optional content if a template is absent; safe rendering is needed for blocking gates.
- validation_or_tests: `tests/test-template-loader.sh` covers path derivation, successful and missing template loads, single/multiple/multiline rendering, unknown placeholders, safe fallback behavior, valid/invalid template directory checks, shell metacharacter literal rendering, and placeholder-injection prevention (`tests/test-template-loader.sh:41-244`, `tests/test-template-loader.sh:257-563`). `tests/test-templates-comprehensive.sh` loads every Markdown template, validates placeholder syntax, checks rendering edge cases, verifies fallback behavior, runs real-template integrations for `wrong-round-number`, `unpushed-commits`, and `goal-tracker-update-section`, and stress-renders all templates with dummy placeholder values (`tests/test-templates-comprehensive.sh:61-194`, `tests/test-templates-comprehensive.sh:262-625`). `tests/test-template-references.sh` protects gate reliability by failing when hook-referenced templates are missing and by requiring critical validators to use `load_and_render_safe` instead of unsafe `load_and_render` (`tests/test-template-references.sh:51-112`, `tests/test-template-references.sh:170-202`). `tests/test-plan-file-handling.sh` includes branch-specific checks that plan-file templates exist and that validators use the expected plan-file templates, including the Bash validator’s `plan-file-staged` path.
