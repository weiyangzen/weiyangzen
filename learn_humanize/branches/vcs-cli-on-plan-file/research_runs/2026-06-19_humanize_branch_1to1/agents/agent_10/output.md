Template injection through `REVIEW_CONTENT` is specifically mitigated by single-pass substitution: placeholder-looking text inside values is emitted literally and not rescanned ([template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/vcs-cli-on-plan-file/hooks/lib/template-loader.sh:39)). A behavioral risk remains that non-critical optional templates loaded with plain `load_template` are not rendered, so any future placeholder added to `push-every-round-note.md`, `post-alignment-action-items.md`, or `goal-tracker-update-request.md` would remain literal unless the caller changes to render them. Current files avoid dynamic placeholders except the base/footer pair. Another edge is prompt inconsistency: Round 0 setup uses an inline push note and code-simplifier wording, while later rounds use these template files; changes to one surface can drift unless tests catch it.
