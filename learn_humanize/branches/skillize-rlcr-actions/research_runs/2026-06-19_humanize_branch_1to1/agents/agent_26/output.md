- algorithmic_behavior: `detect_timeout_impl` chooses the first available implementation in priority order: Homebrew `gtimeout`, GNU `timeout`, `python3`, `python`, then `none` (`scripts/portable-timeout.sh:10`). Native timeout implementations directly execute the command and return their status (`scripts/portable-timeout.sh:38`). Python fallback uses `subprocess.run(sys.argv[1:], timeout=...)`, returns the child return code, and maps `TimeoutExpired` to exit `124` to match GNU timeout (`scripts/portable-timeout.sh:47`). If no timeout implementation exists, it emits a warning and runs the command unbounded (`scripts/portable-timeout.sh:64`).
- inputs_outputs_state: Inputs are the timeout seconds, command path/name, command args, and the current `PATH`/available executables. Output is pass-through stdout/stderr from the child process plus a normalized shell exit code. Internal state is the exported `TIMEOUT_IMPL` variable (`scripts/portable-timeout.sh:73`). There is no filesystem state mutation inside this file.
