- gates_or_invariants: The invariant is that normal loop participants should not read, write, edit, or shell-modify arbitrary `round-*-todos.md` files. `loop-read-validator.sh` blocks Read unless the file is active-loop allowlisted ([loop-read-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/loop-read-validator.sh:36)). `loop-write-validator.sh` and `loop-edit-validator.sh` apply the same check for Write and Edit ([loop-write-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/loop-write-validator.sh:37), [loop-edit-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/loop-edit-validator.sh:36)). `loop-bash-validator.sh` detects shell commands modifying todos files and requires a full active-loop path for the narrow allowlist ([loop-bash-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/loop-bash-validator.sh:344)). The allowlist only includes `round-1-todos.md` and `round-2-todos.md` plus early summaries, so later or wrong-directory todo files are blocked ([loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/loop-common.sh:170)).
