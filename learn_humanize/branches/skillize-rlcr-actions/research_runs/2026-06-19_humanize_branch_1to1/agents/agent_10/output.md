### SKILLIZE_RLCR_ACTIONS-HZ-010 `directory` `prompt-template/block`
- cursor: `[_]`
- core_role: `prompt-template/block` is the shared gate-message catalog for RLCR hook decisions. The templates do not execute logic themselves, but they define the operator-facing contract returned when validators block file access, Bash mutations, plan drift, git hygiene failures, stale schema, round mismatch, and stop-hook prerequisites. The directory is loaded through [hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/template-loader.sh:24), which resolves `prompt-template`, loads files, and renders `{{VAR}}` placeholders.
- algorithmic_behavior: The directory contains 27 leaf Markdown blocks. The loader performs single-pass placeholder substitution, preserves unknown placeholders, and intentionally avoids recursive expansion to prevent injected content from being reinterpreted; see [hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/template-loader.sh:50). Hook logic selects blocks by failure type: read/write/edit/bash validators emit path/round/tool blocks, the stop hook emits plan/git/todo/summary/goal-tracker blocks, and the plan prompt hook emits schema blocks.
- inputs_outputs_state: Inputs are template names plus key-value substitutions such as `CURRENT_ROUND`, `CORRECT_PATH`, `PLAN_FILE`, `BACKUP_PATH`, `GIT_ISSUES`, `AHEAD_COUNT`, `FIELD_NAME`, and `MISSING_ITEMS`. Outputs are Markdown strings written either to stderr with exit code `2` for PreToolUse blocks or wrapped into JSON `decision: block` payloads for UserPromptSubmit/Stop hooks. The templates do not mutate state; they describe state transitions enforced elsewhere, such as preserving `state.md`, entering finalize phase, or requiring the next round summary.
- gates_or_invariants: The directory encodes invariants that protected RLCR files are not user-mutated: `state.md`, `finalize-state.md`, loop `plan.md` backup, prompt files, todo round files, wrong-round summary files, and `goal-tracker.md` after Round 0. It also encodes stop-gate prerequisites: no incomplete todos, no large changed files, clean git state, pushed commits when configured, initialized goal tracker, and unmodified original plan.
- dependencies_and_callers: `hooks/lib/loop-common.sh` centralizes common block calls for todos, prompt, state, finalize-state, summary-Bash, goal-tracker-Bash, git-add, and post-Round-0 goal tracker blocks at [hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/loop-common.sh:190). Direct consumers include `loop-write-validator.sh`, `loop-edit-validator.sh`, `loop-read-validator.sh`, `loop-bash-validator.sh`, `loop-plan-file-validator.sh`, and `loop-codex-stop-hook.sh`. Parent coordination is through `prompt-template` sibling directories: `block/` supplies short gate reasons, while `codex/` and `claude/` supply full review/next-round prompts consumed later in the stop hook.
- edge_cases_or_failure_modes: Missing templates are not fatal: `load_and_render_safe` uses inline fallbacks, so hook blocking remains functional even when a block file is absent. Unknown placeholders survive unchanged, which is safe but can surface raw `{{VAR}}` if callers omit substitutions. The catalog includes split-note partials with leading blank lines (`git-not-clean-humanize-local.md`, `git-not-clean-untracked.md`), which are intentionally concatenated into `SPECIAL_NOTES` in the git-clean stop gate.
- validation_or_tests: Template existence and references are covered by `tests/test-template-references.sh` and `tests/test-templates-comprehensive.sh`, with explicit known-block coverage for `summary-bash-write.md`, `goal-tracker-bash-write.md`, and `goal-tracker-modification.md`. Behavioral use is exercised by `tests/test-plan-file-hooks.sh`, especially plan backup and schema blocks, and by `tests/test-bash-validator-patterns.sh` for Bash mutation detection that routes to these messages.
