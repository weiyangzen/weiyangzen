- cursor: `[_]`
- core_role: Block-message template for the read validator when an agent attempts to read a prompt or summary file from a non-current RLCR round. It is a small but algorithmically important guard surface: it communicates the current round and correct active-loop files when read access is denied.
- algorithmic_behavior: The template renders a deterministic Markdown error message with five variables: `CLAUDE_ROUND`, `FILE_TYPE`, `CURRENT_ROUND`, `ACTIVE_LOOP_DIR`, and `FILE_PATH`. Its text says the requested `round-{{CLAUDE_ROUND}}-{{FILE_TYPE}}.md` is not the current round, lists the current prompt and summary files, and gives an explicit `cat {{FILE_PATH}}` escape hatch only if that exact file is still needed. See [prompt-template/block/wrong-round-file.md](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/prompt-template/block/wrong-round-file.md:1).
- inputs_outputs_state: Inputs are supplied by `hooks/loop-read-validator.sh` after it parses hook JSON, identifies a `Read` tool invocation, extracts `FILE_PATH`, locates the active loop, parses `current_round` from `state.md` or `finalize-state.md`, extracts the requested round number from the filename, and determines whether the file is a summary or prompt. The rendered output goes to stderr and the validator exits `2` to block the read, as shown at [hooks/loop-read-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/loop-read-validator.sh:116).
- gates_or_invariants: Gate condition is `CLAUDE_ROUND != CURRENT_ROUND` and the file is not allowlisted for the active loop. This only runs after earlier checks establish the path is a `round-N-summary.md` or `round-N-prompt.md`, an active loop exists, and the path is inside `.humanize/rlcr`; otherwise other gates handle todos, wrong location, or non-round files. The state parsing and round extraction sequence is at [hooks/loop-read-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/loop-read-validator.sh:61).

Rendering depends on `hooks/lib/template-loader.sh`, whose single-pass `render_template` preserves unresolved placeholders and prevents recursive placeholder substitution. Existence/reference is covered by `tests/test-template-references.sh`.

Allowlisted old files can bypass this template through `is_allowlisted_file`.
