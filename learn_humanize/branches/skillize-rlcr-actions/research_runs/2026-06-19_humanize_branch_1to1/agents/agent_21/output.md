### SKILLIZE_RLCR_ACTIONS-HZ-051 `file` `hooks/lib/template-loader.sh`
- cursor: `[_]`
- core_role: Shared template loading/rendering library for RLCR hook prompts and block messages. It centralizes the algorithm that converts `prompt-template/**` Markdown files plus `VAR=value` assignments into hook-facing text, with safe fallback behavior for missing templates.
- algorithmic_behavior: `get_template_dir` resolves the plugin root by going two directories up from `hooks/lib` and appending `prompt-template` ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/template-loader.sh:26)). `load_template` returns file content when present; otherwise it emits a warning to stderr and returns empty stdout ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/template-loader.sh:36)). `render_template` accepts content plus repeated `VAR=value` arguments, exports them as `TMPL_VAR_*`, and uses a single awk pass to replace `{{VAR}}` placeholders ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/template-loader.sh:58)). Missing placeholders remain literal, and replacement values are written directly to output rather than recursively rescanned ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/template-loader.sh:115)). `load_and_render` composes load plus render only when content exists, while `load_and_render_safe` falls back to an inline message if loading or rendering produces empty content ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/template-loader.sh:136), [hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/template-loader.sh:170)).
- inputs_outputs_state: Inputs are a template directory, template name, optional fallback string, and optional `VAR=value` bindings. Outputs are rendered Markdown/message text on stdout, warnings/errors on stderr for missing files or invalid directories, and return status `0` for normal rendering/fallback flows. It does not maintain persistent state. It temporarily passes substitution state through environment variables prefixed with `TMPL_VAR_`, reducing collision with caller variables ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/template-loader.sh:62)).
- gates_or_invariants: The documented template grammar is `{{VARIABLE_NAME}}`, but the awk implementation accepts any text between `{{` and `}}` as the lookup key; variables not provided remain unchanged. Rendering is intentionally single-pass to prevent placeholder injection when values themselves contain `{{...}}` ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/template-loader.sh:50)). `validate_template_dir` requires the root directory and the `block`, `codex`, and `claude` subdirectories; failure returns `1` and prints an error to stderr ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/template-loader.sh:208)).
- dependencies_and_callers: Depends on POSIX-ish shell utilities plus `awk`, `env`, and `cat`. It is sourced by `hooks/lib/loop-common.sh`, which initializes `TEMPLATE_DIR` and only warns if validation fails so hooks can degrade to inline fallbacks ([hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/loop-common.sh:46), [hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/loop-common.sh:50)). It is used by read/write/edit/bash validators and the stop hook through `load_and_render_safe`; template-reference tests scan those critical scripts for unsafe `load_and_render` usage ([tests/test-template-references.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/tests/test-template-references.sh:56), [tests/test-template-references.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/tests/test-template-references.sh:172)).
- edge_cases_or_failure_modes: A `VAR=value` argument with an empty variable name becomes `TMPL_VAR_`, which the awk loader would expose under an empty key; normal callers use explicit uppercase names. Values can contain `=` because parsing splits only at the first `=`. A malformed placeholder with no closing braces preserves the literal `{{` and continues scanning after it ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/template-loader.sh:102)). Because `load_template` returns success with empty output for missing files, callers that use plain `load_and_render` can silently receive an empty message; critical validators are expected to use `load_and_render_safe`. `append_template` always appends the loaded content, so a missing appended template yields the base content plus an empty trailing section ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/hooks/lib/template-loader.sh:151)).
- validation_or_tests: `tests/test-template-loader.sh` covers path resolution, existing/missing template loading, single/multiple/multiline substitution, special characters, unreplaced variables, safe fallback, and directory validation ([tests/test-template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/tests/test-template-loader.sh:41), [tests/test-template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/tests/test-template-loader.sh:197), [tests/test-template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/tests/test-template-loader.sh:235)). `tests/test-error-scenarios.sh` specifically verifies missing templates/directories return empty without crashing under strict mode and that safe fallback substitutes variables ([tests/test-error-scenarios.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/tests/test-error-scenarios.sh:42), [tests/test-error-scenarios.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/tests/test-error-scenarios.sh:109), [tests/test-error-scenarios.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/tests/test-error-scenarios.sh:134)). `tests/test-templates-comprehensive.sh` includes adjacent-variable and fallback checks ([tests/test-templates-comprehensive.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/skillize-rlcr-actions/tests/test-templates-comprehensive.sh:431)).
