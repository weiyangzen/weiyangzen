The Bash command detector is heuristic and can miss complex quoting, indirection, generated paths, or multi-step shell writes. Missing/invalid transcript input allows stopping because the todo checker treats that as no todos. Corrupt numeric `current_round` removes the state file and stops the loop, which is fail-open for completion but fail-closed for continued automation. Max-iteration expiry removes state even if Codex never confirmed completion.
