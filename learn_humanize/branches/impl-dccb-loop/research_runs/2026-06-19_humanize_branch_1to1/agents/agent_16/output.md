### IMPL_DCCB_LOOP-HZ-016 `file` `hooks/loop-edit-validator.sh`
- cursor: `[_]`
- core_role: PreToolUse validator for Claude `Edit` operations in the RLCR loop state machine. It prevents direct edits to loop-owned control artifacts and enforces that mutable round summaries stay aligned with the active round. The hook is registered in `hooks/hooks.json:14-20` as the command for the `Edit` matcher. The README lists it as one of the lifecycle hooks at `README.md:389-399`.
- algorithmic_behavior: The script reads hook JSON from stdin (`hooks/loop-edit-validator.sh:22`), extracts `.tool_name` with `jq`, and exits successfully unless the tool is exactly `Edit` (`hooks/loop-edit-validator.sh:23-27`). For `Edit`, it extracts `.tool_input.file_path`, lowercases it for case-insensitive pattern checks, then applies gates in order (`hooks/loop-edit-validator.sh:29-44`). It globally blocks `round-N-todos.md` edits and `round-N-prompt.md` edits before checking loop location (`hooks/loop-edit-validator.sh:36-44`). If the target is not under `.humanize-rlcr.local/`, it allows the edit (`hooks/loop-edit-validator.sh:50-52`). For loop-local paths, it finds the newest active RLCR loop directory, reads the current round from `rlcr-state.md`, blocks state-file edits, blocks goal-tracker edits after round 0, and validates summary filenames against the current round (`hooks/loop-edit-validator.sh:58-117`).
- inputs_outputs_state: Inputs are the Claude hook JSON on stdin, `CLAUDE_PROJECT_DIR` or `pwd` as project root, the path being edited, and the active loop state file at `.humanize-rlcr.local/<timestamp>/rlcr-state.md` (`hooks/loop-edit-validator.sh:22-30`, `hooks/loop-edit-validator.sh:58-66`). Outputs are exit code `0` to allow or exit code `2` with a stderr guidance message to block (`hooks/loop-edit-validator.sh:36-44`, `hooks/loop-edit-validator.sh:72-84`, `hooks/loop-edit-validator.sh:101-112`). It does not write persistent state. Its state-machine role is defensive: it preserves loop ownership of prompt, todos, state, tracker, and round-summary surfaces rather than advancing rounds itself.
- gates_or_invariants: Todo round files are not editable through `Edit`; the hook tells Claude to use native todo tooling (`hooks/loop-edit-validator.sh:36-38`, `hooks/lib/loop-common.sh:75-90`). Prompt round files are read-only generated instructions (`hooks/loop-edit-validator.sh:41-43`, `hooks/lib/loop-common.sh:92-108`). Any loop-local `*state.md` path is blocked as managed state (`hooks/loop-edit-validator.sh:72-74`, `hooks/lib/loop-common.sh:166-170`). Goal tracker edits are intended to be allowed only in round 0 and rejected after that, redirecting requests into the current summary (`hooks/loop-edit-validator.sh:81-84`, `hooks/lib/loop-common.sh:203-235`). Summary edits are allowed only when the extracted `round-N` equals `current_round`; stale or premature summary edits are blocked with the correct current summary path (`hooks/loop-edit-validator.sh:91-112`).
- dependencies_and_callers: Depends on Bash strict mode, `jq`, and shared helper library `hooks/lib/loop-common.sh` (`hooks/loop-edit-validator.sh:12-16`, `hooks/loop-edit-validator.sh:23`, `hooks/loop-edit-validator.sh:29`). Shared helpers provide active-loop discovery, current-round parsing, path classification, round extraction, and standard block messages (`hooks/lib/loop-common.sh:15-47`, `hooks/lib/loop-common.sh:49-73`, `hooks/lib/loop-common.sh:75-123`, `hooks/lib/loop-common.sh:160-176`, `hooks/lib/loop-common.sh:203-235`). It coordinates with sibling validators: `loop-write-validator.sh` enforces comparable write-path gates and adds stricter summary location checks (`hooks/loop-write-validator.sh:111-171`), `loop-read-validator.sh` blocks stale prompt/summary reads (`hooks/loop-read-validator.sh:48-130`), and `loop-bash-validator.sh` blocks shell-command bypasses for loop files (`hooks/loop-bash-validator.sh:69-124`).
- edge_cases_or_failure_modes: If `.tool_name` is absent or not `Edit`, the hook permits the operation (`hooks/loop-edit-validator.sh:23-27`). If no active loop is found, it permits loop-local edits (`hooks/loop-edit-validator.sh:60-64`), relying on `find_active_loop`, which only considers the newest timestamp-like directory with `rlcr-state.md` (`hooks/lib/loop-common.sh:11-33`). Prompt and todos blocking is filename-based and applies even outside `.humanize-rlcr.local/` because it happens before the loop-dir check (`hooks/loop-edit-validator.sh:36-44`). The summary gate checks round number but does not verify the edited summary is in the newest active loop directory; an old session path with the same current round can pass this script, unlike the stricter `Write` validator (`hooks/loop-edit-validator.sh:91-115`, `hooks/loop-write-validator.sh:161-171`). A notable tracker-name mismatch exists: setup and docs create/use `rlcr-tracker.md` (`scripts/setup-rlcr-loop.sh:237-248`, `README.md:415-418`), but `is_goal_tracker_path` currently matches `goal-tracker.md$` (`hooks/lib/loop-common.sh:160-164`), so the intended post-round-0 edit block may not catch the actual RLCR tracker filename.
