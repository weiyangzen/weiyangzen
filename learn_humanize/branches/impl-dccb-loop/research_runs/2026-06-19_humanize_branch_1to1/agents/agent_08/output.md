### IMPL_DCCB_LOOP-HZ-008 `file` `commands/cancel-rlcr-loop.md`
- cursor: `[_]`
- core_role: Defines the slash-command workflow for manually terminating an active RLCR loop. Its algorithmic role is state-token deletion: active RLCR loop liveness is represented by `.humanize-rlcr.local/*/rlcr-state.md`, and cancellation removes that marker while preserving the session directory artifacts. The command is RLCR-specific and mirrors the sibling DCCB cancel pattern with different state directory/file names.
- algorithmic_behavior: The command first enumerates candidate state files with `ls .humanize-rlcr.local/*/rlcr-state.md 2>/dev/null || echo "NO_LOOP"` (`commands/cancel-rlcr-loop.md:11-15`). If no glob match exists, the expected user-facing response is `No active RLCR loop found.` (`commands/cancel-rlcr-loop.md:17`). If one or more state files are found, the workflow reads the state file, extracts/report current progress as round `N of M`, removes state files using `rm .humanize-rlcr.local/*/rlcr-state.md`, and reports cancellation (`commands/cancel-rlcr-loop.md:19-22`). It does not delete summaries, prompts, tracker, or review results; preserved loop history is explicit (`commands/cancel-rlcr-loop.md:24`).
- inputs_outputs_state: Inputs are only filesystem state under `.humanize-rlcr.local/<session>/rlcr-state.md` plus its YAML frontmatter values such as `current_round` and `max_iterations`. That state file is created by RLCR setup with keys including `current_round: 0`, `max_iterations`, Codex config, `plan_file`, and `started_at` (`scripts/setup-rlcr-loop.sh:223-234`). Output is conversational status, either no active loop or cancelled at round `N of M`. The state transition is active -> inactive by deleting `rlcr-state.md`; all other session files remain as audit/reference artifacts. After deletion, hooks using `find_active_loop` see no active loop when the newest loop directory lacks `rlcr-state.md` (`hooks/lib/loop-common.sh:15-32`), and the stop hook exits early when no active loop is found (`hooks/loop-codex-stop-hook.sh:50-55`).
- gates_or_invariants: Tool permissions are intentionally narrow: allowed Bash forms are `ls`, `rm`, and `cat` against `.humanize-rlcr.local/*/rlcr-state.md`, plus `Read` (`commands/cancel-rlcr-loop.md:1-4`). The invariant is that cancellation must remove only the RLCR state marker, not the loop directory. The loop system itself reinforces that `rlcr-state.md` controls lifecycle: max-iteration termination removes it (`hooks/loop-codex-stop-hook.sh:471-476`), successful `COMPLETE` removes it (`hooks/loop-codex-stop-hook.sh:820-827`), `STOP` removes it (`hooks/loop-codex-stop-hook.sh:830-857`), and ongoing loops update `current_round` in-place (`hooks/loop-codex-stop-hook.sh:864-867`). The start command also lists user cancellation as one of the formal stopping conditions (`commands/start-rlcr-loop.md:51-55`) while warning the worker not to self-cancel to evade review (`commands/start-rlcr-loop.md:43-49`; `scripts/setup-rlcr-loop.sh:380`).
- dependencies_and_callers: This markdown command depends on Claude/plugin slash-command execution semantics and its `allowed-tools` frontmatter. It is referenced by the RLCR start command as a stopping mechanism (`commands/start-rlcr-loop.md:51-55`), by setup help/output as the cancellation command (`scripts/setup-rlcr-loop.sh:76`, `scripts/setup-rlcr-loop.sh:416`), and by README usage/docs. It coordinates indirectly with `scripts/setup-rlcr-loop.sh`, which creates the state marker, and with RLCR hooks, which detect active loops through `.humanize-rlcr.local` and `rlcr-state.md`. It is not called by shell scripts as a function; it is a user-invoked command definition.
- edge_cases_or_failure_modes: The glob removes every matching `.humanize-rlcr.local/*/rlcr-state.md`, while hook active-loop discovery only considers the newest timestamp directory (`hooks/lib/loop-common.sh:11-32`). This broad deletion is useful for clearing stale state files, but it can cancel multiple surviving state markers at once. If the file exists but is malformed, the cancel command still instructs removal; it may not be able to report accurate `N of M`. If directories contain spaces or unusual names, the raw shell glob remains simple but unquoted; timestamp directories created by setup use safe names (`YYYY-MM-DD_HH-MM-SS`) (`scripts/setup-rlcr-loop.sh:210-214`). If `rm` fails due to permissions or race conditions, the command text does not define retry or failure reporting. If no match exists, `ls` prints `NO_LOOP`, which is the sole no-op gate.
