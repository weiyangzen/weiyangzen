- core_role: This file is the Claude slash-command workflow definition for starting the RLCR implementation loop. It is not the loop engine itself; it is a narrow command wrapper that exposes `/humanize:start-rlcr-loop`, declares accepted arguments, restricts allowed tool execution to the RLCR setup script, and delegates initialization to `${CLAUDE_PLUGIN_ROOT}/scripts/setup-rlcr-loop.sh` with the user-provided `$ARGUMENTS` (`commands/start-rlcr-loop.md:1-14`). In the algorithmic surface, this file is the entrypoint that binds the user command to the script/hook-based RLCR state machine.
- algorithmic_behavior: The command’s direct behavior is deterministic delegation: run `"${CLAUDE_PLUGIN_ROOT}/scripts/setup-rlcr-loop.sh" $ARGUMENTS` (`commands/start-rlcr-loop.md:10-14`). Its embedded workflow description defines the high-level loop: Claude works from the supplied implementation plan, writes a round summary, stop-hook review sends that summary to Codex, Codex feedback either blocks exit and starts another round or ends the loop when its last output is `COMPLETE` (`commands/start-rlcr-loop.md:16-23`). The file also establishes the Goal Tracker algorithm: Round 0 initializes immutable goal and acceptance criteria, later rounds maintain mutable task/progress/evolution sections, and every few rounds Codex performs full alignment checks (`commands/start-rlcr-loop.md:24-42`). The delegated setup script implements this by parsing options, validating inputs, creating `.humanize-rlcr.local/<timestamp>/`, writing `rlcr-state.md`, creating `rlcr-tracker.md`, and emitting `round-0-prompt.md` (`scripts/setup-rlcr-loop.sh:93-157`, `scripts/setup-rlcr-loop.sh:207-234`, `scripts/setup-rlcr-loop.sh:240-387`).
- inputs_outputs_state: Inputs are a required plan path plus optional `--max N`, `--codex-model MODEL:EFFORT`, `--codex-timeout SECONDS`, and `--push-every-round`, declared in the frontmatter argument hint (`commands/start-rlcr-loop.md:3`) and implemented by the setup parser (`scripts/setup-rlcr-loop.sh:93-157`). The setup script normalizes relative plan paths against `${CLAUDE_PROJECT_DIR:-$(pwd)}` (`scripts/setup-rlcr-loop.sh:173-177`). Persistent outputs are session files under `.humanize-rlcr.local/<timestamp>/`: `rlcr-state.md` containing `current_round`, `max_iterations`, Codex config, push policy, `plan_file`, and `started_at` (`scripts/setup-rlcr-loop.sh:223-234`), plus `rlcr-tracker.md` with immutable goal/acceptance criteria and mutable task/evolution/deferral/open-issue tables (`scripts/setup-rlcr-loop.sh:243-328`). It also creates `round-0-prompt.md` and identifies `round-0-summary.md` as the required Claude summary target (`scripts/setup-rlcr-loop.sh:334-387`). Runtime state transitions are owned by the stop hook: if Codex does not output terminal status, it increments `current_round` in `rlcr-state.md` and writes the next `round-N-prompt.md` and expected `round-N-summary.md` (`hooks/loop-codex-stop-hook.sh:860-933`). Terminal states remove `rlcr-state.md`: `COMPLETE` removes it after successful review (`hooks/loop-codex-stop-hook.sh:819-827`), `STOP` removes it for stagnation/circuit-breaker termination (`hooks/loop-codex-stop-hook.sh:830-857`), and max-iteration expiry removes it (`hooks/loop-codex-stop-hook.sh:471-477`). README confirms this state contract: loop state is controlled by presence of `.humanize-rlcr.local/*/rlcr-state.md`, and cancel removes that file (`README.md:147-163`).

Setup gates require exactly one plan file, reject missing or multiple plans, reject unknown options, require `--max` and `--codex-timeout` to be numeric, require the plan file to exist, require at least 5 plan lines, and require `codex` on PATH (`scripts/setup-rlcr-loop.sh:98-144`, `scripts/setup-rlcr-loop.sh:147-152`, `scripts/setup-rlcr-loop.sh:163-201`).
