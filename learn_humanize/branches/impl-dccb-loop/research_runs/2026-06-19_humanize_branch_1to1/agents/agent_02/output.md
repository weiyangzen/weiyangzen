### IMPL_DCCB_LOOP-HZ-002 `directory` `commands`

- core_role: The `commands/` directory is the slash-command entrypoint layer for the plugin’s iterative loop algorithms. It contains four hidden command definitions: DCCB start/cancel and RLCR start/cancel. The files themselves are Markdown command contracts with front matter, allowed tool scopes, argument hints, and operator instructions. They do not implement the review loops directly; they gate and delegate loop initialization to `scripts/setup-dccb-loop.sh` or `scripts/setup-rlcr-loop.sh`, and define the cancellation procedure by removing loop state files.

- algorithmic_behavior: `commands/start-dccb-loop.md` declares the DCCB start command with an argument shape of `[--max N] [--codex-model MODEL:EFFORT] [--codex-timeout SECONDS] [--output-dir DIR]` and restricts execution to `${CLAUDE_PLUGIN_ROOT}/scripts/setup-dccb-loop.sh` via `allowed-tools` in `commands/start-dccb-loop.md:1-5`. Its executable block delegates all initialization to that script with `$ARGUMENTS` in `commands/start-dccb-loop.md:10-14`. The rest of the file defines the DCCB loop contract: Claude produces reconstruction-ready architecture documentation, Codex reviews at stop time, feedback causes another refinement round, and `COMPLETE` ends the loop in `commands/start-dccb-loop.md:16-23`. It also establishes the adaptive documentation invariant: the output structure must scale with codebase size instead of using a fixed layout in `commands/start-dccb-loop.md:34-44`, and it defines de-specialization requirements such as avoiding line references, original file paths, and implementation-specific references in `commands/start-dccb-loop.md:81-99`.

  `commands/cancel-dccb-loop.md` defines the DCCB cancellation algorithm. It checks for `.humanize-dccb.local/*/dccb-state.md`, reports no active loop if none exists, otherwise reads current round state, removes the state file, and reports cancellation while preserving generated docs and review artifacts in `commands/cancel-dccb-loop.md:11-24`. This makes DCCB loop activeness depend on state-file presence, not on deleting the whole loop directory.

  `commands/start-rlcr-loop.md` is the analogous RLCR entrypoint. Its front matter accepts `<path/to/plan.md>` plus Codex/max/push options and only allows `${CLAUDE_PLUGIN_ROOT}/scripts/setup-rlcr-loop.sh` in `commands/start-rlcr-loop.md:1-5`. Its executable block delegates setup with `$ARGUMENTS` in `commands/start-rlcr-loop.md:10-14`. The behavior contract is plan-driven implementation with Codex stop-time review in `commands/start-rlcr-loop.md:16-23`. It adds the Goal Tracker algorithm: an immutable Ultimate Goal and Acceptance Criteria section plus mutable active/completed/deferred/evolution sections, with full alignment checks every fifth-style checkpoint after four rounds of work in `commands/start-rlcr-loop.md:24-42`.

  `commands/cancel-rlcr-loop.md` mirrors DCCB cancellation for `.humanize-rlcr.local/*/rlcr-state.md`: find active state, read round, remove state, report cancellation, and preserve summaries/review results in `commands/cancel-rlcr-loop.md:11-24`.

  The delegated setup scripts supply the actual initialization mechanics. `scripts/setup-dccb-loop.sh` parses `--max`, `--codex-model`, `--codex-timeout`, and `--output-dir`, rejects unexpected positional arguments, and requires Codex CLI availability in `scripts/setup-dccb-loop.sh:107-182`. It creates `.humanize-dccb.local/<timestamp>/`, creates the documentation output directory, writes `dccb-state.md`, initializes `dccb-tracker.md`, creates `round-0-prompt.md`, and emits that prompt back to the active Claude session in `scripts/setup-dccb-loop.sh:188-541`. `scripts/setup-rlcr-loop.sh` parses the plan path and options, requires a real plan file with at least five lines, requires Codex CLI availability, creates `.humanize-rlcr.local/<timestamp>/`, writes `rlcr-state.md`, initializes `rlcr-tracker.md`, creates `round-0-prompt.md`, and emits the prompt in `scripts/setup-rlcr-loop.sh:93-452`.

- inputs_outputs_state: Inputs at the command layer are slash command arguments, `$ARGUMENTS`, `${CLAUDE_PLUGIN_ROOT}`, and the current project directory. DCCB accepts only options and analyzes the current codebase; its setup script explicitly rejects positional arguments in `scripts/setup-dccb-loop.sh:159-168`.   DCCB start outputs a new loop state directory, `dccb-state.md`, `dccb-tracker.md`, `round-0-prompt.md`, and the requested documentation directory.   The primary state transition is absent loop state to active loop state when a start command delegates to setup. The stop hooks then advance rounds by rewriting `current_round`; for DCCB this happens after Codex finds gaps, when `loop-dccb-stop-hook.sh` writes `current_round: $NEXT_ROUND` and creates the next prompt in `hooks/loop-dccb-stop-hook.sh:821-925`.
