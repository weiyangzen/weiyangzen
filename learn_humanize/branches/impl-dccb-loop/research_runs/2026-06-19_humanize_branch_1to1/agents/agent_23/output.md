### IMPL_DCCB_LOOP-HZ-023 `file` `hooks/lib/loop-common.sh`
- cursor: `[_]`
- core_role: Shared Bash utility library for loop hook validation. The file centralizes common RLCR loop path, round, state-file, and blocked-message logic used by the PreToolUse validators and the RLCR stop hook. It is also sourced by the DCCB stop hook, but DCCB currently implements its own active-loop discovery for `.humanize-dccb.local` rather than using the RLCR-specific `find_active_loop` helper.
- algorithmic_behavior: The library provides small deterministic predicates and extractors rather than a full scheduler loop. `find_active_loop` accepts a loop base directory, chooses the lexicographically newest child directory via `ls -1d ... | sort -r | head -1`, and only returns it if that newest directory contains `rlcr-state.md`; older directories are ignored to prevent resurrecting stale loops after abnormal exits (`hooks/lib/loop-common.sh:15`, `hooks/lib/loop-common.sh:25`, `hooks/lib/loop-common.sh:27`). `get_current_round` extracts YAML frontmatter between `---` markers, reads `current_round:`, strips spaces, and defaults to `0` (`hooks/lib/loop-common.sh:37`, `hooks/lib/loop-common.sh:41`, `hooks/lib/loop-common.sh:44`, `hooks/lib/loop-common.sh:46`). `to_lower`, `is_round_file_type`, and `extract_round_number` normalize and classify round files matching `round-N-summary.md`, `round-N-prompt.md`, or `round-N-todos.md` (`hooks/lib/loop-common.sh:50`, `hooks/lib/loop-common.sh:56`, `hooks/lib/loop-common.sh:66`). Message functions emit standardized denial text for blocked todos, prompt writes, state-file modifications, summary writes via Bash, and tracker writes (`hooks/lib/loop-common.sh:77`, `hooks/lib/loop-common.sh:93`, `hooks/lib/loop-common.sh:111`, `hooks/lib/loop-common.sh:128`, `hooks/lib/loop-common.sh:145`, `hooks/lib/loop-common.sh:205`). Predicate helpers classify goal tracker paths, state paths, and `.humanize-rlcr.local` membership (`hooks/lib/loop-common.sh:161`, `hooks/lib/loop-common.sh:167`, `hooks/lib/loop-common.sh:173`). `command_modifies_file` is the main shell-command gate: it tests a lowercased Bash command against regexes for redirection, `tee`, in-place `sed`, in-place `awk`, in-place `perl`, `mv` or `cp` into a protected path, and `dd of=` writes (`hooks/lib/loop-common.sh:181`, `hooks/lib/loop-common.sh:185`).
- inputs_outputs_state: Inputs are plain strings supplied by callers: base directory paths, state-file paths, candidate file paths, filenames, command text, file regex fragments, current round, and expected summary/tracker paths. Outputs are stdout values for lookup/extraction helpers, process return codes for predicates, and markdown-formatted denial messages emitted by message helpers. The library itself does not mutate files or persistent state. Its state model is indirect: validators use the returned active loop directory and round number to decide whether a Claude tool action is allowed, and stop hooks use the same state source to determine whether a loop remains active.
- gates_or_invariants: The active RLCR loop invariant is “only the newest session directory may be active, and only if it has `rlcr-state.md`” (`hooks/lib/loop-common.sh:11`). Round-file gates require `round-[0-9]+-{summary,prompt,todos}.md` naming and compare extracted round numbers to the current round. Access-control gates enforce that todos files are inaccessible through Read/Write/Edit/Bash and should use TodoWrite instead (`hooks/lib/loop-common.sh:80`), prompt files are read-only instructions from Codex to Claude (`hooks/lib/loop-common.sh:93`), `rlcr-state.md` is system-managed (`hooks/lib/loop-common.sh:111`), Bash cannot modify summaries because it bypasses Write/Edit validation (`hooks/lib/loop-common.sh:128`), and after round 0 Claude must request tracker changes through the summary instead of editing `rlcr-tracker.md` directly (`hooks/lib/loop-common.sh:205`). The path predicates encode a looser suffix invariant: any lowercased path ending in `state.md` or `goal-tracker.md` is treated as protected by callers (`hooks/lib/loop-common.sh:163`, `hooks/lib/loop-common.sh:169`).
- dependencies_and_callers: Direct dependencies are standard shell utilities: `ls`, `sort`, `head`, `sed`, `grep`, and `tr`. The library assumes Bash arrays for `command_modifies_file`. It is sourced by all loop validators through `source "$SCRIPT_DIR/lib/loop-common.sh"`: read validator (`hooks/loop-read-validator.sh:16`), write validator (`hooks/loop-write-validator.sh:17`), edit validator (`hooks/loop-edit-validator.sh:16`), bash validator (`hooks/loop-bash-validator.sh:16`), RLCR stop hook (`hooks/loop-codex-stop-hook.sh:48`), and DCCB stop hook (`hooks/loop-dccb-stop-hook.sh:41`). The hook registry wires those validators into Claude PreToolUse and Stop events (`hooks/hooks.json:4`, `hooks/hooks.json:42`). In RLCR callers, the common functions are operationally central: read validation blocks todos, wrong-session, wrong-round, and wrong-directory reads (`hooks/loop-read-validator.sh:36`, `hooks/loop-read-validator.sh:69`, `hooks/loop-read-validator.sh:75`, `hooks/loop-read-validator.sh:118`); write/edit validation blocks prompt/state/tracker and wrong summary targets (`hooks/loop-write-validator.sh:37`, `hooks/loop-write-validator.sh:42`, `hooks/loop-write-validator.sh:92`, `hooks/loop-write-validator.sh:101`, `hooks/loop-edit-validator.sh:72`, `hooks/loop-edit-validator.sh:81`); bash validation blocks protected writes and optional `git push` during RLCR (`hooks/loop-bash-validator.sh:38`, `hooks/loop-bash-validator.sh:74`, `hooks/loop-bash-validator.sh:85`, `hooks/loop-bash-validator.sh:101`, `hooks/loop-bash-validator.sh:111`, `hooks/loop-bash-validator.sh:121`). For DCCB, the stop hook sources this file but performs DCCB-specific active-loop detection by scanning for `dccb-state.md` in `.humanize-dccb.local` (`hooks/loop-dccb-stop-hook.sh:43`, `hooks/loop-dccb-stop-hook.sh:47`, `hooks/loop-dccb-stop-hook.sh:60`), so the common library is a light dependency there rather than the primary DCCB state-machine implementation.
- edge_cases_or_failure_modes: `find_active_loop` intentionally ignores older valid loop directories if the newest child lacks `rlcr-state.md`; that prevents stale loop revival but can also hide an older legitimate active session (`hooks/lib/loop-common.sh:12`, `hooks/lib/loop-common.sh:25`, `hooks/lib/loop-common.sh:27`). It sorts names lexicographically rather than by filesystem mtime despite the comment saying “by timestamp name,” so correctness depends on timestamp-like directory names. Directory names with whitespace can be fragile in callers that iterate unquoted command substitution, though this function itself only echoes the selected path. `get_current_round` silently returns `0` for missing files, malformed frontmatter, absent keys, or command failures, which is safe for avoiding crashes but can make validators apply round-0 permissions to corrupted state (`hooks/lib/loop-common.sh:41`, `hooks/lib/loop-common.sh:46`). `is_state_file_path` matches any `state.md` suffix, not strictly `rlcr-state.md`, so callers may block broader filenames inside the loop directory (`hooks/lib/loop-common.sh:167`). `is_goal_tracker_path` checks `goal-tracker.md`, while generated loop names and messages use `rlcr-tracker.md` in several places; this mismatch means direct Write/Edit protection may not catch `rlcr-tracker.md` by suffix through that predicate, while Bash protection uses `goal-tracker\.md` and has the same naming mismatch (`hooks/lib/loop-common.sh:161`, `hooks/loop-write-validator.sh:101`, `hooks/loop-bash-validator.sh:85`). `is_in_humanize_loop_dir` is hardcoded to `.humanize-rlcr.local/`, so it does not classify `.humanize-dccb.local/` paths (`hooks/lib/loop-common.sh:173`). `command_modifies_file` catches common shell write forms but is regex-based and can miss quoted paths with spaces, alternate shells, command substitutions, `install`, `rsync`, `python -c`, `node -e`, editor commands, multi-line scripts with unusual spacing, or writes hidden behind variables.
