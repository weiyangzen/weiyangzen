- core_role: `hooks/lib` is the shared shell-library layer for loop hook validation. Recursive inspection found one descendant: `hooks/lib/loop-common.sh` at 236 lines / 7007 bytes. The file is sourced by the RLCR read/write/edit/bash validators and both stop-hook scripts via `source "$SCRIPT_DIR/lib/loop-common.sh"` in `hooks/loop-read-validator.sh:15-16`, `hooks/loop-write-validator.sh:16-17`, `hooks/loop-edit-validator.sh:15-16`, `hooks/loop-bash-validator.sh:15-16`, `hooks/loop-codex-stop-hook.sh:47-50`, and `hooks/loop-dccb-stop-hook.sh:39-41`. - algorithmic_behavior: The directory centralizes common path classification, active-loop lookup, state-frontmatter parsing, write-bypass detection, and standard block messages. `find_active_loop` accepts a loop base directory, lists timestamp/session children in descending lexical order, and returns only the newest child that contains `rlcr-state.md`; otherwise it emits an empty string, deliberately avoiding older “zombie” loop sessions (`hooks/lib/loop-common.sh:15-33`). `get_current_round` extracts YAML frontmatter between `---` delimiters, reads `current_round:`, strips spaces, and defaults to `0` when missing or unreadable (`hooks/lib/loop-common.sh:37-47`). `to_lower`, `is_round_file_type`, and `extract_round_number` normalize and classify `round-N-summary.md`, `round-N-prompt.md`, and `round-N-todos.md` filenames (`hooks/lib/loop-common.sh:49-73`). Message helpers emit consistent operator guidance for blocked todos, prompt writes, state edits, summary Bash writes, goal-tracker Bash writes, and post-round-0 goal-tracker edits (`hooks/lib/loop-common.sh:75-158`, `hooks/lib/loop-common.sh:203-235`). Path predicates classify goal tracker, state file, and `.humanize-rlcr.local/` membership (`hooks/lib/loop-common.sh:160-176`). `command_modifies_file` scans a lowercased shell command for common mutation forms: redirection, `tee`, `sed -i`, `awk -i inplace`, `perl -i`, `mv` / `cp`, and `dd of=` (`hooks/lib/loop-common.sh:178-201`).
- inputs_outputs_state: Inputs are shell strings and paths supplied by parent hooks: loop base directories, state-file paths, hook tool paths, command text, and filename patterns. Outputs are stdout-returned scalar values for lookup/parsing helpers, Bash return codes for predicates, and stderr/stdout heredoc block messages consumed by hook scripts before they exit with block status. The library itself has no persistent mutable state and performs no writes. State transitions happen in callers: validators exit `0` to allow or `2` to block, while stop hooks use the shared values to decide whether to continue expensive loop review. `hooks/hooks.json:4-40` wires these validators into `PreToolUse`, and `hooks/hooks.json:42-54` wires the RLCR and DCCB stop hooks into `Stop`.
- gates_or_invariants: The main invariant is that only the newest RLCR loop directory with `rlcr-state.md` is active, which prevents stale sessions from being revived (`hooks/lib/loop-common.sh:11-33`). Todos files are always blocked in read/write/edit/bash flows so agents use native todo tooling instead (`hooks/lib/loop-common.sh:75-90`; applied in `hooks/loop-read-validator.sh:36-39`, `hooks/loop-write-validator.sh:37-40`, `hooks/loop-edit-validator.sh:36-39`, `hooks/loop-bash-validator.sh:121-123`).
