### IMPL_DCCB_LOOP-HZ-017 `file` `hooks/loop-read-validator.sh`
- cursor: `[_]`
- core_role: PreToolUse read guard for loop round artifacts. It validates Claude `Read` tool access to RLCR prompt and summary files so the agent reads only the active loop directory and active round, while also blocking `round-*-todos.md` reads entirely. The hook is registered for the `Read` matcher in `hooks/hooks.json:23-30`, alongside write/edit/bash validators and stop hooks.

- algorithmic_behavior: The script consumes the hook JSON payload from stdin, extracts `.tool_name`, and exits successfully unless the tool is exactly `Read` (`hooks/loop-read-validator.sh:22-27`). For `Read`, it extracts `.tool_input.file_path`, lowercases it through `to_lower`, and first blocks any `round-[0-9]+-todos.md` target with `todos_blocked_message` plus exit code `2` (`hooks/loop-read-validator.sh:29-39`). It then classifies only `round-N-summary.md` and `round-N-prompt.md` as round files; other reads are allowed (`hooks/loop-read-validator.sh:45-55`). For classified round files, it checks whether the path contains `.humanize-rlcr.local/`, locates the newest active RLCR loop with `find_active_loop`, reads `current_round` from `rlcr-state.md`, extracts the requested round number from the basename, and computes the file type (`hooks/loop-read-validator.sh:57-92`). It rejects three mismatch classes with user-facing stderr guidance and exit code `2`: loop file outside `.humanize-rlcr.local/` (`hooks/loop-read-validator.sh:98-112`), wrong round number (`hooks/loop-read-validator.sh:118-131`), or right filename but wrong active loop directory/path (`hooks/loop-read-validator.sh:137-149`). A fully matching current-round active-loop path exits `0` (`hooks/loop-read-validator.sh:151`).

- inputs_outputs_state: Input is Claude hook JSON on stdin, expected to contain `tool_name` and `tool_input.file_path` (`hooks/loop-read-validator.sh:22-30`). Output is no stdout; allowed reads exit `0`. Blocked reads emit Markdown-formatted diagnostic text to stderr and exit `2`, matching Claude hook blocking semantics. State is read-only: it derives `PROJECT_ROOT` from `CLAUDE_PROJECT_DIR` or `pwd`, uses `$PROJECT_ROOT/.humanize-rlcr.local` as the loop base, and reads the active loop state from `$ACTIVE_LOOP_DIR/rlcr-state.md` (`hooks/loop-read-validator.sh:67-75`). It does not mutate files. The shared helper defines active loop discovery as the lexicographically newest timestamp directory that contains `rlcr-state.md`, ignoring older directories to avoid reviving stale loops (`hooks/lib/loop-common.sh:11-33`). Current round is parsed from YAML frontmatter key `current_round:` and defaults to `0` if missing/unparseable (`hooks/lib/loop-common.sh:35-47`).

- gates_or_invariants: Only Claude `Read` is in scope; non-Read tool invocations are pass-through (`hooks/loop-read-validator.sh:25-27`). Todos files are never readable through the `Read` tool regardless of location, enforcing the invariant that native TodoWrite is used instead of `round-*-todos.md` artifacts (`hooks/loop-read-validator.sh:36-39`, `hooks/lib/loop-common.sh:75-90`). Prompt and summary reads must satisfy all of these conditions: filename matches `round-[0-9]+-(prompt|summary).md`, path contains `.humanize-rlcr.local/`, round number equals `current_round`, and full path equals `$ACTIVE_LOOP_DIR/$CLAUDE_FILENAME` (`hooks/loop-read-validator.sh:48-55`, `hooks/loop-read-validator.sh:98-149`). README state docs align with this model: RLCR artifacts live in `.humanize-rlcr.local/<TIMESTAMP>/`, with `rlcr-state.md`, `round-N-prompt.md`, and `round-N-summary.md` among the created files (`README.md:409-421`). The README also states loop state is controlled by presence of `.humanize-rlcr.local/*/rlcr-state.md` (`README.md:149-160`).

- dependencies_and_callers: Depends on `jq` for JSON extraction (`hooks/loop-read-validator.sh:22-30`), shell utilities `basename`, `grep`, `sed`, `tr`, `ls`, `sort`, and `head` through direct and shared-helper logic (`hooks/lib/loop-common.sh:23-72`). It sources `hooks/lib/loop-common.sh` using the script directory resolved from `BASH_SOURCE` (`hooks/loop-read-validator.sh:14-16`). Helper functions used are `to_lower`, `is_round_file_type`, `todos_blocked_message`, `is_in_humanize_loop_dir`, `find_active_loop`, `get_current_round`, and `extract_round_number` (`hooks/lib/loop-common.sh:15-90`, `hooks/lib/loop-common.sh:172-176`). The caller is Claude Code’s hook runner via `hooks/hooks.json:23-30`. Sibling validators enforce related write/edit/bash constraints: write/edit validators use the same shared round-file and current-round helpers, and bash validator blocks shell-side modifications to protected loop files, so this read validator is the read-side part of a broader RLCR artifact access policy.

- edge_cases_or_failure_modes: If no active loop directory exists, the hook allows the read (`hooks/loop-read-validator.sh:69-73`), which avoids blocking normal project reads outside a live loop. If a round-like basename fails round extraction, it allows the read (`hooks/loop-read-validator.sh:81-84`); because prior classification is stricter, this mainly protects malformed/unexpected names. Case handling is mixed: matching and round extraction are case-insensitive because the path or filename is lowercased, but final full-path equality is case-sensitive (`hooks/loop-read-validator.sh:29-30`, `hooks/loop-read-validator.sh:137-149`). `is_in_humanize_loop_dir` is a substring grep for `.humanize-rlcr.local/`, not canonical path resolution, so symlinks, relative path normalization, `..`, or alternate spellings are not resolved before the final equality check (`hooks/lib/loop-common.sh:172-176`). Active loop selection is based on reverse lexical sort of child directory names, which assumes timestamp-like names; a differently named newest directory could win or an older directory with a lexically greater name could be selected (`hooks/lib/loop-common.sh:23-31`). `get_current_round` defaults to `0` if the state file lacks a valid frontmatter value, which could make the validator accept only `round-0-*` paths during state corruption (`hooks/lib/loop-common.sh:40-47`). Although this branch includes DCCB state docs using `.humanize-dccb.local/<TIMESTAMP>/` (`README.md:429-439`), this read validator is RLCR-specific and does not validate DCCB prompt/summary reads.
