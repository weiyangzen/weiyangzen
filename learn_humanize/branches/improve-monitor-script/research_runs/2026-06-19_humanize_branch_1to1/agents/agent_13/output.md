### IMPROVE_MONITOR_SCRIPT-HZ-043 `file` `prompt-template/block/git-push.md`
- cursor: `[_]`
- core_role: Policy template for the “git push blocked” gate during an active RLCR loop. It is small, but algorithmically relevant because it is the user-facing block reason emitted when the Bash validator detects a prohibited `git push` while `push_every_round` is false.
- algorithmic_behavior: The template states that commits should stay local, the loop will handle local commits until completion, and pushing requires starting the loop with `--push-every-round` (`prompt-template/block/git-push.md:1-9`). It does not compute policy by itself; it supplies the rendered enforcement message for the Bash pre-tool gate.
- inputs_outputs_state: Input is implicit template selection by `hooks/loop-bash-validator.sh` when an active loop exists and parsed `STATE_PUSH_EVERY_ROUND` is not `true` (`hooks/loop-bash-validator.sh:36-49`, `hooks/loop-bash-validator.sh:56-59`). The command input inspected by the gate is the Bash tool command, lowercased into `COMMAND_LOWER` (`hooks/loop-bash-validator.sh:22-31`). Output is the Markdown block text written to stderr via `load_and_render_safe`, followed by hook exit code `2`, which blocks the Bash command (`hooks/loop-bash-validator.sh:59-67`).
- gates_or_invariants: The gate invariant is: during an active loop, `git push` is forbidden unless the session state has `push_every_round: true`. State parsing gets that field from `state.md` through `parse_state_file`, with a default of `false` if absent (`hooks/lib/loop-common.sh:120-135`). The setup script writes `push_every_round: $PUSH_EVERY_ROUND` into `state.md` (`scripts/setup-rlcr-loop.sh:485-498`), and only sets it true when `--push-every-round` is provided (`scripts/setup-rlcr-loop.sh:148-151`). The matching expression is anchored to commands beginning with optional whitespace then `git push`, so the direct command is blocked, while unrelated commands are not evaluated by this template (`hooks/loop-bash-validator.sh:59-67`).
- dependencies_and_callers: Direct caller is `hooks/loop-bash-validator.sh`, loaded as the `PreToolUse` Bash hook in `hooks/hooks.json` (`hooks/hooks.json:42-48`). That validator depends on `hooks/lib/loop-common.sh` for active-loop discovery, state parsing, and template loading (`hooks/loop-bash-validator.sh:14-17`, `hooks/lib/loop-common.sh:46-56`). The positive push path is separately reflected in `prompt-template/claude/push-every-round-note.md`, which the stop hook appends to next-round prompts only when `PUSH_EVERY_ROUND` is true (`hooks/loop-codex-stop-hook.sh:1004-1011`).
