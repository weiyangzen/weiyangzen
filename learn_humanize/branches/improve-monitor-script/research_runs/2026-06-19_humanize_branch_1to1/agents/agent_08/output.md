### IMPROVE_MONITOR_SCRIPT-HZ-038 `file` `hooks/lib/template-loader.sh`
- cursor: `[_]`
- core_role: `hooks/lib/template-loader.sh` is the RLCR prompt-template engine. It is not a policy gate by itself, but it is core plumbing for every rendered validator block, Codex review prompt, Claude follow-up prompt, and safe fallback path used by the RLCR state machine.
- algorithmic_behavior: `get_template_dir` computes the plugin root by moving two levels up from `hooks/lib` and returns `prompt-template` at `hooks/lib/template-loader.sh:24-31`. `load_template` concatenates `template_dir/template_name` and emits file contents if present, otherwise warns on stderr and returns an empty string at `hooks/lib/template-loader.sh:33-48`. `render_template` accepts content plus `VAR=value` assignments, maps them into `TMPL_VAR_*` environment variables, and runs an AWK single-pass renderer at `hooks/lib/template-loader.sh:50-132`. The AWK scanner appends literal text until `{{`, finds the next `}}`, extracts the full inner variable name, replaces it if present, and otherwise preserves the original placeholder at `hooks/lib/template-loader.sh:84-128`. `load_and_render` composes load plus render at `hooks/lib/template-loader.sh:134-147`; `append_template` appends another template’s raw content at `hooks/lib/template-loader.sh:149-161`; `load_and_render_safe` falls back when a template is missing or rendering is empty at `hooks/lib/template-loader.sh:167-203`; `validate_template_dir` requires `block`, `codex`, and `claude` subdirectories at `hooks/lib/template-loader.sh:205-222`.
- inputs_outputs_state: Inputs are a template directory, a relative template path such as `block/git-not-clean.md`, template content containing `{{VARIABLE}}` placeholders, and `VAR=value` pairs. Outputs are rendered prompt/block strings on stdout, warnings/errors on stderr, and shell return statuses. It does not persist state directly; it influences RLCR state transitions indirectly by determining the text attached to block decisions and generated prompts.
- gates_or_invariants: Rendering is intentionally single-pass: values containing `{{OTHER_VAR}}` are emitted literally and are not recursively expanded, which prevents review content or user text from injecting new substitutions. Missing variables stay as placeholders. Missing templates return empty from `load_template`, while `load_and_render_safe` ensures critical callers can still emit fallback text. Directory validation is structural only, requiring the three expected template categories, not validating every file’s placeholder set.
- dependencies_and_callers: `loop-common.sh` sources this file and initializes `TEMPLATE_DIR` at `hooks/lib/loop-common.sh:46-56`, making these functions available to every validator. Stop-hook callers render plan-modified, incomplete-todos, git-status, large-files, git-clean, unpushed-commits, work-summary, goal-tracker, Codex review, next-round prompt/footer templates at `hooks/loop-codex-stop-hook.sh:232-395`, `hooks/loop-codex-stop-hook.sh:489-592`, `hooks/loop-codex-stop-hook.sh:641-709`, and `hooks/loop-codex-stop-hook.sh:985-1018`. File validators call it for wrong-location, wrong-round, protected-plan, and schema-outdated blocks at `hooks/loop-read-validator.sh:99-135`, `hooks/loop-write-validator.sh:107-183`, `hooks/loop-edit-validator.sh:92-129`, and `hooks/loop-plan-file-validator.sh:53-69`.
- edge_cases_or_failure_modes: `load_template` exits successfully with empty output for missing files, so unsafe direct callers can accidentally produce empty messages unless they use `load_and_render_safe`. `render_template` preserves unclosed `{{` literally by emitting the opening braces and continuing, and preserves unknown placeholders. The documented placeholder syntax says uppercase/numbers/underscore, but the renderer itself does not enforce the variable-name regex; syntax validation is handled by tests, not by runtime rendering. Multiline values are passed through environment variables into AWK and are covered by tests, but this design depends on shell/environment handling of large values.
