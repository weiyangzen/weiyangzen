### IMPROVE_MONITOR_SCRIPT-HZ-048 `file` `prompt-template/block/incomplete-todos.md`
- cursor: `[_]`
- core_role: Stop-hook block template for the “incomplete todos” gate. It is not executable code, but it defines the user-facing contract that blocks loop shutdown until the latest TodoWrite state has no incomplete items.
- algorithmic_behavior: The template renders a Markdown block headed “Incomplete Todos Detected” (`prompt-template/block/incomplete-todos.md:1`). It interpolates `{{INCOMPLETE_LIST}}` into the reason body (`prompt-template/block/incomplete-todos.md:3-5`) and instructs the agent to complete remaining todos, mark each todo completed through TodoWrite, then write the summary and stop only after all todos are complete (`prompt-template/block/incomplete-todos.md:7-10`). It explicitly prevents proceeding to Codex review while todos remain unfinished (`prompt-template/block/incomplete-todos.md:12`).
- inputs_outputs_state: Input is the rendered variable `INCOMPLETE_LIST`, supplied by `hooks/loop-codex-stop-hook.sh` after running `hooks/check-todos-from-transcript.py` (`hooks/loop-codex-stop-hook.sh:246-283`). The template output becomes the `reason` string in a JSON object with `"decision": "block"` and system message `"Loop: Blocked - incomplete todos detected, please finish all tasks first"` (`hooks/loop-codex-stop-hook.sh:285-292`). It does not maintain state itself; state is derived from the latest TodoWrite call in the Claude transcript.
- gates_or_invariants: The associated stop-hook gate runs before expensive Codex review (`hooks/loop-codex-stop-hook.sh:240-244`). If the todo checker exits `1`, the stop hook blocks immediately and renders this template (`hooks/loop-codex-stop-hook.sh:272-293`). The checker treats any todo whose `status` is not exactly `completed` as incomplete (`hooks/check-todos-from-transcript.py:113-120`). Therefore `pending`, `in_progress`, empty, or unknown statuses all keep the loop from stopping. The invariant expressed by the template is simple: all latest TodoWrite entries must be completed before summary-and-stop can proceed.
- dependencies_and_callers: Direct caller is `hooks/loop-codex-stop-hook.sh`, which loads the template through `load_and_render_safe "$TEMPLATE_DIR" "block/incomplete-todos.md"` (`hooks/loop-codex-stop-hook.sh:277-283`). The data producer is `hooks/check-todos-from-transcript.py`, which reads hook input JSON, expands `transcript_path`, scans JSONL transcript entries, and records the most recent non-empty TodoWrite list (`hooks/check-todos-from-transcript.py:90-108`, `20-87`). Rendering uses `hooks/lib/template-loader.sh`; substitution is single-pass and preserves unknown placeholders rather than recursively expanding injected template syntax (`hooks/lib/template-loader.sh:50-58`, `71-131`, `170-203`).
- edge_cases_or_failure_modes: If the Python checker file is missing, the stop hook skips this gate entirely because it is guarded by `[[ -f "$TODO_CHECKER" ]]` (`hooks/loop-codex-stop-hook.sh:246-248`). Invalid hook input JSON makes the checker exit `2`, and the stop hook returns a different parse-error block instead of this template (`hooks/loop-codex-stop-hook.sh:253-269`; `hooks/check-todos-from-transcript.py:92-97`). Missing `transcript_path`, a nonexistent transcript file, or no TodoWrite entries allow stopping because the checker exits `0` (`hooks/check-todos-from-transcript.py:99-111`, `25-27`). Invalid JSONL transcript lines are ignored (`hooks/check-todos-from-transcript.py:36-39`). Only non-empty TodoWrite lists update `latest_todos`, so an empty TodoWrite call does not clear a prior non-empty list (`hooks/check-todos-from-transcript.py:61-64`, `73-76`, `83-85`). The template has no escaping layer beyond Markdown rendering; malformed or very long todo content would be embedded into the block reason as supplied by the checker.
