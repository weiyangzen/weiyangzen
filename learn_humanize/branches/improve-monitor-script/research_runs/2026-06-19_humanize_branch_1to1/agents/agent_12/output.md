- cursor: `[_]`
- core_role: Slash-command workflow definition for user-initiated RLCR loop cancellation. It provides the operator-facing algorithm for converting the newest active loop from resumable/running state into preserved cancelled state by renaming `.humanize/rlcr/<timestamp>/state.md` to `.humanize/rlcr/<timestamp>/cancel-state.md` (`commands/cancel-rlcr-loop.md:7`, `commands/cancel-rlcr-loop.md:28-31`).
- algorithmic_behavior: The command first selects the current loop directory as the lexicographically newest timestamp folder under `.humanize/rlcr/` using `ls -1d .humanize/rlcr/*/ | sort -r | head -1` (`commands/cancel-rlcr-loop.md:11-16`). It then treats the loop as active only if `${LOOP_DIR}state.md` exists (`commands/cancel-rlcr-loop.md:20-24`). On active state, it reads state metadata for current round/max iterations and performs the transition by moving `state.md` to `cancel-state.md` (`commands/cancel-rlcr-loop.md:28-31`). This matches the shared active-loop detector, which intentionally inspects only the newest loop directory and returns it only when `state.md` exists (`hooks/lib/loop-common.sh:58-79`).
- inputs_outputs_state: Inputs are the local `.humanize/rlcr/*/` directory set, the selected newest loop folder, and its `state.md` contents containing at least current round/max iteration context (`commands/cancel-rlcr-loop.md:11-15`, `commands/cancel-rlcr-loop.md:28-31`). Outputs are user-facing status messages for no loop/no active state/cancel success (`commands/cancel-rlcr-loop.md:18`, `commands/cancel-rlcr-loop.md:26`, `commands/cancel-rlcr-loop.md:31`) plus a filesystem state transition from active `state.md` to terminal `cancel-state.md`. The loop artifacts, summaries, reviews, and state information are preserved in place (`commands/cancel-rlcr-loop.md:35`).
- gates_or_invariants: The key invariant is that the newest timestamp directory is the only current loop candidate, and active status is determined solely by the presence of `state.md` there (`commands/cancel-rlcr-loop.md:33`). Older directories are ignored even if they contain state files; `find_active_loop` documents this as protection against reviving stale loops (`hooks/lib/loop-common.sh:58-60`). Cancellation is non-destructive with respect to loop evidence: it renames the control file rather than deleting the loop folder (`commands/cancel-rlcr-loop.md:30-35`). The command metadata restricts allowed tools to listing loop dirs, moving `state.md` to `cancel-state.md`, reading/catting state, and `Read` (`commands/cancel-rlcr-loop.md:1-4`).
- dependencies_and_callers: The command is surfaced by setup/help text as `/humanize:cancel-rlcr-loop` (`scripts/setup-rlcr-loop.sh:86-88`) and documented as the recommended cancellation path (`README.md:120-127`). Several validators and prompts reference it as the operator escape hatch when plan-file integrity, schema compatibility, or branch invariants block continuation, including `prompt-template/block/plan-file-modified.md`, `prompt-template/block/schema-outdated.md`, and hook messages found by repository search. It depends on the RLCR storage convention `.humanize/rlcr/<timestamp>/state.md`, which is also used by monitoring/status code in `scripts/humanize.sh` and common hook logic.
- edge_cases_or_failure_modes: If no `.humanize/rlcr/*/` directory exists, the workflow reports no active loop (`commands/cancel-rlcr-loop.md:18`). If a newest loop directory exists but lacks `state.md`, it reports that no active loop exists while acknowledging the directory (`commands/cancel-rlcr-loop.md:26`). Because only the newest timestamp is considered, an older directory with `state.md` will not be cancelled by this command; that is consistent with the anti-zombie invariant but can surprise an operator with stale state. The allowed-tool declaration includes a broad globbed `mv .humanize/rlcr/*/state.md .humanize/rlcr/*/cancel-state.md` pattern (`commands/cancel-rlcr-loop.md:3`), while the written algorithm uses a single resolved `LOOP_DIR`; correctness depends on following the algorithm rather than applying the broad glob blindly when multiple state files exist.
