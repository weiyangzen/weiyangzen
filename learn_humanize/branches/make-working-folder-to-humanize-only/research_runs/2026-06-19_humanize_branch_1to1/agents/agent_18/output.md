- core_role: PreToolUse Edit-path guard for the RLCR loop. The hook is registered for the `Edit` matcher in `hooks/hooks.json:24-30`.
- algorithmic_behavior: The hook reads the full hook JSON from stdin, extracts `.tool_name` and `.tool_input.file_path` with `jq`, and exits early unless the tool is exactly `Edit` (`hooks/loop-edit-validator.sh:22-30`). It lowercases the target path for case-insensitive matching of round files and state/goal tracker suffixes (`hooks/loop-edit-validator.sh:29-30`). It globally blocks `round-N-todos.md` edits and `round-N-prompt.md` edits before active-loop detection (`hooks/loop-edit-validator.sh:36-44`). It then only applies deeper loop guards to paths containing `.humanize/rlcr/` (`hooks/loop-edit-validator.sh:50-52`). For loop paths, it locates the active loop as the newest loop directory with `state.md`, reads `current_round`, then applies state-file, plan-backup, goal-tracker, and summary-round gates (`hooks/loop-edit-validator.sh:58-133`).
- inputs_outputs_state: Inputs are hook JSON on stdin, `CLAUDE_PROJECT_DIR` or the current working directory as project root, `.humanize/rlcr/<timestamp>/state.md`, and prompt templates loaded through `hooks/lib/loop-common.sh`. Outputs are only an exit status and optional stderr block message: `0` allows the edit; `2` blocks it with a rendered reason. The validator does not write files or advance state. It preserves the invariant that `state.md` is modified only by loop machinery, while `state.md` itself controls active-loop presence and current-round selection. The repository docs define active-loop state as the presence of `state.md` in the newest `.humanize/rlcr/` directory (`README.md:116-118`, `README.md:244-260`), and setup creates the state frontmatter with `current_round`, max iteration/config fields, plan tracking, branch, and timestamp (`scripts/setup-rlcr-loop.sh:406-418`).
- gates_or_invariants: Todos and prompt files are blocked by round-file suffix regardless of loop activity (`hooks/loop-edit-validator.sh:36-44`). Non-`.humanize/rlcr/` paths are allowed after those global gates (`hooks/loop-edit-validator.sh:50-52`). If no active loop exists, loop-path edits are allowed because `find_active_loop` returns empty unless the newest loop directory has `state.md` (`hooks/loop-edit-validator.sh:60-64`; helper behavior at `hooks/lib/loop-common.sh:23-44`). `state.md` edits under an active loop are always blocked (`hooks/loop-edit-validator.sh:72-75`). `plan.md` inside `.humanize/rlcr/` is blocked as a protected backup (`hooks/loop-edit-validator.sh:81-88`). `goal-tracker.md` is blocked after round 0 and redirected toward the current round summary (`hooks/loop-edit-validator.sh:95-99`). `round-N-summary.md` can only target the current round; wrong-round edits are redirected to `round-${CURRENT_ROUND}-summary.md` (`hooks/loop-edit-validator.sh:105-129`).
