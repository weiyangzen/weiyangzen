- core_role: Codex-side review prompt templates for the RLCR loop. It contains three files: `full-alignment-review.md`, `regular-review.md`, and `goal-tracker-update-section.md`.
- algorithmic_behavior: `regular-review.md` is the default per-round review contract. It requires Codex to read the original plan first, compare Claude’s summary and actual implementation against that plan, reject deferrals as incomplete, and write `COMPLETE` only when all planned work and acceptance criteria are fully done (`prompt-template/codex/regular-review.md:5`, `:11`, `:20`, `:24`, `:33`, `:49`, `:53`). `full-alignment-review.md` is selected every fifth round by the stop hook when `CURRENT_ROUND % 5 == 4` (`hooks/loop-codex-stop-hook.sh:559`). It adds a comprehensive goal tracker audit, forgotten-item detection, deferred-item audit, completion summary, and stagnation check over the last five rounds (`prompt-template/codex/full-alignment-review.md:19`, `:30`, `:36`, `:42`, `:59`, `:74`). `goal-tracker-update-section.md` is embedded into both review types as Part 3 and gives Codex sole responsibility to evaluate Claude’s requested goal-tracker mutations after Round 0 (`prompt-template/codex/goal-tracker-update-section.md:1`, `:3`, `:5`, `:6`).
- inputs_outputs_state: Inputs are rendered placeholders supplied by `hooks/loop-codex-stop-hook.sh`: `CURRENT_ROUND`, `PLAN_FILE`, `PROMPT_FILE`, `SUMMARY_CONTENT`, `GOAL_TRACKER_FILE`, `DOCS_PATH`, `GOAL_TRACKER_UPDATE_SECTION`, `COMPLETED_ITERATIONS`, `LOOP_TIMESTAMP`, previous round indexes, and `REVIEW_RESULT_FILE` (`hooks/loop-codex-stop-hook.sh:596`, `:612`). Outputs are review prompt files at `round-N-review-prompt.md`; Codex then writes `round-N-review-result.md`. The review result’s last non-empty line becomes loop state: exact `COMPLETE` ends as complete; exact `STOP` ends as stop (`hooks/loop-codex-stop-hook.sh:831`, `:838`, `:849`). State transitions governed by the templates are: incomplete findings -> continue loop with action items; all work complete -> `COMPLETE`; stagnation in full alignment -> `STOP`; approved goal-tracker update request -> Codex edits mutable tracker sections, never immutable AC/goal (`prompt-template/codex/goal-tracker-update-section.md:6`, `:10`).
- gates_or_invariants: Main invariant is no false completion: both regular and full templates state that `COMPLETE` is allowed only when all original plan tasks and ACs are fully met, with no deferred or pending work (`prompt-template/codex/regular-review.md:53`, `:54`, `:55`, `:56`; `prompt-template/codex/full-alignment-review.md:88`, `:91`, `:92`, `:93`). Goal tracker invariants require every AC to be audited and forgotten/deferred items to be surfaced (`prompt-template/codex/full-alignment-review.md:21`, `:24`, `:30`, `:36`; `prompt-template/codex/regular-review.md:35`). The embedded update section prohibits Codex from modifying the immutable Ultimate Goal and Acceptance Criteria after initialization (`prompt-template/codex/goal-tracker-update-section.md:10`). Full-alignment rounds add a circuit-breaker gate: if stagnation is detected, `STOP` must be the last line instead of `COMPLETE` (`prompt-template/codex/full-alignment-review.md:76`, `:84`, `:90`).
- dependencies_and_callers: Primary caller is `hooks/loop-codex-stop-hook.sh`, which builds `GOAL_TRACKER_UPDATE_SECTION` from `codex/goal-tracker-update-section.md`, then chooses `codex/full-alignment-review.md` or `codex/regular-review.md` (`hooks/loop-codex-stop-hook.sh:553`, `:556`, `:594`, `:596`, `:612`).
