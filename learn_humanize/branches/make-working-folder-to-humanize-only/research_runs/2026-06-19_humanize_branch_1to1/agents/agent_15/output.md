- core_role: `prompt-template/block/plan-file-modified.md` is the user-facing block template emitted when the RLCR stop hook detects that the original plan file no longer matches the loop’s backup copy. It is a small template, but it defines a key algorithmic contract: plan files are immutable during an active RLCR loop unless the loop is cancelled and restarted.
- algorithmic_behavior: The template has two placeholders, `{{PLAN_FILE}}` and `{{BACKUP_PATH}}` (`prompt-template/block/plan-file-modified.md:3`, `prompt-template/block/plan-file-modified.md:12`). It explains that the plan file was modified after loop start, forbids plan mutation during active RLCR, gives the recovery transition sequence, and points to the backup. The prescribed transition is: cancel the current loop, update the plan file, then start a new loop against that plan (`prompt-template/block/plan-file-modified.md:7-10`). This message is loaded by `loop-codex-stop-hook.sh` only after a content diff failure between `$PROJECT_ROOT/$PLAN_FILE` and `$LOOP_DIR/plan.md` (`hooks/loop-codex-stop-hook.sh:196-215`).

Output is markdown text embedded as the `reason` field in a JSON block response with system message `Loop: Blocked - plan file modified` (`hooks/loop-codex-stop-hook.sh:213-214`). The relevant state inputs are `plan_file` and `plan_tracked`, parsed from state frontmatter (`hooks/loop-codex-stop-hook.sh:69-77`), and the backup file path `$LOOP_DIR/plan.md` (`hooks/loop-codex-stop-hook.sh:148-149`).
