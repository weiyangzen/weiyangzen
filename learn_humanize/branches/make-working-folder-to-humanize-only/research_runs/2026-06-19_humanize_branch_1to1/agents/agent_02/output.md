- core_role: `hooks/lib/loop-common.sh` is the shared RLCR hook policy library. It centralizes active-loop discovery, round parsing, path classification, block-message rendering, shell-command mutation detection, and terminal loop-state renaming for validators and the Stop hook.
- algorithmic_behavior: On source, it locates `hooks/lib`, sources `template-loader.sh`, derives `TEMPLATE_DIR`, and validates template directory shape with warning-only fallback (`hooks/lib/loop-common.sh:11-21`). `find_active_loop` accepts a loop base path, selects the newest timestamp-named child directory by reverse lexical sort, and returns it only if that newest directory contains `state.md`; older directories are intentionally ignored (`hooks/lib/loop-common.sh:23-44`). `get_current_round` extracts YAML-ish frontmatter between `---` delimiters and returns `current_round`, defaulting to `0` (`hooks/lib/loop-common.sh:46-58`). The file type helpers normalize case, match `round-N-summary.md`, `round-N-prompt.md`, and `round-N-todos.md`, and extract round numbers portably via `sed` (`hooks/lib/loop-common.sh:60-84`). Message helpers load block templates with inline fallbacks for todos, prompt writes, state edits, summary Bash writes, goal-tracker Bash writes, and post-Round 0 goal-tracker edits (`hooks/lib/loop-common.sh:86-135`, `hooks/lib/loop-common.sh:185-197`). `command_modifies_file` scans lowercased Bash command text for common mutation forms: redirection, append, `tee`, `sed -i`, in-place `awk`, `perl -i`, `mv/cp`, `rm`, `dd of=`, `truncate`, `printf >`, and `exec >` (`hooks/lib/loop-common.sh:155-183`). `end_loop` validates terminal reason and renames `state.md` to `<reason>-state.md` for `complete`, `cancel`, `maxiter`, `stop`, or `unexpected` (`hooks/lib/loop-common.sh:199-232`).
- inputs_outputs_state: Inputs are paths, filenames, hook command strings, template variables, and a state file. Outputs are stdout values for discovery/round helpers, stderr block text for message helpers, process return codes for predicates, and state-file renames for `end_loop`. The main state transition exposed by this file is `state.md -> complete-state.md|cancel-state.md|maxiter-state.md|stop-state.md|unexpected-state.md` (`hooks/lib/loop-common.sh:206-232`). Path gates distinguish round prompt/summary/todos files by suffix and reject non-current round access through callers. State-file mutation is always blocked by consumers through `is_state_file_path` and `state_file_blocked_message` (`hooks/lib/loop-common.sh:106-113`, `hooks/lib/loop-common.sh:143-147`). With `push_every_round: false`, Bash `git push` is blocked and commits are expected to remain local (`hooks/loop-bash-validator.sh:49-65`). This keeps the prompt contract and hook enforcement aligned: the prompt note is only emitted when pushing is both permitted and required.
