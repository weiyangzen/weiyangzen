### MAKE_WORKING_FOLDER_TO_HUMANIZE_ONLY-HZ-016 `file` `hooks/loop-bash-validator.sh`
- cursor: `[_]`
- core_role: Bash PreToolUse validator for the RLCR loop. It prevents Claude-side shell commands from bypassing the normal Write/Edit/Read hook policy while a loop is active. The hook is registered as the Bash matcher in `hooks/hooks.json:42-48`, and the README lists it under lifecycle hooks at `README.md:224-233`.
- algorithmic_behavior: The script reads hook JSON from stdin, extracts `.tool_name` and `.tool_input.command` with `jq`, and exits immediately unless the tool is `Bash` (`hooks/loop-bash-validator.sh:22-30`). It derives `PROJECT_ROOT` from `CLAUDE_PROJECT_DIR` or `pwd`, finds the newest active loop under `.humanize/rlcr`, and exits `0` when no active loop exists (`hooks/loop-bash-validator.sh:36-43`). Once active, it reads the current round and state file (`hooks/loop-bash-validator.sh:45-47`), then applies ordered gates: git push, `state.md`, plan backup, `goal-tracker.md`, round prompt, round summary, and round todos (`hooks/loop-bash-validator.sh:53-134`). Each violation writes a rendered block message to stderr and exits `2`; non-violating commands exit `0` at `hooks/loop-bash-validator.sh:137`.
- inputs_outputs_state: Input is a Claude hook event JSON on stdin. Required fields are `tool_name` and `tool_input.command`; missing values default to empty strings (`hooks/loop-bash-validator.sh:22-30`). Runtime state comes from the latest `.humanize/rlcr/<loop>/state.md`, especially `current_round` via `get_current_round` and `push_every_round` via grep (`hooks/loop-bash-validator.sh:45-55`). Outputs are only process status and stderr policy text. State transitions are guard-level transitions, not file mutations: inactive loop or non-Bash command -> allow; active Bash command targeting protected artifacts -> blocked; `goal-tracker.md` has round-sensitive guidance, where round 0 directs use of Write/Edit and later rounds direct the request into the current summary (`hooks/loop-bash-validator.sh:96-104`).
- gates_or_invariants: The hook enforces that `state.md` is system-managed and never modified by Bash (`hooks/loop-bash-validator.sh:72-75`), `plan.md` backup is read-only (`hooks/loop-bash-validator.sh:83-88`), `round-*-prompt.md` is read-only because it contains instructions to Claude (`hooks/loop-bash-validator.sh:112-115`), `round-*-summary.md` and `goal-tracker.md` must go through Write/Edit or summary-request flows (`hooks/loop-bash-validator.sh:96-125`), and `round-*-todos.md` is not Bash-managed (`hooks/loop-bash-validator.sh:132-135`). The git push gate defaults to local-only round commits unless `push_every_round: true` is present (`hooks/loop-bash-validator.sh:53-64`). The matching engine is delegated to `command_modifies_file`, which catches redirection, append redirection, `tee`, in-place `sed/awk/perl`, `mv/cp/rm`, `dd of=`, `truncate`, `printf >`, and FD redirection (`hooks/lib/loop-common.sh:155-183`).
- dependencies_and_callers: Depends on `bash`, `jq`, `grep`, `sed`, `find_active_loop`, `get_current_round`, `to_lower`, `command_modifies_file`, and shared message helpers from `hooks/lib/loop-common.sh` (`hooks/loop-bash-validator.sh:12-16`). `find_active_loop` only considers the newest timestamp directory with `state.md`, preventing old loop directories from being revived (`hooks/lib/loop-common.sh:23-44`). It uses shared templates through `load_and_render_safe`; for prompt writes that path is `prompt_write_blocked_message` in `hooks/lib/loop-common.sh:97-104`. Caller is Claude’s hook system through `hooks/hooks.json:42-48`. Related stop-loop behavior also reads `push_every_round`, showing the Bash push gate is part of a broader local-vs-push round policy (`hooks/loop-codex-stop-hook.sh` references found around push checks).
- edge_cases_or_failure_modes: If `jq` is unavailable or invalid JSON is supplied, `set -euo pipefail` makes the hook fail rather than silently allow. If no active loop directory exists, all Bash commands are allowed by design (`hooks/loop-bash-validator.sh:40-43`). If the template directory is invalid, shared helpers warn and fall back to inline messages rather than disabling the guard (`hooks/lib/loop-common.sh:18-21`). Matching is regex/string based on the lowercased command before execution, so it covers many common bypass forms but is not a full shell parser. Tests explicitly treat legacy `.humanize-loop.local` as non-loop and allowed (`tests/test-plan-file-hooks.sh:737-749`). The `git push` gate only matches commands beginning with `git push` after whitespace, so wrapped forms like `command git push` or aliases are outside this specific regex gate.
- validation_or_tests: `tests/test-template-references.sh:57-64` includes this script in template-reference scanning, and `tests/test-template-references.sh:152-159` verifies shared common templates. `tests/test-plan-file-hooks.sh:228-266` verifies Bash blocks writes/removal to `plan.md`, including direct `.humanize/rlcr/plan.md`. `tests/test-plan-file-hooks.sh:272-335` covers command substitution, glob, brace expansion, pipe-to-tee, and backtick bypass attempts against plan modification. `tests/test-plan-file-hooks.sh:737-749` validates the legacy-path negative case.
