It is registered for Bash tool use in `hooks/hooks.json:42-48`.
- algorithmic_behavior: The hook reads JSON from stdin, exits immediately unless `.tool_name == "Bash"`, extracts `.tool_input.command`, lowercases it, finds the newest active loop under `${CLAUDE_PROJECT_DIR:-$(pwd)}/.humanize/rlcr`, and exits open when no active loop exists (`hooks/loop-bash-validator.sh:22-43`). It selects `finalize-state.md` over `state.md` when present, parses state frontmatter, and uses `current_round` and `push_every_round` to drive policy (`hooks/loop-bash-validator.sh:45-63`). Its gates then run in priority order: direct `git push` block when `push_every_round` is not true (`:63-72`), `finalize-state.md` and `state.md` modification blocks (`:87-104`), source-side `mv`/`cp` state-file protection using segment splitting and redirection stripping (`:117-252`), shell-wrapper bypass checks for `sh -c`/`bash -c` (`:254-276`), plan backup protection (`:284-289`), goal-tracker protection with round-specific messaging (`:297-305`), prompt-file write protection (`:313-316`), summary Bash-write protection (`:323-326`), and path-restricted todos modification handling (`:333-340`).

State inputs come from `state.md` or `finalize-state.md` frontmatter parsed by `parse_state_file`, especially `current_round` and `push_every_round`. Outputs are process exit code `0` to allow the command, or exit code `2` with a rendered block message on stderr to deny it. The script itself does not mutate repository or loop state; it only allows a later Bash command to mutate state when `is_cancel_authorized` approves moving the active `state.md` or `finalize-state.md` to `cancel-state.md`.
- gates_or_invariants: The active-loop invariant is delegated to `find_active_loop`, which only treats the newest loop directory with `state.md` or `finalize-state.md` as active. The cancel exception requires an active-loop-local `.cancel-requested`, rejects command substitution/backticks/newlines/shell operators, permits only `mv`, requires exactly two args, normalizes `$loop_dir`/`${loop_dir}` and `/./`, and requires source exactly active `state.md` or `finalize-state.md` and destination exactly active `cancel-state.md` in `hooks/lib/loop-common.sh:274-412`. The todos gate is a path allowlist: Bash modifications to `round-[0-9]+-todos.md` are allowed only when the command text contains the exact active loop path and `round-1-todos.md` or `round-2-todos.md`.
