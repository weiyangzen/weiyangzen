- algorithmic_behavior: The directory has three direct descendants and no deeper children: `full-alignment-review.md`, `regular-review.md`, and `goal-tracker-update-section.md`. `regular-review.md` is the default per-round review prompt: it requires reading the original plan, comparing Claude’s summary to actual implementation, checking goal tracker drift, rejecting deferred or pending work as incomplete, and allowing `COMPLETE` only when all original tasks and acceptance criteria are fully done (`prompt-template/codex/regular-review.md:1`, `:5`, `:20`, `:33`, `:49-57`). `full-alignment-review.md` is selected every fifth logical checkpoint, when `CURRENT_ROUND % 5 == 4`; it adds acceptance-criteria audit tables, forgotten-item detection, deferred-item audit, historical last-five-round stagnation analysis, and permits `STOP` as the final-line circuit-breaker marker (`prompt-template/codex/full-alignment-review.md:1-3`, `:19-48`, `:59-85`, `:86-93`). `goal-tracker-update-section.md` is embedded into both review modes and delegates mutable goal-tracker updates to Codex, while explicitly forbidding modifications to the immutable Ultimate Goal and Acceptance Criteria (`prompt-template/codex/goal-tracker-update-section.md:1-17`).
- inputs_outputs_state: Inputs are rendered placeholders from `hooks/loop-codex-stop-hook.sh`: round number, plan path, current prompt path for regular reviews, Claude summary content, goal tracker path, docs path, derived loop timestamp/history values, and review result path (`hooks/loop-codex-stop-hook.sh:671-681`, `:689-748`). The templates enforce “no deferrals” for completion, despite adjacent command docs using looser wording about “validly deferred” completion (`prompt-template/codex/regular-review.md:53-57`, `prompt-template/codex/full-alignment-review.md:91-93`, `commands/start-rlcr-loop.md:51-55`). Final marker parsing is strict: only the last non-empty line, trimmed, may equal `COMPLETE` or `STOP`; this avoids false positives such as prose containing “cannot complete” (`hooks/loop-codex-stop-hook.sh:955-960`).
