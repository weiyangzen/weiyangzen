### CANCEL_WHEN_FINALIZE-HZ-020 `file` `hooks/loop-read-validator.sh`
- cursor: `[_]`
- core_role: PreToolUse `Read` validator for RLCR loop state artifacts. It prevents the assistant from reading stale or misplaced loop files, especially `round-N-summary.md`, `round-N-prompt.md`, and most `round-N-todos.md` files. It is wired into the Claude hook config at `hooks/hooks.json:33-40` for matcher `Read`.

- algorithmic_behavior: The script reads hook JSON from stdin, extracts `.tool_name`, and exits successfully for non-`Read` tools (`hooks/loop-read-validator.sh:22-27`). For `Read`, it extracts `.tool_input.file_path`, lowercases the path, then applies two major validation paths:
  - Todos files are handled first. Any basename matching `round-[0-9]+-todos.md` is blocked unless it is explicitly allowlisted in the active loop directory (`hooks/loop-read-validator.sh:36-43`). The block message comes from `todos_blocked_message`, whose fallback instructs agents to use native TodoWrite instead of round todos files (`hooks/lib/loop-common.sh:190-199`).
  - Summary and prompt files are then validated. Non-summary and non-prompt reads exit successfully (`hooks/loop-read-validator.sh:50-52`). For round prompt/summary files, it finds the active loop, parses current round state, extracts the requested round number, determines file type, and enforces active-loop location, current-round correctness, and exact directory path (`hooks/loop-read-validator.sh:54-146`).

- inputs_outputs_state: Input is Claude hook JSON on stdin with `tool_name` and `tool_input.file_path` (`hooks/loop-read-validator.sh:22-30`). The validator also reads environment-derived project state: `PROJECT_ROOT` or `CLAUDE_PROJECT_DIR`, `LOOP_BASE_DIR`, optional pre-set `LOOP_DIR`, and `.humanize/rlcr/<timestamp>/{state.md,finalize-state.md}` (`hooks/loop-read-validator.sh:37-39`, `61-76`). Output is only process status plus stderr guidance for blocks. Exit `0` means allow; exit `2` means block (`hooks/loop-read-validator.sh:25-27`, `40-43`, `109`, `128`, `145`, `148`). It does not mutate repository or loop state.

- gates_or_invariants: Active loop selection is delegated to `find_active_loop`, which only considers the newest `.humanize/rlcr/*/` directory active when it contains `state.md` or `finalize-state.md`; completed/cancelled/other terminal state files are not active (`hooks/lib/loop-common.sh:58-81`). The validator prefers `finalize-state.md` over `state.md` when present, so Finalize Phase keeps the same current-round protections (`hooks/loop-read-validator.sh:69-76`). Round file recognition is strict suffix matching on lowercase path: `round-[0-9]+-{summary|prompt|todos}.md` (`hooks/lib/loop-common.sh:146-153`). Wrong-location round prompt/summary reads are blocked before round validation if the path is not under `.humanize/rlcr/` (`hooks/loop-read-validator.sh:100-110`). Wrong-round reads are blocked unless allowlisted (`hooks/loop-read-validator.sh:116-129`). Even if the filename and round are acceptable, the final invariant requires the path to equal `$ACTIVE_LOOP_DIR/$CLAUDE_FILENAME`, preventing same-basename files in old loop directories or other roots (`hooks/loop-read-validator.sh:135-146`). The allowlist is path-specific and limited to `round-1-todos.md`, `round-2-todos.md`, `round-0-summary.md`, and `round-1-summary.md` in the active loop dir (`hooks/lib/loop-common.sh:167-188`).

- dependencies_and_callers: Direct dependencies are `jq`, `basename`, shell utilities, and `hooks/lib/loop-common.sh` sourced at startup (`hooks/loop-read-validator.sh:14-16`, `22-30`). Through `loop-common.sh`, it depends on `hooks/lib/template-loader.sh` for `load_and_render_safe`, template root discovery, and fallback-safe rendering (`hooks/lib/loop-common.sh:46-56`; `hooks/lib/template-loader.sh:24-31`, `167-203`). It renders `prompt-template/block/wrong-file-location.md`, `prompt-template/block/wrong-round-file.md`, and `prompt-template/block/wrong-directory-path.md` when available, with inline fallbacks otherwise (`hooks/loop-read-validator.sh:102-108`, `117-127`, `138-144`). Its caller is the hook framework configured in `hooks/hooks.json:33-40`.

- edge_cases_or_failure_modes: If there is no active loop, summary/prompt reads are allowed because the validator exits at `hooks/loop-read-validator.sh:65-67`; todos reads are still blocked unless there is an active loop and the exact allowlisted path matches (`hooks/loop-read-validator.sh:36-43`). If a summary/prompt filename lacks an extractable round number, the script exits allow (`hooks/loop-read-validator.sh:83-86`), which is safe for nonconforming names but means only canonical round filenames are governed by the round gate. The `.humanize/rlcr/` directory test is substring-based rather than canonical path resolution (`hooks/lib/loop-common.sh:415-419`), but the later exact path equality gate blocks wrong active-loop paths for canonical prompt/summary files (`hooks/loop-read-validator.sh:135-146`). `parse_state_file` returns defaults when fields are missing, so malformed state can make `CURRENT_ROUND` default to `0` rather than blocking here (`hooks/lib/loop-common.sh:112-139`); deeper schema validation appears to live in the Stop hook. If `jq` is unavailable or invalid JSON is provided, `set -euo pipefail` will cause an early failure rather than a structured allow/block response.

### CANCEL_WHEN_FINALIZE-HZ-050 `file` `prompt-template/block/goal-tracker-not-initialized.md`

- core_role: Block-response template for the Round 0 goal-tracker initialization gate. It is not executable code, but it defines the user-facing remediation contract when the Stop hook detects that `goal-tracker.md` still contains placeholders in required goal-tracking sections.

- algorithmic_behavior: The template renders a block titled “Goal Tracker Not Initialized,” states that the loop is in Round 0, lists `{{MISSING_ITEMS}}`, and instructs the agent to read `{{GOAL_TRACKER_FILE}}`, replace placeholders with actual Ultimate Goal, 3-7 Acceptance Criteria, and Active Tasks mapped to ACs, then write the updated tracker (`prompt-template/block/goal-tracker-not-initialized.md:1-15`). It also encodes a state-transition warning: the immutable section can only be set in Round 0 and becomes read-only after that (`prompt-template/block/goal-tracker-not-initialized.md:16-18`).

- inputs_outputs_state: Template inputs are `GOAL_TRACKER_FILE` and `MISSING_ITEMS`, passed by `hooks/loop-codex-stop-hook.sh` through `load_and_render_safe` (`hooks/loop-codex-stop-hook.sh:611-618`). The rendered output becomes the `reason` field in a JSON block decision with system message `Loop: Goal Tracker not initialized in Round 0` (`hooks/loop-codex-stop-hook.sh:620-627`). The template itself does not inspect or mutate state; the state gate around it is in the Stop hook.

- gates_or_invariants: The caller only reaches this template when all of these are true: the loop is not in Finalize Phase, `CURRENT_ROUND` equals `0`, and `goal-tracker.md` exists (`hooks/loop-codex-stop-hook.sh:556-563`). The Stop hook extracts three sections from the tracker: `### Ultimate Goal`, `### Acceptance Criteria`, and `#### Active Tasks` (`hooks/loop-codex-stop-hook.sh:573-590`). Each section is checked for the generic placeholder pattern `\[To be [a-z]`, avoiding hard-coded wording while limiting the scan to the relevant section (`hooks/loop-codex-stop-hook.sh:565-592`). If any placeholder remains, `MISSING_ITEMS` receives section-specific bullets for Ultimate Goal, Acceptance Criteria, and/or Active Tasks (`hooks/loop-codex-stop-hook.sh:596-609`), then the rendered template blocks exit (`hooks/loop-codex-stop-hook.sh:611-628`). This makes initialization a required Round 0 exit condition, not just a prompt suggestion.

- dependencies_and_callers: The direct caller is `hooks/loop-codex-stop-hook.sh` at `hooks/loop-codex-stop-hook.sh:616`. Rendering uses the shared template loader: placeholders use `{{VARIABLE_NAME}}`, missing variables are preserved, and substitution is single-pass to avoid placeholder injection from values (`hooks/lib/template-loader.sh:7-14`, `50-58`, `71-131`). If the template is missing or empty, `load_and_render_safe` falls back to a shorter inline message that still includes `GOAL_TRACKER_FILE` and `MISSING_ITEMS` (`hooks/loop-codex-stop-hook.sh:611-618`; `hooks/lib/template-loader.sh:167-203`). The template aligns with setup-time instructions that create `goal-tracker.md` and the initial Round 0 prompt (`scripts/setup-rlcr-loop.sh:504-594`, `602-653`).

- edge_cases_or_failure_modes: The gate only runs if `goal-tracker.md` exists; if the file is absent, this specific template is not emitted (`hooks/loop-codex-stop-hook.sh:560-563`). The placeholder detection is intentionally generic and section-scoped, but only placeholders matching lowercase text after `[To be ` are detected (`hooks/loop-codex-stop-hook.sh:576-584`, `591-592`); differently formatted placeholder text could bypass this gate. The `awk` section extraction stops at headings matching `^##`, so markdown structure changes can affect detection (`hooks/loop-codex-stop-hook.sh:573-590`).
