### CANCEL_WHEN_FINALIZE-HZ-021 `file` `hooks/loop-write-validator.sh`
- cursor: `[_]`
- core_role: PreToolUse `Write` hook for the RLCR loop. It is registered by `hooks/hooks.json:14-21` and protects loop-managed files from Claude direct writes while allowing the one expected write target for each phase. It is core state-machine enforcement because it guards round summaries, todo files, prompt files, state files, finalize state, finalize summaries, goal tracking, and plan backups.
- algorithmic_behavior: Reads hook JSON from stdin, extracts `.tool_name`, and exits allow for non-`Write` tools (`hooks/loop-write-validator.sh:23-28`). For `Write`, it extracts `.tool_input.file_path`, normalizes a lowercase copy, then applies ordered guards. It blocks round todo files unless they are in the active loop allowlist (`:37-45`), blocks round prompt files unconditionally (`:47-50`), classifies summary/finalize-summary/humanize-loop paths (`:56-63`), then ignores unrelated paths. For relevant loop paths, it finds the newest active loop under `.humanize/rlcr`, switches to finalize parsing when `finalize-state.md` exists (`:79-94`), parses `current_round` from the active state (`:96-99`), blocks `finalize-state.md` before generic `state.md` because the generic state matcher also catches finalize state (`:101-113`), allows only the active loop’s exact `finalize-summary.md` (`:115-126`), blocks `plan.md` loop backups (`:128-139`), blocks `goal-tracker.md` after round 0 (`:141-149`), blocks summary files outside `.humanize/rlcr` (`:151-163`), extracts the loop-relative target name (`:169-175`), checks round-number consistency for summary files with an allowlist escape (`:181-199`), and finally enforces that the target path equals `$ACTIVE_LOOP_DIR/$CLAUDE_FILENAME` (`:205-216`).
- inputs_outputs_state: Input is Claude hook JSON on stdin with `tool_name` and `tool_input.file_path`. State inputs come from `CLAUDE_PROJECT_DIR` or `pwd`, `.humanize/rlcr`, the newest active loop directory, and `state.md` or `finalize-state.md`. Output is only process status plus stderr block text. Exit `0` means allow; exit `2` means block with a rendered reason. The script does not mutate state; it enforces valid transitions by preventing writes to files that only loop machinery or the Codex/finalize workflow should own.
- gates_or_invariants: Todos are not normally written through `Write`; Claude should use native TodoWrite, with a small active-loop allowlist from `hooks/lib/loop-common.sh:170-188`. Prompt files are read-only instructions. `state.md` and `finalize-state.md` are system-managed. During finalize, `finalize-summary.md` is the allowed summary artifact only when it is exactly in the active loop directory. Round summaries must match `current_round` unless allowlisted. Summary-like writes outside `.humanize/rlcr` are redirected to the active loop summary path. `goal-tracker.md` becomes immutable to Claude after round 0; updates must be requested through the current round summary. `plan.md` inside the loop is treated as a protected backup.
- dependencies_and_callers: Depends on `jq`, `sed`, `grep`, `basename`, Bash strict mode, and `hooks/lib/loop-common.sh`. Key shared functions are `find_active_loop` (`hooks/lib/loop-common.sh:63-81`), `parse_state_file` (`:112-139`), `to_lower` (`:141-144`), `is_round_file_type` (`:148-153`), `extract_round_number` (`:158-165`), `is_allowlisted_file` (`:170-188`), block-message helpers (`:190-226`, `:451-463`), and path predicates for goal/state/finalize/humanize loop paths (`:250-271`, `:415-419`). The hook is invoked through Claude’s `PreToolUse` `Write` matcher in `hooks/hooks.json:14-21`.
- edge_cases_or_failure_modes: If there is no active loop, protected-looking writes may be allowed because there is no state to validate against (`hooks/loop-write-validator.sh:84-86`). The active loop is the newest timestamp-sorted directory only, so older loops are ignored even if they retain state (`hooks/lib/loop-common.sh:58-81`). `finalize-state.md` must be checked before `state.md`; the script explicitly documents this overlap (`hooks/loop-write-validator.sh:101-105`). ### CANCEL_WHEN_FINALIZE-HZ-051 `file` `prompt-template/block/incomplete-todos.md`
- cursor: `[_]`
- core_role: Block-message template used by the Stop hook when Claude attempts to stop while the latest TodoWrite state still contains incomplete todos. It is a gate contract rather than executable logic: it defines the human/agent-facing reason and required recovery action for the todo-completion invariant.
- algorithmic_behavior: The template declares “Incomplete Todos Detected,” renders an injected `{{INCOMPLETE_LIST}}`, and instructs the agent to complete all remaining todos, mark each one completed using TodoWrite, and only then write the summary and stop (`prompt-template/block/incomplete-todos.md:1-12`). The executable transition is in `hooks/loop-codex-stop-hook.sh`: before expensive review, it runs `hooks/check-todos-from-transcript.py` against the transcript (`hooks/loop-codex-stop-hook.sh:252-263`). If the checker exits with incomplete-todo status, the stop hook extracts the listed todos, renders this template with `INCOMPLETE_LIST`, returns a JSON block decision, and exits before Codex review (`:284-305`).
- inputs_outputs_state: Template input is `INCOMPLETE_LIST`, supplied from the todo checker output after stripping its first marker line (`hooks/loop-codex-stop-hook.sh:284-295`). The underlying state source is the latest TodoWrite call found in Claude’s JSONL transcript. The checker supports Claude assistant messages, alternate message entries, and direct tool-use entries (`hooks/check-todos-from-transcript.py:51-86`). Output is not a file write; it becomes the `reason` string inside the Stop hook’s JSON response with `"decision": "block"` (`hooks/loop-codex-stop-hook.sh:297-304`).
- gates_or_invariants: Stop is blocked unless the latest TodoWrite list has no item whose status differs from `completed`. The checker treats any non-`completed` status as incomplete and formats it as `- [status] content` (`hooks/check-todos-from-transcript.py:113-126`). This preserves the invariant that the agent cannot proceed to Codex review, final summary, or loop stop while its own task list is unfinished. The template reinforces that TodoWrite is the authoritative completion surface for these transient todos.
- dependencies_and_callers: Called by `hooks/loop-codex-stop-hook.sh:294-295` through `load_and_render_safe`. Depends indirectly on `hooks/check-todos-from-transcript.py`, Python JSON parsing, transcript path from Stop hook input, and the template loader. `load_and_render_safe` substitutes placeholders single-pass and falls back to inline text if the template is missing (`hooks/lib/template-loader.sh:54-57`, `:167-203`). The Stop hook itself is registered in `hooks/hooks.json:52-59`.
- edge_cases_or_failure_modes: If the transcript path is missing or no TodoWrite calls are found, the checker exits allow (`hooks/check-todos-from-transcript.py:99-111`), so this template is not used. Malformed hook input yields checker exit code `2`, and the Stop hook emits a parse-error block instead of this template (`hooks/loop-codex-stop-hook.sh:265-281`). Invalid JSONL transcript lines are skipped by the checker (`hooks/check-todos-from-transcript.py:36-39`), so a corrupted latest TodoWrite line could make an older valid TodoWrite call authoritative. Missing `content` or unusual statuses still render as incomplete if status is not exactly `completed` (`:115-120`).
- validation_or_tests: `tests/test-todo-checker.sh:106-142` covers all-complete versus incomplete transcripts and verifies incomplete task details are output. `tests/test-finalize-phase.sh:502-540` specifically verifies finalize phase blocks exit when todos are incomplete and does not invoke Codex before that check. `tests/test-template-references.sh:121-142` scans templates and `:83-110` verifies direct template references exist, covering this template’s call path from the Stop hook.
