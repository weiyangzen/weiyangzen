- core_role: Shared runtime compatibility library for bounded shell execution across macOS and Linux. - algorithmic_behavior: At source time, `detect_timeout_impl` selects the first available timeout backend in priority order: Homebrew `gtimeout`, GNU `timeout`, `python3`, `python`, or `none` (`scripts/portable-timeout.sh:10-27`). `run_with_timeout` accepts a timeout value, shifts it away, preserves the remaining command as a Bash array, and dispatches by backend (`scripts/portable-timeout.sh:33-38`). Native backends execute `gtimeout "$timeout_secs" "${cmd[@]}"` or `timeout "$timeout_secs" "${cmd[@]}"` and return the child exit code (`scripts/portable-timeout.sh:39-45`). Python backends run `subprocess.run(sys.argv[1:], timeout=$timeout_secs)`, map `TimeoutExpired` to exit `124`, and map other wrapper exceptions to exit `1` after printing an error (`scripts/portable-timeout.sh:47-61`). Output is the wrapped command's stdout/stderr and exit status, with `124` used as the timeout sentinel for GNU/Python-compatible consumers (`scripts/portable-timeout.sh:31-33`, `scripts/portable-timeout.sh:56-57`). Persistent state is minimal: `TIMEOUT_IMPL` is computed once when sourced, so later PATH changes do not alter backend choice unless the file is re-sourced (`scripts/portable-timeout.sh:29`). It does not mutate repository state directly.
- gates_or_invariants: The backend gate rejects non-GNU `timeout` by requiring `timeout --version` to succeed; otherwise it falls through to Python or `none` (`scripts/portable-timeout.sh:13-20`).
