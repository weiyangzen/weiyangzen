- algorithmic_behavior: The template renders a “Wrong Round File” message, names the attempted `round-{{CLAUDE_ROUND}}-{{FILE_TYPE}}.md`, states the active `{{CURRENT_ROUND}}`, lists the current prompt and summary paths, and gives a fallback direct `cat {{FILE_PATH}}` command for intentional historical access (`prompt-template/block/wrong-round-file.md:1-9`).
- inputs_outputs_state: Inputs are `CLAUDE_ROUND`, `FILE_TYPE`, `CURRENT_ROUND`, `ACTIVE_LOOP_DIR`, and `FILE_PATH`, supplied by `hooks/loop-read-validator.sh` when the Read tool targets `round-N-summary.md` or `round-N-prompt.md`. Output is rendered markdown to stderr, followed by validator exit code `2`; no state file is mutated (`hooks/loop-read-validator.sh:116-128`).
- gates_or_invariants: The template is selected only after an active loop is found, the target is a summary or prompt file, the current round is parsed from `state.md` or `finalize-state.md`, and the requested round differs from current round while not being allowlisted (`hooks/loop-read-validator.sh:50-77`, `hooks/loop-read-validator.sh:83-94`, `hooks/loop-read-validator.sh:116-128`). - dependencies_and_callers: Primary caller is `hooks/loop-read-validator.sh`, through `load_and_render_safe "$TEMPLATE_DIR" "block/wrong-round-file.md"` (`hooks/loop-read-validator.sh:122-127`). It depends on `extract_round_number`, `is_round_file_type`, `is_allowlisted_file`, `parse_state_file`, and template rendering from `hooks/lib/loop-common.sh` and `hooks/lib/template-loader.sh`.
- edge_cases_or_failure_modes: If there is no active loop, the read is allowed (`hooks/loop-read-validator.sh:65-67`). If the file is outside `.humanize/rlcr`, a different `wrong-file-location.md` template is used first (`hooks/loop-read-validator.sh:100-109`). If the filename does not yield a round number, validation exits without blocking (`hooks/loop-read-validator.sh:83-86`). If the template is missing, the read validator has an inline fallback (`hooks/loop-read-validator.sh:116-128`). The template’s `cat` instruction is intentional: this block governs the Read tool, while Bash-based reading can still be used when historical file inspection is explicitly needed.
