It turns GitHub issue comments, inline review comments, and PR review submissions into one markdown digest used by PR-loop setup and later human/Codex review. `scripts/setup-pr-loop.sh` invokes it to create `round-0-pr-comment.md` (`scripts/setup-pr-loop.sh:425-431`).
- algorithmic_behavior: It parses positional `pr_number` and `output_file`, plus `--after` and `--bots` (`scripts/fetch-pr-comments.sh:22-90`). It validates required args, numeric PR number, and presence of `gh`/`jq` (`scripts/fetch-pr-comments.sh:92-120`). It resolves the correct PR base repo by checking current repo first and parent repo for fork scenarios (`scripts/fetch-pr-comments.sh:126-162`). It fetches three GitHub API endpoints with three retries each: issue comments, PR review comments, and PR reviews (`scripts/fetch-pr-comments.sh:176-220`). It transforms each source into a common JSON shape, deduplicates by `id`, optionally filters by timestamp, sorts humans before bots and newest-first within groups, then emits markdown (`scripts/fetch-pr-comments.sh:260-343`, `scripts/fetch-pr-comments.sh:345-443`).
- inputs_outputs_state: Inputs are GitHub CLI repo context, PR number, output path, optional ISO timestamp, and optional active bot list. Outputs are a markdown file headed with PR number, fetch time, and resolved repository (`scripts/fetch-pr-comments.sh:245-254`). Comments include type, author, timestamp, body, optional file/line, and optional review state. When `--bots` is set, bot sections are grouped by mapped bot author; `codex` maps to `chatgpt-codex-connector[bot]`, and other bot names map to `<bot>[bot]` (`scripts/fetch-pr-comments.sh:381-422`). No persistent repo state is changed beyond the requested output file.
- gates_or_invariants: Required gates are numeric PR number, writable/usable output target by shell redirection, installed `gh` and `jq`, resolvable base repo, and parseable owner/name (`scripts/fetch-pr-comments.sh:92-162`). API endpoint failures are not fatal after retry; they increment `API_FAILURES`, write an empty JSON array for that source, and later append a warning to the output (`scripts/fetch-pr-comments.sh:183-205`, `scripts/fetch-pr-comments.sh:445-450`). Empty PR reviews with approval-only bodies are converted to a review-state placeholder, preserving approval signals that otherwise have no text body (`scripts/fetch-pr-comments.sh:300-320`).
- dependencies_and_callers: Called by `scripts/setup-pr-loop.sh:430-431`, exercised by `tests/test-pr-loop-scripts.sh:259-326`, fixture-tested by `tests/test-pr-loop-hooks.sh:1242-1319`, and robustness-tested by `tests/robustness/test-pr-loop-api-robustness.sh:348-550`. It depends on `gh api --paginate`, `gh repo view`, `gh pr view`, `jq`, `mktemp`, `sed`, `date`, and shell traps.
- edge_cases_or_failure_modes: Fork PRs are handled by parent repo lookup (`scripts/fetch-pr-comments.sh:126-154`). Missing `--after`/`--bots` values are rejected at parse time (`scripts/fetch-pr-comments.sh:29-43`). Unknown options and extra arguments are rejected (`scripts/fetch-pr-comments.sh:74-86`). Null `created_at` entries are filtered before sorting to avoid `fromdateiso8601` errors (`scripts/fetch-pr-comments.sh:334-343`).
