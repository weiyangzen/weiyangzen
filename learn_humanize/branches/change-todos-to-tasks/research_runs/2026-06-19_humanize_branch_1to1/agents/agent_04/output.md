### CHANGE_TODOS_TO_TASKS-HZ-034 `file` `scripts/check-bot-reactions.sh`
- cursor: `[_]`
- core_role: Runtime helper for the PR loop that converts GitHub reaction state into loop decisions. It detects Codex approval via `+1` on the PR body and Claude readiness via `eyes` on a trigger comment (`scripts/check-bot-reactions.sh:3-16`).
- algorithmic_behavior: The script is a two-command dispatcher (`scripts/check-bot-reactions.sh:86-308`). `codex-thumbsup` parses `<pr_number>` plus optional `--after`, resolves the PR base repo for fork support, fetches issue reactions with pagination, merges pages with `jq -s 'add // []'`, selects the first reaction from `chatgpt-codex-connector[bot]` with content `+1`, optionally rejects reactions older than `--after`, and prints the reaction JSON on success (`87-188`). `claude-eyes` parses `<comment_id>`, `--pr`, `--retry`, and `--delay`, resolves the base repo when a PR number is supplied, sleeps before each attempt, fetches comment reactions with pagination, selects `claude[bot]` `eyes`, and retries until found or exhausted (`190-301`).
- inputs_outputs_state: Inputs are CLI args, `gh` authentication/context, GitHub PR/comment reaction APIs, `jq`, and the portable timeout wrapper sourced from `scripts/portable-timeout.sh` (`31-33`). Outputs are reaction JSON on stdout and exit codes: `0` found, `1` not found/timeout, `2` argument or API error (`13-16`, `57-60`). It does not persist local state directly; PR loop state changes are performed by callers based on its result.
- gates_or_invariants: It uses timeout-wrapped `gh` calls (`GH_TIMEOUT=30`, `24-25`) and paginated reaction fetches (`152-159`, `269-275`) so large reaction sets are not silently truncated. Codex `--after` filtering is a key stale-approval gate (`175-183`). Claude eyes retries are deliberately bounded (`264-300`), so the stop hook can block quickly when Claude bot configuration is missing.
- dependencies_and_callers: Called by `hooks/pr-loop-stop-hook.sh` for pre-poll Codex `+1` approval (`675`), Claude trigger acknowledgement (`776`), and polling-loop Codex `+1` approval (`998`). `scripts/setup-pr-loop.sh:585-599` also calls `claude-eyes` at loop setup time. Tests in `tests/test-pr-loop-system.sh:199-230` cover direct command behavior.
- edge_cases_or_failure_modes: Missing args/unknown options exit `2` (`98-117`, `211-230`, `303-307`). Fork PRs are handled by trying current repo then parent repo (`119-150`, `232-262`). If base repo cannot be resolved, it falls back to `CURRENT_REPO`, which can still fail at API fetch and exit `2` for Codex or keep retrying for Claude. Timestamp comparison is string-based ISO-8601 comparison (`178-181`), so correctness depends on normalized sortable timestamps. `claude-eyes` intentionally sleeps before the first check (`264-267`), which makes even `--retry 1` wait for `--delay` seconds unless delay is set to `0`.
- validation_or_tests: Direct tests verify Codex `+1`, `--after` rejection, and Claude eyes detection in `tests/test-pr-loop-system.sh:199-230`. Integration tests verify the script’s stop-hook effects: Codex `+1` approval in `tests/test-pr-loop-stophook.sh:896-1009`, Claude eyes timeout blocking in `1011-1143`, and stale trigger/force-push behavior in `21-138` and `539-640`.
