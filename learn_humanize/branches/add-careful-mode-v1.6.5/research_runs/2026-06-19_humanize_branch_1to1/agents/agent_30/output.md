- gates_or_invariants: The template encodes several invariants: ACs must be testable with positive and negative cases; feasibility hints are explicitly non-prescriptive; dependencies describe relative order rather than dates; implementation code/comments must avoid plan-specific workflow terms such as `AC-`, `Milestone`, `Step`, and `Phase` (`prompt-template/plan/gen-plan-template.md:46-74`). - edge_cases_or_failure_modes: Template absence is treated as plugin configuration failure by the IO validator. The template is a scaffold with placeholders, so completion depends on the command replacing placeholders while preserving the appended draft. For drafts with fixed deterministic designs, the template expects upper and lower bounds to converge and allowed choices to state the fixed choice rather than invent alternatives (`prompt-template/plan/gen-plan-template.md:44`). The implementation-notes terminology ban is easy for later implementers to miss unless downstream review hooks or humans enforce it.
