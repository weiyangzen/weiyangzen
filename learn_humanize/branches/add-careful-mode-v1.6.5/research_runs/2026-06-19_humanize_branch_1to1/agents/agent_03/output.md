### ADD_CAREFUL_MODE_V1_6_5-HZ-033 `file` `scripts/cancel-rlcr-loop.sh`
- cursor: `[_]`
- core_role: Runtime cancellation script for active RLCR sessions. It is the authoritative mutation path for converting an active `.humanize/rlcr/<timestamp>/state.md` or `finalize-state.md` session into `cancel-state.md` while preserving loop evidence.
- algorithmic_behavior: The script parses only `--force` and help (`scripts/cancel-rlcr-loop.sh:26`). It resolves `PROJECT_ROOT` from `CLAUDE_PROJECT_DIR` or `pwd`, sets the base to `.humanize/rlcr`, selects the newest timestamp-like child directory by descending lexical sort, and then detects active state by checking `state.md` first and `finalize-state.md` second (`scripts/cancel-rlcr-loop.sh:69`). It extracts `current_round` and `max_iterations` from the active state file (`scripts/cancel-rlcr-loop.sh:105`). If the loop is in finalize phase and `--force` was not supplied, it emits `FINALIZE_NEEDS_CONFIRM` and exits with code 2 (`scripts/cancel-rlcr-loop.sh:117`). Otherwise it creates `.cancel-requested`, renames the active state file to `cancel-state.md`, and emits either `CANCELLED` or `CANCELLED_FINALIZE` (`scripts/cancel-rlcr-loop.sh:134`).
- inputs_outputs_state: Inputs are `--force`, the current working directory or `CLAUDE_PROJECT_DIR`, and state files under `.humanize/rlcr/<newest>/`. Outputs are structured first-line status tokens: `NO_LOOP`, `NO_ACTIVE_LOOP`, `FINALIZE_NEEDS_CONFIRM`, `CANCELLED`, or `CANCELLED_FINALIZE`. State transition is `state.md -> cancel-state.md` for normal loops and `finalize-state.md -> cancel-state.md` for forced finalize cancellation; `.cancel-requested` is created before the rename so validators can authorize the otherwise blocked state-file move.
- gates_or_invariants: The script uses `set -euo pipefail` (`scripts/cancel-rlcr-loop.sh:18`), rejects unknown options with exit code 3 (`scripts/cancel-rlcr-loop.sh:57`), treats only the newest loop directory as relevant (`scripts/cancel-rlcr-loop.sh:72`), and refuses non-forced finalize cancellation with an explicit confirmation-required status (`scripts/cancel-rlcr-loop.sh:117`). The command wrapper requires user confirmation before rerunning with `--force` (`commands/cancel-rlcr-loop.md:23`).
- dependencies_and_callers: Called by `/humanize:cancel-rlcr-loop` as specified in `commands/cancel-rlcr-loop.md:13`. Shared loop discovery in `hooks/lib/loop-common.sh:157` also treats `state.md` and `finalize-state.md` as active states, matching this script’s state model. Bash validators normally block state-file movement but allow authorized cancellation when `.cancel-requested` exists and `is_cancel_authorized` validates the exact `mv state.md|finalize-state.md cancel-state.md` shape (`hooks/lib/loop-common.sh:557`, `hooks/loop-bash-validator.sh:139`).
- edge_cases_or_failure_modes: If `.humanize/rlcr` is absent or contains no child directories, it emits `NO_LOOP` and exits 1 (`scripts/cancel-rlcr-loop.sh:75`). If newest loop directory exists but lacks both active state files, it emits `NO_ACTIVE_LOOP` and exits 1 (`scripts/cancel-rlcr-loop.sh:95`). Missing `current_round` or `max_iterations` becomes `?` rather than failure (`scripts/cancel-rlcr-loop.sh:109`). A preexisting `cancel-state.md` could be overwritten by `mv` because the script does not guard destination existence. Directory selection relies on lexically sortable timestamp names; nonconforming names could affect “newest” selection.
- validation_or_tests: Finalize behavior and validator interaction are covered by `tests/test-finalize-phase.sh` and cancellation-signal tests. The shared authorization helper is heavily stress-tested in `tests/test-cancel-signal-file.sh` and robustness tests, including finalize-state source handling, symlink rejection, shell injection, command substitution, mixed quotes, and wrong-directory moves. The assigned PR-loop script tests do not directly exercise this RLCR script.
