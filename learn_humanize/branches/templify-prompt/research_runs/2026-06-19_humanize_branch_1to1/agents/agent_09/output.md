If any violations exist, it renders this file with `MAX_LINES` and `LARGE_FILES`, then returns a JSON block decision ([loop-codex-stop-hook.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/loop-codex-stop-hook.sh:156)). The template’s transition contract is: split oversized files, preserve behavior or documentation coherence, commit changes, then attempt exit again ([large-files.md](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/prompt-template/block/large-files.md:11)).
- inputs_outputs_state: Inputs are `{{MAX_LINES}}`, used in the heading and remediation thresholds, and `{{LARGE_FILES}}`, the hook-built bullet list of violating paths with line counts and file type ([large-files.md](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/prompt-template/block/large-files.md:3)). The template states each split file should be `< {{MAX_LINES}} lines` ([large-files.md](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/prompt-template/block/large-files.md:14)), which is slightly stricter than the hook’s numeric violation condition only at the boundary.
