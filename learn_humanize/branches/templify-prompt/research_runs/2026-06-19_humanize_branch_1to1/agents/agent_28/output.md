### TEMPLIFY_PROMPT-HZ-028 `file` `hooks/lib/loop-common.sh`
- cursor: `[_]`
- core_role: Shared policy and utility layer for RLCR loop hooks. The file is sourced by the read, write, edit, bash, and Codex stop hooks via `source "$SCRIPT_DIR/lib/loop-common.sh"` (`hooks/loop-read-validator.sh:16`, `hooks/loop-write-validator.sh:17`, `hooks/loop-edit-validator.sh:16`, `hooks/loop-bash-validator.sh:16`, `hooks/loop-codex-stop-hook.sh:48`). It centralizes active-loop discovery, round parsing, protected path classification, shell-command mutation detection, and rendered block messages.
- algorithmic_behavior: On load, it locates `hooks/lib`, sources `template-loader.sh`, derives `TEMPLATE_DIR`, and validates the prompt-template tree with warning-only degradation (`hooks/lib/loop-common.sh:11-21`). `find_active_loop` inspects only the newest subdirectory under `.humanize-loop.local` and accepts it only if `state.md` exists, deliberately ignoring older directories to avoid reviving stale loops (`hooks/lib/loop-common.sh:23-44`). `get_current_round` extracts YAML-like frontmatter between `---` delimiters and returns `current_round`, defaulting to `0` (`hooks/lib/loop-common.sh:46-58`). Path helpers normalize and identify `round-N-summary.md`, `round-N-prompt.md`, `round-N-todos.md`, `goal-tracker.md`, `state.md`, and membership under `.humanize-loop.local/` (`hooks/lib/loop-common.sh:60-153`). `command_modifies_file` scans lowercased Bash commands for redirection, `tee`, in-place edit forms, copy/move into target paths, and `dd of=...` writes (`hooks/lib/loop-common.sh:155-178`). Message helpers render block templates with inline fallbacks for todos access, prompt writes, state writes, summary Bash writes, and goal-tracker writes (`hooks/lib/loop-common.sh:86-134`, `hooks/lib/loop-common.sh:180-192`).
- inputs_outputs_state: Inputs are `loop_base_dir`, `state_file`, lowercased path strings, raw filenames, lowercased shell commands, file regex patterns, and current-round or summary-path values supplied by caller hooks. Outputs are stdout values such as active loop path, round number, lowercase strings, extracted round number, or rendered Markdown block messages; predicate helpers return shell status codes. It reads loop state through `state.md` but does not mutate state. State transitions are enforced indirectly by caller hooks: read/write/edit/bash validators block access or allow tool execution, while the stop hook uses `find_active_loop` to decide whether the RLCR loop is active (`hooks/loop-codex-stop-hook.sh:52-57`).
- gates_or_invariants: The newest loop directory is the only active candidate, and it must contain `state.md` (`hooks/lib/loop-common.sh:35-43`). Round file naming must match `round-[0-9]+-{summary|prompt|todos}.md` for classification and numeric extraction (`hooks/lib/loop-common.sh:65-84`). Templates are preferred, but missing templates must not make validators fail silently because `load_and_render_safe` is used with inline fallback strings (`hooks/lib/loop-common.sh:88-134`, `hooks/lib/template-loader.sh:155-188`). Bash mutation detection is pattern-based and only checks the command text passed by hook input (`hooks/lib/loop-common.sh:158-178`). Goal tracker edits after Round 0 are routed to a summary-file update request instead of direct mutation (`hooks/lib/loop-common.sh:180-192`, `hooks/loop-write-validator.sh:94-98`, `hooks/loop-edit-validator.sh:81-85`, `hooks/loop-bash-validator.sh:83-92`).
- dependencies_and_callers: Depends on `hooks/lib/template-loader.sh`, specifically `get_template_dir`, `validate_template_dir`, and `load_and_render_safe` (`hooks/lib/loop-common.sh:11-20`; `hooks/lib/template-loader.sh:11-16`, `hooks/lib/template-loader.sh:155-207`). Read validator uses it to block todos reads, stale/wrong-location prompt or summary reads, and non-active loop paths (`hooks/loop-read-validator.sh:36-132`). Write validator uses it to block todos and prompt writes, state writes, post-Round-0 goal tracker writes, wrong summary locations, wrong round summaries, and wrong active-loop directory paths (`hooks/loop-write-validator.sh:37-164`). Stop hook uses active-loop discovery before parsing state and rendering Codex prompts (`hooks/loop-codex-stop-hook.sh:43-57`).
- edge_cases_or_failure_modes: If the loop base directory is missing, `find_active_loop` returns empty and caller hooks generally allow normal operation (`hooks/lib/loop-common.sh:30-33`; examples at `hooks/loop-read-validator.sh:60-62`, `hooks/loop-bash-validator.sh:40-43`). ### TEMPLIFY_PROMPT-HZ-058 `file` `prompt-template/codex/regular-review.md`

- core_role: Codex review prompt template for ordinary RLCR rounds. It defines the review contract that determines whether the Claude implementation loop stops or continues. The stop hook renders this template for non-full-alignment rounds (`hooks/loop-codex-stop-hook.sh:475-505`) and then runs `codex exec` with the rendered prompt (`hooks/loop-codex-stop-hook.sh:570-597`).
- algorithmic_behavior: The template requires Codex to read the original implementation plan first through `@{{PLAN_FILE}}`, then evaluate Claude’s claimed completion using both the original plan and current round prompt `@{{PROMPT_FILE}}` (`prompt-template/codex/regular-review.md:3-12`). It embeds Claude’s summary between explicit start/end comments through `{{SUMMARY_CONTENT}}` (`prompt-template/codex/regular-review.md:13-18`). Part 1 requires skeptical implementation review focused on gaps between plan design and actual implementation, with special handling for deferrals and admitted unfinished work: Codex must not accept deferrals as completion and must produce a singular, directive implementation plan for remaining work (`prompt-template/codex/regular-review.md:20-31`). Part 2 mandates goal alignment against `@{{GOAL_TRACKER_FILE}}`, including acceptance criteria progress, forgotten items, deferrals, and plan evolution (`prompt-template/codex/regular-review.md:33-45`). Part 3 injects a shared goal-tracker update contract via `{{GOAL_TRACKER_UPDATE_SECTION}}` (`prompt-template/codex/regular-review.md:47`; source at `prompt-template/codex/goal-tracker-update-section.md:1-17`). Part 4 defines output requirements and the stop condition: only output `COMPLETE` as the final line when all original plan tasks and acceptance criteria are fully complete with no deferred or pending work (`prompt-template/codex/regular-review.md:49-57`).
- inputs_outputs_state: Template inputs are `CURRENT_ROUND`, `PLAN_FILE`, `PROMPT_FILE`, `SUMMARY_CONTENT`, `GOAL_TRACKER_FILE`, `DOCS_PATH`, `GOAL_TRACKER_UPDATE_SECTION`, and `REVIEW_RESULT_FILE`, supplied by `load_and_render_safe` in the stop hook (`hooks/loop-codex-stop-hook.sh:493-505`).
