### TEMPLIFY_PROMPT-HZ-020 `file` `hooks/loop-write-validator.sh`
- cursor: `[_]`
- core_role: Write-side PreToolUse validator for the RLCR loop. It is registered as the `Write` hook command in `hooks/hooks.json:4-11` and protects the loop’s generated/control files from being written in the wrong phase, directory, or round.
- algorithmic_behavior: The script reads the hook JSON from stdin, extracts `.tool_name` and `.tool_input.file_path` via `jq`, and exits successfully for any non-`Write` tool (`hooks/loop-write-validator.sh:23-30`). It normalizes the path to lowercase for pattern matching (`hooks/loop-write-validator.sh:30-31`), then applies ordered gates: block `round-*-todos.md` writes (`hooks/loop-write-validator.sh:37-40`), block `round-*-prompt.md` writes (`hooks/loop-write-validator.sh:42-45`), classify summary files and `.humanize-loop.local` membership (`hooks/loop-write-validator.sh:51-52`), allow unrelated files (`hooks/loop-write-validator.sh:54-57`), allow non-control loop-local files except `state.md` and `goal-tracker.md` (`hooks/loop-write-validator.sh:59-65`), discover the active loop and current round (`hooks/loop-write-validator.sh:71-80`), then validate state, goal tracker, summary location, round number, and exact active directory path (`hooks/loop-write-validator.sh:85-167`).
- inputs_outputs_state: Input is Claude hook JSON on stdin plus environment/context: `CLAUDE_PROJECT_DIR` or current working directory for the project root (`hooks/loop-write-validator.sh:71`), active loop files under `.humanize-loop.local`, and the shared template directory resolved through `hooks/lib/loop-common.sh`. Output is process exit status and stderr block messages. Exit `0` allows the write; exit `2` blocks the write with a rendered prompt block. State read is the newest active loop directory selected by `find_active_loop`, which only considers the newest timestamped child containing `state.md` (`hooks/lib/loop-common.sh:23-44`), and current round from YAML frontmatter key `current_round:` in `state.md`, defaulting to `0` (`hooks/lib/loop-common.sh:46-58`). The hook itself does not mutate state.
- gates_or_invariants: Round todos are never written through `Write`; the common helper says to use TodoWrite instead (`hooks/lib/loop-common.sh:86-95`). Round prompts are read-only/generated by Codex (`hooks/lib/loop-common.sh:97-104`). `state.md` writes are always blocked once an active loop exists (`hooks/loop-write-validator.sh:85-88`). `goal-tracker.md` is writable only during round `0`; after that the hook redirects the actor to include update requests in the current summary (`hooks/loop-write-validator.sh:94-98`, `hooks/lib/loop-common.sh:180-192`). Summary files must be under the active `.humanize-loop.local/<loop>/` directory (`hooks/loop-write-validator.sh:104-112`), must use the current round number (`hooks/loop-write-validator.sh:130-148`), and must exactly match the active loop directory, not an older sibling or flattened loop path (`hooks/loop-write-validator.sh:118-124`, `hooks/loop-write-validator.sh:154-165`).
- dependencies_and_callers: Direct dependency on `bash`, `jq`, `sed`, `grep`, `sort`, `head`, `tr`, and the sourced `hooks/lib/loop-common.sh` (`hooks/loop-write-validator.sh:13-17`). The shared library depends on `hooks/lib/template-loader.sh` (`hooks/lib/loop-common.sh:11-17`) for block rendering with `{{VARIABLE}}` placeholders and safe fallbacks (`hooks/lib/template-loader.sh:152-188`). Template dependencies include `block/todos-file-access.md`, `block/prompt-file-write.md`, `block/state-file-modification.md`, `block/goal-tracker-modification.md`, `block/wrong-summary-location.md`, `block/wrong-round-number.md`, and `block/wrong-directory-path.md`, as referenced in this hook and common helpers. It is one of the lifecycle hooks described in the repo structure (`README.md:223-232`) and has sibling validators for `Edit`, `Read`, and `Bash` that share the same library (`hooks/loop-edit-validator.sh:14-17`, `hooks/loop-read-validator.sh:14-17`).
- edge_cases_or_failure_modes: If there is no `.humanize-loop.local` base or the newest child lacks `state.md`, `find_active_loop` returns empty and the hook allows the write (`hooks/lib/loop-common.sh:30-43`, `hooks/loop-write-validator.sh:75-77`); this is a deliberate graceful no-active-loop behavior, but it means protection depends on active-loop discovery. If `state.md` lacks parseable `current_round`, the current round defaults to `0` (`hooks/lib/loop-common.sh:51-57`), so goal tracker writes remain allowed. Filename detection is strict and suffix-based: only paths ending in `round-[0-9]+-summary.md`, `round-[0-9]+-prompt.md`, or `round-[0-9]+-todos.md` are treated as round files (`hooks/lib/loop-common.sh:65-72`). Path membership is substring-based on `.humanize-loop.local/` (`hooks/lib/loop-common.sh:149-153`), so semantically equivalent paths with symlinks or unresolved `..` segments are not canonicalized. Round-file path extraction has two regex forms for nested timestamp directories and flattened loop-local files (`hooks/loop-write-validator.sh:118-121`); if neither matches, validation exits `0` (`hooks/loop-write-validator.sh:122-124`). With `set -euo pipefail`, malformed JSON or missing `jq` will fail the hook rather than fall back. Template loading is safe against missing templates because `load_and_render_safe` uses inline fallbacks if the file is missing or empty (`hooks/lib/template-loader.sh:155-188`).
