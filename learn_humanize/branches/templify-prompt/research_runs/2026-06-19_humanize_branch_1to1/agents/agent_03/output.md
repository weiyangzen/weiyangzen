### TEMPLIFY_PROMPT-HZ-033 `file` `prompt-template/block/git-not-clean.md`
- cursor: `[_]`
- core_role: `prompt-template/block/git-not-clean.md` is the stop-hook block message template used when the active RLCR loop tries to stop while Git has uncommitted changes. It is part of the algorithmic gate that forces work to be saved before Codex review or loop termination.
- algorithmic_behavior: The template contains placeholders `{{GIT_ISSUES}}` and `{{SPECIAL_NOTES}}` (`prompt-template/block/git-not-clean.md:3-4`). `hooks/loop-codex-stop-hook.sh:187-216` computes those values from `git status --porcelain`, setting `GIT_ISSUES` to `uncommitted changes` and adding special notes for untracked `.humanize-loop.local` or other untracked artifacts. `loop-codex-stop-hook.sh:219-239` renders this template via `load_and_render_safe` and returns a JSON block decision with the rendered reason and system message.
- inputs_outputs_state: Inputs are the detected git issue summary and optional notes. Output is Markdown feedback embedded in the Stop-hook JSON `"reason"`. It does not mutate state directly; it shapes the user-facing transition from “attempted stop” back to “must stage/commit or ignore artifacts first.” The template also instructs use of a `code-simplifier` plugin if installed, review of untracked files, staging real changes, and creating a descriptive commit (`git-not-clean.md:5-9`).
- gates_or_invariants: The template reinforces commit hygiene: commit message must follow project conventions, AI tools must not be credited as authors, and `Co-Authored-By: Claude` or similar attribution must not be included (`git-not-clean.md:11-14`). The stop hook reaches this template only when Git is available, the working directory is inside a Git repo, and porcelain status is non-empty (`loop-codex-stop-hook.sh:185-193`). It blocks before Codex review, so dirty work cannot be considered complete by review alone.
- dependencies_and_callers: Direct caller is `hooks/loop-codex-stop-hook.sh:227-229`. Rendering depends on `hooks/lib/template-loader.sh:155-188`, which falls back to an inline message if the template is missing or renders empty. Related optional note templates are `prompt-template/block/git-not-clean-humanize-local.md` and `prompt-template/block/git-not-clean-untracked.md`, loaded at `loop-codex-stop-hook.sh:200-214`. Template existence is covered by the reference scanner in `tests/test-template-references.sh:83-110`.
- edge_cases_or_failure_modes: If `SPECIAL_NOTES` is empty, the blank placeholder line still remains after the issue line; this is harmless but may leave extra vertical spacing. The template assumes the appropriate action is to commit all real changes; if the dirty state is caused only by intentionally untracked loop files, the special note templates must clarify whether to ignore rather than commit them. If the template is missing, `load_and_render_safe` prevents an empty block reason by using the fallback defined at `loop-codex-stop-hook.sh:221-226`.
