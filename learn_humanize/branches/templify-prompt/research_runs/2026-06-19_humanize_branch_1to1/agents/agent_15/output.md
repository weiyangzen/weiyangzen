- core_role: Declarative hook manifest for the RLCR loop. It is not the validator logic itself; it is the routing table that binds Claude lifecycle/tool events to the shell validators and stop controller. `hooks/hooks.json:4` starts the `PreToolUse` hook set, and `hooks/hooks.json:42` starts the `Stop` hook set.
- algorithmic_behavior: On `PreToolUse`, dispatches by tool matcher: `Write` to `hooks/loop-write-validator.sh` (`hooks/hooks.json:6-10`), `Edit` to `hooks/loop-edit-validator.sh` (`hooks/hooks.json:15-19`), `Read` to `hooks/loop-read-validator.sh` (`hooks/hooks.json:24-28`), and `Bash` to `hooks/loop-bash-validator.sh` (`hooks/hooks.json:33-37`). On `Stop`, dispatches to `hooks/loop-codex-stop-hook.sh` with a 7200-second hook timeout (`hooks/hooks.json:42-49`). This creates the RLCR state-machine boundary: tool access is checked before use, and exit is intercepted for summary, git, todo, large-file, and Codex-review gates.
- inputs_outputs_state: Input is Claude hook event metadata matched by lifecycle key and matcher name. Output is indirect: each command emits either normal exit/allow behavior or a block response/error via the invoked script. State is external, primarily `.humanize-loop.local/<timestamp>/state.md`, round files, summary files, git status, and transcript-derived todos. The manifest itself has no mutable state; it maps hook event names to executable commands under `${CLAUDE_PLUGIN_ROOT}`.
- gates_or_invariants: The manifest enforces that all relevant RLCR mutations pass through validators before tool execution. The invoked validators enforce round-file and loop-directory invariants: Read blocks todos and wrong round/location prompt or summary reads (`hooks/loop-read-validator.sh:36-39`, `hooks/loop-read-validator.sh:87-96`, `hooks/loop-read-validator.sh:103-115`, `hooks/loop-read-validator.sh:124-132`); Edit blocks todos, prompt files, state files, and post-round-0 goal tracker edits (`hooks/loop-edit-validator.sh:36-44`, `hooks/loop-edit-validator.sh:72-84`); Bash blocks bypass writes such as redirection, `tee`, in-place edits, git push when disabled, and shell writes to state/goal/prompt/summary/todos files (`hooks/loop-bash-validator.sh:55-64`, `hooks/loop-bash-validator.sh:72-121`). The Stop hook gates exit on incomplete todos, large changed files, dirty git state, missing summary, uninitialized goal tracker, Codex review failure, or Codex review not ending with `COMPLETE`/`STOP` (`hooks/loop-codex-stop-hook.sh:67-95`, `hooks/loop-codex-stop-hook.sh:104-176`, `hooks/loop-codex-stop-hook.sh:218-239`, `hooks/loop-codex-stop-hook.sh:319-339`, `hooks/loop-codex-stop-hook.sh:348-400`, `hooks/loop-codex-stop-hook.sh:712-757`).

### TEMPLIFY_PROMPT-HZ-045 `file` `prompt-template/block/work-summary-missing.md`
- cursor: `[_]`
- core_role: Block-message template for the Stop-hook summary-existence gate. It defines the user-facing contract when the agent attempts to exit an active RLCR loop without creating the required round summary file. The template title is `Work Summary Missing` (`prompt-template/block/work-summary-missing.md:1`), and the required output path is injected via `{{SUMMARY_FILE}}` (`prompt-template/block/work-summary-missing.md:5-8`).
- algorithmic_behavior: The Stop hook computes `SUMMARY_FILE="$LOOP_DIR/round-${CURRENT_ROUND}-summary.md"` and checks file existence (`hooks/loop-codex-stop-hook.sh:319-321`). If missing, it renders this template with `SUMMARY_FILE=$SUMMARY_FILE` (`hooks/loop-codex-stop-hook.sh:325-329`) and emits a JSON block decision with that rendered markdown as `reason` plus system message `Loop: Summary file missing for round ...` (`hooks/loop-codex-stop-hook.sh:331-338`). This prevents the state machine from entering the Codex review phase without Claude’s round summary.
- inputs_outputs_state: Input placeholder is `SUMMARY_FILE`, supplied by the Stop hook from active loop state and current round. Output is rendered markdown telling the agent to write the summary file and include implementation, changed files, tests, and remaining items (`prompt-template/block/work-summary-missing.md:10-14`). State transition is negative/blocked: no round increment, no Codex review prompt, and no loop completion until a summary file exists. The next exit attempt can proceed past this gate if the expected file path exists.
- gates_or_invariants: Enforces the invariant that every exit attempt from an active RLCR round must have a round-specific summary at `.humanize-loop.local/<active>/round-N-summary.md` before Codex review. The template also narrows acceptable summary content to four categories: implemented work, files created or modified, tests added or passed, and remaining items (`prompt-template/block/work-summary-missing.md:10-14`). The location invariant depends on the Stop hook’s derived `SUMMARY_FILE`, while write/edit validators separately prevent writing summaries for the wrong round (`hooks/loop-edit-validator.sh:91-115`) and prevent Bash from modifying summary files directly (`hooks/loop-bash-validator.sh:109-112`).
- dependencies_and_callers: Direct caller is `hooks/loop-codex-stop-hook.sh` through `load_and_render_safe "$TEMPLATE_DIR" "block/work-summary-missing.md" ...` (`hooks/loop-codex-stop-hook.sh:325-329`). Rendering is handled by `hooks/lib/template-loader.sh`, whose `load_and_render_safe` loads the template, falls back if missing or empty, and performs placeholder substitution (`hooks/lib/template-loader.sh:155-188`). `TEMPLATE_DIR` is initialized from `hooks/lib/loop-common.sh` via `get_template_dir` (`hooks/lib/loop-common.sh:11-17`; `hooks/lib/template-loader.sh:11-16`).
- edge_cases_or_failure_modes: If this template is missing or empty, `load_and_render_safe` uses the inline fallback `# Work Summary Missing` message with the same `SUMMARY_FILE` substitution (`hooks/loop-codex-stop-hook.sh:325-329`; `hooks/lib/template-loader.sh:164-180`). If the template keeps an unresolved placeholder or renames `{{SUMMARY_FILE}}`, the block reason may fail to provide the correct path. If `CURRENT_ROUND` in `state.md` is corrupted, the Stop hook removes the state file and exits before this gate (`hooks/loop-codex-stop-hook.sh:303-308`), so this template is bypassed. If an empty summary file exists, this specific gate passes because it checks only `-f`; later review quality gates may still fail, but this template is not responsible for content validation.
- validation_or_tests: `tests/test-template-references.sh` scans `loop-codex-stop-hook.sh` and would fail if `block/work-summary-missing.md` were referenced but absent (`tests/test-template-references.sh:57-64`, `tests/test-template-references.sh:83-110`, `tests/test-template-references.sh:213-229`). `tests/test-templates-comprehensive.sh` loads all markdown templates and validates placeholder syntax, including uppercase double-brace placeholders such as `{{SUMMARY_FILE}}` (`tests/test-templates-comprehensive.sh:90-107`, `tests/test-templates-comprehensive.sh:122-194`). `tests/test-error-scenarios.sh` documents the risk of empty feedback when unsafe rendering is used, while this caller uses the safe loader with fallback (`tests/test-error-scenarios.sh:114-120`; `hooks/loop-codex-stop-hook.sh:328`).
