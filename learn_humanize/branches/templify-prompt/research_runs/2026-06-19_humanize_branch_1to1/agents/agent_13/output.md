- core_role: Prompt/template block used as the canonical user-facing denial message when an agent tries to create, read, edit, write, or shell-modify legacy `round-*-todos.md` files. - algorithmic_behavior: The template itself is static text: it instructs “Do NOT create or access” matching round todo files and requires native TodoWrite instead. The enforcement algorithm lives in hooks: shared helper `is_round_file_type` matches `round-[number]-todos.md`; `todos_blocked_message` loads this template through `load_and_render_safe`; Read, Write, Edit, and Bash validators call that helper and exit with code 2 to block the attempted operation. See [prompt-template/block/todos-file-access.md](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/prompt-template/block/todos-file-access.md:3), [hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/lib/loop-common.sh:65), [hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/lib/loop-common.sh:86).
- inputs_outputs_state: Inputs are attempted tool operations whose path or command text targets a `round-*-todos.md` filename. Outputs are blocking stderr messages rendered from this template, plus hook exit code 2. The state transition prevented here is the creation of a parallel todo-file state; task state must remain in Claude Code’s native TodoWrite system. Stop-hook logic later reads the transcript’s latest TodoWrite state to block exit while todos remain incomplete, so the template supports that single-source-of-truth design.
