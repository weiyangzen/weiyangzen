### TEMPLIFY_PROMPT-HZ-019 `file` `hooks/loop-read-validator.sh`
- cursor: `[_]`
- core_role: PreToolUse `Read` validator for RLCR loop artifacts. It prevents the assistant from reading stale, misplaced, or scheduler-owned loop files, especially `round-N-summary.md`, `round-N-prompt.md`, and `round-N-todos.md`. It is registered under the `Read` matcher in [hooks/hooks.json](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/hooks.json:24).
- algorithmic_behavior: The hook reads JSON from stdin, extracts `.tool_name` and `.tool_input.file_path` with `jq`, and exits immediately unless the tool is `Read` ([hooks/loop-read-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/loop-read-validator.sh:22)). It lowercases the path for pattern checks, blocks `round-*-todos.md` reads with `todos_blocked_message`, ignores non-summary/non-prompt paths, then resolves the active loop from `${CLAUDE_PROJECT_DIR:-$(pwd)}/.humanize-loop.local` ([hooks/loop-read-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/loop-read-validator.sh:36), [hooks/loop-read-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/loop-read-validator.sh:56)). For round summary/prompt files, it extracts the attempted round from the basename, determines file type, and applies three gates: file must be inside `.humanize-loop.local`, round number must equal the active state round, and the full path must exactly equal `$ACTIVE_LOOP_DIR/$CLAUDE_FILENAME`.
- inputs_outputs_state: Inputs are hook JSON on stdin, `CLAUDE_PROJECT_DIR` or current working directory, the `.humanize-loop.local/<session>/state.md` frontmatter, and the requested file path. Outputs are process status only: `0` allows the read, `2` blocks the read and writes a rendered markdown diagnostic to stderr. The hook does not mutate loop state; it projects current state into access decisions. Current round is parsed from `current_round:` frontmatter and defaults to `0` if absent via [hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/lib/loop-common.sh:48).
- gates_or_invariants: `round-*-todos.md` must not be read through `Read`; native todo tooling is expected instead. Summary/prompt reads are allowed only for the active loop directory and active round. Wrong locations render `block/wrong-file-location.md`, wrong round reads render `block/wrong-round-file.md`, and old-session or otherwise non-exact paths render `block/wrong-directory-path.md` ([hooks/loop-read-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/loop-read-validator.sh:87), [hooks/loop-read-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/loop-read-validator.sh:103), [hooks/loop-read-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/loop-read-validator.sh:124)). The exact-path invariant is string-based, not inode/canonical-path based.
- dependencies_and_callers: Depends on `jq`, `basename`, standard shell tools, and shared helpers from [hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/lib/loop-common.sh:11). `find_active_loop` selects the newest loop subdirectory by sorted name and only accepts it if `state.md` exists ([hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/lib/loop-common.sh:27)). File type and round extraction come from `is_round_file_type` and `extract_round_number` ([hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/lib/loop-common.sh:67), [hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/lib/loop-common.sh:77)). Diagnostics use `load_and_render_safe`, which falls back if a template is missing or empty ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/lib/template-loader.sh:155)).
- edge_cases_or_failure_modes: If no active loop is found, the hook allows the read, so validation is inactive before loop initialization or after malformed active-loop discovery. If JSON is malformed or `jq` is unavailable, `set -euo pipefail` can fail the hook before a friendly diagnostic. Round matching accepts only filenames ending in `round-[0-9]+-{summary,prompt,todos}.md`; nonconforming files are ignored. `is_in_humanize_loop_dir` is substring based, so canonical path safety depends on the later exact-path check. Relative paths, symlink paths, or case-variant paths may be blocked even if they resolve to the same file because the final directory gate compares raw strings.

### TEMPLIFY_PROMPT-HZ-049 `file` `prompt-template/block/wrong-round-number.md`
- cursor: `[_]`
- core_role: Markdown diagnostic template for the RLCR round-number gate used when a write/edit targets a summary file for a non-current round. It is a small contract surface: tell the agent the attempted round, the authoritative current round, the correct path, and that it must not increment the round itself.
- algorithmic_behavior: The template contains five placeholders: `ACTION`, `CLAUDE_ROUND`, `FILE_TYPE`, `CURRENT_ROUND`, and `CORRECT_PATH` ([prompt-template/block/wrong-round-number.md](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/prompt-template/block/wrong-round-number.md:3)). It renders a blocking message such as “trying to edit `round-3-summary.md`, but the current round is 5,” followed by a correct path and an explicit “Do NOT increment” instruction ([prompt-template/block/wrong-round-number.md](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/prompt-template/block/wrong-round-number.md:5)).
- inputs_outputs_state: Inputs are the caller-supplied placeholder values. Output is rendered markdown, normally written to stderr by a validator immediately before exit code `2`. The template has no independent state transitions; it supports the validator’s state projection from `state.md` current round into a user-facing block reason.
- gates_or_invariants: The invariant encoded here is that summary writes/edits must target the active `CURRENT_ROUND`, and the agent must not self-advance by choosing a future `round-N-summary.md`. The template itself is not enforcement; enforcement comes from the sibling validators that render it only after comparing extracted round to current round.
- dependencies_and_callers: Called by [hooks/loop-write-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/loop-write-validator.sh:130) for wrong-round summary writes, passing `ACTION=write to`, `FILE_TYPE=summary`, and a computed correct path ([hooks/loop-write-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/loop-write-validator.sh:140)). Called by [hooks/loop-edit-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/loop-edit-validator.sh:91) for wrong-round summary edits, passing `ACTION=edit` and the same round/path data ([hooks/loop-edit-validator.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/loop-edit-validator.sh:108)). Rendering is handled by the single-pass template loader, which leaves unknown placeholders intact and avoids recursive placeholder injection ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/templify-prompt/hooks/lib/template-loader.sh:39)).
- edge_cases_or_failure_modes: The template is generic enough to render any `FILE_TYPE`, but current callers reach it only for summary files; prompt writes/edits are blocked earlier by separate gates. If a caller omits a placeholder, the literal `{{PLACEHOLDER}}` remains in the message because the renderer preserves unknown variables. If the template is missing or empty, callers use inline fallbacks through `load_and_render_safe`, so blocked operations should still receive a diagnostic.
