### ADD_GEN_PLAN_COMMAND-HZ-013 `directory` `prompt-template/codex`
- cursor: `[_]`
- core_role: Codex review prompt layer for the RLCR loop. The directory contains three templates: `regular-review.md`, `full-alignment-review.md`, and `goal-tracker-update-section.md`. These templates define how Claude's implementation summary is converted into a Codex review contract and how Codex can advance, block, or stop the loop through review-result output.
- algorithmic_behavior: `hooks/loop-codex-stop-hook.sh:671-748` constructs `round-N-review-prompt.md` from these templates after a Claude summary exists. `regular-review.md:1-57` defines ordinary per-round review: read the original plan, compare Claude's claimed work with actual implementation, audit goal alignment, and only end with `COMPLETE` if all original work is fully done with no deferrals or pending items. `full-alignment-review.md:1-93` defines every-fifth-round alignment review: audit all acceptance criteria, forgotten work, deferred work, implementation quality, and stagnation over historical round files; it may end with `STOP` for circuit-breaker termination. `goal-tracker-update-section.md:1-17` is embedded into both review templates and delegates goal-tracker edits to Codex-side evaluation, while preserving the immutable ultimate goal and acceptance criteria.
- inputs_outputs_state: Inputs are `CURRENT_ROUND`, `PLAN_FILE`, `PROMPT_FILE`, `SUMMARY_CONTENT`, `GOAL_TRACKER_FILE`, `DOCS_PATH`, `REVIEW_RESULT_FILE`, `LOOP_TIMESTAMP`, `COMPLETED_ITERATIONS`, and prior-round indexes, all supplied in `loop-codex-stop-hook.sh:718-748`. Outputs are rendered review prompts written to `round-N-review-prompt.md`, then Codex output written or copied to `round-N-review-result.md` via `loop-codex-stop-hook.sh:897-953`. State transition is marker-driven: `COMPLETE` as the last non-empty line enters Finalize Phase by renaming state to `finalize-state.md` (`loop-codex-stop-hook.sh:961-1020`); `STOP` terminates with the stop exit reason (`loop-codex-stop-hook.sh:1023-1050`); any other review content increments `current_round` and creates the next Claude prompt (`loop-codex-stop-hook.sh:1057-1108`).
- gates_or_invariants: Both review templates make deferrals incomplete: `regular-review.md:53-56` and `full-alignment-review.md:91-93` require all tasks and ACs to be fully met before `COMPLETE`. `full-alignment-review.md:74-85` adds a stagnation audit and requires `STOP` as a single final line when repeated failure patterns are found. The hook enforces strict final-line marker parsing with whitespace trimming only (`loop-codex-stop-hook.sh:955-960`), preventing accidental strings such as "cannot complete" from advancing the loop.
- dependencies_and_callers: Called by `hooks/loop-codex-stop-hook.sh` after summary validation and before Codex execution. Rendering depends on `hooks/lib/template-loader.sh`, whose single-pass placeholder substitution keeps injected `{{VAR}}` in values from being recursively expanded (`template-loader.sh:50-131`) and whose safe loader provides fallback text when templates are missing (`template-loader.sh:170-203`). `tests/test-templates-comprehensive.sh:531-539` directly validates rendering for `goal-tracker-update-section.md`.
- edge_cases_or_failure_modes: Missing templates do not hard-fail because `load_and_render_safe` falls back (`template-loader.sh:170-203`), but weaker fallback prompts may reduce review rigor. If Codex exits nonzero, fails to create the review result, or writes an empty result, the hook blocks the loop with debug paths (`loop-codex-stop-hook.sh:878-950`). `STOP` outside a full-alignment round is still honored but logged as unusual (`loop-codex-stop-hook.sh:1038-1047`). `COMPLETE` at max iterations bypasses Finalize Phase and ends as max-iteration termination (`loop-codex-stop-hook.sh:961-968`).
- validation_or_tests: Template rendering is covered by `tests/test-templates-comprehensive.sh:531-539`; template loader behavior and special-character substitution are broadly tested in `tests/test-template-loader.sh` and `tests/test-error-scenarios.sh` by reference; `tests/test-template-references.sh:172-200` enforces safe rendering usage for critical validators, though its listed critical scripts do not include the stop hook. Stop-hook JSON and marker/state behavior are referenced in `tests/test-hook-preservation.sh` and `tests/test-finalize-phase.sh`.
