- inputs_outputs_state: Inputs are Claude hook events and their hook JSON payloads; this file passes them to scripts through stdin by command registration. It relies on `CLAUDE_PLUGIN_ROOT` resolving to the plugin root in every command path (`hooks/hooks.json:9`, `:20`, `:29`, `:38`, `:47`, `:57`). It produces no direct output and stores no state. State effects are delegated: plan validator blocks via JSON decisions, PreToolUse validators block by exit `2`, and the Stop hook can advance rounds, enter finalize phase, or end the loop.
- gates_or_invariants: Ensures every core interaction surface has a gate: prompt submission checks schema/branch/plan tracking, tool use checks protected RLCR file access, and stop checks completion through Codex.
