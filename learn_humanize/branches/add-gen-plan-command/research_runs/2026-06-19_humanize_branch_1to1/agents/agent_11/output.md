- core_role: Specific block template for the plan-integrity gate in the RLCR Stop hook. It tells the user the source plan changed after loop start and gives the required recovery path: cancel current loop, update plan, restart loop with that plan, and reference the backup path.
- algorithmic_behavior: The template is loaded when the Stop hook verifies that the current plan file content differs from the loop backup. The hook first checks tracked-plan uncommitted modifications with `git status --porcelain` when `PLAN_TRACKED=true`, then always performs `diff -q "$FULL_PLAN_PATH" "$BACKUP_PLAN"` to catch content drift, including committed changes in tracked files (`hooks/loop-codex-stop-hook.sh:214`, `hooks/loop-codex-stop-hook.sh:230`, `hooks/loop-codex-stop-hook.sh:231`). The template itself contains placeholders at `plan-file-modified.md:3`, `plan-file-modified.md:10`, and `plan-file-modified.md:12`.
- inputs_outputs_state: Inputs are `{{PLAN_FILE}}` and `{{BACKUP_PATH}}`, supplied from parsed loop state and the active loop backup path. Output is rendered Markdown used as the JSON `reason`. The template does not mutate state; the caller exits successfully from the hook process after emitting a block decision, which tells Claude Code to stop the attempted exit and continue the active loop under unchanged plan assumptions.
