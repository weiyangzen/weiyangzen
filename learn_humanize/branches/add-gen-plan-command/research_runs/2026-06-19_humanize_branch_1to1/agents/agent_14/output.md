- core_role: Provides the user-facing cancellation algorithm for an active RLCR loop. - algorithmic_behavior: The skill discovers the current loop by selecting the newest timestamped `.humanize/rlcr/*/` directory with `ls -1d ... | sort -r | head -1` (`skills/cancel-rlcr-loop/SKILL.md:20-25`). In finalize mode it first asks for confirmation, then creates `.cancel-requested` and renames `finalize-state.md` to `cancel-state.md` only if the user confirms (`skills/cancel-rlcr-loop/SKILL.md:43-56`).
- inputs_outputs_state: Inputs are the local `.humanize/rlcr` session tree, the newest timestamp directory, either `state.md` or `finalize-state.md`, and in finalize mode a user confirmation answer. Outputs are a `.cancel-requested` signal file plus preserved terminal loop state in `cancel-state.md`; user-facing output reports no active loop, already-ended loop, cancelled normal loop, cancelled finalize phase, or continue-finalize (`skills/cancel-rlcr-loop/SKILL.md:27`, `skills/cancel-rlcr-loop/SKILL.md:35`, `skills/cancel-rlcr-loop/SKILL.md:41`, `skills/cancel-rlcr-loop/SKILL.md:54-56`). The state transition is active normal/finalize state file to cancelled preserved state file, with cancellation authorization signal written before the move.

Shared hook code defines active-loop discovery using newest directory and `state.md`/`finalize-state.md` only (`hooks/lib/loop-common.sh:58-81`), path predicates for state files (`hooks/lib/loop-common.sh:256-266`), and generic `end_loop` state preservation behavior (`hooks/lib/loop-common.sh:615-648`).
