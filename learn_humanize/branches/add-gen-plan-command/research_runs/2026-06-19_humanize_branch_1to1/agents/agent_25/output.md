It does not mutate loop state; it only reads loop state to choose gates. The state-machine read is: active loop discovery -> normal/finalize state selection -> current round parse -> path-specific allow/block.

### ADD_GEN_PLAN_COMMAND-HZ-055 `file` `hooks/lib/loop-common.sh`
- cursor: `[_]`
- core_role: Shared RLCR hook algorithm library. It centralizes loop discovery, state frontmatter parsing, file/path classification, allowlist policy, block-message rendering, cancel authorization, `.humanize` Git-add protection, shell write-detection heuristics, and terminal state renaming. It is used by read/write/edit/bash validators, the plan-file validator, and the Codex stop hook.
- algorithmic_behavior: Initialization defines canonical state fields and exit reasons (`hooks/lib/loop-common.sh:15-40`), sources `template-loader.sh`, computes `TEMPLATE_DIR`, and warns rather than hard-failing when templates are unavailable (`hooks/lib/loop-common.sh:46-56`). `find_active_loop` selects only the newest timestamp-like child directory under `.humanize/rlcr` and treats it as active only if it has `state.md` or `finalize-state.md` (`hooks/lib/loop-common.sh:58-80`). `parse_state_file` extracts YAML-like frontmatter with `sed`, populates global `STATE_*` variables, strips legacy quotes for some fields, and defaults `current_round=0`, `max_iterations=10`, and `push_every_round=false` (`hooks/lib/loop-common.sh:98-138`). Path helpers classify round files, extract round numbers, identify goal/state/finalize files, and check whether a path lives under `.humanize/rlcr` (`hooks/lib/loop-common.sh:141-188`, `250-272`, `415-419`). Message helpers render template-backed block text with fallbacks (`hooks/lib/loop-common.sh:190-248`, `541-568`, `601-613`). `is_cancel_authorized` validates the only allowed state mutation by shell command: an explicitly signaled `mv state.md|finalize-state.md cancel-state.md` in the active loop (`hooks/lib/loop-common.sh:274-413`). `git_adds_humanize` detects unsafe `git add` forms that could stage `.humanize` loop state (`hooks/lib/loop-common.sh:421-539`). `command_modifies_file` is a generalized regex detector for shell writes to protected files (`hooks/lib/loop-common.sh:571-599`). `end_loop` performs the actual loop terminal transition by renaming `state.md` to `<reason>-state.md` (`hooks/lib/loop-common.sh:615-648`).
- inputs_outputs_state: Inputs are loop base paths, active loop paths, state file paths, lowercased commands, and candidate file paths. Outputs vary by function: `find_active_loop`, `get_current_round`, and message helpers write to stdout; classifiers return shell status; `parse_state_file` sets global variables; `end_loop` mutates state by moving `state.md` to `complete-state.md`, `cancel-state.md`, `maxiter-state.md`, `stop-state.md`, or `unexpected-state.md`. The library does not itself advance rounds; the Codex stop hook increments `current_round` with `sed` after review feedback (`hooks/loop-codex-stop-hook.sh:1059-1060`).
- gates_or_invariants: Active-loop invariant is newest directory only; older directories are ignored to prevent “zombie” loops (`hooks/lib/loop-common.sh:58-61`). State parsing expects frontmatter delimited by `---` and field names listed in constants (`hooks/lib/loop-common.sh:15-24`, `119-131`). The allowlist is intentionally exact-path and exact-basename: `round-1-todos.md`, `round-2-todos.md`, `round-0-summary.md`, and `round-1-summary.md` in the active loop only (`hooks/lib/loop-common.sh:170-187`). Cancel authorization requires a `.cancel-requested` signal, rejects command substitution, backticks, newlines, shell chaining, residual `$` variables, extra args, and non-exact source/destination paths (`hooks/lib/loop-common.sh:288-410`). `.humanize` Git protection blocks direct `.humanize` references, force broad adds, all-scope adds when `.humanize` exists, and broad `git add .` / `*` when `.humanize` is not gitignored (`hooks/lib/loop-common.sh:480-535`). Terminal loop reasons are constrained to `complete`, `cancel`, `maxiter`, `stop`, and `unexpected` (`hooks/lib/loop-common.sh:627-635`).
- dependencies_and_callers: Depends on `hooks/lib/template-loader.sh`, plus standard shell tools `sed`, `grep`, `sort`, `head`, `tr`, `basename`, and `git check-ignore` for part of Git-add protection. Callers found include `hooks/loop-read-validator.sh`, `hooks/loop-write-validator.sh`, `hooks/loop-edit-validator.sh`, `hooks/loop-bash-validator.sh`, `hooks/loop-plan-file-validator.sh`, and `hooks/loop-codex-stop-hook.sh`. `hooks/hooks.json:14-59` shows those validators and stop hook are the plugin’s runtime hook surface.
