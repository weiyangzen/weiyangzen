### ADD_GEN_PLAN_COMMAND-HZ-056 `file` `hooks/lib/template-loader.sh`
- cursor: `[_]`
- core_role: Shared prompt/block template rendering library for RLCR hooks. It is not itself a state-machine transition hook, but it is core support for the validators and stop hook because it supplies deterministic message rendering and safe fallbacks for block reasons, review prompts, and next-round prompts.
- algorithmic_behavior: `get_template_dir` resolves the plugin `prompt-template` directory from a script/lib path by walking two levels up (`hooks/lib/template-loader.sh:24-31`). `load_template` returns file contents when present or emits a warning and empty content when absent (`hooks/lib/template-loader.sh:33-48`). `render_template` performs single-pass replacement of `{{VAR}}` placeholders using environment variables prefixed with `TMPL_VAR_`, deliberately avoiding recursive expansion of placeholders inside values (`hooks/lib/template-loader.sh:50-132`). `load_and_render` composes load plus render and returns nothing for missing/empty templates (`hooks/lib/template-loader.sh:134-147`). `append_template` concatenates base and additional loaded content (`hooks/lib/template-loader.sh:149-161`). `load_and_render_safe` falls back to an inline message when a template is missing or renders empty (`hooks/lib/template-loader.sh:167-203`). `validate_template_dir` requires the root plus `block`, `codex`, and `claude` subdirectories (`hooks/lib/template-loader.sh:205-222`).
- inputs_outputs_state: Inputs are a template directory, template name, template content, and `VAR=value` assignments. Outputs are rendered text on stdout, warnings/errors on stderr, and status codes for directory validation. No persistent state is mutated. Missing placeholders remain unchanged by design (`hooks/lib/template-loader.sh:119-122`). Values containing `=` are preserved after the first delimiter via `${var_assignment#*=}` (`hooks/lib/template-loader.sh:65-68`). The renderer processes line by line through awk and prints rendered lines, so output text is normalized through awk’s `print` path (`hooks/lib/template-loader.sh:84-129`).
- gates_or_invariants: Template placeholder syntax is documented as `{{VARIABLE_NAME}}`, uppercase/numbers/underscores, but the implementation actually accepts any text between `{{` and `}}` as the lookup key (`hooks/lib/template-loader.sh:7-13`, `hooks/lib/template-loader.sh:111-122`). The primary safety invariant is single-pass rendering: substituted values are emitted directly and never rescanned, preventing review content or fallback values from injecting additional template substitutions (`hooks/lib/template-loader.sh:54-57`, `hooks/lib/template-loader.sh:115-125`). The safe-load invariant is that critical callers can always produce a nonempty block reason if either the template exists or an inline fallback exists (`hooks/lib/template-loader.sh:170-203`).
- dependencies_and_callers: `hooks/lib/loop-common.sh` sources this file, initializes `TEMPLATE_DIR`, and warns rather than aborting if template validation fails (`hooks/lib/loop-common.sh:46-56`). Callers include `loop-plan-file-validator.sh` for schema-outdated blocks, read/write/edit/bash validators for access blocks, and `loop-codex-stop-hook.sh` for plan integrity, git cleanliness, Codex review prompts, Finalize Phase prompt, and next-round prompt/footer generation (`hooks/loop-plan-file-validator.sh:62`; `hooks/loop-codex-stop-hook.sh:244`, `hooks/loop-codex-stop-hook.sh:720-743`, `hooks/loop-codex-stop-hook.sh:1005-1008`, `hooks/loop-codex-stop-hook.sh:1075-1092`). Template reference scanning is codified in `tests/test-template-references.sh:56-110`.
- edge_cases_or_failure_modes: `load_template` missing-file behavior is intentionally nonfatal, but unsafe callers using `load_and_render` without fallback can silently get empty output (`hooks/lib/template-loader.sh:36-47`, `hooks/lib/template-loader.sh:141-146`). `validate_template_dir` only checks the three top-level subdirectories, not individual files (`hooks/lib/template-loader.sh:216-221`). Invalid `VAR=value` assignments without `=` become a variable whose value equals the whole string, because shell parameter expansion returns the original string for both name and value paths. Newlines and shell metacharacters in values are passed through environment variables into awk; tests cover many such cases, but environment-size limits remain a practical boundary for very large rendered values. `append_template` is not safe/fallback-aware and will append empty content plus a blank line if the additional template is missing (`hooks/lib/template-loader.sh:151-160`).

It also specifically covers placeholder-in-value injection prevention and realistic review content containing `{{GOAL_TRACKER_FILE}}`/`{{PLAN_FILE}}` (`tests/test-template-loader.sh:500-543`). `tests/test-templates-comprehensive.sh` adds fallback, invalid/incomplete directory, real-template rendering, and broader edge coverage (`tests/test-templates-comprehensive.sh:440-510`). `tests/test-error-scenarios.sh` covers missing templates and fallback behavior.
