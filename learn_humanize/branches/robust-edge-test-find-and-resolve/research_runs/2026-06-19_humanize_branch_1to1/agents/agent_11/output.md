- core_role: Executable shell specification for two robustness fixes: preventing accidental or forced version-control inclusion of `.humanize` loop state, and replacing shell-glob iteration with `find`-based iteration to avoid zsh “no matches found” failures.
- algorithmic_behavior: The test imports production helpers from [hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/robust-edge-test-find-and-resolve/hooks/lib/loop-common.sh:18), lowercases candidate commands, and asserts whether `git_adds_humanize` blocks or allows them. Group 6 specifies allowed cases: normal file additions, non-add git commands, patch mode, and similarly named files like `.humanizeconfig`. Groups 7-8 simulate the `find` loops used by the monitor for empty directories, dotfiles-only directories, missing state files, state-file detection, and timestamped session directory selection.
- inputs_outputs_state: Inputs are command strings and temporary filesystem layouts under `/tmp/test-humanize-*`. Outputs are colored PASS/FAIL lines and process exit status: zero when `TESTS_FAILED` remains zero, one otherwise. Test state is held in counters `TESTS_PASSED` and `TESTS_FAILED`, plus temporary directories created and removed inside each scenario. - gates_or_invariants: The primary invariant is that commands capable of staging `.humanize` local loop state must be detected and blocked. Direct references are blocked regardless of filesystem state; `-A`/`--all` cases are blocked when `.humanize` exists; force plus broad scope is blocked because it bypasses ignore protection. The complementary invariant is no over-blocking for regular files, `.gitignore`, patch mode, non-add commands, and names that merely begin with `.humanize`. For zsh safety, iteration must return zero items on empty or irrelevant directories instead of expanding an unmatched glob.
- dependencies_and_callers: The tested production function is [hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/robust-edge-test-find-and-resolve/hooks/lib/loop-common.sh:644), which splits command segments on shell operators, detects `git ... add`, strips quotes for path matching, identifies direct `.humanize` references, recognizes force/all/broad-scope flags, and consults `git check-ignore` for broad non-force adds.
