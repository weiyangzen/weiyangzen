### IMPROVE_GEN_PLAN_COMMAND-HZ-032 `file` `scripts/cancel-rlcr-loop.sh`
- cursor: `[_]`
- core_role: User-facing cancellation script for an active RLCR loop. It implements the authorized cancellation state transition from active `state.md` or `finalize-state.md` to `cancel-state.md`, guarded by a `.cancel-requested` signal file.
- algorithmic_behavior: The script parses only `--force`, `-h`, and `--help`; unknown options exit `3` at `scripts/cancel-rlcr-loop.sh:24-63`. It resolves `PROJECT_ROOT` from `CLAUDE_PROJECT_DIR` or `pwd`, sets `LOOP_BASE_DIR="$PROJECT_ROOT/.humanize/rlcr"`, selects the newest loop directory by reverse sort, and exits `NO_LOOP` with code `1` when none exists at `scripts/cancel-rlcr-loop.sh:69-79`. It then classifies the active state as normal if `state.md` exists, finalize if `finalize-state.md` exists, or `NO_ACTIVE_LOOP` when neither exists at `scripts/cancel-rlcr-loop.sh:85-99`.
- inputs_outputs_state: Inputs are CLI flags, `CLAUDE_PROJECT_DIR`, active loop files under `.humanize/rlcr/<timestamp>/`, and optional `current_round:` / `max_iterations:` fields parsed by grep/sed at `scripts/cancel-rlcr-loop.sh:105-111`. Outputs are status lines such as `NO_LOOP`, `NO_ACTIVE_LOOP`, `FINALIZE_NEEDS_CONFIRM`, `CANCELLED`, or `CANCELLED_FINALIZE`; exit codes are documented at `scripts/cancel-rlcr-loop.sh:11-15`. State transition is `touch "$LOOP_DIR/.cancel-requested"` followed by `mv "$ACTIVE_STATE_FILE" "$LOOP_DIR/cancel-state.md"` at `scripts/cancel-rlcr-loop.sh:134-138`.
- gates_or_invariants: Finalize phase requires confirmation unless `--force` is supplied. Without force, the script prints `FINALIZE_NEEDS_CONFIRM`, loop path, round metadata, and exits `2` at `scripts/cancel-rlcr-loop.sh:117-128`. The user-facing slash command enforces this by asking a confirmation question and rerunning with `--force` only on affirmative answer at `commands/cancel-rlcr-loop.md:17-34`. The script’s core invariant is that cancellation preserves the active state as `cancel-state.md`, never deleting loop history.
- dependencies_and_callers: Direct caller is `commands/cancel-rlcr-loop.md`, which allows exactly the plain and `--force` Bash invocations and states that active means newest loop directory containing `state.md` or `finalize-state.md` at `commands/cancel-rlcr-loop.md:1-4` and `commands/cancel-rlcr-loop.md:36-38`. Runtime safety also depends on `hooks/loop-bash-validator.sh`, which blocks state-file shell mutation except when `is_cancel_authorized` accepts the signal-authorized move at `hooks/loop-bash-validator.sh:139-155`; the authorization parser lives in `hooks/lib/loop-common.sh:437-614`.
- edge_cases_or_failure_modes: If `.humanize/rlcr` has older active sessions but the newest directory lacks `state.md` and `finalize-state.md`, this script selects the newest directory and reports no active loop. Missing round fields are rendered as `?` at `scripts/cancel-rlcr-loop.sh:109-111`. `mv` can overwrite an existing `cancel-state.md` because no existence guard is present. Newest directory selection relies on lexicographic timestamp naming. Finalize cancellation without `--force` intentionally blocks even though the state file exists.
- validation_or_tests: `tests/test-cancel-signal-file.sh` verifies that the Bash validator allows `mv state.md cancel-state.md` only with a signal, rejects absent or wrong-directory signals, and rejects other mutations even with the signal at `tests/test-cancel-signal-file.sh:75-215`. The assigned robustness test expands this coverage through direct `is_cancel_authorized` cases in `tests/robustness/test-cancel-security-robustness.sh:39-405`.
