It then asserts validator exit code `0` for authorized cancel-shaped `mv` commands and exit code `2` for blocked state mutation or bypass attempts.

The modeled transition is `state.md` to `cancel-state.md`, authorized only when the active loop directory contains `.cancel-requested`.

- edge_cases_or_failure_modes: Explicitly covers missing active loop, which must allow commands because no RLCR state is active (`tests/test-cancel-signal-file.sh:1233-1249`). It also covers source-side bypasses such as moving `state.md` away from the loop, command prefixes such as `sudo`/`env`/`command`, and redirection-prefix parsing where shell syntax can hide the real command (`tests/test-cancel-signal-file.sh:552-1206`).

### ASK_CODEX_SKILL-HZ-086 `file` `hooks/lib/template-loader.sh`
- cursor: `[_]`
- core_role: Shared prompt-template loading/rendering library for RLCR/PR validators and stop hooks. It turns algorithmic block decisions into stable user-facing messages while preserving fallback behavior when templates are unavailable.
- algorithmic_behavior: `get_template_dir` resolves plugin root from `hooks/lib` to `prompt-template` (`hooks/lib/template-loader.sh:24-31`). `load_template` cats a requested template or emits a warning and empty output if absent (`hooks/lib/template-loader.sh:33-48`). `render_template` builds `TMPL_VAR_*` environment values and uses a single-pass `awk` scanner over `{{VAR}}` placeholders, replacing known variables and leaving unknown placeholders literal (`hooks/lib/template-loader.sh:50-132`). `load_and_render`, `append_template`, and `load_and_render_safe` compose loading/rendering and fallback behavior (`hooks/lib/template-loader.sh:134-203`). `validate_template_dir` requires `block`, `codex`, and `claude` subdirectories (`hooks/lib/template-loader.sh:205-222`).
- inputs_outputs_state: Inputs are a template directory, relative template name, template content, and `VAR=value` assignments. Outputs are rendered text to stdout and warnings/errors to stderr. No persistent state is mutated; the only state-like behavior is the environment passed to `awk` for one render call.
- gates_or_invariants: Important invariant is single-pass substitution: values containing `{{OTHER}}` are emitted without recursive expansion, preventing prompt corruption or placeholder injection (`hooks/lib/template-loader.sh:54-57`, `71-129`). Missing variables remain as placeholders (`hooks/lib/template-loader.sh:115-122`). Safe loading falls back if the template is missing or renders empty (`hooks/lib/template-loader.sh:170-203`).
- dependencies_and_callers: Sourced by `hooks/lib/loop-common.sh` during library setup (`hooks/lib/loop-common.sh:156-165`), so most validators use it for block messages. Direct callers include `loop-read-validator.sh` for wrong round/location blocks (`hooks/loop-read-validator.sh:122-150`) and `loop-bash-validator.sh` for git-push and file-protection messages (`hooks/loop-bash-validator.sh:96-105`, `336-340`). Tests include comprehensive template loading/syntax checks (`tests/test-templates-comprehensive.sh:59-194`) and error/fallback robustness (`tests/test-error-scenarios.sh:42-180`, `tests/robustness/test-template-error-robustness.sh:34-220`).
- edge_cases_or_failure_modes: Missing template returns empty for raw load but fallback for safe load. Unclosed `{{` is treated literally by the scanner. The comments define uppercase placeholder convention (`hooks/lib/template-loader.sh:7-13`), while enforcement of malformed/lowercase placeholders is handled by tests rather than by `render_template` itself. Template names are concatenated under the template directory, so callers must pass trusted repo-relative template names.
- validation_or_tests: Validated by template tests that load all `prompt-template/*.md`, check placeholder syntax, verify special characters, missing-template fallbacks, non-recursive substitution, BOM/empty-template handling, and strict-mode behavior.
- skip_candidate: `no`
