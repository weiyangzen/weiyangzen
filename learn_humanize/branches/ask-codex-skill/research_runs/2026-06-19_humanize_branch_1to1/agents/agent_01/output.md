It extracts `.session_id` with `jq` (`hooks/loop-post-bash-hook.sh:94`) and patches an empty `session_id:` line with `awk` (`hooks/loop-post-bash-hook.sh:105`).
- inputs_outputs_state: Input is Claude hook JSON, especially `.tool_input.command` and `.session_id`. Persistent input/output is `.humanize/.pending-session-id`, whose first line is `state.md` path and second line is the full resolved setup script path. State output is replacement of `session_id:` in `state.md` with `session_id: <id>` when currently empty. The signal file is removed after successful recording (`hooks/loop-post-bash-hook.sh:122`).
- gates_or_invariants: No signal means no-op exit 0 (`hooks/loop-post-bash-hook.sh:35`). Invalid or missing state file causes signal cleanup and exit 0 (`hooks/loop-post-bash-hook.sh:50`). If command signature exists, the hook refuses to consume the signal unless the observed Bash command is a real setup invocation (`hooks/loop-post-bash-hook.sh:60`). If `session_id` is missing, it preserves the signal for a later attempt (`hooks/loop-post-bash-hook.sh:100`). - edge_cases_or_failure_modes: Requires `jq`; without it, command and session extraction remain empty and the hook exits without consuming the signal. It rejects false positives such as `echo /path/setup-rlcr-loop.sh`, basename-only references, and quoted-prefix concatenation by requiring command start plus whitespace/end boundary.
