It classifies the active loop as normal when `state.md` exists or finalize phase when `finalize-state.md` exists (`:99-113`), extracts `current_round` and `max_iterations` with fallback `?` values (`:119-125`), requires explicit `--force` in finalize phase (`:131-142`), then touches `.cancel-requested`, removes `.humanize/.pending-session-id`, and renames the active state file to `cancel-state.md` (`:148-155`).
- inputs_outputs_state: Inputs are CLI args, `CLAUDE_PROJECT_DIR` or `pwd`, the newest active loop directory, `state.md` or `finalize-state.md`, and the shared `find_active_loop` function. Outputs are status sentinels on stdout: `NO_LOOP`, `NO_ACTIVE_LOOP`, `FINALIZE_NEEDS_CONFIRM`, `CANCELLED`, or `CANCELLED_FINALIZE` (`:89-92`, `:110-112`, `:131-141`, `:161-169`). State changes are `.cancel-requested` creation, pending-session-id cleanup, and active state rename to `cancel-state.md`.
- gates_or_invariants: Exit codes are part of the contract: `0` success, `1` no active loop, `2` finalize confirmation required, `3` bad args/other error (`:11-15`, `:43-47`). Finalize phase cannot be canceled accidentally unless `--force` is present (`:131-142`). It relies on `find_active_loop` newest-directory semantics to avoid reviving stale older loops. The `.cancel-requested` signal also aligns with Bash validator authorization, which only allows state-file renames for cancellation when the signal is present.
- dependencies_and_callers: Depends on `hooks/lib/loop-common.sh` (`:72-74`). Called by `commands/cancel-rlcr-loop.md:11-36`, whose slash-command contract maps first-line output to user messaging and asks for confirmation before rerunning with `--force`.
- edge_cases_or_failure_modes: If the newest loop directory exists but has neither `state.md` nor `finalize-state.md`, the script reports `NO_ACTIVE_LOOP` and does not scan older loops (`:103-113`). If round fields are missing, output remains useful with `?` placeholders. Because `set -euo pipefail` is active, a failed `touch`, `rm`, or `mv` aborts rather than pretending cancellation succeeded. If `touch "$CANCEL_SIGNAL"` succeeds but `mv` fails, a stale `.cancel-requested` could remain, but the intended subsequent state rename would not be reported as successful. The script intentionally ignores session identity, documented as a product decision at `:76-87`.
