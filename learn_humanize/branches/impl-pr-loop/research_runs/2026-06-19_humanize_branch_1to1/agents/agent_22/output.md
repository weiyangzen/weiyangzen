### IMPL_PR_LOOP-HZ-022 `file` `hooks/check-todos-from-transcript.py`
- cursor: `[_]`
- core_role: Stop-hook validator helper for RLCR/Codex-loop sessions. It reads Claude Code transcript state and enforces the invariant that the agent cannot exit while native `TodoWrite` todos remain incomplete. The direct implementation is in [hooks/check-todos-from-transcript.py](/Users/wangweiyang/GitHub/humanize_branch_worktrees/impl-pr-loop/hooks/check-todos-from-transcript.py:20), and the stop-hook caller is [hooks/loop-codex-stop-hook.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/impl-pr-loop/hooks/loop-codex-stop-hook.sh:261).
- algorithmic_behavior: `find_latest_todos(transcript_path)` streams transcript JSONL line by line, ignores blank and malformed JSONL records, and keeps the latest non-empty `TodoWrite` payload it recognizes. It supports three transcript shapes: Claude Code assistant messages with `message.content[].tool_use`, alternate `type: message` entries with `content[].tool_use`, and direct `type: tool_use` entries. `main()` reads hook input JSON from stdin, extracts `transcript_path`, expands `~`, loads the latest todos, and classifies every todo with `status != "completed"` as incomplete.
- inputs_outputs_state: Input is hook JSON on stdin, normally containing `transcript_path`; secondary input is the transcript JSONL file. Output is process status plus optional text: exit `0` means no blocking todos or no transcript/todos; exit `1` prints `INCOMPLETE_TODOS` followed by incomplete todo lines; exit `2` prints `PARSE_ERROR` to stderr for invalid hook input JSON. It does not mutate repository or loop state itself; the caller converts exit `1` or `2` into a JSON stop-hook block decision.
- gates_or_invariants: The gate is fail-closed for malformed hook input and incomplete todos, but permissive when no transcript path, missing transcript file, empty transcript, or no recognized todos exist. Incomplete means any status other than exact `completed`, including `pending`, `in_progress`, missing status, or empty status. The caller blocks before expensive Codex review when exit `1` occurs and renders `prompt-template/block/incomplete-todos.md` with the incomplete list at [hooks/loop-codex-stop-hook.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/impl-pr-loop/hooks/loop-codex-stop-hook.sh:287).
- dependencies_and_callers: Uses only Python stdlib modules `json`, `sys`, and `pathlib`. Called by `hooks/loop-codex-stop-hook.sh`, which sets `TODO_CHECKER`, pipes the original hook input to Python, checks exit codes, and emits block JSON for parse errors or incomplete todos. Related validators discourage file-based todo lists and push agents toward native `TodoWrite`, visible in `hooks/loop-read-validator.sh`, `hooks/loop-edit-validator.sh`, and `hooks/loop-write-validator.sh` search references.
- edge_cases_or_failure_modes: Missing transcript path and non-existent transcript are allowed, which prevents false blocks when transcript metadata is unavailable. Malformed JSONL lines inside the transcript are ignored, but malformed stdin hook JSON blocks. A later `TodoWrite` call with an empty `todos` list is ignored because the script only updates `latest_todos` when `todos` is truthy; this means an older non-empty todo list can remain authoritative. A transcript path that exists but is unreadable or is a directory is not caught explicitly and would raise an unhandled Python exception rather than one of the documented exit codes.
- validation_or_tests: Covered by [tests/test-todo-checker.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/impl-pr-loop/tests/test-todo-checker.sh:51), including invalid stdin JSON, missing transcript path, non-existent transcript, all-completed todos, incomplete statuses, invalid JSONL lines, multiple `TodoWrite` calls using the latest non-empty call, direct `tool_use` format, alternate `message` format, missing status, empty content, and Unicode content. `tests/test-finalize-phase.sh` also contains transcript fixtures using `TodoWrite`.
