- core_role: Startup-case classifier for PR loop setup and later re-evaluation. It analyzes all bot reviewer activity across issue comments, inline review comments, and PR review submissions, then emits JSON describing which bots have commented, which are missing, whether reviews are stale relative to the latest commit, and which startup case applies.
- algorithmic_behavior: Parses `<pr_number> --bots <bot1,bot2>` (`scripts/check-pr-reviewer-status.sh:49-85`), maps logical bot names to GitHub author logins (`codex -> chatgpt-codex-connector[bot]`) at `94-100`, resolves the PR base repo for fork-safe API calls (`113-137`), fetches latest commit metadata (`139-148`), fetches issue comments, review comments, and PR review submissions (`150-165`), then loops over configured bots to compute `REVIEWERS_COMMENTED`, `REVIEWERS_MISSING`, `REVIEWERS_STALE`, and global newest review timestamp (`171-202`).
- inputs_outputs_state: Inputs are GitHub CLI state, current/fork repository metadata, PR number, comma-separated bot list, and GitHub API comment/review streams. Output is JSON via `jq -n` with `case`, `reviewers_commented`, `reviewers_missing`, `latest_commit_sha`, `latest_commit_at`, `newest_review_at`, and `has_commits_after_reviews` (`256-272`). It does not mutate repository files or loop state directly.
- gates_or_invariants: Requires PR number and `--bots`; unknown flags or duplicate PR numbers fail (`49-85`). Uses `run_with_timeout` for `gh` calls (`35-40`). Classifies staleness per bot, not by global newest review, so one fresh bot cannot mask another stale bot (`189-197`, `233-237`). Case mapping is: no comments -> case 1; partial comments fresh -> case 2; all commented fresh -> case 3; all commented with stale review -> case 4; partial comments with stale review -> case 5 (`217-241`).
- dependencies_and_callers: Called by `scripts/setup-pr-loop.sh:418-428` to seed `startup_case`, latest commit SHA/date, and trigger behavior. Called again by `hooks/pr-loop-stop-hook.sh:1531-1543` after Codex analysis to dynamically refresh `startup_case`. Depends on `scripts/portable-timeout.sh:31-71`, `gh`, and `jq`.
- edge_cases_or_failure_modes: Fork PR lookup first tries current repo, then parent repo, falling back to current repo with a warning if base resolution fails (`119-137`). Comment API failures default individual comment sources to `[]` (`153-162`), but failure to fetch commit info exits hard (`140-145`). Timestamp comparison is lexicographic ISO-8601 string comparison (`191`), which is valid for normalized `YYYY-MM-DDTHH:MM:SSZ` values but relies on GitHub timestamp normalization.
- validation_or_tests: The main runner sources script tests, and the larger PR system tests explicitly include reviewer-status cases: no comments, partial comments, all commented, PR reviews as valid reviewer activity, and startup cases 4/5 (`tests/test-pr-loop-system.sh:151`, `172`, `235`, `492`, `660`, `1314`, `1346`).
