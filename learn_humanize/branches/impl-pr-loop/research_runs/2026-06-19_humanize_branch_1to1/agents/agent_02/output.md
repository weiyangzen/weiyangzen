### IMPL_PR_LOOP-HZ-032 `file` `scripts/cancel-rlcr-loop.sh`
- cursor: `[_]`
- core_role: Operator-facing RLCR cancellation entrypoint. It converts an active RLCR loop from `state.md` or `finalize-state.md` into `cancel-state.md` and drops `.cancel-requested` so polling hooks can stop cooperatively.
- algorithmic_behavior: Parses `--force` and help, discovers the newest loop directory under `${CLAUDE_PROJECT_DIR:-$(pwd)}/.humanize/rlcr`, chooses active state from `state.md` or `finalize-state.md`, extracts `current_round` and `max_iterations`, then either blocks finalize cancellation without force or performs cancellation (`scripts/cancel-rlcr-loop.sh:24-63`, `69-99`, `105-128`, `134-151`).
- inputs_outputs_state: Inputs are current working directory or `CLAUDE_PROJECT_DIR`, optional `--force`, and the newest timestamped RLCR loop directory (`scripts/cancel-rlcr-loop.sh:69-87`). Outputs are stable stdout status tokens: `NO_LOOP`, `NO_ACTIVE_LOOP`, `FINALIZE_NEEDS_CONFIRM`, `CANCELLED`, or `CANCELLED_FINALIZE` (`scripts/cancel-rlcr-loop.sh:75-78`, `96-98`, `117-127`, `144-151`). State transition is `state.md -> cancel-state.md` for normal loops or `finalize-state.md -> cancel-state.md` for forced finalize cancellation, with `.cancel-requested` created first (`scripts/cancel-rlcr-loop.sh:134-138`).
- gates_or_invariants: It only inspects the newest RLCR loop directory (`scripts/cancel-rlcr-loop.sh:72-79`). It refuses finalize cancellation unless `--force` is present (`scripts/cancel-rlcr-loop.sh:117-128`). It exits `1` for no active loop, `2` for finalize confirmation required, and `3` for argument errors (`scripts/cancel-rlcr-loop.sh:11-15`, `57-60`).
- dependencies_and_callers: Cancellation safety is enforced downstream by hook validators using `.cancel-requested` plus `is_cancel_authorized` in `hooks/lib/loop-common.sh`; that helper requires the signal file and restricts the command to moving `state.md` or `finalize-state.md` to `cancel-state.md` (`hooks/lib/loop-common.sh:437-455`, `461-614`). The stop hook notices `.cancel-requested` during polling and exits the poll loop (`hooks/pr-loop-stop-hook.sh:941-945`). Related PR-specific cancellation is implemented separately in `scripts/cancel-pr-loop.sh`, which targets `.humanize/pr-loop` and only handles `state.md` (`scripts/cancel-pr-loop.sh:72-122`).
- edge_cases_or_failure_modes: If a loop directory exists but neither active state file exists, it reports `NO_ACTIVE_LOOP` and exits `1` (`scripts/cancel-rlcr-loop.sh:89-99`). If `grep` finds no round metadata, printed round/max values default to `?` (`scripts/cancel-rlcr-loop.sh:105-112`). It does not itself reject symlinked state files; that protection lives in the hook authorization helper (`hooks/lib/loop-common.sh:600-610`). Discovery by `ls | sort -r | head -1` means malformed timestamp names can affect which loop is considered newest (`scripts/cancel-rlcr-loop.sh:72-73`).
- validation_or_tests: Direct cancellation-signal behavior is covered by `tests/test-cancel-signal-file.sh`, including allowed move with signal, blocked move without signal, blocked edits even with signal, wrong destination, wrong signal directory, and injection cases (`tests/test-cancel-signal-file.sh:75-193`, `196-215`).
