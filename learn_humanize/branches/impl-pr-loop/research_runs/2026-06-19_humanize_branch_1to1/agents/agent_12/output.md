### IMPL_PR_LOOP-HZ-072 `file` `hooks/lib/template-loader.sh`
- cursor: `[_]`
- core_role: Shared template loading and rendering library for RLCR and PR-loop hooks. It centralizes prompt/block template resolution, variable substitution, fallback rendering, and template directory validation, so hook validators and stop hooks can emit consistent, non-empty instructions.
- algorithmic_behavior: `get_template_dir` derives the template root from a caller-supplied hooks/lib path by moving two levels up and appending `prompt-template` (`hooks/lib/template-loader.sh:24-31`). `load_template` reads a template file if present, otherwise writes a warning to stderr and returns an empty string (`36-47`). `render_template` performs single-pass substitution over `{{VAR}}` placeholders: it converts each `VAR=value` argument into a `TMPL_VAR_VAR=value` environment entry (`62-69`), runs an `awk` scanner over the template (`71-129`), replaces placeholders found in `ENVIRON`, leaves missing placeholders unchanged, and does not rescan replacement values (`115-125`). `load_and_render` composes load plus render but only renders non-empty content (`134-147`). `append_template` concatenates base content and another template (`149-161`). `load_and_render_safe` falls back to a caller-provided message if the template is missing, empty, or renders empty (`167-203`). `validate_template_dir` checks that the root exists and contains `block`, `codex`, and `claude` subdirectories (`205-222`).
- inputs_outputs_state: Inputs are a template directory path, relative template names, base content, fallback strings, and variable assignments in `NAME=value` form. Outputs are rendered Markdown/text on stdout and warnings/errors on stderr. It has no persistent state; its only state-like behavior is deterministic substitution based on process environment passed to `awk`. It deliberately preserves unknown placeholders as literal `{{VAR}}` text (`hooks/lib/template-loader.sh:119-122`), which allows callers/tests to detect missing substitutions downstream.
- gates_or_invariants: The key invariant is single-pass rendering to prevent placeholder injection: if an inserted value contains `{{OTHER_VAR}}`, that nested placeholder is not expanded (`hooks/lib/template-loader.sh:50-57`, `71-73`, `117-118`). Another invariant is graceful degradation: critical hook call sites can use `load_and_render_safe` and still produce fallback text when a template is unavailable (`170-203`). Directory validation is intentionally soft in `hooks/lib/loop-common.sh`; it warns but does not fail so hooks can proceed with inline fallbacks (`hooks/lib/loop-common.sh:135-145`).
- dependencies_and_callers: Uses Bash plus `cat`, `cd`, `pwd`, `env`, and `awk`. It is sourced by `hooks/lib/loop-common.sh:135-140`, making it available to loop read/write/edit/bash validators and stop hooks. Direct call sites include `hooks/loop-read-validator.sh:127-149`, `hooks/loop-write-validator.sh`, `hooks/loop-edit-validator.sh`, `hooks/loop-bash-validator.sh`, `hooks/loop-codex-stop-hook.sh:683-751`, `hooks/pr-loop-stop-hook.sh`, and `scripts/setup-pr-loop.sh`. Tests source it directly in `tests/test-template-loader.sh:10-12`, `tests/test-error-scenarios.sh`, `tests/test-templates-comprehensive.sh`, and robustness tests.
- edge_cases_or_failure_modes: Missing templates return empty from `load_template`; unsafe callers of `load_and_render` can therefore emit no block reason, which is why critical validators are expected to use `load_and_render_safe`. `render_template` splits assignments at the first `=`, so values may contain additional equals signs but names may not. The header comment says variables are uppercase letters, numbers, and underscores (`hooks/lib/template-loader.sh:7-13`), but the renderer itself extracts any text between `{{` and `}}`; placeholder-shape validation is handled by tests/template hygiene rather than enforced inside this function. Unclosed `{{` is treated literally by the scanner (`102-108`). Command substitution around the `awk` call strips trailing newlines before `echo` emits output, so exact trailing-newline preservation is not guaranteed. If a fallback string itself contains placeholders and assignments are provided, the safe loader renders the fallback too (`181-185`, `193-198`).
- validation_or_tests: `tests/test-template-loader.sh` covers path derivation, loading, missing-file behavior, single/multiple/multiline substitution, unresolved placeholders, safe fallback, and directory validation (`tests/test-template-loader.sh:41-230`). The same file includes shell metacharacter tests for ampersands, backslashes, dollar signs, backticks, pipes, semicolons, glob patterns, quotes, JSON-like values, multiline values, and placeholder injection prevention (`tests/test-template-loader.sh:250-543`). `tests/test-templates-comprehensive.sh` adds broader edge-case rendering and validates every template can load and render (`tests/test-templates-comprehensive.sh:262-490`, `562-606`). `tests/test-template-references.sh` scans hook template references and verifies critical validators use safe rendering (`tests/test-template-references.sh:56-111`, `172-200`).
