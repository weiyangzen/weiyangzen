- algorithmic_behavior: The script parses a command mode, then dispatches to `codex-thumbsup` or `claude-eyes` (`scripts/check-bot-reactions.sh:79-87`, `scripts/check-bot-reactions.sh:187`). In `claude-eyes`, it accepts a comment id plus optional `--pr`, `--retry`, and `--delay`, resolves the base repo when a PR number is available, then sleeps and polls `repos/$PR_BASE_REPO/issues/comments/$COMMENT_ID/reactions` until it finds `claude[bot]` with content `eyes` or exhausts attempts (`scripts/check-bot-reactions.sh:187-295`).
- inputs_outputs_state: Inputs are command-line arguments, GitHub CLI authentication/context, `jq`, and `portable-timeout.sh` sourced from the script directory (`scripts/check-bot-reactions.sh:31-33`). Outputs are JSON for the matching reaction on success (`scripts/check-bot-reactions.sh:182-184`, `scripts/check-bot-reactions.sh:281-284`), stderr diagnostics for missing reactions/errors, and exit codes: `0` found, `1` not found/timeout, `2` argument/API error (`scripts/check-bot-reactions.sh:13-16`, `scripts/check-bot-reactions.sh:57-60`). It does not persist repo state.
- gates_or_invariants: Requires a PR number for `codex-thumbsup` and a comment id for `claude-eyes` (`scripts/check-bot-reactions.sh:114-117`, `scripts/check-bot-reactions.sh:224-227`). Reaction matching is strict on GitHub login and content, so only the intended bot signal passes (`scripts/check-bot-reactions.sh:161-165`, `scripts/check-bot-reactions.sh:275-279`). Timestamp filtering uses ISO string comparison, assuming normalized UTC timestamps (`scripts/check-bot-reactions.sh:174-180`).
- dependencies_and_callers: Called by `scripts/setup-pr-loop.sh` to verify Claude eyes for startup cases 4/5 after posting a trigger comment (`scripts/setup-pr-loop.sh:484-520`). Called by `hooks/pr-loop-stop-hook.sh` for round-0 Codex PR-body approval and trigger-comment Claude acknowledgement (`hooks/pr-loop-stop-hook.sh:459`, `hooks/pr-loop-stop-hook.sh:766`, `hooks/pr-loop-stop-hook.sh:985`).
