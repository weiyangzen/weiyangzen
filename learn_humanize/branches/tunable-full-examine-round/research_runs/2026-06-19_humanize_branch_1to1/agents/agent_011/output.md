- core_role: The directory is the Claude-facing prompt template set for the RLCR/humanize loop’s implementation, review-fix, next-round, and finalize phases. It is not executable code itself, but it is core algorithm configuration: `hooks/loop-codex-stop-hook.sh` renders these Markdown fragments into blocking hook responses and round prompt files that drive Claude’s next action. The directory contains eight direct child templates and no nested child directories:
  - `prompt-template/claude/finalize-phase-prompt.md`: clean review passed; enter finalize/simplification phase.
  - `prompt-template/claude/finalize-phase-skipped-prompt.md`: review skipped; enter finalize with manual verification warning.
  - `prompt-template/claude/review-phase-prompt.md`: Codex found `[P0-9]` issues; instruct Claude to fix only review findings.
  - `prompt-template/claude/next-round-prompt.md`: general continuation prompt after review feedback.
  - `prompt-template/claude/next-round-footer.md`: common completion footer for next-round prompts.
  - `prompt-template/claude/post-alignment-action-items.md`: optional addendum after a full alignment check.
  - `prompt-template/claude/push-every-round-note.md`: optional push instruction when `--push-every-round` is enabled.
  - `prompt-template/claude/goal-tracker-update-request.md`: optional summary section Claude should include when the read-only goal tracker needs Codex-mediated updates.

- algorithmic_behavior: The directory participates in a template-rendering state machine:
  - Template syntax is `{{VARIABLE_NAME}}`; variables are uppercase/digit/underscore placeholders, missing variables remain literal, and substitution is single-pass to avoid placeholder injection from values like review content (`hooks/lib/template-loader.sh:7`, `hooks/lib/template-loader.sh:12`, `hooks/lib/template-loader.sh:54`).
  - `load_template` reads a template by path under `prompt-template`; missing files emit a warning and empty output (`hooks/lib/template-loader.sh:33`, `hooks/lib/template-loader.sh:41`).
  - `load_and_render_safe` is the runtime gate for critical templates: if a Claude template is missing or renders empty, the hook falls back to an inline prompt instead of returning an empty block reason (`hooks/lib/template-loader.sh:167`, `hooks/lib/template-loader.sh:179`, `hooks/lib/template-loader.sh:192`).
