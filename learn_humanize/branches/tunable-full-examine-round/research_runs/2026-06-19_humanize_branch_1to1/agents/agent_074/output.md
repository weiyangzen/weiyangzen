- algorithmic_behavior: The file provides six core operations. `get_template_dir(script_dir)` resolves plugin root by going two directories above the caller-provided hooks/lib path and returns `<plugin_root>/prompt-template` ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/tunable-full-examine-round/hooks/lib/template-loader.sh:26)). `load_template(template_dir, template_name)` concatenates path segments, returns file contents on `-f`, otherwise warns to stderr and returns empty stdout ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/tunable-full-examine-round/hooks/lib/template-loader.sh:36)). `render_template(content, VAR=value...)` builds prefixed environment variables `TMPL_VAR_<VAR>` and runs a single awk pass over the template ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/tunable-full-examine-round/hooks/lib/template-loader.sh:58)). The awk scanner walks each line, finds `{{`, searches for the next `}}`, extracts the literal substring between braces as `var_name`, replaces only if that exact key exists in `vars`, and appends replacement values directly to output without rescanning them ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/tunable-full-examine-round/hooks/lib/template-loader.sh:84)). `load_and_render` composes `load_template` and `render_template` but emits nothing for missing/empty templates ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/tunable-full-examine-round/hooks/lib/template-loader.sh:136)). `append_template` simply emits base content and then loaded additional content, without rendering ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/tunable-full-examine-round/hooks/lib/template-loader.sh:151)). `load_and_render_safe` suppresses load warnings, falls back when the template is missing/empty, renders fallback variables when supplied, and also falls back if rendering produces empty output ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/tunable-full-examine-round/hooks/lib/template-loader.sh:170)). `validate_template_dir` gates only directory shape: root exists and has `block`, `codex`, and `claude` children ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/tunable-full-examine-round/hooks/lib/template-loader.sh:208)).

Newlines in variable values are passed through environment variables into awk and can produce multi-line rendered output; tests cover this behavior. Shell metacharacters in values are not interpreted because values are provided via environment variables and emitted from awk variables, avoiding sed/gsub replacement pitfalls. Values with embedded NUL cannot be represented in shell environment variables. Very large content is supported only within shell/environment/awk practical limits; stress tests cover 100KB values/templates but not unbounded prompt sizes.
