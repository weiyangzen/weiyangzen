- core_role: This is a guardrail prompt block used when a non-Codex loop actor attempts to modify `goal-tracker.md` after Round 0. Its algorithmic role is not computation, but enforcement messaging: it preserves a single-writer contract for the Goal Tracker by redirecting requested changes into the round summary for later Codex review. The core rule is explicit at `prompt-template/block/goal-tracker-modification.md:3-5`: after Round 0, only Codex may modify the Goal Tracker, and direct Write/Edit/Bash modification is prohibited.
- algorithmic_behavior: The block is rendered by `goal_tracker_blocked_message()` in `hooks/lib/loop-common.sh:1098-1110`. That function passes `CURRENT_ROUND` and `SUMMARY_FILE` into `load_and_render_safe`, targeting this template with a shorter fallback if the file is missing or empty. The template tells the blocked actor to add a `## Goal Tracker Update Request` section to `{{SUMMARY_FILE}}`, then include requested changes and justification (`prompt-template/block/goal-tracker-modification.md:7-23`). It therefore converts a forbidden direct state mutation into a structured request queue for Codex.
- inputs_outputs_state: Inputs are `CURRENT_ROUND` and `SUMMARY_FILE`, rendered into the title and summary path (`prompt-template/block/goal-tracker-modification.md:1,9-10`). Runtime state is supplied by hook state parsing: validators assign `CURRENT_ROUND` from `STATE_CURRENT_ROUND` in `hooks/loop-write-validator.sh:145`, `hooks/loop-edit-validator.sh:110`, and `hooks/loop-bash-validator.sh:86`; `STATE_CURRENT_ROUND` is parsed/defaulted in `hooks/lib/loop-common.sh:225-237`. The persistent state transition is deferred: `goal-tracker.md` remains unchanged, while the requested transition is recorded in the current round summary for Codex to review.
- gates_or_invariants: The primary gate is `CURRENT_ROUND > 0` for Write/Edit attempts targeting a path ending in `goal-tracker.md`: `is_goal_tracker_path()` matches `goal-tracker\.md$` in `hooks/lib/loop-common.sh:503-507`, and both Write and Edit validators block only when that path predicate is true and the round is greater than zero (`hooks/loop-write-validator.sh:192-195`, `hooks/loop-edit-validator.sh:145-148`). Bash modification is blocked in all rounds, but branches by round: Round 0 emits the Bash-specific “use Write/Edit” message, while Round > 0 emits this template (`hooks/loop-bash-validator.sh:343-357`).
