### TUNABLE_FULL_EXAMINE_ROUND-HZ-114 `file` `prompt-template/claude/review-phase-prompt.md`

- core_role: This file is the Claude-facing remediation prompt for the RLCR Review Phase. It is emitted only after `codex review` has found code-review issues with `[P0-9]` severity markers. The template turns raw review output into a constrained fix round: show findings, require fixes only, require a commit, require a round summary, and explicitly keep the loop in review mode until the automated review has no priority markers. Direct evidence is in `prompt-template/claude/review-phase-prompt.md:1`, `prompt-template/claude/review-phase-prompt.md:3`, `prompt-template/claude/review-phase-prompt.md:11`, and `prompt-template/claude/review-phase-prompt.md:25`.

- algorithmic_behavior: The prompt has four operational sections. `# Code Review Findings` identifies the phase (`review-phase-prompt.md:1-3`). `## Review Results` injects `{{REVIEW_CONTENT}}`, the extracted Codex review issue text (`review-phase-prompt.md:5-7`). `## Instructions` defines the action contract: address all `[P0-9]` issues, avoid unrelated work, commit fixes, and write the summary to `{{SUMMARY_FILE}}` (`review-phase-prompt.md:9-14`). `## Important Notes` defines the review-loop transition rule: `COMPLETE` is ignored in this phase, Codex reviews again after commit and summary, and the loop repeats until no `[P0-9]` issues remain (`review-phase-prompt.md:23-28`). The actual phase driver is `continue_review_loop_with_issues()` in `hooks/loop-codex-stop-hook.sh:1116-1160`, which renders this template into `round-<round>-prompt.md`.

- inputs_outputs_state: Inputs are `{{REVIEW_CONTENT}}` and `{{SUMMARY_FILE}}`. The caller passes these at render time in `hooks/loop-codex-stop-hook.sh:1148-1150`. `review_content` originates from `detect_review_issues()` output when Codex review markers are found; the review handler captures it at `hooks/loop-codex-stop-hook.sh:1013-1022`. The summary path is computed as `"$LOOP_DIR/round-${round}-summary.md"` at `hooks/loop-codex-stop-hook.sh:1130-1132`. The output is a rendered prompt written to `"$LOOP_DIR/round-${round}-prompt.md"` (`hooks/loop-codex-stop-hook.sh:1129-1150`) and returned to Claude as a blocking hook response with system message `Loop: Review Phase Round $round - Fix code review issues` (`hooks/loop-codex-stop-hook.sh:1152-1159`). State transition: the caller updates `current_round` in `state.md` before prompt generation (`hooks/loop-codex-stop-hook.sh:1124-1127`). Entry into Review Phase sets `review_started: true` and creates `.review-phase-started` (`hooks/loop-codex-stop-hook.sh:1421-1428`), then invokes review using `CURRENT_ROUND + 1` (`hooks/loop-codex-stop-hook.sh:1430-1433`). On subsequent review exits, `COMPLETE` is ignored if `REVIEW_STARTED=true` (`hooks/loop-codex-stop-hook.sh:1394-1399`), and review is rerun every exit attempt (`hooks/loop-codex-stop-hook.sh:1440-1468`).

- gates_or_invariants: The main gate is detection of `[P0-9]` issues. `detect_review_issues()` scans the combined review log and returns `0` when issues are found, `1` when no issues are found, and `2` for missing or empty log hard errors (`hooks/lib/loop-common.sh:332-348`). Its marker rule is precise: `[P?]` must appear within the first 10 characters of a line, where `?` is a digit (`hooks/lib/loop-common.sh:341-345`, `hooks/lib/loop-common.sh:364-372`). If issues are found, it extracts from the first matching line to the end, writes `round-<round>-review-result.md`, and prints `## Codex Review Issues` plus content (`hooks/lib/loop-common.sh:374-387`). `run_and_handle_code_review()` enforces the gate: command failure blocks, issue output continues the review loop, and no issues transitions to Finalize Phase (`hooks/loop-codex-stop-hook.sh:999-1027`). Review Phase also has a state-integrity invariant: `review_started=true` is valid only if `.review-phase-started` exists; otherwise the hook blocks as inconsistent manual state (`hooks/loop-codex-stop-hook.sh:1446-1462`). The template reinforces the invariant at the human-agent level by stating that `COMPLETE` has no effect and `[P0-9]` clearance is the only way forward (`review-phase-prompt.md:25-28`).
