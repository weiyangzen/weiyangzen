### TUNABLE_FULL_EXAMINE_ROUND-HZ-027 `file` `hooks/loop-edit-validator.sh`
- cursor: `[_]`
- core_role: PreToolUse `Edit` hook gate for the RLCR and PR-loop state machines. It does not advance loop state itself; it prevents Claude `Edit` operations from mutating state-owned, prompt-owned, wrong-round, or post-bootstrap loop files before the edit reaches the filesystem. It is registered as the `Edit` matcher hook in `hooks/hooks.json:24-30`.
- algorithmic_behavior: The hook reads hook JSON from stdin, extracts `.tool_name`, and exits immediately unless the tool is `Edit` (`hooks/loop-edit-validator.sh:24-31`). It lowercases the target path for pattern checks (`hooks/loop-edit-validator.sh:31-32`). It first handles global round-file protections: `round-N-todos.md` is blocked unless it is one of the small active-loop allowlist entries, and `round-N-prompt.md` is always blocked (`hooks/loop-edit-validator.sh:38-51`). It then applies PR-loop protections for paths under `.humanize/pr-loop/`: block `state.md`, block generated/read-only PR artifacts, and allow `round-N-pr-resolve.md` only when its round matches the active PR-loop `current_round` (`hooks/loop-edit-validator.sh:57-77`). For RLCR paths, it finds the active loop, prefers `finalize-state.md` over `state.md`, strictly parses the selected state file, and uses `STATE_CURRENT_ROUND` as the current state-machine cursor (`hooks/loop-edit-validator.sh:83-110`). After that, it blocks `finalize-state.md`, `state.md`, `.humanize/rlcr/**/plan.md`, and `goal-tracker.md` when the loop has advanced beyond round 0 (`hooks/loop-edit-validator.sh:117-149`). Finally, it validates `round-N-summary.md` edits: a summary edit is allowed only for the current round unless the exact target is allowlisted for early historical files (`hooks/loop-edit-validator.sh:155-180`).
- inputs_outputs_state: Primary input is Claude hook JSON on stdin, especially `tool_name` and `tool_input.file_path` (`hooks/loop-edit-validator.sh:24-31`). Runtime inputs include `CLAUDE_PROJECT_DIR`, `PROJECT_ROOT`, `LOOP_BASE_DIR`, and optional `LOOP_DIR` overrides used to locate `.humanize/rlcr` (`hooks/loop-edit-validator.sh:39-41`, `hooks/loop-edit-validator.sh:91-93`). Filesystem state inputs are the active RLCR `state.md` or `finalize-state.md` and the active PR-loop `state.md` through shared helpers. Outputs are process exit status and rendered stderr block messages: `0` means allow, `2` is a deliberate hook block for protected edits, and `1` can occur for malformed RLCR state or malformed JSON under `set -euo pipefail` (`hooks/loop-edit-validator.sh:14`, `hooks/loop-edit-validator.sh:105-109`). The hook makes no file writes and performs no state transition; it only reads state and gates the proposed edit.
- gates_or_invariants: Core invariant is that generated loop control files remain system-owned. Todos should be handled by native TodoWrite except the explicit active-loop allowlist (`hooks/loop-edit-validator.sh:38-45`; allowlist in `hooks/lib/loop-common.sh:423-441`). Prompt files are read-only instructions generated by Codex (`hooks/loop-edit-validator.sh:48-51`). PR-loop `state.md` and generated PR-loop files are immutable to Edit (`hooks/loop-edit-validator.sh:59-70`; PR read-only file classifier in `hooks/lib/loop-common.sh:825-835`). RLCR state files are blocked, with `finalize-state.md` checked before generic `state.md` because the generic state matcher also matches the finalize filename (`hooks/loop-edit-validator.sh:113-124`). The active state file must pass strict frontmatter validation before round-sensitive gates are evaluated (`hooks/loop-edit-validator.sh:99-110`; parser requirements in `hooks/lib/loop-common.sh:255-329`). `goal-tracker.md` is mutable only during round 0, then blocked after `CURRENT_ROUND > 0` (`hooks/loop-edit-validator.sh:141-149`). Summary edits must target `round-${CURRENT_ROUND}-summary.md` unless the exact target is allowlisted (`hooks/loop-edit-validator.sh:155-180`).
