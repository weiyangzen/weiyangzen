### TUNABLE_FULL_EXAMINE_ROUND-HZ-069 `file` `tests/test-template-references.sh`
- cursor: `[_]`
- core_role: Executable specification for template-reference integrity across the RLCR hook layer. The file is not production runtime code, but it encodes a core safety invariant: any validator or stop hook that blocks an action must be able to load a non-empty, existing markdown prompt template, or at least use safe fallback rendering. This prevents blocked actions from producing empty or missing guidance for Claude/Codex-facing loop behavior. See `tests/test-template-references.sh:3-9`.

- algorithmic_behavior: The script runs four sequential validation phases. First, it computes `SCRIPT_DIR`, `PROJECT_ROOT`, and `TEMPLATE_DIR` from its own path, so it is repo-root independent when invoked from another working directory (`tests/test-template-references.sh:14-16`). It initializes pass/fail/warning counters and helper functions that mutate those counters (`tests/test-template-references.sh:25-42`). Section 1 scans a fixed allowlist of hook scripts for literal calls to `load_template`, `load_and_render`, or `load_and_render_safe` that use the exact first argument `"$TEMPLATE_DIR"` (`tests/test-template-references.sh:54-64`, `tests/test-template-references.sh:87-110`). For each matching non-comment line, it extracts the second quoted argument with `sed`, builds `$TEMPLATE_DIR/$template_path`, and records pass/fail according to file existence (`tests/test-template-references.sh:93-107`). Section 2 enumerates all `*.md` files below `prompt-template` with null-delimited `find`, then greps `hooks/` for direct quoted references; unreferenced templates only emit warnings (`tests/test-template-references.sh:119-142`). Section 3 hardcodes the common block-message template names expected by `hooks/lib/loop-common.sh` and verifies they exist (`tests/test-template-references.sh:147-167`). Section 4 enforces safe rendering for critical validators by counting `load_and_render "$TEMPLATE_DIR"` calls not containing `load_and_render_safe` in `loop-read-validator.sh`, `loop-write-validator.sh`, and `loop-edit-validator.sh` (`tests/test-template-references.sh:172-201`). The summary reports counters and exits success only when `TESTS_FAILED` is zero (`tests/test-template-references.sh:206-229`).

- inputs_outputs_state: Inputs are the repository filesystem, specifically the fixed `SCRIPTS_TO_CHECK` array, the `prompt-template` directory, and three critical validator scripts. The script reads shell source lines and markdown template paths; it does not write files. State is internal shell arrays and counters: `MISSING_TEMPLATES`, `FOUND_REFERENCES`, `TEMPLATE_FILES`, `UNREFERENCED`, `TESTS_PASSED`, `TESTS_FAILED`, and `WARNINGS` (`tests/test-template-references.sh:25-27`, `tests/test-template-references.sh:71-72`, `tests/test-template-references.sh:121-132`). Outputs are colored terminal diagnostics, per-template pass/fail/warn lines, an optional critical missing-template list, and process exit code `0` or `1` (`tests/test-template-references.sh:29-49`, `tests/test-template-references.sh:213-229`). In this branch, running `bash tests/test-template-references.sh` produced `98` passes, `0` failures, `7` warnings, and final success.

- gates_or_invariants: Missing directly referenced templates are fatal because each missing template increments `TESTS_FAILED` and is later printed as a critical missing file (`tests/test-template-references.sh:102-107`, `tests/test-template-references.sh:213-219`). Missing scripts in the fixed scan list are warnings, not failures, which keeps the test tolerant of absent optional hooks (`tests/test-template-references.sh:74-78`). Unreferenced markdown templates are warnings only because dynamic or non-hook usage is allowed (`tests/test-template-references.sh:130-141`). The common template list is a stricter existence gate for shared message functions such as todos blocking, prompt write blocking, state blocking, summary bash blocking, goal-tracker bash blocking, and goal-tracker modification blocking (`tests/test-template-references.sh:152-167`). Critical read/write/edit validators must use `load_and_render_safe` for all template rendering; any direct unsafe `load_and_render` call in those scripts is fatal (`tests/test-template-references.sh:176-200`). The script uses `set -uo pipefail` but not `set -e`, so validation proceeds after failures and reports a complete failure summary instead of stopping on the first problem (`tests/test-template-references.sh:12`).

- dependencies_and_callers: The test depends on the template loader API names implemented in `hooks/lib/template-loader.sh`, especially `load_template`, `load_and_render`, and `load_and_render_safe` (`hooks/lib/template-loader.sh:33-48`, `hooks/lib/template-loader.sh:134-147`, `hooks/lib/template-loader.sh:167-203`). `load_template` returns empty output for missing files, and `load_and_render_safe` falls back when the loaded or rendered content is empty, which is the behavioral risk this test is designed to protect (`hooks/lib/template-loader.sh:41-47`, `hooks/lib/template-loader.sh:176-202`). The scanned hook callers include `hooks/loop-codex-stop-hook.sh`, `hooks/loop-read-validator.sh`, `hooks/loop-write-validator.sh`, `hooks/loop-edit-validator.sh`, `hooks/loop-bash-validator.sh`, and `hooks/lib/loop-common.sh` (`tests/test-template-references.sh:57-64`). Examples validated by the test include read-block templates for wrong file location, wrong round file, and wrong directory (`hooks/loop-read-validator.sh:122-167`); write-block templates for plan backups, wrong summary location, wrong round number, and wrong directory (`hooks/loop-write-validator.sh:179-263`); edit-block templates for plan backups and wrong round number (`hooks/loop-edit-validator.sh:131-178`); and common helper templates in `loop-common.sh` (`hooks/lib/loop-common.sh:443-500`). The test also overlaps with broader template validation in `tests/test-template-loader.sh`, which verifies rendering, fallback behavior, literal metacharacter handling, single-pass substitution, and template directory validation.

- edge_cases_or_failure_modes: The extraction logic only detects literal quoted invocations whose first argument is exactly `"$TEMPLATE_DIR"` and whose template path is the next quoted string (`tests/test-template-references.sh:93-97`). It will miss dynamically constructed template names, alternate variable names, single-quoted arguments, multiline calls where `"$TEMPLATE_DIR"` and the template path are split differently, or scripts outside the fixed `SCRIPTS_TO_CHECK` list. Section 2 only searches under `hooks/`, so templates used solely by setup scripts or commands can be warned as unreferenced; this happened for six `pr-loop/round-0...` and `critical-requirements...` templates plus `plan/gen-plan-template.md` in this branch, while the test still passed. The unsafe-render check is scoped only to three critical validator scripts, so `load_and_render` usage in other contexts is not fatal. Comments are skipped only when the full line begins with optional whitespace followed by `#`; inline comments after a real call are still scanned, which is acceptable for this narrow pattern (`tests/test-template-references.sh:87-91`). Duplicate references increment pass counters repeatedly, so pass totals represent checks performed, not unique templates. Because `find "$TEMPLATE_DIR"` is not guarded, a missing template directory would cause the process-substitution loop to receive no files and likely print a `find` error, but fatality would come mainly through missing referenced-template checks.
