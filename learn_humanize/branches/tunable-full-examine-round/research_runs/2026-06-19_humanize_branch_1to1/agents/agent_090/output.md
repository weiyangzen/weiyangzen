- core_role: This file is a PR-loop block-message template used when the loop cannot proceed because no valid trigger comment has been found. It is not executable code, but it is part of the user-facing enforcement surface for the trigger gate. The template title and body establish the blocked state: `prompt-template/block/no-trigger-comment.md:1` names the block, `prompt-template/block/no-trigger-comment.md:3` says the PR loop is waiting for a trigger comment, and `prompt-template/block/no-trigger-comment.md:9-15` gives the required action and example mention format.
- algorithmic_behavior: The template is rendered by `hooks/pr-loop-stop-hook.sh` when the hook has determined `REQUIRE_TRIGGER=true` and `PR_LAST_TRIGGER_AT` is empty. The hook computes trigger requirements from round number, startup case, and new-commit detection at `hooks/pr-loop-stop-hook.sh:677-708`: round `> 0` always requires a trigger, new commits detected during the poll require a fresh trigger, and round 0 startup cases `4|5` require a trigger while cases `1|2|3` do not. The missing-trigger gate itself is at `hooks/pr-loop-stop-hook.sh:717-735`; it builds `STARTUP_CASE_DESC`, renders `block/no-trigger-comment.md`, and emits a JSON hook response with `"decision": "block"`. The template therefore supplies the explanatory output for an algorithmic transition from “attempting to exit/proceed” to “blocked until human posts bot mention.”
- inputs_outputs_state: Inputs are placeholder values passed to the template renderer: `STARTUP_CASE`, `STARTUP_CASE_DESC`, `CURRENT_ROUND`, and `BOT_MENTION_STRING`, all visible in `prompt-template/block/no-trigger-comment.md:6-14` and passed by the caller at `hooks/pr-loop-stop-hook.sh:729-731`.
