### ADD_COMMIT_PLAN_FILE_CLI-HZ-020 `file` `hooks/loop-read-validator.sh`
- cursor: `[_]`
- core_role: Read-side RLCR loop guard for Claude Code. It is registered as the `PreToolUse` hook for `Read` in `hooks/hooks.json:33-40`, and its job is to prevent stale or wrong-surface reads of loop-managed files. It protects the RLCR state machine by forcing prompt and summary reads to target the active `.humanize-loop.local/<session>/round-N-*` files, and by blocking legacy `round-*-todos.md` file access entirely in favor of native TodoWrite state.
- algorithmic_behavior: The script reads hook JSON from stdin, extracts `.tool_name`, and exits immediately unless the tool is `Read` (`hooks/loop-read-validator.sh:22-27`). It then extracts `.tool_input.file_path`, lowercases it through `to_lower`, and first applies a hard todo-file gate: any path matching `round-[0-9]+-todos.md` emits the shared todo-access block and exits `2` (`hooks/loop-read-validator.sh:29-39`). For non-todo paths, it only handles round `summary` and `prompt` files; all other reads exit `0` (`hooks/loop-read-validator.sh:45-47`). For prompt/summary paths, it resolves the project root from `CLAUDE_PROJECT_DIR` or `pwd`, finds the newest active loop under `.humanize-loop.local`, gets the current round from `state.md`, extracts the requested round number, infers the requested file type, and validates three conditions in sequence: location is inside `.humanize-loop.local`, requested round equals current round, and requested full path equals the active loop directory plus the requested filename (`hooks/loop-read-validator.sh:49-133`).
- inputs_outputs_state: Inputs are the Claude hook payload on stdin, especially `tool_name` and `tool_input.file_path` (`hooks/loop-read-validator.sh:22-30`), plus environment/context values `CLAUDE_PROJECT_DIR`, current working directory, `.humanize-loop.local`, and active `state.md` (`hooks/loop-read-validator.sh:56-64`). Outputs are process exit codes and stderr diagnostics: `0` allows the read, `2` blocks the read with a rendered message (`hooks/loop-read-validator.sh:36-39`, `87-96`, `103-115`, `124-132`). It does not mutate repository or loop state. State is read from the active loop discovered by `find_active_loop`; `get_current_round` parses YAML frontmatter `current_round` and defaults to `0` when absent (`hooks/lib/loop-common.sh:27-58`).
- gates_or_invariants: Todo files are never readable through file access once the filename matches the round-todos pattern (`hooks/loop-read-validator.sh:36-39`). Prompt and summary files are only considered protected when their basename matches `round-N-prompt.md` or `round-N-summary.md` (`hooks/lib/loop-common.sh:65-83`). If no active loop exists, read access is allowed (`hooks/loop-read-validator.sh:60-62`). If a prompt/summary-like path lacks an extractable round number, access is allowed (`hooks/loop-read-validator.sh:70-73`). The active-loop invariant is newest timestamp-named child directory with `state.md`; older loop directories are ignored by design to avoid reviving stale sessions (`hooks/lib/loop-common.sh:23-44`). The final directory invariant is strict string equality between the requested path and `$ACTIVE_LOOP_DIR/$CLAUDE_FILENAME` (`hooks/loop-read-validator.sh:122-132`).
- dependencies_and_callers: Called by Claude Code through `hooks/hooks.json` as a `PreToolUse` `Read` command (`hooks/hooks.json:33-40`). Depends on `jq` for JSON parsing (`hooks/loop-read-validator.sh:23`, `29`), Bash with `set -euo pipefail` (`hooks/loop-read-validator.sh:12`), and shared helpers from `hooks/lib/loop-common.sh` (`hooks/loop-read-validator.sh:14-16`). Shared helper dependencies include `template-loader.sh` for safe template rendering (`hooks/lib/loop-common.sh:11-17`), `is_round_file_type`, `extract_round_number`, `is_in_humanize_loop_dir`, `find_active_loop`, and `get_current_round` (`hooks/lib/loop-common.sh:27-83`, `149-153`). The todo block goes through `todos_blocked_message`, which renders `prompt-template/block/todos-file-access.md` using `load_and_render_safe` (`hooks/lib/loop-common.sh:86-95`). Sibling validators enforce analogous constraints for writes, edits, and Bash bypass attempts, including shared todo-file blocking in `hooks/loop-write-validator.sh:37-40` and `hooks/loop-bash-validator.sh:132-135`.
- edge_cases_or_failure_modes: If `jq` is missing or hook JSON is malformed, `set -euo pipefail` can make the hook fail before a controlled allow/block result. If `CLAUDE_PROJECT_DIR` is unset and the hook runs from a different directory, active-loop discovery may target the wrong root (`hooks/loop-read-validator.sh:56-58`). Relative paths or symlinked paths may fail the strict final equality check even when they resolve to the intended file (`hooks/loop-read-validator.sh:122-132`). A path containing `.humanize-loop.local/` but outside the active session can pass the broad location test and then be blocked only by the strict final path check (`hooks/loop-read-validator.sh:49-50`, `122-132`). Case-insensitive matching is used for classification via lowercasing, but the final path comparison is case-sensitive (`hooks/loop-read-validator.sh:29-30`, `124`). Reads of stale filenames that do not match the exact `round-N-summary.md` or `round-N-prompt.md` pattern are outside this guard and are allowed (`hooks/loop-read-validator.sh:45-47`, `70-73`).
