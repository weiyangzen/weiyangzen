- algorithmic_behavior: The script reads hook JSON from stdin, extracts `.tool_name` and `.tool_input.file_path` with `jq`, and exits allowfully unless `tool_name == "Edit"` (`hooks/loop-edit-validator.sh:22-30`). It lowercases the target path for case-insensitive classifier checks (`hooks/loop-edit-validator.sh:29-30`). It blocks `round-N-todos.md` edits immediately because todos should be managed through TodoWrite (`hooks/loop-edit-validator.sh:36-39`), and blocks `round-N-prompt.md` edits because prompts are generated instructions (`hooks/loop-edit-validator.sh:41-44`). For other targets, it only applies loop-specific gates if the path contains `.humanize-loop.local/` (`hooks/loop-edit-validator.sh:50-52`). It finds the newest active loop via `find_active_loop` and reads `current_round` from that loop’s `state.md` (`hooks/loop-edit-validator.sh:58-67`). It blocks `state.md` edits (`hooks/loop-edit-validator.sh:72-75`), blocks loop backup `plan.md` edits (`hooks/loop-edit-validator.sh:81-89`), blocks `goal-tracker.md` after round 0 (`hooks/loop-edit-validator.sh:95-99`), and blocks summary edits whose filename round does not match the active `current_round` (`hooks/loop-edit-validator.sh:105-131`).

Output is no stdout on allowed edits, stderr explanatory markdown on blocked edits, and process status `0` for allow or `2` for block. State read is `.humanize-loop.local/<latest>/state.md`; the active loop is the newest timestamped directory with `state.md`, per `find_active_loop` (`hooks/lib/loop-common.sh:27-44`). `current_round` is parsed from YAML frontmatter with default `0` (`hooks/lib/loop-common.sh:46-58`). This validator does not write state; it preserves loop state by refusing edits to loop-owned files.

The state-file gate is unconditional once a path is inside `.humanize-loop.local/` and an active loop exists (`hooks/loop-edit-validator.sh:50-75`). The summary gate enforces round consistency by comparing extracted filename round to `CURRENT_ROUND` and redirecting to `round-${CURRENT_ROUND}-summary.md` (`hooks/loop-edit-validator.sh:113-128`). The active-loop invariant depends on `state.md` presence: README says loop state is controlled by the presence of `state.md` in the newest loop directory (`README.md:116-118`), and exit renames use `complete-state.md`, `cancel-state.md`, etc. (`README.md:255-260`).
