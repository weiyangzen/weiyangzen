- core_role: `prompt-template/` is the message/prompt data layer for the RLCR loop state machine. It contains 29 Markdown templates, grouped as `block/` validator/stop-block messages, `codex/` review prompts and shared review sections, and `claude/` next-round prompt fragments. The directory is discovered through `get_template_dir()` as `<plugin_root>/prompt-template` and validated for required subdirectories `block`, `codex`, and `claude` in [hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/add-commit-plan-file-cli/hooks/lib/template-loader.sh:11) and [hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/add-commit-plan-file-cli/hooks/lib/template-loader.sh:193).
- algorithmic_behavior: The templates externalize hook decisions into stable, renderable contracts. `block/` templates explain why a tool action or stop attempt is blocked, including wrong loop-file access, wrong round, protected state/plan files, incomplete todos, large files, dirty git, unpushed commits, summary missing, and goal-tracker violations. `codex/regular-review.md` and `codex/full-alignment-review.md` define the review gates that determine whether the loop exits, continues, or triggers stagnation stop. `claude/next-round-prompt.md`, footer, push note, post-alignment action items, and goal-tracker request format assemble the next work prompt when review finds issues.
- inputs_outputs_state: Inputs are template names plus `{{VAR}}` substitutions from hooks: paths, round numbers, review content, summary content, git counts, Codex debug file paths, and goal tracker path. Output is rendered Markdown used either as stderr hook feedback, JSON `reason`/`systemMessage`, prompt files such as `round-N-review-prompt.md`, or next-round prompt files. Rendering is single-pass via `render_template`, intentionally preventing values containing `{{OTHER_VAR}}` from being recursively substituted ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/add-commit-plan-file-cli/hooks/lib/template-loader.sh:35)).
- gates_or_invariants: The directory must contain `block`, `codex`, and `claude` subdirectories. Missing templates loaded with `load_and_render_safe` fall back to inline messages rather than producing empty blocker text ([hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/add-commit-plan-file-cli/hooks/lib/template-loader.sh:155)). The Codex review prompt must end with a review result whose last non-empty line is exactly `COMPLETE` or `STOP` to transition to completion/stop; anything else advances the loop and generates the next Claude prompt ([hooks/loop-codex-stop-hook.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/add-commit-plan-file-cli/hooks/loop-codex-stop-hook.sh:831)).

Stop-hook examples include plan modification, todos, large-file, dirty-git, unpushed-commit, summary-missing, goal-tracker initialization, Codex review prompt generation, and next-round prompt generation at [hooks/loop-codex-stop-hook.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/add-commit-plan-file-cli/hooks/loop-codex-stop-hook.sh:210), [hooks/loop-codex-stop-hook.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/add-commit-plan-file-cli/hooks/loop-codex-stop-hook.sh:556), and [hooks/loop-codex-stop-hook.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/add-commit-plan-file-cli/hooks/loop-codex-stop-hook.sh:900).
