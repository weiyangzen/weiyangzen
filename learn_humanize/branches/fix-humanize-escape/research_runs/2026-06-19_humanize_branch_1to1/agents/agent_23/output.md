### FIX_HUMANIZE_ESCAPE-HZ-023 `file` `scripts/portable-timeout.sh`
- cursor: `[_]`
- core_role: Provides the repository’s portable command-timeout abstraction for RLCR loop scripts and hooks, especially git-state validation gates that must fail closed instead of hanging indefinitely. The script is meant to be sourced, then called as `run_with_timeout <seconds> <command> [args...]` (`scripts/portable-timeout.sh:4`, `scripts/portable-timeout.sh:33`).
- algorithmic_behavior: On source, it detects the best available timeout implementation once via `detect_timeout_impl`: Homebrew `gtimeout`, GNU `timeout`, `python3`, `python`, or `none`, in that priority order (`scripts/portable-timeout.sh:9-27`, `scripts/portable-timeout.sh:29`). `run_with_timeout` stores the first argument as the timeout budget, shifts it off, preserves the remaining command/arguments as a Bash array, then dispatches by `$TIMEOUT_IMPL` (`scripts/portable-timeout.sh:33-38`). The GNU paths delegate directly to `gtimeout "$timeout_secs" "${cmd[@]}"` or `timeout "$timeout_secs" "${cmd[@]}"` and return the wrapped exit code (`scripts/portable-timeout.sh:39-46`). The Python path runs `subprocess.run(sys.argv[1:], timeout=$timeout_secs)`, exits with the child return code, and maps `subprocess.TimeoutExpired` to `124` to match GNU timeout semantics (`scripts/portable-timeout.sh:47-61`). The `none` path warns to stderr, runs the command without an enforced timeout, and returns the command’s status (`scripts/portable-timeout.sh:64-69`).
- inputs_outputs_state: Inputs are the timeout duration string and the command argv passed by callers (`scripts/portable-timeout.sh:31-36`). Output is whatever the wrapped command writes to stdout/stderr, plus a warning on stderr only when no timeout implementation exists (`scripts/portable-timeout.sh:64-67`). State is the exported `TIMEOUT_IMPL` variable, computed at source time and made visible to sourcing scripts (`scripts/portable-timeout.sh:29`, `scripts/portable-timeout.sh:73-76`). Exit status is the wrapped command’s exit status, `124` on Python-enforced timeout, `1` for unexpected Python wrapper exceptions, or the native timeout tool’s status (`scripts/portable-timeout.sh:39-62`).
- gates_or_invariants: The wrapper preserves argv boundaries by using a Bash array for command arguments, so paths with spaces are not split by the wrapper itself (`scripts/portable-timeout.sh:36`, `scripts/portable-timeout.sh:40`, `scripts/portable-timeout.sh:44`). It explicitly rejects non-GNU `timeout` implementations by requiring `timeout --version` to succeed before selecting `timeout` (`scripts/portable-timeout.sh:13-19`). Timeout exit code `124` is an invariant consumed by callers as a fail-closed signal; for example, `loop-plan-file-validator.sh` treats `124` from git commands as a blocking timeout for branch/plan-file validation (`hooks/loop-plan-file-validator.sh:123-131`, `hooks/loop-plan-file-validator.sh:147-156`, `hooks/loop-plan-file-validator.sh:193-201`), and `setup-rlcr-loop.sh` exits setup on plan-file git timeout (`scripts/setup-rlcr-loop.sh:311-327`).
