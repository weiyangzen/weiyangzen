- cursor: `[_]`
- core_role: PreToolUse `Read` validator for the RLCR loop. It prevents an agent from reading stale, wrong-location, wrong-session, or prohibited RLCR loop files, especially `round-*-prompt.md`, `round-*-summary.md`, and `round-*-todos.md`. It is registered as the `Read` matcher hook in `hooks/hooks.json:33-39`.
- algorithmic_behavior: The hook reads JSON from stdin, extracts `.tool_name`, and exits successfully unless the tool is exactly `Read` (`hooks/loop-read-validator.sh:22-27`). For a `Read`, it extracts `.tool_input.file_path`, lowercases it, and first applies a todos-file gate (`hooks/loop-read-validator.sh:29-44`). It then only continues deeper validation for `summary` or `prompt` round files; all other reads pass through (`hooks/loop-read-validator.sh:50-52`). For summary/prompt files, it finds the active RLCR loop, parses the current round from `state.md` or `finalize-state.md`, extracts the round number from the target filename, derives the target file type, and enforces three gates: inside `.humanize/rlcr/`, matching current round unless allowlisted, and exact active-loop directory path (`hooks/loop-read-validator.sh:54-146`).
- inputs_outputs_state: Input is a Claude hook payload on stdin with `tool_name` and `tool_input.file_path`. Output is either no output plus exit `0` for allowed/non-applicable reads, or a rendered block message on stderr plus exit `2` for blocked reads. State is read from `PROJECT_ROOT` or `CLAUDE_PROJECT_DIR`, the active loop under `.humanize/rlcr`, and either `state.md` or `finalize-state.md` (`hooks/loop-read-validator.sh:61-77`). It does not mutate files or loop state.
- gates_or_invariants: Non-`Read` tools must not be affected. Non-round files must not be affected. `round-*-todos.md` reads are blocked unless the file is allowlisted in the active loop (`hooks/loop-read-validator.sh:36-43`). Summary/prompt reads must be in `.humanize/rlcr/`, must use the active loop directory, and must match `current_round` unless allowlisted (`hooks/loop-read-validator.sh:100-146`). The shared allowlist is exact path-based and includes only `round-1-todos.md`, `round-2-todos.md`, `round-0-summary.md`, and `round-1-summary.md` in the active loop (`hooks/lib/loop-common.sh:170-188`).
- dependencies_and_callers: Depends on `jq`, Bash, and `hooks/lib/loop-common.sh` (`hooks/loop-read-validator.sh:14-16`). Shared functions used include `to_lower`, `is_round_file_type`, `find_active_loop`, `is_allowlisted_file`, `parse_state_file`, `extract_round_number`, `is_in_humanize_loop_dir`, `todos_blocked_message`, and `load_and_render_safe`. Template dependencies include `block/todos-file-access.md`, `block/wrong-file-location.md`, `block/wrong-round-file.md`, and `block/wrong-directory-path.md` through common/template loader paths. Caller is the Claude hook runtime via `hooks/hooks.json:33-39`.
- edge_cases_or_failure_modes: If there is no active loop directory, summary/prompt reads pass (`hooks/loop-read-validator.sh:65-67`); for todos, no active loop blocks unless allowlisted, because the todos gate treats empty loop as invalid (`hooks/loop-read-validator.sh:36-43`). During Finalize Phase, `finalize-state.md` overrides `state.md` as the state source (`hooks/loop-read-validator.sh:69-77`). A file with a round-like path but no extractable round number passes (`hooks/loop-read-validator.sh:83-86`). The directory membership check is substring-style through `.humanize/rlcr/` (`hooks/lib/loop-common.sh:415-419`), then later exact active-loop path matching closes most false-location cases. The allowlist bypasses the round mismatch gate but not arbitrary path matching, because `is_allowlisted_file` requires exact active-loop path.

The same test invokes `hooks/loop-read-validator.sh` to allow `round-1-todos.md`, allow historical `round-0-summary.md`, block `round-3-todos.md`, and block `round-3-summary.md` (`tests/test-allowlist-validators.sh:232-281`). `tests/test-finalize-phase.sh` includes coverage that the read validator parses `finalize-state.md` and allows the current round (`tests/test-finalize-phase.sh:639-650`). `tests/test-template-references.sh` includes `loop-read-validator.sh` in template-reference scanning and critical safe-render checks (`tests/test-template-references.sh:56-64`, `tests/test-template-references.sh:172-190`).
