- core_role: PreToolUse `Edit` validator for the RLCR loop. It protects `.humanize/rlcr` loop state files and round artifacts from edits that would bypass the loop state machine or corrupt Codex/Claude handoff files. It is registered for Claude `Edit` tool calls in `hooks/hooks.json:24-30`.
- algorithmic_behavior: The hook reads JSON from stdin, extracts `.tool_name`, and exits successfully for non-`Edit` tools (`hooks/loop-edit-validator.sh:22-27`). For `Edit`, it extracts `.tool_input.file_path`, lowercases a copy for matching, then applies ordered gates:
  - Blocks `round-*-todos.md` edits unless the target is one of the active-loop allowlisted files (`round-1-todos.md`, `round-2-todos.md`) in the exact active loop directory (`hooks/loop-edit-validator.sh:36-44`, helper allowlist at `hooks/lib/loop-common.sh:170-188`).
  - Blocks all `round-*-prompt.md` edits because prompts are generated instructions from Codex to Claude (`hooks/loop-edit-validator.sh:46-49`).
  - Allows ordinary files outside `.humanize/rlcr/` after the global round todos/prompt checks (`hooks/loop-edit-validator.sh:55-57`).
  - Locates the active loop and parses `state.md` or `finalize-state.md` to determine `CURRENT_ROUND` (`hooks/loop-edit-validator.sh:63-79`).
  - Blocks `finalize-state.md` before `state.md` because the broader `state.md` regex also matches `finalize-state.md` (`hooks/loop-edit-validator.sh:82-94`).
  - Blocks edits to loop `plan.md` backups (`hooks/loop-edit-validator.sh:100-108`).
  - Blocks `goal-tracker.md` edits after Round 0 and directs updates into the current round summary (`hooks/loop-edit-validator.sh:114-118`).
  - For `round-*-summary.md`, extracts the round number and blocks edits to non-current summaries unless the exact file is allowlisted (`hooks/loop-edit-validator.sh:124-150`).
- inputs_outputs_state: Input is the hook JSON payload plus environment-derived paths: `CLAUDE_PROJECT_DIR`, optional `PROJECT_ROOT`, optional `LOOP_BASE_DIR`, and optional `LOOP_DIR`. It reads the newest active `.humanize/rlcr/<timestamp>/state.md` or `finalize-state.md` via `find_active_loop` (`hooks/lib/loop-common.sh:63-81`) and `parse_state_file` (`hooks/lib/loop-common.sh:112-139`). It writes no state. Output is exit `0` to allow the edit, or exit `2` with a rendered block message on stderr to deny the edit.
- gates_or_invariants: Maintains these invariants: round prompt files are read-only for Claude; round todo files should be managed through TodoWrite except narrow historical allowlist cases; `state.md` and `finalize-state.md` are hook-managed; loop `plan.md` backups are immutable; after Round 0 only Codex owns the goal tracker; Claude edits only the current round summary unless a historical file is explicitly allowlisted. The active loop selection intentionally uses only the newest loop directory with `state.md` or `finalize-state.md`, preventing older “zombie” loops from being revived (`hooks/lib/loop-common.sh:58-81`).
- dependencies_and_callers: Depends on `jq`, `sed`, `grep`, Bash, and `hooks/lib/loop-common.sh` (`hooks/loop-edit-validator.sh:14-16`). Shared helpers bring in `hooks/lib/template-loader.sh`, with safe fallback rendering for block messages (`hooks/lib/template-loader.sh:170-203`). Called by the Claude hook system as a `PreToolUse` validator for matcher `Edit` (`hooks/hooks.json:24-30`). It coordinates with sibling validators: `loop-write-validator.sh` enforces similar write gates, and `loop-bash-validator.sh` prevents shell bypasses.
- edge_cases_or_failure_modes: Non-`Edit` payloads and empty non-matching file paths are allowed. Todos/prompt filename checks happen before `.humanize/rlcr` containment, so a file named like `round-3-todos.md` outside the loop can still be blocked unless it matches the exact allowlist path. If no active loop exists, internal `.humanize/rlcr` non-round files are allowed after the early gates (`hooks/loop-edit-validator.sh:67-69`). The allowlist is exact-path sensitive, so an allowlisted filename in another directory is still rejected.
