### FIX_HUMANIZE_ESCAPE-HZ-064 `file` `prompt-template/block/summary-bash-write.md`
- cursor: `[_]`
- core_role: This block template is the specific guard message for Bash-based writes to round summary files. It is core to the loop’s validation contract because summary files must be written through Write/Edit tooling so round-number and location validators can run.
- algorithmic_behavior: The template states the block reason, injects the current valid summary path through `{{CORRECT_PATH}}`, and explains that commands like `cat`, `echo`, `sed`, and `awk` bypass validation hooks (`prompt-template/block/summary-bash-write.md:1-8`). It is rendered by `summary_bash_blocked_message` (`hooks/lib/loop-common.sh:228-237`).
- inputs_outputs_state: Input is `CORRECT_PATH`, computed as the active loop directory plus `round-${CURRENT_ROUND}-summary.md` in the Bash validator (`hooks/loop-bash-validator.sh:334-336`). Output is a rendered markdown block sent to stderr, followed by hook exit code 2 (`hooks/loop-bash-validator.sh:334-337`). State is not mutated; the transition is a pre-tool-use denial that keeps summary state unchanged.
- gates_or_invariants: The invariant is that Bash commands must not create or modify `round-N-summary.md`. The Bash validator detects summary modifications via `command_modifies_file "$COMMAND_LOWER" "round-[0-9]+-summary\.md"` (`hooks/loop-bash-validator.sh:330-337`). The underlying detector covers redirection, `tee`, in-place `sed`, `awk -i inplace`, `perl -i`, `mv`, `cp`, `rm`, `dd`, `truncate`, `printf >`, and `exec >` patterns (`hooks/lib/loop-common.sh:571-599`).
- dependencies_and_callers: Depends on `hooks/lib/template-loader.sh` for safe rendering and fallback behavior, `hooks/lib/loop-common.sh` for the message function, `find_active_loop` and `parse_state_file` to compute the active round (`hooks/loop-bash-validator.sh:36-53`), and `hooks/loop-bash-validator.sh` as the enforcement caller. It coordinates with `hooks/loop-write-validator.sh`, which separately handles wrong summary location and wrong round number for Write/Edit operations (`hooks/loop-write-validator.sh:151-198`).
- edge_cases_or_failure_modes: The guard only applies when the tool is Bash and the command matches known modification patterns; novel shell constructs outside `command_modifies_file` could be missed. Conversely, conservative matching may block commands that mention summary paths in write-like forms even if the user intended a harmless operation. The template itself has one required placeholder; if missing, the loader would leave `{{CORRECT_PATH}}` literal, but normal callers pass it.
- validation_or_tests: `tests/test-template-references.sh:152-159` includes `block/summary-bash-write.md` in the common-template existence checks. `tests/test-templates-comprehensive.sh:563-606` ensures all templates, including this one, load and render with dummy placeholder values. Its runtime path is validated indirectly by Bash validator tests where summary-file Bash writes should be blocked by this message path.
- skip_candidate: `no`
