### FIX_HUMANIZE_ESCAPE-HZ-045 `file` `hooks/lib/template-loader.sh`
- cursor: `[_]`
- core_role: Shared prompt-template loading and rendering library for RLCR validators and stop-hook prompts. It is core support for the RLCR state machine because blocking decisions and next-round prompts are emitted through templates, and this library defines how placeholders are resolved safely. See [hooks/lib/template-loader.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/fix-humanize-escape/hooks/lib/template-loader.sh:1) lines 1-22.
- algorithmic_behavior: `get_template_dir` derives `<plugin-root>/prompt-template` from the caller’s `hooks/lib` directory. `load_template` reads a relative template path or emits an empty string with a warning. `render_template` performs single-pass placeholder replacement for `{{VAR}}` tokens using `TMPL_VAR_` environment variables and awk, preserving unresolved placeholders and avoiding recursive replacement of injected `{{...}}` content. `load_and_render` composes load plus render. `load_and_render_safe` adds fallback rendering when the template is missing or renders empty. `validate_template_dir` checks that `block`, `codex`, and `claude` template subdirectories exist.
- inputs_outputs_state: Inputs are a template directory, template name, raw template content, and `VAR=value` assignments. Outputs are rendered prompt/block text on stdout and warnings/errors on stderr. It does not persist RLCR state directly. The state-machine effect is indirect: validators call it to produce block reasons, and the stop hook calls it to produce next-round/finalize prompts.
- gates_or_invariants: The key invariant is single-pass substitution, documented at lines 12-13 and implemented in awk at lines 74-129. Values are injected from environment variables, which avoids sed-style escaping failure for shell metacharacters. Missing variables stay visible as placeholders at lines 119-122. Missing templates fail soft in `load_template` lines 41-47, while `load_and_render_safe` supplies a fallback at lines 176-203. Directory validation is intentionally structural, not exhaustive, at lines 208-222.
- dependencies_and_callers: `hooks/lib/loop-common.sh` sources this library and initializes `TEMPLATE_DIR` at [hooks/lib/loop-common.sh](/Users/wangweiyang/GitHub/humanize_branch_worktrees/fix-humanize-escape/hooks/lib/loop-common.sh:46) lines 46-55. Shared block-message helpers then call `load_and_render_safe`, for example todos/prompt/state/finalize-state/summary/goal-tracker messages at lines 190-248 and git-add/goal-tracker messages at lines 568-612. `hooks/loop-codex-stop-hook.sh` uses the same loader for review prompts, block messages, finalize prompt, next-round prompt, footer, push note, and goal-tracker update request.
- edge_cases_or_failure_modes: `load_template` allows any concatenated relative template path; it does not enforce path normalization or deny traversal. Command substitution can normalize trailing newlines, so exact EOF newline preservation is not guaranteed. `render_template` accepts whatever appears before the first `=` as a variable name; comments document uppercase-only placeholders, but the function does not enforce that. Very large replacement values are bounded by environment-size limits. An unclosed `{{` is treated literally rather than failing, which favors prompt continuity over strict template validation.
