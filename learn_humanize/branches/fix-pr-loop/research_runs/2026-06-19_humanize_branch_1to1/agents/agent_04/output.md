### FIX_PR_LOOP-HZ-034 `file` `scripts/check-bot-reactions.sh`
- cursor: `[_]`
- core_role: Runtime helper for PR loop reaction gates. It converts GitHub reactions into machine-readable approvals or acknowledgements for the PR stop hook and setup script.
- algorithmic_behavior: The script has two commands. `codex-thumbsup` accepts a PR number plus optional `--after`, resolves the PR base repository, fetches all PR issue reactions with pagination, filters for `chatgpt-codex-connector[bot]` with `+1`, rejects reactions older than `--after`, and emits the reaction JSON (`scripts/check-bot-reactions.sh:86-187`). `claude-eyes` accepts a comment ID, optional `--pr`, retry count, and delay; it resolves the base repository, polls the comment reactions endpoint, filters for `claude[bot]` with `eyes`, and emits reaction JSON if found (`scripts/check-bot-reactions.sh:190-300`).
- inputs_outputs_state: Inputs are CLI args, current repository identity from `gh repo view`, PR existence from `gh pr view`, GitHub reactions from `gh api`, and retry options. Output is JSON reaction info on stdout when found. Exit codes are part of the API contract: `0` found, `1` not found, `2` usage/API error (`scripts/check-bot-reactions.sh:13-16`, `scripts/check-bot-reactions.sh:57-60`). It does not modify repo files or local state.
- gates_or_invariants: Requires a PR number for `codex-thumbsup` (`scripts/check-bot-reactions.sh:114-117`) and a comment ID for `claude-eyes` (`scripts/check-bot-reactions.sh:227-230`). Unknown options or duplicate positional args fail with code `2` (`scripts/check-bot-reactions.sh:98-107`, `scripts/check-bot-reactions.sh:211-220`). Both modes use base-repo resolution for fork PRs by trying current repo first and parent repo second (`scripts/check-bot-reactions.sh:119-150`, `scripts/check-bot-reactions.sh:232-262`). Paginated reaction arrays are merged with `jq -s 'add // []'` (`scripts/check-bot-reactions.sh:157-159`, `scripts/check-bot-reactions.sh:273-275`).
- dependencies_and_callers: Depends on `gh`, `jq`, `sleep`, and `scripts/portable-timeout.sh` (`scripts/check-bot-reactions.sh:31-33`). Called by `hooks/pr-loop-stop-hook.sh` for Codex thumbs-up before and during polling (`hooks/pr-loop-stop-hook.sh:675`, `hooks/pr-loop-stop-hook.sh:998`) and for Claude eyes verification (`hooks/pr-loop-stop-hook.sh:776`). Also called during PR loop setup when auto-triggering cases that require a Claude acknowledgement (`scripts/setup-pr-loop.sh:526`).
- edge_cases_or_failure_modes: If a PR cannot be found in current or parent repo, the base repo falls back to current repo and the later API call may fail (`scripts/check-bot-reactions.sh:147-162`, `scripts/check-bot-reactions.sh:259-277`). `codex-thumbsup` only examines the first matching Codex reaction after filtering (`scripts/check-bot-reactions.sh:164-168`); if the first matching reaction is older than `--after`, a later newer matching reaction would not be considered. `claude-eyes` sleeps before each attempt, so even a zero-delay test path differs from default production behavior (`scripts/check-bot-reactions.sh:264-268`). API failures in `claude-eyes` continue to the next attempt, while API failure in `codex-thumbsup` is an error.
- validation_or_tests: Covered by `tests/test-pr-loop-system.sh` reaction scenarios referenced at `tests/test-pr-loop-system.sh:201-226`, and by the assigned stop-hook spec through Codex thumbs-up tests (`tests/test-pr-loop-stophook.sh:896-1009`) and Claude eyes timeout tests (`tests/test-pr-loop-stophook.sh:1011-1143`). `tests/robustness/test-pr-loop-api-robustness.sh` also exercises API robustness around PR-loop scripts.
