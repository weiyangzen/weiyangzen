### FIX_PR_LOOP-HZ-035 `file` `scripts/check-pr-reviewer-status.sh`
- cursor: `[_]`
- core_role: Runtime classifier for PR-loop startup/re-review cases. It determines whether monitored bots have reviewed, whether reviews are stale relative to the latest PR commit, and emits JSON consumed by PR-loop setup/stop logic.
- algorithmic_behavior: Parses `<pr_number> --bots <bot1,bot2>` (`scripts/check-pr-reviewer-status.sh:49-85`), maps bot aliases to GitHub authors with special Codex mapping (`scripts/check-pr-reviewer-status.sh:91-100`), resolves fork/base repository by trying current repo then parent (`scripts/check-pr-reviewer-status.sh:109-140`), fetches latest commit info (`scripts/check-pr-reviewer-status.sh:142-151`), gathers issue comments, inline review comments, and PR review submissions (`scripts/check-pr-reviewer-status.sh:153-168`), then computes per-bot commented/missing/stale sets (`scripts/check-pr-reviewer-status.sh:174-205`).
- inputs_outputs_state: Inputs are PR number, comma-separated bot list, GitHub CLI context, `jq`, and network/API data. Output is JSON with `case`, `reviewers_commented`, `reviewers_missing`, `latest_commit_sha`, `latest_commit_at`, `newest_review_at`, and `has_commits_after_reviews` (`scripts/check-pr-reviewer-status.sh:259-275`). It writes no files, but drives `startup_case` and trigger behavior in `scripts/setup-pr-loop.sh:437-447` and later dynamic re-evaluation in `hooks/pr-loop-stop-hook.sh:1544-1555`.
- gates_or_invariants: Argument validation is fail-fast for missing PR, missing `--bots`, unknown options, and multiple PR numbers (`scripts/check-pr-reviewer-status.sh:49-85`). Commit fetch failure is fatal (`scripts/check-pr-reviewer-status.sh:142-148`). Comment fetch failures degrade to empty arrays (`scripts/check-pr-reviewer-status.sh:156-165`). Staleness is per-bot, comparing latest commit timestamp to that bot’s newest review timestamp, not global newest review (`scripts/check-pr-reviewer-status.sh:189-200`).
- dependencies_and_callers: Depends on `scripts/portable-timeout.sh` via `run_with_timeout` (`scripts/check-pr-reviewer-status.sh:38-40`), `gh`, and `jq`. Called by `setup-pr-loop.sh` to initialize case and trigger comments (`scripts/setup-pr-loop.sh:437-468`) and by `pr-loop-stop-hook.sh` to refresh case as reviews arrive (`hooks/pr-loop-stop-hook.sh:1544-1547`). Tests use the mock GitHub CLI in `tests/mocks/gh`, which maps PR commits, comments, reviews, and reactions from fixtures and env vars (`tests/mocks/gh:58-72`, `tests/mocks/gh:154-188`).
- edge_cases_or_failure_modes: Bot CSV entries are split by comma without trimming whitespace (`scripts/check-pr-reviewer-status.sh:106-107`), so `"claude, codex"` would look for `" codex[bot]"`. Case documentation says case 5 is future-distinction “all reviewers” (`scripts/check-pr-reviewer-status.sh:22-27`), but implementation uses case 5 for partial comments with stale commented bots (`scripts/check-pr-reviewer-status.sh:223-230`). Timestamp comparison is lexical and assumes normalized ISO-8601 UTC strings (`scripts/check-pr-reviewer-status.sh:194`). If base repo cannot be resolved, it falls back to `CURRENT_REPO`, which may be empty (`scripts/check-pr-reviewer-status.sh:137-140`).
- validation_or_tests: Covered by `tests/test-pr-loop-system.sh`: case 1 no comments (`tests/test-pr-loop-system.sh:152-170`), case 2 partial comments (`tests/test-pr-loop-system.sh:173-192`), PR review submissions counted as comments (`tests/test-pr-loop-system.sh:236-250`), case 3 all commented (`tests/test-pr-loop-system.sh:493-506`), case 4 all commented plus new commits (`tests/test-pr-loop-system.sh:1315-1341`), and case 5 partial plus new commits (`tests/test-pr-loop-system.sh:1347-1378`).
