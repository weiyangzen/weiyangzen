- algorithmic_behavior: Reads hook JSON from stdin, validates it with shared JSON guards, ignores non-`Write` tools, requires `tool_input.file_path`, then classifies the path. The early gates block round todo writes unless allowlisted, block round prompt writes, and apply PR-loop protections before RLCR protections (`hooks/loop-write-validator.sh:25-93`). For RLCR paths, it finds the newest active loop, detects finalize mode, strictly parses `state.md` or `finalize-state.md`, blocks direct state writes, allows only the active `finalize-summary.md`, blocks loop `plan.md`, blocks `goal-tracker.md` after round 0, rejects summaries outside `.humanize/rlcr`, enforces current summary round, and enforces the active loop directory (`hooks/loop-write-validator.sh:123-265`).

Outputs are process exit status plus stderr block text: `0` allows, `1` fails closed on invalid hook/state input, and `2` blocks a prohibited write. The hook does not mutate state; it turns a proposed write into an allow/block transition.
- gates_or_invariants: Loop state files are system-owned; generated prompts and PR comment/check/feedback files are read-only; PR `round-N-pr-resolve.md` is writable only for the active round; RLCR summaries must be in the active loop directory and match `current_round`; `goal-tracker.md` is only directly writable in round 0; `plan.md` backup is protected; malformed state blocks for safety (`hooks/loop-write-validator.sh:140-195`).
