- cursor: `[_]`
- core_role: Cross-platform timeout abstraction for long-running Git, GitHub CLI, Codex, and hook validation subprocesses. It normalizes timeout behavior on macOS/Linux by selecting `gtimeout`, GNU `timeout`, Python, or no-timeout fallback.
- algorithmic_behavior: `detect_timeout_impl` checks tool availability in priority order: Homebrew `gtimeout`, GNU `timeout` verified by `timeout --version`, `python3`, `python`, then `none` at lines 10-27. The global `TIMEOUT_IMPL` is computed immediately when sourced at line 29. `run_with_timeout` accepts `<seconds> <command> [args...]`, stores the remaining argv as an array, and dispatches by implementation at lines 33-70. `gtimeout` and GNU `timeout` are invoked directly at lines 39-45. - inputs_outputs_state: Inputs are ambient PATH/tool availability, the timeout seconds, and command argv. Outputs are the child command stdout/stderr and exit code. State is the exported `TIMEOUT_IMPL` variable in the sourcing shell environment. No files are written by this script.
- gates_or_invariants: Timeout implementation is chosen once at source time, not per call. GNU-compatible timeout is required for `timeout`; BSD/non-GNU `timeout` is rejected as `none` unless Python exists. Python fallback intentionally matches GNU timeout’s `124` exit on timeout. Command arguments are passed as arrays/direct argv, avoiding shell string re-tokenization for normal use. With `TIMEOUT_IMPL=none`, timeout is advisory only and commands can run indefinitely.
